**摘要**

城市绿色物流配送是当前城市物流发展的重要方向，车辆路径规划问题（VRP）是其核心优化问题之一。本文针对某城市物流配送中心的车辆调度问题，考虑配送成本最小化目标、多车型容量约束、时间窗约束以及绿色配送区的环保限制，建立了一套完整的数学模型并设计两种高效求解算法。

针对问题一（静态环境配送优化），本文分析了98个客户点的配送需求，设计并实现了两种求解算法：**遗传算法（GA）和**自适应大邻域搜索算法（ALNS）** 。遗传算法采用MATLAB实现，使用排列编码、锦标赛选择、PMX交叉和交换变异，种群规模100，迭代500代，采用高装载率解码策略；ALNS算法采用Python实现，结合多种破坏算子和修复算子，配合模拟退火接受准则。两种算法均考虑时变车速、载重对能耗的影响、时间窗约束等因素。最终求得最优配送方案。

本文提出的遗传算法和ALNS算法各有优势：遗传算法全局搜索能力强，适合理论分析；ALNS算法收敛速度快、解质量高，更适合实际工程应用。研究成果可为城市物流配送企业提供决策支持，具有重要的理论意义和实用价值。

**关键词**：车辆路径问题；遗传算法；自适应大邻域搜索；时间窗约束；绿色物流；城市配送

## 一、问题重述

### 1.1 问题背景

随着电子商务的快速发展和城市居民消费水平的提升，城市物流配送需求日益增长。然而，传统物流配送模式存在车辆空载率高、路径规划不合理、碳排放量大等问题，不仅增加了物流企业的运营成本，也加剧了城市交通拥堵和环境污染。因此，如何优化城市物流配送路径，在降低配送成本的同时减少碳排放，实现经济效益与环境效益的双赢，成为学术界和产业界共同关注的重要课题。

车辆路径问题（Vehicle Routing Problem，VRP）是运筹学与组合优化领域的经典问题，于1959年由Dantzig和Ramser首次提出。该问题旨在为一系列配送需求点安排合理的车辆行驶路线，在满足各种约束条件的前提下，最小化总配送成本。VRP问题及其变体在物流配送、运输调度、供应链管理等领域有着广泛的应用价值。

本文研究的问题来源于某城市物流配送中心的实际调度场景。该配送中心位于城市坐标$(20, 20)$ km处，市中心位于$(0, 0)$ km，需要为分布在城市各处的98个客户点提供配送服务。配送区域覆盖半径约40km，其中距市中心10km范围内为绿色配送区，对配送车辆的排放类型有特殊限制。配送中心拥有5种车型共185辆配送车辆（含燃油车和电动车），需在8:00-21:00时段内完成所有配送任务。

### 1.2 问题回顾

**问题1：静态环境配送优化**
> 在不考虑绿色配送区限行政策的情况下，如何安排配送路线使总配送成本最小化？要求同时满足车辆容量约束（载重、体积）和客户时间窗约束。

#### 1.3.4 油耗与电耗模型

车辆油耗成本采用FPK（Fuel Per Kilometer）公式计算：

$$FPK(v) = 0.0025v^2 - 0.2554v + 31.75 \quad \text{(L/100km)}$$

其中，$v$为车辆行驶速度（km/h）。考虑载重影响，实际油耗需乘以载荷系数：

$$F_{实际} = FPK \times (1 + 0.4 \times \alpha)$$

其中，$\alpha$为载重率（满载为1.0）。

车辆电耗成本采用EPK（Energy Per Kilometer）公式计算：

$$EPK(v) = 0.0014v^2 - 0.12v + 36.19 \quad \text{(kWh/100km)}$$

考虑载重影响：

$$E_{实际} = EPK \times (1 + 0.35 \times \alpha)$$

## 二、问题分析

### 2.1 问题一分析：静态环境配送优化

#### 2.1.1 问题特征识别

问题一定义了在静态环境下的车辆路径规划问题，其主要特征包括：

**（1）多约束耦合性**

该问题同时包含多种约束条件，形成复杂的约束耦合关系：

- **容量约束**：每辆车的总装载量（重量和体积）不得超过其最大容量
- **时间窗约束**：每个客户点都有指定的服务时间窗口$[e_i, l_i]$，车辆必须在该时间窗口内到达
- **车辆数量约束**：使用的车辆总数不得超过配送中心可用车辆数（185辆）

约束的数学表达如下：

$$\sum_{i \in R_k} w_i \leq W_k \quad \forall k \in K \quad \text{(载重约束)}$$

$$\sum_{i \in R_k} v_i \leq V_k \quad \forall k \in K \quad \text{(体积约束)}$$

$$e_i \leq t_i \leq l_i \quad \forall i \in C \quad \text{(时间窗约束)}$$

**（2）NP难复杂性**

VRP问题是组合优化领域的经典NP难问题。随着客户数量的增加，问题的可行解空间呈指数级增长。对于98个客户点的问题，即使采用最先进的精确算法也难以在合理时间内求得全局最优解。

**（3）多成本目标**

总配送成本由多个成本组件构成：

$$C_{total} = C_{start} + C_{fuel} + C_{carbon} + C_{wait} + C_{late}$$

- $C_{start}$：车辆启动成本（固定成本）
- $C_{fuel}$：燃油/电费成本（与行驶距离成正比）
- $C_{carbon}$：碳排放成本（与能源消耗成正比）
- $C_{wait}$：早到等待成本（与等待时间成正比）
- $C_{late}$：晚到惩罚成本（与延误时间成正比）

#### 2.1.2 数据特点分析

**客户分布特征**：
- 客户坐标分布范围：X方向[-33.61, 35.98] km，Y方向[-37.57, 38.87] km
- 配送中心位于(20, 20) km，距市中心20km
- 15个客户位于绿色配送区内（距市中心<10km）

**需求量分布特征**：
- 需求量差异悬殊：最大客户需求超过10,000 kg，需拆分配送
- 绿色配送区内客户需求相对集中
- 平均客户需求：约3,240 kg/客户

**时间窗分布特征**：
- 时间窗跨度大：最早9:09，最晚20:58
- 时间窗宽度不一：从约40分钟到超过100分钟不等

## 四、模型建立

### 4.1 目标函数

总配送成本最小化：

$$\min C_{total} = C_{start} + C_{energy} + C_{carbon} + C_{time}$$

其中：
- $C_{start} = \sum_{k \in K} c_{start} \cdot y_k$：车辆启动成本
- $C_{energy}$：能耗成本（燃油/电力）
- $C_{carbon}$：碳排放成本
- $C_{time}$：时间窗惩罚成本

### 4.2 约束条件

**（1）载重约束**

$$\sum_{i \in R_k} w_i \leq W_k \quad \forall k \in K$$

**（2）体积约束**

$$\sum_{i \in R_k} v_i \leq V_k \quad \forall k \in K$$

**（3）时间窗约束**

$$e_i \leq t_i \leq l_i \quad \forall i \in C \setminus \{0\}$$

**（4）路径连续性约束**

$$\sum_{j \in C} x_{ijk} = \sum_{j \in C} x_{jik} = 1 \quad \forall i \in C \setminus \{0\}, k \in K$$

**（5）车辆访问约束**

每个客户点必须且仅被一辆车访问一次：

$$\sum_{k \in K} \sum_{j \in C} x_{ijk} = 1 \quad \forall i \in C \setminus \{0\}$$

#### 4.3.2 核心代码实现

**主函数** (`solve_vrp_ga.m`)：

```matlab
function [best_cost, best_routes, best_veh_types, best_order] = solve_vrp_ga()
% 问题1：静态环境下的车辆调度（无绿色配送区限制，车辆数量不限，高装载率）

    % ---------- 加载数据 ----------
    load('matlab.mat', 'd__ij_', 'timeline', 'wei_vol');
    load('vehicle_types.mat', 'vehicles');
    
    % ---------- 遗传算法参数 ----------
    pop_size = 100;    % 种群规模
    max_gen = 500;     % 最大代数
    pc = 0.8;          % 交叉概率
    pm = 0.15;         % 变异概率
    
    % 初始化种群
    pop = zeros(pop_size, num_cust);
    for i = 1:pop_size
        pop(i,:) = randperm(num_cust);
    end
    
    % ---------- 遗传算法主循环 ----------
    for gen = 1:max_gen
        % 评估适应度
        for i = 1:pop_size
            [c, r, vt] = evaluate_solution(pop(i,:), clients, dist, tw, vehicles, green_flag_zero);
            costs(i) = c;
        end
        
        % 选择、交叉、变异
        new_pop = zeros(pop_size, num_cust);
        for i = 1:2:pop_size
            p1 = tournament_select(costs, pop);
            p2 = tournament_select(costs, pop);
            if rand < pc
                [c1, c2] = pmx_crossover(p1, p2);
            else
                c1 = p1; c2 = p2;
            end
            c1 = swap_mutation(c1, pm);
            c2 = swap_mutation(c2, pm);
            new_pop(i,:) = c1;
            new_pop(i+1,:) = c2;
        end
        pop = new_pop;
        
        fprintf('Generation %d, Best Cost = %.2f\n', gen, best_cost);
    end
end
```

**路径评估函数** (`eval_single_route.m`)：

```matlab
function [total_cost, details] = eval_single_route(route, veh_type_id, clients, dist_matrix, time_windows, vehicles, green_flag)
% 评估单条路径的总成本（不含启动费），支持绿色配送区硬约束
    details = struct();
    details.energy_cost = 0;
    details.carbon_cost = 0;
    details.time_penalty = 0;
    
    veh = vehicles(veh_type_id);
    
    % 初始载重 = 路径上所有客户的总需求
    customer_ids = route(2:end-1);
    current_load_kg = sum([clients(customer_ids).weight]);
    
    t = 0; % 8:00出发
    for i = 1:length(route)-1
        from = route(i);
        to = route(i+1);
        d = dist_matrix(from+1, to+1);
        
        [arrive, segs] = travel_segments(t, d);
        
        % 计算能耗和碳排放
        for s = 1:size(segs,1)
            v = segs(s,1);
            dist_seg = segs(s,2);
            load_ratio = current_load_kg / veh.capacity_kg;
            
            if veh.type == 1   % 燃油车
                base_rate = FPK_fun(v);
                actual_rate = base_rate * (1 + veh.load_penalty * load_ratio);
                energy_L = actual_rate * dist_seg / 100;
                energy_cost = energy_L * veh.energy_price;
                carbon_cost = energy_L * veh.emission_factor * 0.65;
            else              % 电动车
                base_rate = EPK_fun(v);
                actual_rate = base_rate * (1 + veh.load_penalty * load_ratio);
                energy_kWh = actual_rate * dist_seg / 100;
                energy_cost = energy_kWh * veh.energy_price;
                carbon_cost = energy_kWh * veh.emission_factor * 0.65;
            end
            details.energy_cost = details.energy_cost + energy_cost;
            details.carbon_cost = details.carbon_cost + carbon_cost;
        end
        
        % 时间窗检查与服务
        if to ~= 0
            [e, l] = deal(time_windows(to,1), time_windows(to,2));
            if arrive < e
                wait = e - arrive;
                service_start = e;
            elseif arrive > l
                late = arrive - l;
            end
            details.time_penalty = details.time_penalty + (wait/60)*20 + (late/60)*50;
            current_load_kg = current_load_kg - clients(to).weight;
            t = service_start + 20;
        else
            t = arrive;
        end
    end
    
    total_cost = details.energy_cost + details.carbon_cost + details.time_penalty;
end
```

**高装载率解码** (`evaluate_solution.m`)：

```matlab
function [total_cost, final_routes, best_assign] = evaluate_solution(order, clients, dist, tw, vehicles, green_flag)
% 高装载率解码：一辆车尽可能装更多客户后才结束路线
    total_cost = 0;
    routes = {};
    served = false(1, length(order));
    
    ptr = 1;
    while ~all(served)
        % 找到第一个未被服务的客户
        while ptr <= length(order) && served(order(ptr))
            ptr = ptr + 1;
        end
        if ptr > length(order), break; end
        
        best_route = [];
        best_total_cost = inf;
        best_v = 0;
        
        % 尝试每种车型装一条路线
        for v = 1:length(vehicles)
            [route, new_served] = build_highload_route(order, ptr, v, clients, dist, tw, vehicles(v), served, green_flag);
            if ~isempty(route)
                rc = eval_single_route([0, route, 0], v, clients, dist, tw, vehicles, green_flag);
                if rc < best_total_cost
                    best_total_cost = rc;
                    best_route = route;
                    best_v = v;
                    best_new_served = new_served;
                end
            end
        end
        
        served = best_new_served;
        routes{end+1} = [0, best_route, 0];
        total_cost = total_cost + best_total_cost + vehicles(best_v).start_cost;
    end
    
    final_routes = routes;
end
```

```python
from dataclasses import dataclass
from enum import Enum

# 交通时段枚举
class TrafficPeriod(Enum):
    CONGESTED = "拥堵"    # 08:00–09:00, 11:30–13:00
    SMOOTH    = "顺畅"    # 09:00–10:00, 13:00–15:00
    MODERATE  = "一般"    # 10:00–11:30, 15:00–17:00
    EVENING   = "夜间"    # 17:00 以后

# 车辆类型定义
@dataclass(frozen=True)
class VehicleType:
    type_id: str        # "O1"/"O2"/"O3"/"E1"/"E2"
    category: str       # "fuel" | "electric"
    max_weight: float   # 最大载重（kg）
    max_volume: float   # 最大容积（m³）
    fleet_count: int    # 车队总数量
    load_factor: float  # 满载额外能耗比例
    
    def base_consumption(self, speed: float) -> float:
        """空载百公里能耗"""
        if self.category == "fuel":
            return 0.0025 * speed ** 2 - 0.2554 * speed + 31.75
        return 0.0014 * speed ** 2 - 0.12 * speed + 36.19
    
    def consumption(self, speed: float, load_ratio: float) -> float:
        """实际百公里能耗，考虑负载修正"""
        return self.base_consumption(speed) * (1.0 + self.load_factor * load_ratio)
```

#### 4.4.3 客户拆分策略

```python
def split_overloaded_customers(net: Network):
    """将超量客户拆分为多个虚拟客户"""
    virtual_customers = []
    next_id = 99
    
    # 拆分阈值（对齐O2中型车容量）
    split_weight_limit = 1500.0   # kg
    split_volume_limit = 10.8     # m³
    
    for cid, cust in net.customers.items():
        trips = max(
            math.ceil(cust.demand_weight / split_weight_limit),
            math.ceil(cust.demand_volume / split_volume_limit),
        )
        
        if trips <= 1:
            virtual_customers.append({
                'original_id': cid,
                'virtual_id': cid,
                'weight': cust.demand_weight,
                'volume': cust.demand_volume,
                ...
            })
        else:
            weight_per_trip = cust.demand_weight / trips
            volume_per_trip = cust.demand_volume / trips
            
            for t in range(trips):
                vid = next_id
                next_id += 1
                virtual_customers.append({
                    'original_id': cid,
                    'virtual_id': vid,
                    'weight': weight_per_trip,
                    'volume': volume_per_trip,
                    ...
                })
    
    return virtual_customers
```

#### 4.4.4 路径模拟与评估

```python
def simulate_route(route_customers: List[int], depart_time: float, vehicle_type: VehicleType):
    """模拟车辆路径，返回成本信息"""
    info = {
        'total_cost': 0.0,
        'energy_cost': 0.0,
        'carbon_cost': 0.0,
        'time_penalty': 0.0,
        'feasible': True
    }
    
    t_now = depart_time
    load_weight = sum(customers[c]['weight'] for c in route_customers)
    
    prev = 0
    for cust in route_customers:
        distance = dist_matrix[prev][cust]
        travel_time = distance / get_speed(t_now) * 60
        arrive_time = t_now + travel_time
        
        tw_start = customers[cust]['tw_start']
        tw_end = customers[cust]['tw_end']
        
        if arrive_time < tw_start:
            wait_time = tw_start - arrive_time
            info['time_penalty'] += wait_time * WAIT_RATE
            service_start = tw_start
        elif arrive_time > tw_end:
            late_time = arrive_time - tw_end
            info['time_penalty'] += late_time * LATE_RATE
            service_start = arrive_time
        else:
            service_start = arrive_time
        
        speed = get_speed(t_now)
        load_ratio = load_weight / vehicle_type.max_weight
        info['energy_cost'] += vehicle_type.energy_cost(distance, speed, load_ratio)
        info['carbon_cost'] += vehicle_type.carbon_cost(distance, speed, load_ratio) * CARBON_PRICE
        
        load_weight -= customers[cust]['weight']
        t_now = service_start + SERVICE_TIME
        prev = cust
    
    info['total_cost'] = info['energy_cost'] + info['carbon_cost'] + info['time_penalty']
    return info
```

#### 4.4.5 出发时间优化

```python
def optimize_depart_time(route_customers: List[int], vehicle_type: VehicleType):
    """优化单条路径的出发时间（离散枚举）"""
    best_depart = 0.0
    best_info = {'total_cost': float('inf')}
    
    for depart in range(0, 17*60, 5):  # 每5分钟枚举
        info = simulate_route(route_customers, depart, vehicle_type)
        if info['feasible'] and info['total_cost'] < best_info['total_cost']:
            best_info = info
            best_depart = depart
    
    return best_depart, best_info
```
