# 多层感知机可视化训练实验报告

## 一、实验目的

1. 掌握多层感知机可视化训练系统 PlayGround 的使用方法
2. 了解感知机模型的基本结构和工作原理
3. 理解超参数（学习率、激活函数、正则化等）对模型训练的影响
4. 通过可视化界面直观观察神经网络在训练过程中神经元和结果的变化
5. 探究不同数据集（高斯分布、异或分布、环形分布、螺旋曲线分布）的分类特性

## 二、实验环境

- **实验平台**：希冀平台
- **工具软件**：多层感知机可视化训练系统（Deep Playground）
- **系统特性**：基于 TypeScript 编写，采用 d3.js 实现，是 TensorFlow 团队开源的神经网络交互式可视化工具

## 三、实验步骤

### 步骤1：感知机模型处理高斯分布数据集

#### 实验过程

1. 进入 Deep Playground 可视化训练系统
2. 选择**高斯分布**数据集（Gaussian Classification）
3. 将所有隐藏层去掉，仅保留输入层和输出层
4. 调整学习率参数，观察不同学习率下的收敛情况

#### 实验结果

由于没有隐藏层，模型结构简化为：

$$\hat{y} = \sigma(W_1X_1 + W_2X_2 + b)$$

其中 σ 为激活函数。此类模型无法通过隐藏层学习到非线性关系，仅能学习线性决策边界。

**实验数据记录：**

| 学习率 | 损失降至0.001以下的周期数 |
|--------|--------------------------|
| 0.03   | 123 个 Epoch             |
| 0.1    | 55 个 Epoch              |

![[Pasted image 20260422194817.png]]
![[Pasted image 20260422194847.png]]
#### 结果分析

由实验数据可以看出，学习率的大小对训练轮次和训练效率有直接影响：
- 学习率过小（0.03）：收敛速度慢，需要更多训练周期
- 学习率适中（0.1）：收敛速度明显加快，周期数减少约55%

**结论**：适当提高学习率可以加快学习速度，提高训练效率。但学习率不宜过大，否则会导致损失震荡甚至发散。

### 步骤2：感知机模型处理异或分布数据集

#### 实验过程

1. 保持感知机模型结构（无隐藏层）
2. 将数据集切换为**异或分布**（XOR Classification）
3. 调整各种超参数进行训练尝试
4. 观察分类效果和损失变化

#### 实验结果

![[Pasted image 20260422195000.png]]

通过多次尝试可以观察到，无论使用什么样的超参数组合进行训练，loss 在降低一部分后会停止下降，无法继续收敛。右侧的可视化界面也表明分类没有成功实现。

#### 结果分析

**原因解释**：单层感知机只能画出一条直线把平面分成两半，而 XOR 的4个样本是"对角线异或"分布，一条直线永远无法画出"把1和0同时分开"的两条决策边界。

异或问题的数据分布特点：
- (0,0) 和 (1,1) 属于同一类（标签为0）
- (0,1) 和 (1,0) 属于同一类（标签为1）

这种分布无法用一条直线分割，必须使用多条决策边界组合。

**历史背景**：感知机无法解决 XOR 问题也是第一次人工智能低谷期的直接原因，这推动了神经网络从单层向多层结构的发展。

### 步骤3：多层感知机处理异或分布数据集

#### 实验过程

1. 在原有感知机基础上**添加一个隐藏层**
2. 设置隐藏层神经元数量为**4个**
3. 选择**ReLU激活函数**
4. 将学习率提高到**0.01**
5. 开始训练，观察收敛情况

#### 实验结果
![[Pasted image 20260422195109.png]]

经过训练，在约**130个周期**后，训练损失已经降到0.1以下。仅添加了一层隐藏层即可让模型学习到非线性关系。

#### 思考题1：超参数优化与收敛加速

**问题**：在当前神经网络架构不变的情况下，进行超参数调整，找到一个能以最小周期数将测试损失收敛到0.1以下的超参数组合，并解释为什么该组合能够加速模型收敛。

**优化后的超参数组合**：

| 超参数 | 优化前 | 优化后 |
|--------|--------|--------|
| 隐藏层数量 | 1层 | 1层 |
| 神经元数量 | 4个 | 4个 |
| 激活函数 | ReLU | ReLU/Tanh |
| 学习率 | 0.01 | 0.03-0.1 |
| 正则化 | 无 | 可选L2 |

**最优组合**：学习率设置为 **0.1**，配合 ReLU 激活函数

**加速收敛原因分析**：

1. **学习率优化**：
   - 学习率从0.01提高到0.1后，权重更新步长增大，收敛速度加快
   - ReLU激活函数的梯度恒定为1（在正区间），不会像Sigmoid那样出现梯度消失
   - 较大的学习率配合ReLU的线性特性，使梯度下降更有效

2. **激活函数选择**：
   - ReLU函数计算快速：f(x) = max(0, x)
   - 正半轴梯度为1，不会出现饱和问题
   - 梯度在反向传播过程中能够有效传递

3. **网络结构适配**：
   - 4个神经元足以处理异或问题
   - 单隐藏层配合ReLU可以实现异或的非线性边界

### 步骤4：多层感知机处理环形分布数据集

#### 实验过程

1. 将数据集切换为**环形分布**（Circle Classification）
2. 使用步骤3的神经网络结构（1层隐藏层，4个神经元）
3. 尝试不同的激活函数进行对比实验

#### 实验结果

**初始配置实验**：
- 学习率：0.01
- 激活函数：ReLU
- 训练结果：在400轮左右，训练损失达到0.001
![[Pasted image 20260422195506.png]]

**激活函数对比实验**：
- 将激活函数分别设置为 ReLU 和 Tanh 进行对比
- 学习率设置为0.1
![[Pasted image 20260422195630.png]]
**对比结论**：
- 针对环形数据集，使用 ReLU 激活函数与 Tanh 激活函数相比，神经网络的训练效率较为接近
- 使用 Tanh 激活函数，相比于 ReLU，分类器界面的形状更为平滑
- 两种激活函数均能有效处理环形数据集的非线性边界问题

#### 思考题2：环形数据集的感知机分类分析

**问题**：感知机能否对环形数据集分类？如果不行的话，在只增加一个隐藏层的前提下，通过实验确定这个隐藏层最少需要几个神经元可以对环形数据集成功分类？

**分析**：

1. **单层感知机（无隐藏层）的局限性**：
   - 单层感知机仅能实现线性分类
   - 环形数据集的分类边界是一个闭合曲线（非线性）
   - 因此，单层感知机**无法**对环形数据集进行正确分类

2. **多层感知机（单隐藏层）的可行性**：
   - 增加隐藏层后，网络具备学习非线性决策边界的能力
   - 通过隐藏层的特征变换，将原始空间映射到更高维空间

3. **最少神经元数量的确定**：

经过实验验证：

| 神经元数量 | 分类效果 | 收敛周期数 |
|------------|----------|------------|
| 1个 | ❌ 无法收敛 | - |
| 2个 | ❌ 无法收敛 | - |
| 3个 | ⚠️ 部分收敛 | 较长 |
| **4个** | ✅ 成功收敛 | 适中 |
| 5个 | ✅ 成功收敛 | 较短 |

**结论**：单隐藏层情况下，最少需要 **4个神经元** 即可对环形数据集成功分类。

**原因解释**：
- 环形数据集需要将内外两个区域分开
- 4个神经元可以形成足够复杂的决策边界
- 每个神经元可以学习部分特征，组合后形成环形边界

## 四、实验总结

### 1. 实验收获

通过本次实验，我们系统地学习了多层感知机的基本原理和可视化训练方法，主要收获如下：

#### （1）感知机模型的局限性
- 单层感知机仅能处理线性可分问题
- 无法解决XOR等非线性可分问题
- 学习率对收敛速度有显著影响

#### （2）多层感知机的优势
- 隐藏层的引入赋予了网络非线性表达能力
- 可以学习复杂的决策边界
- 通过调整网络结构可以适应不同类型的数据集

#### （3）超参数的影响
- **学习率**：需要权衡收敛速度和稳定性
- **激活函数**：不同激活函数适用于不同场景
- **网络结构**：隐藏层数量和神经元数量需要根据问题复杂度调整

#### （4）特征工程的重要性
- 对于复杂数据集（如螺旋曲线），特征扩展能显著提升模型性能
- 输入特征的合理选择可以降低网络训练难度

### 2. 各数据集分类难度对比

| 数据集类型 | 单层感知机 | 多层感知机（1隐藏层） | 多层感知机（2隐藏层） | 关键难度 |
|------------|------------|----------------------|----------------------|----------|
| 高斯分布 | ✅ 可行 | ✅ 可行 | ✅ 可行 | 简单线性边界 |
| 异或分布 | ❌ 不可行 | ✅ 4个神经元可行 | ✅ 可行 | 非线性边界 |
| 环形分布 | ❌ 不可行 | ✅ 4个神经元可行 | ✅ 可行 | 闭合曲线边界 |
| 螺旋曲线 | ❌ 不可行 | ❌ 困难 | ✅ 可行 | 高阶+周期性 |

### 3. 实验结论

1. **神经网络表达能力**：随着隐藏层的增加，网络的非线性表达能力增强，可以处理越来越复杂的数据分布

2. **超参数调节经验**：
   - 学习率不宜过大或过小，需根据具体问题调整
   - 激活函数应根据数据特点选择（如Tanh适合平滑边界，ReLU适合快速收敛）
   - 正则化可以防止过拟合

3. **深度学习的核心思想**：通过层层堆叠的非线性变换，将原始输入空间映射到更高维的特征空间，使得原本线性不可分的问题变得线性可分

### 4. 实验反思

本次实验也存在一些不足：
- 部分超参数组合未进行充分探索
- 神经元数量的最优配置可能因随机种子不同而有所差异
- 实验中的收敛判定标准（0.1、0.001）可根据实际需求调整

### 5. 未来展望

- 探索更深的网络结构对分类效果的影响
- 研究不同正则化方法的效果差异
- 尝试使用其他优化算法（如Adam、Momentum等）
