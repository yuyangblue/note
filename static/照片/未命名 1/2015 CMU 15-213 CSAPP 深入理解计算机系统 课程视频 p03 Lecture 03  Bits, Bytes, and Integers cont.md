### 2. 无符号加法 (Unsigned Addition)

*Content-[01:50]*

讲师回顾了上次的内容：位模式可以解释为无符号数（范围 $0$ 到 $2^w - 1$）或补码有符号数。今天重点关注算术运算。

#### 2.1 模运算概念
无符号加法本质上是一种模运算（Modular Arithmetic）。当两个 $w$ 位数相加时，结果可能需要 $w+1$ 位来表示。但在计算机中，我们丢弃最高位（进位），只保留低 $w$ 位。
*   公式：$s = UAdd_w(u, v) = u + v \pmod{2^w}$
*   这意味着如果结果超过最大值，它会“回绕”（Wrap Around）到零。

*![](/static/screenshots/screenshot_001_891b540f-4d45-412c-ba4d-dbde8488c896.jpg)*

#### 2.2 溢出示例 (4-bit)
以 4 位字长为例，范围是 $0$ 到 $15$。
*   例子：$13 + 5$
    *   二进制计算：$1101 + 0101$
    *   真实和：$18$ ($10010_2$)，需要 5 位。
    *   截断后：丢弃最高位，得到 $0010_2$，即 $2$。
    *   数学解释：$18 \pmod{16} = 2$。
*   可视化：想象一个悬崖，从 $15$ 加 $1$ 直接掉落到 $0$。

*![](/static/screenshots/screenshot_002_c4c1264c-0398-4dc6-9687-c94c334eba90.jpg)*

### 3. 补码加法 (Two's Complement Addition)

*Content-[06:00]*

#### 3.1 硬件一致性
补码加法在比特层面（Bit-Level Behavior）与无符号加法完全相同。这意味着 CPU 可以使用同一套硬件电路来处理有符号和无符号数的加法。
*   减法也可以通过加法实现：$a - b = a + (-b)$，其中 $-b$ 是 $b$ 的补码表示。

#### 3.2 补码加法示例
*   4 位补码范围：$-8$ 到 $+7$。
*   例子：$-3 + 5$
    *   $-3$ 的补码：$1101$
    *   $5$ 的二进制：$0101$
    *   相加：$1101 + 0101 = 10010$
    *   丢弃进位：$0010$，即 $2$。正确。

*![](/static/screenshots/screenshot_003_9c2067fe-67c5-4838-b1d2-996f428b448e.jpg)*

#### 3.3 补码溢出 (Overflow)
补码溢出有两种情况，与无符号数不同：
1.  **正溢出 (Positive Overflow)**：两个正数相加，结果为负。
    *   例子：$7 + 5$
    *   $0111 + 0101 = 1100$
    *   解释：$12$ 无法用 4 位补码表示（最大为 $7$）。截断后 $1100$ 被解释为 $-4$。
2.  **负溢出 (Negative Overflow)**：两个负数相加，结果为正。
    *   例子：$-3 + (-6)$
    *   $1101 + 1010 = 10111$
    *   截断后：$0111$，即 $7$。
    *   解释：$-9$ 无法表示（最小为 $-8$）。

*![](/static/screenshots/screenshot_004_cce189ab-8974-4057-b30d-3ccdd9e1e86f.jpg)*

### 4. 乘法 (Multiplication)

*Content-[14:00]*

#### 4.1 基本规则
乘法的结果通常比操作数大得多。两个 $w$ 位数相乘，结果最多需要 $2w$ 位。
*   为了保持字长固定，同样会截断高 $w$ 位，只保留低 $w$ 位。
*   对于无符号数：$UMult_w(u, v) = u \cdot v \pmod{2^w}$。
*   对于补码数：低位部分与无符号乘法相同，但高位截断会影响符号位的解释。

#### 4.2 乘法示例
*   无符号：$3 \times 5 = 15$ ($01111_2$)，在 4 位范围内可表示。
*   无符号溢出：$5 \times 5 = 25$ ($11001_2$)。截断后为 $1001_2 = 9$。即 $25 \pmod{16} = 9$。
*   补码异常：两个正数相乘可能得到负数。
    *   例子：$5 \times 5$ 在 4 位补码中。
    *   $5$ 是 $0101$。
    *   乘积 $25$ 是 $11001$。
    *   截断低 4 位：$1001$。
    *   解释：符号位为 $1$，所以是负数。具体值为 $-7$ ($1001$ 是 $-7$ 的补码)。
    *   原因：$25 \equiv -7 \pmod{16}$。

### 5. 移位操作 (Shift Operations)

*Content-[22:00]*

#### 5.1 左移 (Left Shift)
*   操作：$u << k$ 等价于乘以 $2^k$。
*   原理：将位向左移动 $k$ 位，低位补 $0$。这相当于增加每个位的权重。
*   优化：编译器会将乘以 $2^k$ 转换为左移指令，因为移位比乘法快得多（历史上如此，现代机器仍有差距）。

*![](/static/screenshots/screenshot_006_62be6adf-6dd3-4a5b-9084-7d4c4e4f7865.jpg)*

#### 5.2 右移 (Right Shift)
*   **逻辑右移 (Logical Shift)**：用于无符号数。高位补 $0$。
    *   等价于除以 $2^k$（向下取整/向零舍入）。
*   **算术右移 (Arithmetic Shift)**：用于补码有符号数。高位补符号位（复制最高位）。
    *   目的：保持数字的符号不变。
    *   例子：$-6$ ($1010$) 右移一位。
        *   逻辑右移：$0101$ ($5$) —— 错误。
        *   算术右移：$1101$ ($-3$) —— 正确。

#### 5.3 除法与舍入问题
*   算术右移对于负数除法实际上是向负无穷方向舍入（例如 $-3 / 2 = -2$ 而不是 $-1$）。
*   **技巧 (Bias Trick)**：为了实现向零舍入（C 语言标准），编译器会在右移前给负数加上一个偏移量（Bias）。
    *   例如除以 $16$：先加 $15$，再右移 $4$ 位。

*![](/static/screenshots/screenshot_007_60444ad6-7dfc-4a7c-904c-3828f931e4c5.jpg)*

### 6. 数字取反 (Negation)

*Content-[31:40]*

如何手动计算一个数的负数？
*   方法：**按位取反加一 (Complement and Increment)**。
*   公式：$-x = \sim x + 1$。
*   例子：$6$ ($0110$) -> 取反 $1001$ -> 加 $1$ -> $1010$ ($-6$)。

### 7. 为什么使用无符号数 (Why Use Unsigned?)

*Content-[35:00]*

#### 7.1 陷阱与风险
混合使用有符号和无符号数是 C 语言中常见的 Bug 来源。
*   **隐式转换**：当一个有符号数和一个无符号数比较或运算时，有符号数会被隐式转换为无符号数。
*   **经典案例**：
    ```c
    unsigned i;
    for (i = cnt - 2; i >= 0; i--) // 死循环
        a[i] += a[i+1];
    ```
    *   当 $i=0$ 时，$i--$ 变为 $-1$。
    *   $-1$ 作为无符号数是一个巨大的正数（$UMax$）。
    *   条件 $i >= 0$ 永远为真，导致无限循环甚至数组越界访问。

#### 7.2 最佳实践
*   **Robert Seacord 的建议**：避免使用无符号数进行递减计数。
*   **推荐做法**：使用 `size_t` 类型或者确保循环变量是有符号的，并在比较时显式转换。
*   **安全编码**：C 标准保证无符号运算是模运算（不会像有符号数那样定义溢出行为），这使得某些底层算法（如加密、位集操作）更安全可控。

### 8. 内存组织 (Memory Organization)

*Content-[44:00]*

#### 8.1 字节寻址
*   内存被视为一个巨大的字节数组，通过地址索引。
*   **64 位机器**：地址空间理论上是 $2^{64}$ 字节，但实际上通常限制在 $2^{47}$ 字节左右（约 128 TB）。
*   **虚拟内存**：操作系统提供这种大数组的幻觉，实际数据可能在物理内存或磁盘之间交换。

#### 8.2 机器字长 (Word Size)
*   指处理器处理数据的自然单位大小（通常是指针的大小）。
*   **32 位机器**：字长 32 位，地址空间 4GB。
*   **64 位机器**：字长 64 位，地址空间巨大。
*   **对齐 (Alignment)**：多字节的数据（如 int, long）应存储在地址为字节大小倍数的位置上（例如 32 位整数地址必须是 4 的倍数），以提高硬件访问效率。

#### 8.3 字节序 (Endianness)
*   **大端模式 (Big-Endian)**：高字节存低地址（符合人类书写习惯，网络传输顺序）。
*   **小端模式 (Little-Endian)**：低字节存低地址（Intel x86, ARM 默认）。
*   **影响**：不同字节序的机器上读取同一个多字节整数，其字节排列看起来是反转的。

### 9. 课堂谜题 (Puzzles)

*Content-[45:00]*

最后，讲师通过几个布尔表达式来测试学生对整数性质的理解（判断是否为永真、永假或有时成立）：
1.  $x == -x$：只有当 $x=0$ 或 $x=TMin$ 时成立。
2.  $x > y \implies -x < -y$：不总是成立（例如 $x=TMin$ 时，$-x$ 未定义/溢出）。
3.  $(x \ll 2) == (x \times 4)$：不总是成立（溢出时失效）。
4.  $x \cdot x \ge 0$：不总是成立（补码乘法溢出可能导致负数）。
