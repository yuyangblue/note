## 1. 引言与比特基础

### 课程介绍
本课程是 Carnegie Mellon University (CMU) 15-213《深入理解计算机系统》(CSAPP) 的一部分。讲师 Randy E. Bryant 将探讨数据的表示方式，特别是数字如何以不同形式（如整数、浮点数）存储，以及这些表示对算术运算属性的影响。我们将重点关注二进制表示及其在处理溢出等边缘情况时的表现。

*   **历史背景**: 人类习惯使用十进制（基于手指），早期的电子计算机如 ENIAC (1946年) 甚至使用十进制进行算术运算（通过真空管）。直到 1948 年左右，人们才广泛转向二进制。
*   **为什么是比特 (Bits)?**:
    *   在数字世界中，我们可以将模拟信号量化为离散的电压范围（例如低电压代表 0，高电压代表 1）。
    *   **优势**: 只要信号不超过阈值，即使存在噪音或不完美，也能获得干净的信号。这使得存储和传输数字信息比模拟信息更容易且更可靠。

*   **二进制表示**:
    *   整数基于 2 的幂次方。
    *   小数点在二进制中称为“二元点” (binary point)。左侧权重为 $2^0, 2^{-1}, \dots$，右侧为 $2^{-1}, 2^{-2}, \dots$（即 1, 1/2, 1/4...）。
    *   **十六进制 (Hex)**: 为了便于阅读长串的二进制数，通常每 4 位一组转换为十六进制（0-F）。
        *   例如：`FA1D37B` 在 C 语言中写作 `0xFA1D37B`。

## 2. 字节与数据表示

### 字节定义与 C 数据类型
*   **Byte**: 定义为 8 个位 (bits)。
*   **字长 (Word Size)**: 现代机器通常为 32 位或 64 位。地址空间大小由字长决定（例如 64 位机器有 64 位地址值，即 8 字节）。

*C 语言中的数据类型大小 (x86-64)*:

| C Data Type | Typical 32-bit | Typical 64-bit | x86-64 |
| :--- | :---: | :---: | :---: |
| `char` | 1 | 1 | 1 |
| `short` | 2 | 2 | 2 |
| `int` | 4 | 4 | 4 |
| `long` | 4 | 8 | 8 |
| `float` | 4 | 4 | 4 |
| `double` | 8 | 8 | 8 |
| `long double` | - | - | 10/16 |
| `pointer` | 4 | 8 | 8 |

*   **注意**: `long` 的大小在不同架构上差异很大（32 位系统为 4 字节，64 位 Linux 系统为 8 字节）。`float` 通常为 32 位，`double` 为 64 位。
*   **Intel 特殊扩展**: Intel 机器中存在一种 10 字节 (80 位) 的浮点表示（`long double`），在 64 位机器上使用时会浪费额外的 6 位，并导致数据按 16 字节对齐。

## 3. 布尔代数与位操作

### 历史与基础逻辑
*   **乔治·布尔 (George Boole)**: 19 世纪提出布尔代数，将逻辑（真/假）与代数结合。
*   **克劳德·香农 (Claude Shannon)**: 将布尔代数应用于数字电路设计（继电器开关），这是现代计算机的基础。

### 基本位操作 (Bitwise Operations)
这些操作直接作用于位的集合，遵循布尔代数的规则：

*   **AND (&)**: 两个输入都为 1 时结果为 1 (交集)。
*   **OR (|)**: 任一输入为 1 时结果为 1 (并集)。
*   **XOR (^)**: 仅当输入不同时结果为 1 (对称差)。
*   **NOT (~)**: 翻转所有位 (补集)。

*   **集合表示**: 位向量可以用来表示集合。例如，一个 8 位字节可以表示 $\{0, ..., 7\}$ 的子集。第 $i$ 位为 1 表示元素 $i$ 在集合中。
    *   `&` 对应集合交集。
    *   `|` 对应集合并集。
    *   `^` 对应对称差。
    *   `~` 对应集合补集。

*   **Screencast Example**: 文件权限标志 (Read, Write, Execute) 常使用这种位掩码技术表示。

### 逻辑操作 vs 位操作
在 C 语言中，区分单符号（位操作）和双符号（逻辑操作）至关重要，初学者常混淆：

| 操作符     | 名称          | 行为          | 返回值         |         |            |
| :------ | :---------- | :---------- | :---------- | ------- | ---------- |
| `&`, `  | `, `^`, `~` | **Bitwise** | 逐位操作        | 保留原始位模式 |            |
| `&&`, ` |             | `, `!`      | **Logical** | 整体真/假判断 | 总是返回 0 或 1 |

*   **短路求值 (Short-circuiting)**:
    *   `&&`: 如果左边为假，右边不执行。
    *   `||`: 如果左边为真，右边不执行。
    *   **用途**: 常用于防止空指针解引用，例如 `if (p != NULL && p->field == 5)`。

## 4. 移位操作

### 左移 (`<<`) 和右移 (`>>`)
*   **左移 (`x << y`)**: 将 $x$ 的位向左移动 $y$ 位，低位补 0。
    *   对于无符号数和有符号正数，相当于乘以 $2^y$。
    *   **注意**: 如果移位量 $\ge$ 字长，行为未定义（取决于硬件，通常取模）。

*   **右移**: 有两种类型：
    1.  **逻辑右移 (Logical Shift)**: 高位补 0。适用于无符号数。
    2.  **算术右移 (Arithmetic Shift)**: 高位复制符号位（最高有效位）。适用于有符号补码数，保持数值符号不变。

*   **例子**:
    *   `x << 3`: 丢弃高位，低位补 0。
    *   算术右移 `-2 >> 1`: 符号位是 1，所以填充 1，结果仍为负数。

## 5. 整数编码：无符号与补码

### 核心公式
假设字长为 $w$，位模式为 $x_{w-1} ... x_0$。

1.  **无符号 (Unsigned) 编码**:
    $$ B2U(X) = \sum_{i=0}^{w-1} x_i \cdot 2^i $$
    所有位均为正权重。

2.  **补码 (Two's Complement) 编码**:
    $$ B2T(X) = -x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2} x_i \cdot 2^i $$
    最高位（符号位）具有负权重 $-2^{w-1}$。

### 数值范围
*   **无符号 (Unsigned)**:
    *   $UMin = 0$
    *   $UMax = 2^w - 1$
*   **补码 (Signed)**:
    *   $TMin = -2^{w-1}$
    *   $TMax = 2^{w-1} - 1$

*   **不对称性**: 补码范围中，负数比正数多一个（因为 0 是非负的）。例如 8 位补码范围是 -128 到 127。
    *   $|TMin| = TMax + 1$

*   **特殊值**:
    *   全 0: 0
    *   全 1: -1 (补码) 或 $UMax$ (无符号)
    *   $TMin$ 的二进制形式为 $100...0$。

## 6. 有符号与无符号的转换及陷阱

### 映射关系 (Mapping)
*   **T2U (Two's Complement to Unsigned)**: 保持位模式不变，重新解释。
    *   如果 $x \ge 0$, $T2U(x) = x$.
    *   如果 $x < 0$, $T2U(x) = x + 2^w$.
*   **U2T (Unsigned to Two's Complement)**: 保持位模式不变，重新解释。
    *   如果 $ux < 2^{w-1}$, $U2T(ux) = ux$.
    *   如果 $ux \ge 2^{w-1}$, $U2T(ux) = ux - 2^w$.

### C 语言中的隐式转换与陷阱
在表达式中混合有符号和无符号数时，**有符号数会被隐式转换为无符号数**。这是一个常见的错误来源。

*   **比较陷阱**:
    *   `(-1 < 0)` -> True (有符号比较)
    *   `(-1 < 0U)` -> False (无符号比较)。因为 -1 被转换为 $UMax$ (极大的正数)。
    *   `(int)-1 < 0U` -> `false`
    *   `(unsigned)-1 > 0` -> `true` (因为 -1 变成了最大无符号数)

*   **循环陷阱**:
    ```c
    // 危险代码
    for (int i = n - 1; i >= 0; i--) { ... }
    ```
    如果 $n=0$，则 $i$ 变为 $-1$。但在无符号比较中，$-1$ (即 $UMax$) $\ge 0$ 永远成立，导致死循环。
    *修正*: 使用无符号索引或改变循环条件。

*   **截断 (Truncation)**:
    *   从宽字长截断到窄字长（例如 `int` 转 `short`）只是简单地丢弃高位。
    *   这等同于模运算：结果等于原值对 $2^k$ 取模（其中 $k$ 是目标位数）。

## 7. AI 总结

本次讲座深入探讨了计算机系统中整数的底层表示和处理机制。核心要点包括：

1.  **基础表示**: 计算机内部所有数据均基于比特 (bits)，二进制因其抗噪性和易实现性成为标准。十六进制用于方便地表示二进制数据。
2.  **数据类型**: C 语言中不同类型的数据（char, int, long 等）在不同架构下的字节大小有所不同，特别是在 32 位与 64 位系统之间。
3.  **位操作与逻辑**: 区分了位操作符 (`&`, `|`, `^`, `~`) 与逻辑操作符 (`&&`, `||`, `!`)，前者处理位模式，后者处理真假值并支持短路求值。
4.  **整数编码**: 重点讲解了**无符号数**和**补码 (Two's Complement)**。补码利用最高位作为负权重的符号位，解决了正负数统一表示的问题，但导致了范围不对称 ($|TMin| = TMax + 1$)。
5.  **转换陷阱**: 在 C 语言中，有符号数与无符号数混合运算时，有符号数会隐式转换为无符号数。这会导致看似荒谬的比较结果（如 `-1 < 0U` 为假）和潜在的无限循环，是编程中极易出错的地方。
