# 第一章 背包 DP

> 背包问题就是"给定容量，选物品使价值最大"——吃饭下馆子兜里只有这么多钱，怎么点最爽。

---

## 一、0/1 背包

### 大白话

有 N 个物品，每个只能拿一次（拿或不拿），背包容量 V。每个物品有体积 w[i] 和价值 v[i]。问最多能装多少价值？

像吃自助餐，每道菜只能吃一次，肚子容量有限，怎么吃最回本。

### 核心思路

**状态定义**：`dp[j]` = 容量为 j 的背包能装的最大价值

为什么要倒着遍历容量？因为每个物品只能用一次，正着遍历的话同一个物品会被重复使用（那就变成完全背包了）。

**关键逻辑**：
- 对每个物品 i，枚举容量 j 从 V 到 w[i]
- `dp[j] = max(dp[j], dp[j - w[i]] + v[i])`
- 取或不取，二者选优

### 代码

```python
def knapsack_01(N, V, w, v):
    """
    0/1 背包
    N: 物品数量
    V: 背包容量
    w: 体积列表，长度 N
    v: 价值列表，长度 N
    """
    dp = [0] * (V + 1)          # dp[j]：容量 j 的最大价值，初始 0
    
    for i in range(N):          # 遍历每个物品
        for j in range(V, w[i] - 1, -1):   # 倒序遍历容量，保证每个物品只用一次
            dp[j] = max(dp[j], dp[j - w[i]] + v[i])
            # 要么不拿（dp[j]），要么拿（腾出 w[i] 空间 + 当前价值）
    
    return dp[V]                # 容量为 V 时的最大价值
```

## 二、完全背包

### 大白话

和 0/1 背包的区别：每个物品可以拿无限次。

像去自助餐厅，鸡翅可以一次拿一盘，拿完可以再去拿，吃到饱为止。

### 核心思路

**状态定义**：`dp[j]` = 容量为 j 的背包能装的最大价值

为什么这次要**正着**遍历？因为允许重复使用一个物品，正着遍历会让 dp 值「累积」当前物品，从而允许无限取。

**关键逻辑**：
- 对每个物品 i，枚举容量 j 从 w[i] 到 V
- `dp[j] = max(dp[j], dp[j - w[i]] + v[i])`
- 正序遍历 = 可以多次选择同一个物品

### 代码

```python
def knapsack_complete(N, V, w, v):
    """
    完全背包
    区别：正序遍历容量
    """
    dp = [0] * (V + 1)
    
    for i in range(N):
        for j in range(w[i], V + 1):       # 正序遍历！
            dp[j] = max(dp[j], dp[j - w[i]] + v[i])
    
    return dp[V]
```

## 三、多重背包

### 大白话

每个物品有数量限制——比如餐厅鸡翅限拿 3 盘。多了不让拿。

本质上是在 0/1 背包的基础上，把每个物品拆成若干个"单个物品"来处理。但直接拆太慢，需要用**二进制优化**。

### 核心思路

**二进制优化**思想：
- 一个物品有 k 个，不需要拆成 k 个 1
- 拆成 1, 2, 4, 8, ... 和剩余数
- 这些数可以组合出 1~k 的任意个数，且只需要 O(log k) 个物品
- 然后对这些"打包后"的物品做 0/1 背包

比如一个物品有 13 个 → 拆成 1, 2, 4, 6（因为 1+2+4=7，剩下 13-7=6）
想拿 5 个就用 1+4，想拿 10 个就用 4+6 → 任何 0~13 都能组合出来

### 代码

```python
def knapsack_multiple(N, V, w, v, c):
    """
    多重背包（二进制优化）
    c[i]: 第 i 个物品的数量
    """
    # 二进制拆分
    goods = []      # 拆完后的 (体积, 价值) 列表
    for i in range(N):
        k = c[i]                       # 数量
        base = 1
        while k >= base:
            goods.append((w[i] * base, v[i] * base))
            k -= base
            base <<= 1                 # 乘 2
        if k > 0:                      # 剩下的零头
            goods.append((w[i] * k, v[i] * k))
    
    # 拆完后做 0/1 背包
    dp = [0] * (V + 1)
    for weight, value in goods:
        for j in range(V, weight - 1, -1):
            dp[j] = max(dp[j], dp[j - weight] + value)
    
    return dp[V]
```

## 四、分组背包

### 大白话

物品分成了若干组，每组只能选一个。比如甜品类——提拉米苏、布丁、冰淇淋，只能选一个（你只能吃一份甜品）。

每组相当于组内做一次决策——选哪个，或者都不选。

### 核心思路

**状态定义**：`dp[j]` = 容量 j 的最大价值

**关键逻辑**：
- 外层循环：枚举组
- 中层循环：枚举容量（倒序）
- 内层循环：枚举组内每个物品
- 对每组的物品，在同一个容量下做 max 比较

三层循环，容量倒序确保每组最多选一个。

### 代码

```python
def knapsack_group(N, V, groups):
    """
    分组背包
    groups: 列表，每个元素是 [(体积, 价值), ...] 表示一组
    N: 组数
    """
    dp = [0] * (V + 1)
    
    for i in range(N):              # 遍历每组
        for j in range(V, -1, -1):  # 容量倒序
            for w_item, v_item in groups[i]:    # 遍历组内物品
                if j >= w_item:
                    dp[j] = max(dp[j], dp[j - w_item] + v_item)
    
    return dp[V]
```

## 总结

| 类型 | 遍历方向 | 限制 | 核心技巧 |
|------|----------|------|----------|
| 0/1 背包 | 容量倒序 | 每物品最多1个 | 基础模板 |
| 完全背包 | 容量正序 | 无限取用 | 正序即无限 |
| 多重背包 | 拆后倒序 | 每物品有限个 | 二进制优化 |
| 分组背包 | 容量倒序×组内 | 每组最多1个 | 三层循环 |

**一句话记忆**：
- 正序无限（完全）
- 倒序唯一（0/1）
- 二进制拆数量（多重）
- 三层循环选组内（分组）
