# 01 复杂度、枚举、模拟

> 算法竞赛的第一步不是炫技，是能写出**能跑的代码**。枚举和模拟是最基本的——暴力出奇迹之前，至少得先会暴力。

---

## 一、时间复杂度

### 大白话

时间复杂度就是：**数据规模翻倍时，你的程序要多算几倍的时间。**

- O(1)：不管数据多大，时间都不变。比如取数组第一个元素。
- O(n)：数据翻倍，时间也翻倍。比如遍历一次数组。
- O(n²)：数据翻倍，时间变成 4 倍。比如双重循环。
- O(log n)：数据翻倍，时间只多一个常数。比如二分查找。
- O(n log n)：比 O(n) 慢一点，但比 O(n²) 快得多。比如归并排序。

### 常见数据范围与能接受的复杂度

| 数据规模 n | 能接受的复杂度 | 例子 |
|-----------|---------------|------|
| n ≤ 10 | O(n!), O(n⁶) | 暴力枚举排列 |
| n ≤ 20 | O(2ⁿ), O(n⁵) | 状压 DP |
| n ≤ 100 | O(n³) | Floyd 最短路 |
| n ≤ 10³ | O(n²) | 朴素 LIS、暴力双重循环 |
| n ≤ 10⁵ | O(n log n) | 排序、二分、Dijkstra |
| n ≤ 10⁶ | O(n) | 线性扫描 |
| n ≤ 10⁷ | O(n) 极简操作 | 简单的数组遍历 |
| n ≤ 10⁸+ | O(log n), O(1) | 数学公式、二分 |

**关键**：拿到题先看数据范围，算一下你的算法能不能过。10⁵ 的数据上 O(n²) 一定会超时。

### 一个经验法则

Python 大约 1 秒能跑 10⁷ ~ 10⁸ 次简单操作。竞赛时限通常是 1~2 秒，所以：
- n = 10⁵ → 尽量 O(n log n) 以内
- n = 10³ → O(n²) 可以
- 有常数/大循环 → 适当减半预期

## 二、枚举

### 大白话

枚举就是**把所有可能的情况都试一遍**，找出符合条件的那个。

像破译三位数密码锁——从 000 试到 999，总有一个对的。

### 适用场景

- 数据范围很小（n ≤ 20 或更小）
- 没有更好的算法（或者没想到）
- 用来对拍验证正解

### 常见枚举技巧

#### 1. 普通循环枚举

直接 for 循环遍历所有可能。

```python
# 求 1~n 中所有能被 3 整除的数
n = 100
ans = []
for i in range(1, n + 1):
    if i % 3 == 0:
        ans.append(i)
```

#### 2. 嵌套枚举（多重循环）

枚举所有组合，比如两数之和。

```python
# 数组中找两个数的和为 target
def two_sum(arr, target):
    n = len(arr)
    for i in range(n):
        for j in range(i + 1, n):   # j 从 i+1 开始，避免重复
            if arr[i] + arr[j] == target:
                return (i, j)
    return None
```

#### 3. 枚举子集（二进制枚举）

对于 n 个元素，每个元素选或不选，共 2ⁿ 种状态。用位运算枚举。

```python
def enumerate_subset(arr):
    """枚举 arr 的所有非空子集"""
    n = len(arr)
    res = []
    for mask in range(1, 1 << n):            # 1 ~ 2ⁿ-1
        subset = []
        for i in range(n):
            if mask >> i & 1:                # 第 i 位为 1 表示选
                subset.append(arr[i])
        res.append(subset)
    return res
```

#### 4. 枚举排列（全排列）

用 `itertools.permutations` 或自己写 DFS。

```python
import itertools

def enumerate_permutation(arr):
    """枚举 arr 的所有排列"""
    for perm in itertools.permutations(arr):
        print(perm)
```

## 三、模拟

### 大白话

模拟就是**题目怎么说，你就怎么写。** 不需要找规律，不需要巧妙的算法，纯按题意一步一步操作。

像照着菜谱做菜——上面写的每一步你都照着做，不用自己发明做法。

### 适用场景

- 题目描述了一个具体的过程或规则
- 数据范围允许一步步模拟
- 考察代码实现能力而非算法思维

### 模拟题的常见坑

1. **边界条件**：数组越界、循环结束条件
2. **死循环**：while 循环忘记更新变量
3. **输入格式**：多行输入、空格分隔、可能有多余空格
4. **大模拟的状态管理**：用类和函数拆分，不要全写在一个 main 里
5. **时间超限**：检查是不是有更快的模拟方式，不要真的按步数跑完

### 举个例子：约瑟夫问题

```python
def josephus(n, k):
    """
    n 个人围成一圈，从 1 开始报数，报到 k 的人出列
    返回最后剩下的人的编号
    模拟法 O(nk)，当 n 较大时需要用数学公式 O(n)
    """
    people = list(range(1, n + 1))
    idx = 0
    while len(people) > 1:
        idx = (idx + k - 1) % len(people)   # 报数到 k 的人
        people.pop(idx)                      # 出列
    return people[0]
```

### 另一例：日期模拟

```python
# 判断闰年
def is_leap(year):
    return year % 400 == 0 or (year % 4 == 0 and year % 100 != 0)

# 某月的天数
def days_in_month(year, month):
    days = [0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31]
    if month == 2 and is_leap(year):
        return 29
    return days[month]

# 给定年月日，算星期几（基姆拉尔森公式）
def day_of_week(y, m, d):
    """返回 0=周日, 1=周一, ..., 6=周六"""
    if m <= 2:           # 把 1、2 月看成上一年 13、14 月
        m += 12
        y -= 1
    return (d + 2*m + 3*(m+1)//5 + y + y//4 - y//100 + y//400 + 1) % 7
```

## 四、一个完整例子：验证哥德巴赫猜想

### 题目

验证 4~n 之间的所有偶数都能写成两个素数之和。

### 思路

1. 用筛法先生成素数表
2. 枚举偶数，找两个素数加起来等于它
3. 找到就继续，找不到就输出反例

```python
def sieve(n):
    """埃氏筛，返回 n 以内的素数列表"""
    is_prime = [True] * (n + 1)
    is_prime[0] = is_prime[1] = False
    for i in range(2, int(n ** 0.5) + 1):
        if is_prime[i]:
            for j in range(i * i, n + 1, i):
                is_prime[j] = False
    return [i for i in range(n + 1) if is_prime[i]]

def goldbach(n):
    """验证哥德巴赫猜想 4~n"""
    primes = sieve(n)
    prime_set = set(primes)           # 用集合 O(1) 查素数
    
    for x in range(4, n + 1, 2):     # 枚举偶数
        found = False
        for p in primes:
            if p > x // 2:           # 超过一半就不用找了
                break
            if (x - p) in prime_set: # 另一半也是素数
                found = True
                break
        if not found:
            return f"反例：{x}"
    
    return f"4~{n} 范围内哥德巴赫猜想成立"
```

这里的思路就是**枚举 + 预处理**——先用筛法（后面数论章节会细讲）列出素数，再枚举每个偶数找配对。

## 总结

- **复杂度**：知道数据范围能接受什么算法，不用盲猜
- **枚举**：暴力出奇迹，数据小的时候最管用
- **模拟**：别想复杂了，题目说啥你做啥

**一句话**：会算复杂度、会暴力枚举、会按题意模拟——这三样是写所有算法题的基础。
