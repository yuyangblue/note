## 一、二分查找 — "猜数字"的高级版

### 1.1 大白话引入

小时候玩过猜数字吗？A 心里想一个 1~100 的数，你每次猜一个数，A 告诉你"大了"或"小了"。最优策略是什么？

——每次猜**中间**那个数，一次排除掉一半。

这就是二分查找（Binary Search）。**在有序序列中，每次把搜索范围对半砍**，复杂度 O(log n)。100 个数最多猜 7 次，10 亿个数最多猜 30 次。

### 1.2 核心思想

二分的本质不是"在一个有序数组里找某个数"，而是：

> **在一个具有「单调性」的决策序列上，找到满足/不满足某个条件的「分界点」。**

什么意思？想象一条线，左边都是"不满足条件"的状态，右边都是"满足条件"的状态：

```
不满足 ······ | ······ 满足
             ↑
         分界点 ← 二分要找的就是它
```

只要你有一个判定函数 `check(mid)`，使得：
- `check(x) = false` 时，答案一定在左边（或右边）
- `check(x) = true` 时，答案一定在另一边

那就可以用二分。

### 1.3 两种模板 — 理解它们的区别比背更重要

**模板一：找左边界（第一个满足条件的）**

```cpp
// 在 [l, r] 上找第一个满足 check(x) 的位置
int l = 0, r = n - 1;
while (l < r) {
    int mid = l + (r - l) / 2;  // 靠左的中位数
    if (check(mid))
        r = mid;     // mid 可能是答案，保留它
    else
        l = mid + 1; // mid 不可能是答案，跳过它
}
// 退出时 l == r，就是第一个满足条件的位置
```

**模板二：找右边界（最后一个满足条件的）**

```cpp
// 在 [l, r] 上找最后一个满足 check(x) 的位置
int l = 0, r = n - 1;
while (l < r) {
    int mid = l + (r - l + 1) / 2;  // ⚠️ 靠右的中位数（防止死循环）
    if (check(mid))
        l = mid;     // mid 可能是答案，保留它
    else
        r = mid - 1; // mid 不可能是答案，跳过它
}
```

> 🧠 **关键理解**：
> - 模板一 `mid = (l+r)/2`（下取整），`l = mid+1`，`r = mid` → 找第一个
> - 模板二 `mid = (l+r+1)/2`（上取整），`l = mid`，`r = mid-1` → 找最后一个
> - **什么时候上取整？** 当 `l = mid` 时，必须上取整，否则 `l=3, r=4` 时 `mid=3` 会死循环！
> - 不用死记，画个区间想一下：l 和 r 相邻时，mid 取哪个才不会无限循环？

### 1.4 实数二分

整数二分需要小心边界，但**实数二分不用**——因为实数没有"挨着"的概念。

```cpp
// 二分答案，精度要求 1e-6
double l = 0, r = 1e9;
while (r - l > 1e-7) {  // 一般比要求精度多两位
    double mid = (l + r) / 2;
    if (check(mid))
        r = mid;
    else
        l = mid;
}
// 也可以固定迭代 100 次（更稳妥，不怕精度要求变化）
for (int i = 0; i < 100; i++) {
    double mid = (l + r) / 2;
    if (check(mid)) r = mid;
    else l = mid;
}
```

**为什么实数二分不用操心死循环？** 因为 l 和 r 是浮点数，每次区间减半，永远不会卡在相邻两个整数上下不来。

## 二、二分答案 — 最常见的应用

### 2.1 什么时候用二分答案？

如果你碰到这种问题：
> "求 **最大值最小**" 或 "**最小值最大**"

或者换个说法：
> "x 可行吗？"比"x 是多少？"好回答

那就该用二分答案。

**典型场景**：
- 把 n 个数分成 k 段，每段和的最大值最小 → 猜一个最大和，检查能不能分成 k 段
- 在一条直线上放 m 个点，使最近距离最大 → 猜一个最近距离，检查能不能放 m 个点
- 切木头/分巧克力，每块长度一样且尽可能大 → 猜一个长度，检查能切出几块

### 2.2 二分答案的标准流程

```
1. 确定答案的范围 [L, R]
2. while (l < r) 或 for (迭代):
   a. mid = (l + r) / 2
   b. 写一个 check(mid)，回答"mid 作为答案是否可行？"
   c. 根据 check 结果缩小范围
3. 输出 l (或 r，退出时相等)
```

**核心技巧**：把 check 函数写得越简单越好。check 是"验证答案是否可行"，不是"直接求答案"。

### 2.3 例题启发：切绳子

> 有 n 条绳子，长度分别为 L[i]。要切成 k 条长度相等的绳子（可以浪费），求每段最长能有多长？

**思路**：
- 如果答案是 x，能切出的段数 = sum(L[i] / x)（下取整）
- 这个数关于 x 是单调递减的：x 越大，能切的段数越少
- 因此二分 x，检查能不能切出至少 k 段

```cpp
#include <bits/stdc++.h>
using namespace std;

const int N = 10005;
double L[N];
int n, k;

// check(x)：每段长度为 x 时，能不能切出至少 k 段？
bool check(double x) {
    int cnt = 0;
    for (int i = 0; i < n; i++) {
        cnt += (int)(L[i] / x);  // 每条绳子能切出几段
    }
    return cnt >= k;  // 切够了就可行
}

int main() {
    cin >> n >> k;
    double l = 0, r = 0;
    for (int i = 0; i < n; i++) {
        cin >> L[i];
        r = max(r, L[i]);  // 上界：最长的那条绳子
    }

    // 实数二分，迭代 100 次保证精度
    for (int i = 0; i < 100; i++) {
        double mid = (l + r) / 2;
        if (check(mid))
            l = mid;      // 能切出 k 段，试试更长的
        else
            r = mid;      // 切不出 k 段，需要更短
    }

    cout << fixed << setprecision(2) << l << endl;
    return 0;
}
```

> 💡 **启发**：check 函数只回答"行不行"，不回答"最优是多少"。把"求最优"拆成"猜答案 + 验证"两个步骤，往往比直接求简单得多。

## 三、二分查找在有序数组中的应用

### 3.1 在有序数组里找 target

最基础的二分，熟了之后可以闭着眼写：

```cpp
#include <bits/stdc++.h>
using namespace std;

// 标准二分查找：在有序数组 a 中找 target，返回下标，找不到返回 -1
int binarySearch(vector<int>& a, int target) {
    int l = 0, r = (int)a.size() - 1;
    while (l <= r) {  // 注意这里是 <=，因为区间 [l, r] 有效
        int mid = l + (r - l) / 2;  // 防溢出写法
        if (a[mid] == target)
            return mid;
        else if (a[mid] < target)
            l = mid + 1;   // target 在右边，缩小左边界
        else
            r = mid - 1;   // target 在左边，缩小右边界
    }
    return -1;
}
```

> **为什么不用 `(l+r)/2`？** 当 l 和 r 都很大时，l + r 可能溢出 int。`l + (r-l)/2` 等价但安全。

### 3.2 C++ STL 中的二分三剑客

写算法竞赛时，STL 提供的二分函数比你手写更快更稳：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    vector<int> a = {1, 3, 5, 5, 5, 7, 9};

    // 1. lower_bound: 第一个 >= target 的位置
    auto it1 = lower_bound(a.begin(), a.end(), 5);
    // *it1 == 5, 下标是 2（第一个 >= 5 的元素）
    int idx1 = it1 - a.begin();  // 2

    // 2. upper_bound: 第一个 > target 的位置
    auto it2 = upper_bound(a.begin(), a.end(), 5);
    // *it2 == 7, 下标是 5（第一个 > 5 的元素）
    int idx2 = it2 - a.begin();  // 5

    // 3. 利用 lower_bound + upper_bound 求 target 出现的次数
    int cnt = upper_bound(a.begin(), a.end(), 5)
            - lower_bound(a.begin(), a.end(), 5);
    // cnt == 3（5 出现了 3 次）

    // 4. binary_search: 只判断存在与否，返回 bool
    bool exists = binary_search(a.begin(), a.end(), 4);
    // exists == false

    return 0;
}
```

**记忆口诀**：
- `lower_bound` = 下边界（第一个 ≥ 的）
- `upper_bound` = 上边界（第一个 > 的）
- 开区间 `[lower, upper)` 内全是 target

> **竞赛实用技巧**：需要找"第一个 > target"但不等于 target 的元素时，用 `upper_bound` 而不是自己写二分。需要找"最后一个 < target"的元素时，`lower_bound` 的前一个就是。

## 四、三分法 — 二分的好兄弟

### 4.1 什么时候用三分？

二分解决的是**单调**问题（一个方向一直好，另一个方向一直不好）。但有些函数不是单调的，而是**单峰**的：

- 先增后减（凸函数，有个最大值）
- 先减后增（凹函数，有个最小值）

对于这样的函数，用**三分法（Ternary Search）**来找极值点。

### 4.2 三分的思路

在区间 `[l, r]` 上取两个点 `m1` 和 `m2`（三等分或中间偏左/偏右）：

```
l -- m1 -- m2 -- r
```

比较 `f(m1)` 和 `f(m2)`：
- 如果 `f(m1) < f(m2)`：最高点在 m1 右侧 → `l = m1`
- 如果 `f(m1) > f(m2)`：最高点在 m2 左侧 → `r = m2`
- 如果 `f(m1) == f(m2)`：极值点在中间 → 两边都可以缩

每次缩掉约 1/3 的区间，复杂度 O(log_{3/2} n)。

### 4.3 三分求最大值模板

```cpp
#include <bits/stdc++.h>
using namespace std;

// 要求最大值的函数（先增后减的单峰函数）
double f(double x) {
    // 例如：-x^2 + 2x + 1，顶点在 x=1
    return -x * x + 2 * x + 1;
}

int main() {
    double l = -10, r = 10;

    // 实数三分，迭代 100 次
    for (int i = 0; i < 100; i++) {
        double m1 = l + (r - l) / 3;        // 左边 1/3 处
        double m2 = r - (r - l) / 3;        // 右边 1/3 处
        if (f(m1) < f(m2))
            l = m1;   // f(m1) 更小，最大值在 m1 右边
        else
            r = m2;   // f(m2) 更小，最大值在 m2 左边
    }

    cout << fixed << setprecision(6) << l << " " << f(l) << endl;
    return 0;
}
```

### 4.4 可视化理解三分的收敛过程

```
初始:  [l                r]
第1步:  l---m1----m2----r    f(m1) < f(m2) → 丢弃左边
第2步:       [m1         r]
             l--m1--m2--r    f(m1) > f(m2) → 丢弃右边
第3步:       [l---m1---]
                 ...
最终:          ↑ 极值点
```

> 💡 **三分的直觉**：就像爬山，你左右各看一眼，哪边更高就往哪边挪。"看看两边、走向高处"重复足够多次，就能到山顶。

### 4.5 三分的整数版

整数三分和实数略微不同，因为相邻整数之间没有"三等分点"：

```cpp
// 整数三分求最大值（凸函数）
int l = 0, r = n - 1;
while (r - l > 2) {  // 区间长度 > 2 时才缩
    int m1 = l + (r - l) / 3;
    int m2 = r - (r - l) / 3;
    if (f(m1) < f(m2))
        l = m1;
    else
        r = m2;
}
// 最后区间很小了，在 [l, r] 里暴力找最大值
int ans = f(l);
for (int i = l + 1; i <= r; i++) ans = max(ans, f(i));
```

> **为什么最后要暴力？** 整数三分缩到 `r - l <= 2` 时不能再缩了（三等分点无法区分），但最多只剩 3 个数，暴力检查一下就好。

## 五、经典例题精讲

### 5.1 洛谷 P1824 进击的奶牛 — "最小值最大"的经典

> 在一条直线上有 n 个牛棚位置，要放 m 头牛（m ≤ n），使牛之间的最近距离尽可能大。

**思考过程**：

1. **"最小值最大"** → 标准的二分答案信号
2. 猜一个最近距离 `x`，检查能不能放 m 头牛
3. check(x)：第一头牛放最左边，然后每头牛放在"距离上一头 ≥ x"的最左边的牛棚
4. 能放下 m 头 → x 可行，试试更大；放不下 → x 太大，试试更小

```cpp
#include <bits/stdc++.h>
using namespace std;

const int N = 100005;
int a[N], n, m;

// check(x)：最近距离为 x 时，能不能放下 m 头牛？
bool check(int x) {
    int cnt = 1;          // 第一头牛放在 a[0]
    int last = a[0];      // 上一头牛的位置
    for (int i = 1; i < n; i++) {
        if (a[i] - last >= x) {
            cnt++;        // a[i] 这个位置可以放牛
            last = a[i];  // 更新上一头牛的位置
        }
    }
    return cnt >= m;      // 能放 m 头及以上就可行
}

int main() {
    cin >> n >> m;
    for (int i = 0; i < n; i++) cin >> a[i];
    sort(a, a + n);  // 牛棚位置不一定有序，先排序

    // 二分答案：最近距离的范围 [1, 最大间距]
    int l = 1, r = a[n - 1] - a[0];
    while (l < r) {
        // 这里用模板二（找右边界）：最后一个可行的 x
        int mid = l + (r - l + 1) / 2;
        if (check(mid))
            l = mid;      // 可行，试试更大
        else
            r = mid - 1;  // 不可行，需要更小
    }
    cout << l << endl;
    return 0;
}
```

> **关键启发**：二分答案的关键是 **check 函数的设计**。check 越简单、越直观，你的代码就越不容易出错。这里的 check 就是贪心放牛——贪心在 check 里非常常见。

### 5.2 洛谷 P2678 跳石头 — "移除石头，使最小跳跃距离最大"

> 一条河从起点到终点有若干块石头，可以移走 m 块，求移走后相邻石头间最小距离的最大值。

**思路**：
- 跟放牛几乎一样，只不过方向反了：牛棚 → 石头，放牛 → 判断是否要移除
- check(x)：最小跳跃距离为 x 时，需要移走多少块石头？

```cpp
#include <bits/stdc++.h>
using namespace std;

const int N = 50005;
int d[N], L, n, m;

// check(x)：最小跳跃距离为 x，需要移走几块石头？
bool check(int x) {
    int last = 0;      // 起点位置，相当于上一块没被移走的石头
    int removed = 0;   // 已移走的石头数量
    for (int i = 0; i < n; i++) {
        if (d[i] - last < x) {
            // 这块石头不能留——留着会导致跳跃距离小于 x
            removed++;
        } else {
            last = d[i];  // 保留这块石头，作为下一次跳跃的起点
        }
    }
    // 还要检查最后一步到终点的距离
    if (L - last < x) removed++;
    return removed <= m;  // 移走的石头不超过 m 块就可行
}

int main() {
    cin >> L >> n >> m;
    for (int i = 0; i < n; i++) cin >> d[i];
    sort(d, d + n);  // 石头位置升序

    // 二分答案
    int l = 1, r = L;
    while (l < r) {
        int mid = l + (r - l + 1) / 2;
        if (check(mid))
            l = mid;
        else
            r = mid - 1;
    }
    cout << l << endl;
    return 0;
}
```

> 💡 **放牛 vs 跳石头**：两个问题看似不同，但 check 函数的思路一模一样——**贪心地从左到右扫描**。放牛是统计能放几个，跳石头是统计要移除几个。这就是二分答案题的共性：问题不同，check 的套路相似。

### 5.3 洛谷 P3382 三分法模板

完整的三分模板题：

```cpp
#include <bits/stdc++.h>
using namespace std;

const int N = 15;
double a[N];   // 多项式系数
int n;

// 计算 f(x) = a[0]*x^n + a[1]*x^{n-1} + ... + a[n]
double f(double x) {
    double res = 0;
    for (int i = 0; i <= n; i++) {
        res = res * x + a[i];  // 秦九韶算法：高效且精确
    }
    return res;
}

int main() {
    double l, r;
    cin >> n >> l >> r;
    for (int i = 0; i <= n; i++) cin >> a[i];

    // 三分法求单峰函数最大值
    for (int i = 0; i < 100; i++) {
        double m1 = l + (r - l) / 3;
        double m2 = r - (r - l) / 3;
        if (f(m1) < f(m2))
            l = m1;   // 最大值在 m1 右侧
        else
            r = m2;   // 最大值在 m2 左侧
    }
    cout << fixed << setprecision(5) << l << endl;
    return 0;
}
```

> **秦九韶算法**：就是把多项式 `a[0]x³ + a[1]x² + a[2]x + a[3]` 写成 `((a[0]*x + a[1])*x + a[2])*x + a[3]`，省去每次算幂的开销。

## 六、易错点 & 避坑指南

### 6.1 整数二分的死循环

```cpp
// ❌ 错误写法：会导致死循环
int l = 0, r = n - 1;
while (l < r) {
    int mid = (l + r) / 2;         // 下取整
    if (check(mid)) l = mid;       // l 没有变化时无限循环！
    else r = mid - 1;
}
// 当 l=3, r=4, mid=(3+4)/2=3, check(3)为真后 l=3, 死循环！
```

**修复**：当 `l = mid` 时，mid 必须上取整 `(l+r+1)/2`。

### 6.2 实数的精度问题

- 二分到 `r - l > eps` 时，eps 至少比要求精度小两个数量级（要求 1e-4 就设 1e-6）
- 或者直接用 `for (int i = 0; i < 100; i++)` 迭代固定次数——**100 次绝对够**，区间长度缩到最初的 `2^(-100)`，比 double 精度还高

### 6.3 什么时候不能用二分？

- 决策没有单调性 → 二分失效
- check 函数本身比直接求答案还难写 → 换个思路
- check 是 NP-hard 的 → 二分也救不了你

### 6.4 三分的局限性

- 函数必须**严格单峰**（只有一个极值点）
- 多峰函数不能用三分（凹凸交替出现时三分会找错极值点）
- 整数三分注意最后暴力收尾

## 七、总结

### 二分一句话

> **有序就能二分，单调就能二分答案。**

看到"最大值最小"或"最小值最大"，第一反应就是二分答案。先写出 check，再套模板，最后调整边界。

### 三分一句话

> **单峰函数求极值，左右各看三分之一，哪边高往哪边走。**

### 快速判断

```
问题特征                    → 用什么
─────────────────────────────────────────
有序数组里找某个值           → 标准二分 O(log n)
"最大值最小" / "最小值最大"  → 二分答案 O(log range × check)
实数范围内求方程的解         → 实数二分 / 牛顿迭代
单峰函数求最值              → 三分法 O(log n)
单调但 check 不好写          → 先别二分，想清楚 check 怎么设计
```
