# 第三章 区间 DP

> 区间 DP 是在一段连续区间上做决策，大区间的解由小区间合并得来。

---

## 一、石子合并（环形）

### 大白话

N 堆石子排成**环形**，每次选相邻两堆合并，合并的代价是两堆的重量和。问合并成一堆的最小总代价。

像排队打饭——两个人先合并成一个人去打，人越合并越少，每次合并消耗的是两人的饭量之和，怎么排能让总消耗最小。

### 核心思路

**状态定义**：`dp[i][j]` = 合并区间 [i, j] 的石子的最小代价

**转移方程**：
```
dp[i][j] = min(dp[i][k] + dp[k+1][j]) + sum(i, j)   (i ≤ k < j)
```
其中 sum(i, j) 是区间 [i, j] 的石子和，用前缀和 O(1) 求。

**环形处理**：把数组复制一份拼在后面（长度 2N），枚举所有长度为 N 的区间取最小值。

**枚举顺序**：先枚举区间长度 len，再枚举左端点 i，最后枚举分割点 k。

为什么要按长度枚举？因为大区间依赖小区间，按长度递增保证小区间先算好。

### 代码

```python
def merge_stones(arr):
    """
    环形石子合并
    arr: 每堆石子重量
    返回最小合并代价
    """
    n = len(arr)
    # 环形 -> 复制一倍
    a = arr + arr
    m = 2 * n
    # 前缀和
    prefix = [0] * (m + 1)
    for i in range(m):
        prefix[i + 1] = prefix[i] + a[i]
    
    # dp[i][j]：合并 [i, j] 的最小代价
    INF = float('inf')
    dp = [[0] * m for _ in range(m)]
    
    # 按长度枚举
    for length in range(2, n + 1):            # 区间长度从 2 开始
        for i in range(m - length + 1):       # 左端点
            j = i + length - 1                # 右端点
            dp[i][j] = INF
            for k in range(i, j):             # 分割点
                cost = dp[i][k] + dp[k + 1][j] + (prefix[j + 1] - prefix[i])
                dp[i][j] = min(dp[i][j], cost)
    
    # 取所有长度为 n 的区间最小值
    ans = min(dp[i][i + n - 1] for i in range(n))
    return ans
```

## 二、括号匹配

### 大白话

给一个由 `()`、`[]`、`{}` 组成的字符串，问最长合法子序列（或最少加几个括号使其合法，或能否完全匹配）。

合法括号序列的定义：
- 空串合法
- `A` 合法则 `(A)`、`[A]`、`{A}` 合法
- `A` 和 `B` 合法则 `AB` 合法

像写代码检查括号是否配对——你写的 `[{()}]` 就是对的，`[(])` 就错了。

### 核心思路

**状态定义**：`dp[i][j]` = 区间 [i, j] 的最长合法括号子序列长度（或最少需要添加的括号数）

**转移**（以最长合法子序列为例）：
```
如果 s[i] 和 s[j] 匹配： dp[i][j] = dp[i+1][j-1] + 2
然后：                    dp[i][j] = max(dp[i][j], dp[i][k] + dp[k+1][j])   (i ≤ k < j)
```

两种情况：
1. 两端的括号匹配了 → 里面的 + 外面的一对
2. 分成两段合法序列拼起来（取最大值）

### 代码

```python
def longest_valid_brackets(s):
    """
    最长合法括号子序列长度
    子序列可以不连续
    """
    n = len(s)
    dp = [[0] * n for _ in range(n)]
    
    def match(a, b):
        return (a == '(' and b == ')') or \
               (a == '[' and b == ']') or \
               (a == '{' and b == '}')
    
    for length in range(2, n + 1):           # 区间长度
        for i in range(n - length + 1):
            j = i + length - 1
            if match(s[i], s[j]):
                inner = dp[i + 1][j - 1] if i + 1 <= j - 1 else 0
                dp[i][j] = inner + 2        # 两端匹配 + 里面匹配的长度
            # 分割点枚举
            for k in range(i, j):
                dp[i][j] = max(dp[i][j], dp[i][k] + dp[k + 1][j])
    
    return dp[0][n - 1]
```

## 三、四边形不等式优化（选读）

### 大白话

有些区间 DP 的转移中，分割点 k 具有**单调性**——区间越大，最优分割点只会右移不会左移。

利用这个性质可以砍掉一层循环，从 O(n³) 降到 O(n²)。竞赛中如果 n ≤ 500 可以不优化，n 到几千就需要了。

**关键**：记录 `opt[i][j]` 为区间 [i, j] 的最优分割点，下次枚举 k 的范围缩小到 `[opt[i][j-1], opt[i+1][j]]`。

## 总结

| 题目类型 | 状态 | 转移思路 | 复杂度 |
|----------|------|----------|--------|
| 石子合并 | dp[i][j] 最小代价 | 枚举分界点 k | O(n³) |
| 括号匹配 | dp[i][j] 最长合法 | 两端匹配 + 分段取 max | O(n³) |
| 四边形优化 | dp[i][j] 优化版 | 缩小 k 枚举范围 | O(n²) |

**区间 DP 三板斧**：
1. 按长度枚举（大区间依赖小区间）
2. 枚举左端点，确定右端点
3. 枚举分割点 / 判断两端关系
