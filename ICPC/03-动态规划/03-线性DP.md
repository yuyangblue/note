# 第二章 线性 DP

> 线性 DP 就是在一维/二维数组上按顺序递推，每个状态依赖前面若干个状态。

---

## 一、LIS（最长上升子序列）

### 大白话

在一个序列里找一个**递增**的子序列（可以不连续），使它的长度最长。

比如 `[2, 1, 5, 3, 6]` → 上升子序列有 `[2, 5, 6]`、`[1, 3, 6]`，最长长度为 3。

像选朋友圈照片——从一堆照片里挑一组，后一张要比前一张好看，挑得越多越好。

### 核心思路

**状态定义**：`dp[i]` = 以第 i 个元素结尾的 LIS 长度

**转移方程**：
```
dp[i] = max(dp[j] + 1)  (j < i 且 a[j] < a[i])
dp[i] 至少为 1（自己单独成一个序列）
```

**O(n²) 朴素版** vs **O(n log n) 贪心+二分优化版**：
- 朴素好理解，但 n 上万就不行了
- 优化版维护一个 `tails` 数组：`tails[k]` = 长度为 k+1 的上升子序列的**最小结尾元素**

### 代码

**O(n²) 朴素版**：

```python
def LIS_n2(arr):
    """最长上升子序列 O(n²)"""
    n = len(arr)
    dp = [1] * n               # 至少包含自己
    for i in range(n):
        for j in range(i):
            if arr[j] < arr[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)
```

**O(n log n) 优化版**：

```python
import bisect

def LIS_optimized(arr):
    """
    最长上升子序列 O(n log n)
    用二分查找维护 tails 数组
    """
    tails = []                 # tails[k] = 长度为 k+1 的 LIS 的最小末尾
    for x in arr:
        pos = bisect.bisect_left(tails, x)   # 找第一个 >= x 的位置
        if pos == len(tails):
            tails.append(x)                  # x 比所有末尾都大，延长 LIS
        else:
            tails[pos] = x                   # 替换，让末尾更小
    return len(tails)
```

## 二、LCS（最长公共子序列）

### 大白话

两个字符串，找它们都有的**最长子序列**（可以不连续）。

比如 `"abcde"` 和 `"ace"` → LCS 是 `"ace"`，长度 3。

像两个人点了不同的外卖，但都爱吃的那几道菜的清单——越找越长越好。

### 核心思路

**状态定义**：`dp[i][j]` = 字符串 A 的前 i 个字符和字符串 B 的前 j 个字符的 LCS 长度

**转移方程**：
```
如果 A[i-1] == B[j-1]： dp[i][j] = dp[i-1][j-1] + 1
否则：                 dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

**关键逻辑**：
- 字符相等 → 在前一个状态基础上 +1（匹配上了）
- 不等 → 要么跳 A 的字符，要么跳 B 的字符，取较大值

### 代码

```python
def LCS(A, B):
    """最长公共子序列"""
    n, m = len(A), len(B)
    dp = [[0] * (m + 1) for _ in range(n + 1)]
    
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if A[i - 1] == B[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1   # 匹配了
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])  # 跳一个
    
    return dp[n][m]
```

## 三、编辑距离（Levenshtein Distance）

### 大白话

把一个字符串变成另一个字符串，最少需要多少次操作。操作有三种：插入、删除、替换。

比如 `"horse"` → `"ros"`，需要 3 步：
```
horse → rorse (替换 h→r)
rorse → rose  (删除 r)
rose  → ros   (删除 e)
```

像打字校对——把打错的句子改成正确的，最少敲几次退格/插入/替换。

### 核心思路

**状态定义**：`dp[i][j]` = 把 A 的前 i 个字符变成 B 的前 j 个字符的最少操作数

**转移方程**：
```
如果 A[i-1] == B[j-1]： dp[i][j] = dp[i-1][j-1]      （不用操作）
否则：                   dp[i][j] = min(
    dp[i-1][j] + 1,      删除 A[i-1]
    dp[i][j-1] + 1,      在 A 末尾插入 B[j-1]
    dp[i-1][j-1] + 1     将 A[i-1] 替换为 B[j-1]
)
```

**初始化**：dp[0][j] = j（插入 j 次），dp[i][0] = i（删除 i 次）

### 代码

```python
def edit_distance(A, B):
    """
    编辑距离（最少编辑操作数）
    操作：插入、删除、替换
    """
    n, m = len(A), len(B)
    dp = [[0] * (m + 1) for _ in range(n + 1)]
    
    # 初始化边界
    for i in range(n + 1):
        dp[i][0] = i        # 全删除
    for j in range(m + 1):
        dp[0][j] = j        # 全插入
    
    # DP
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if A[i - 1] == B[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]         # 一样，不用操作
            else:
                dp[i][j] = min(
                    dp[i - 1][j] + 1,                # 删除 A[i-1]
                    dp[i][j - 1] + 1,                # 插入 B[j-1]
                    dp[i - 1][j - 1] + 1             # 替换
                )
    
    return dp[n][m]
```

## 总结

| 类型 | 定义 | 复杂度 | 关键点 |
|------|------|--------|--------|
| LIS | 最长上升子序列 | O(n²) / O(n log n) | 优化用二分+tails |
| LCS | 最长公共子序列 | O(nm) | 相等斜 +1，不等取 max |
| 编辑距离 | 最少编辑操作 | O(nm) | 三种操作取 min |

**一句话记忆**：
- LIS 往左比，LCS 对角走，编辑距离三个方向取最小
- 排列的 LCS 可以转 LIS 优化
