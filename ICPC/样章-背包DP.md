## 一、0/1 背包

### 1.1 问题模型

> 有 $N$ 件物品和一个容量为 $V$ 的背包。每件物品**最多选一次**（拿或者不拿）。第 $i$ 件物品的体积是 $w_i$，价值是 $v_i$。问背包能装下的最大总价值是多少？

这就是所谓的 **"选或不选"** 问题。每一件物品，我们面临且仅面临两个选择：**拿**或**不拿**。

### 1.2 状态定义：为什么是 `dp[j]` 而不是 `dp[i][j]`？

先思考二维版本，我们更自然：

$$
dp[i][j] = \text{考虑前 i 件物品，容量为 j 时能装的最大价值}
$$

转移方程也很好理解：

- **不拿第 i 件物品：** $dp[i][j] = dp[i-1][j]$
- **拿第 i 件物品（前提 j ≥ w_i）：** $dp[i][j] = dp[i-1][j - w_i] + v_i$

两者取 max。

但是，二维数组需要 $O(N \times V)$ 的空间，当 $N=10^5$ 时直接爆炸。观察发现，第 $i$ 行只依赖第 $i-1$ 行，所以我们可以**滚动数组**，只保留一行：

$$
dp[j] = \text{当前"考虑过的物品"情况下，容量为 j 时能装的最大价值}
$$

**为什么可以省掉第一维？** 因为内层循环倒序保证了每个物品**只被考虑一次**，天然替我们做到了"第 i 轮看到的是前 i 件物品的结果"。下面会详细解释。

### 1.3 转移方程

$$
dp[j] = \max(dp[j],\; dp[j - w] + v) \quad (j \text{ 从 V 倒序遍历到 } w)
$$

### 1.4 核心问题：为什么内层循环要**倒序**？

这是最经典的面试/笔试问题。让我们用一个具体的例子来感受。

> **情景：** 容量 $V=4$，只有一件物品，体积 $w=2$，价值 $v=3$。
>
> 初始 `dp = [0, 0, 0, 0, 0]`

#### 如果正序枚举（错误的做法）：

```
j = 2 → dp[2] = max(dp[2], dp[0]+3) = max(0, 3) = 3
j = 3 → dp[3] = max(dp[3], dp[1]+3) = max(0, 3) = 3
j = 4 → dp[4] = max(dp[4], dp[2]+3) = max(0, 3+3) = 6  ← 出事了！
```

**问题出在哪里？** 当 $j=4$ 时，`dp[2]` 已经**在这一轮被更新成了 3**（代表已经选了这件物品）。于是 `dp[4] = dp[2] + 3 = 3 + 3 = 6`，等于**同一件物品选了两次**！这就是 0/1 背包中不允许的。

#### 用倒序枚举（正确的做法）：

```
j = 4 → dp[4] = max(dp[4], dp[2]+3) = max(0, 0+3) = 3  ← dp[2] 还是旧的 0
j = 3 → dp[3] = max(dp[3], dp[1]+3) = max(0, 0+3) = 3   ← dp[1] 还是旧的 0
j = 2 → dp[2] = max(dp[2], dp[0]+3) = max(0, 0+3) = 3   ← dp[0] 还是旧的 0
```

**倒序保证：** 计算 `dp[j]` 时，`dp[j-w]` 的值还没有被这一轮更新过，所以它还是**上一轮**（没选这件物品时）的数据。这样每件物品最多用一次。

**一句话总结：** 倒序 = 用"旧数据"更新 → 每件物品只用一次；正序 = 用"新数据"更新 → 允许重复用。

### 1.5 初始化：为什么 `dp = [0] * (V+1)`？

先想想 `dp[j]` 的含义：**容量为 j 的背包能装的最大价值**。

- 如果题目**没有要求"恰好装满"**，那么什么都不装价值就是 0，所以 `dp[0..V] = 0` 完全合理。
- 如果要求**恰好装满**（比如"凑出恰好 100 元"），初始化就要变了：
  - `dp[0] = 0`（容量 0 恰好装满，价值 0）
  - `dp[1..V] = -∞`（其他容量不可能恰好装满，用一个很小的负数表示"还没达到"）
  - 最后检查 `dp[V]` 是否为负数，如果还是负数说明无解。

**所以默认 `dp = [0] * (V+1)` 适用于"不超过容量"的最值问题。**

### 1.6 时间复杂度

两重循环：外层 $N$ 件物品 × 内层 $V$ 容量 → $O(N \times V)$。

当 $N \times V$ 约 $10^7$ 级别时能在一秒内跑完；超过 $10^8$ 就要考虑优化或换算法了。

### 1.7 模板代码（逐行注释）

```python
def zero_one_knapsack(N, V, weights, values):
    """
    0/1 背包通用模板
    :param N: 物品数量
    :param V: 背包容量
    :param weights: 物品体积列表，长度 N
    :param values:  物品价值列表，长度 N
    :return: 最大总价值
    """
    # dp[j] 表示 "当前考虑过的物品" 中，容量 j 能装的最大价值
    dp = [0] * (V + 1)

    # 逐个考虑每件物品
    for i in range(N):
        w = weights[i]   # 当前物品的体积
        v = values[i]    # 当前物品的价值

        # 倒序枚举容量，保证每个物品只选一次
        # 只枚举到 w，因为 j < w 时装不下，没必要更新
        for j in range(V, w - 1, -1):
            # 拿或不拿，取较大值
            # dp[j]（不拿） vs dp[j-w] + v（拿）
            dp[j] = max(dp[j], dp[j - w] + v)

    # dp[V] 就是容量 V 能装的最大价值
    return dp[V]
```

### 1.8 例题

#### P1048 [NOIP2005 普及组] 采药

> **题意：** 采药人有限的时间 $T$，有 $M$ 株药，每株采需要时间 $t_i$，价值 $v_i$。每株只能采一次。问最大总价值。
>
> **分析：** 这就是裸的 0/1 背包。时间 $T$ 就是容量，每株药就是一个物品。

```python
# 读入：T = 总时间（容量），M = 药的数量
T, M = map(int, input().split())

# dp[j]: 花 j 时间能获得的最大价值
dp = [0] * (T + 1)

# 遍历每株药
for _ in range(M):
    t, v = map(int, input().split())  # 采这株药需要 t 时间，价值 v

    # 倒序容量，确保一株药只考虑一次
    for j in range(T, t - 1, -1):
        dp[j] = max(dp[j], dp[j - t] + v)

# 输出总时间 T 内的最大价值
print(dp[T])
```

#### P1060 [NOIP2006 普及组] 开心的金明

> **题意：** 金明有 $N$ 元预算，想买 $m$ 件物品。每件价格 $v$，重要度 $p$。他定义价值 = $v \times p$（价格×重要度）。求最大总价值。
>
> **分析：** 标准 0/1 背包，只是物品的价值变成了 $v \times p$。

```python
# 读入：N = 总预算（容量），m = 物品数
N, m = map(int, input().split())

# dp[j]: 花 j 元能获得的最大价值
dp = [0] * (N + 1)

# 遍历每件物品
for _ in range(m):
    v, p = map(int, input().split())  # 价格 v，重要度 p

    # 价值 = 价格 × 重要度
    value = v * p

    # 倒序背包容量
    for j in range(N, v - 1, -1):
        dp[j] = max(dp[j], dp[j - v] + value)

print(dp[N])
```

## 二、完全背包

### 2.1 问题模型

> 有 $N$ 种物品和一个容量为 $V$ 的背包。**每种物品可以拿无限次**。第 $i$ 种物品的体积 $w_i$，价值 $v_i$。问最大总价值。

完全背包和 0/1 背包唯一的区别就是：**0/1 一件只能拿一次，完全背包可以拿无数次**。

### 2.2 转移方程

$$
dp[j] = \max(dp[j],\; dp[j - w] + v) \quad (j \text{ 从 w 正序遍历到 V})
$$

和 0/1 背包长得一模一样，唯一的区别是：**内层循环变成正序了**。

### 2.3 为什么正序枚举就允许无限次选？

还记得 0/1 背包我们说过**正序会导致同一件物品被选多次**吗？在完全背包里，我们就是要这个效果！

用之前的例子再看一次：

> 容量 $V=4$，一件物品 $w=2, v=3$，可以无限次拿。

正序枚举：

```
j = 2 → dp[2] = max(0, 0+3) = 3          ← 拿第 1 次
j = 3 → dp[3] = max(0, 0+3) = 3          ← 装不下第二件
j = 4 → dp[4] = max(0, 3+3) = 6          ← dp[2] 已经是拿过 1 次的值，再拿 1 次 = 拿 2 次
```

**流程就是：** 计算 `dp[4]` 时，`dp[2]` 已经被本轮更新成了"已选 1 件"的状态 → `dp[2]+v` = "再选 1 件" = 选 2 件。这个过程可以持续下去，相当于允许了无限次选取。

**数学理解：** 正序枚举时，`dp[j]` 的更新依赖于本轮已经更新过的 `dp[j-w]`，这就相当于允许同一物品被**叠加**到之前已经选过的状态上。

### 2.4 另一种理解方式

完全背包可以这样想：

要装满容量 $j$，考虑**最后一件物品**选了什么。如果最后选了第 $i$ 种物品，那么状态就从 $j-w_i$ 转移过来。而 $j-w_i$ 这个子问题**仍然可以选择第 $i$ 种物品**（因为无限次），所以内层正序天然支持这种"重复选"的逻辑。

### 2.5 时间复杂度

$O(N \times V)$，和 0/1 一样。

### 2.6 模板代码（逐行注释）

```python
def complete_knapsack(N, V, weights, values):
    """
    完全背包通用模板
    :param N: 物品种类数
    :param V: 背包容量
    :param weights: 每种物品的体积，长度 N
    :param values:  每种物品的价值，长度 N
    :return: 最大总价值
    """
    # dp[j]: 容量 j 能装的最大价值
    dp = [0] * (V + 1)

    # 遍历每种物品
    for i in range(N):
        w = weights[i]   # 当前物品体积
        v = values[i]    # 当前物品价值

        # 正序枚举容量！和 0/1 背包唯一的不同
        # 正序允许同一种物品被反复选
        for j in range(w, V + 1):
            dp[j] = max(dp[j], dp[j - w] + v)

    return dp[V]
```

### 2.7 例题

#### P1616 疯狂的采药

> **题意：** 采药人有 $t$ 时间，有 $m$ 种药，每种无限供应。采第 $i$ 种需要 $a_i$ 时间，价值 $b_i$。求最大总价值。
>
> **分析：** 和 P1048 采药的区别仅仅在于——这里每种药无限采 → 完全背包。

```python
# 读入：t = 总时间（容量），m = 药品种类数
t, m = map(int, input().split())

# dp[j]: 花 j 时间能获得的最大价值
dp = [0] * (t + 1)

# 遍历每种药
for _ in range(m):
    a, b = map(int, input().split())  # 时间 a，价值 b

    # 正序枚举！每种药可以无限采
    for j in range(a, t + 1):
        dp[j] = max(dp[j], dp[j - a] + b)

print(dp[t])
```

## 三、多重背包（二进制优化）

### 3.1 问题模型

> 有 $N$ 种物品和一个容量为 $V$ 的背包。第 $i$ 种物品有 **$c_i$ 件**（有限的），每件体积 $w_i$，价值 $v_i$。问最大总价值。

**和 0/1 的区别：** 0/1 是每件 1 个，多重背包是每件有 $c_i$ 个相同的。
**和完全背包的区别：** 完全背包无限个，多重背包有限个。

### 3.2 朴素思路（太慢了）

如果能想到的办法是：把 $c_i$ 件物品拆成 $c_i$ 个独立的物品，然后跑 0/1 背包。时间复杂度 $O(V \times \sum c_i)$，当 $c_i$ 很大时（比如 $10^5$ 件同种物品）就炸了。

**二进制优化就是用来解决这个问题的。**

### 3.3 二进制优化的核心思想

我们希望把 $c$ 件相同的物品，拆成少数几组，使得：

> 这几组物品做 0/1 背包时，**能组合出 $0 \sim c$ 之间的任意数量**。

最直接的拆法是拆成 **1, 2, 4, 8, ...**（2 的幂），最后剩一个余数。

比如 $c = 13$：

```
13 = 1 + 2 + 4 + 6
    ↑   ↑   ↑   ↑
    1   2   4   6（剩余）
```

拆成了 4 组，每组对应一个"打包物品"：

| 组 | 包含原物品个数 | 体积 | 价值 |
|----|--------------|------|------|
| 1  | 1 件 | 1w | 1v |
| 2  | 2 件 | 2w | 2v |
| 3  | 4 件 | 4w | 4v |
| 4  | 6 件 | 6w | 6v |

**为什么这 4 组能凑出 0~13 任意数量？**

- 用 1, 2, 4 可以凑出 0~7 任意数。
- 加上一个 6，就能凑出 6~13。
- 而 0~6 本身就能用 1,2,4 覆盖，所以 0~13 全覆盖。

### 3.4 数学证明

> 证明：任意正整数 $c$，可以拆分为：
> $$1, 2, 4, \dots, 2^{t}, \; r$$
> 其中 $t = \lfloor \log_2(c+1) \rfloor - 1$，$r = c - (2^{t+1} - 1)$，且 $0 \le r < 2^{t+1}$。
>
> 这组数能通过 0/1 选或不选，表示 $0 \sim c$ 的所有整数。

**证明思路：**

设前 $t+1$ 个数是 $1, 2, 4, \dots, 2^{t}$。这些数可以表示 $0 \sim 2^{t+1} - 1$ 的所有整数（二进制表示原理，每个 2 的幂次选或不选）。

再加上余数 $r$（$0 \le r < 2^{t+1}$，因为 $c < 2^{t+2}$）：

- $0 \sim r - 1$：直接用前 $t+1$ 个数表示（不选 $r$）。
- $r \sim 2^{t+1} - 1 + r$：选上 $r$，再从前 $t+1$ 个数中选合适的数量补上。
- 注意到 $2^{t+1} - 1 + r = c$，所以 $0 \sim c$ 全都能表示。

**证毕。** 🎉

### 3.5 复杂度分析

原来拆成 $c$ 件 → $O(V \times c)$。
优化后拆成 $\lceil \log_2 c \rceil$ 组 → $O(V \times \log c)$。

对于 $c = 10^5$，$\log_2 10^5 \approx 17$，快了近 6000 倍！

### 3.6 模板代码（逐行注释）

```python
def multiple_knapsack(N, V, weights, values, counts):
    """
    多重背包（二进制优化）
    :param N: 物品种类数
    :param V: 背包容量
    :param weights: 每种物品单件体积
    :param values:  每种物品单件价值
    :param counts:  每种物品的数量
    :return: 最大总价值
    """
    # 第一步：二进制拆分，把每种物品拆成 log(c) 组
    items = []  # 存拆分后的 (体积, 价值)

    for i in range(N):
        w = weights[i]   # 单件体积
        v = values[i]    # 单件价值
        c = counts[i]    # 该种物品的总数量

        k = 1
        # 用 1, 2, 4, ... 拆分，直到剩余不够下一个 2 的幂
        while k <= c:
            items.append((w * k, v * k))  # 把 k 件打包成一组
            c -= k                        # 从总数中减掉已拆的
            k <<= 1                       # k = k * 2，准备下一个 2 的幂
        # 如果还有剩余，单独成组
        if c > 0:
            items.append((w * c, v * c))

    # 第二步：对拆分后的所有组跑 0/1 背包（倒序容量）
    dp = [0] * (V + 1)
    for w, v in items:
        for j in range(V, w - 1, -1):
            dp[j] = max(dp[j], dp[j - w] + v)

    return dp[V]
```

### 3.7 例题

#### P1776 宝物筛选

> **题意：** 有 $n$ 种宝物，第 $i$ 种价值 $v_i$，重量 $w_i$，数量 $m_i$。背包最大载重 $W$。求最大总价值。
>
> **分析：** 标准的裸多重背包。数据范围较大会卡朴素 $O(W \times \sum m_i)$，必须用二进制优化。

```python
# 读入：W = 背包承重（容量），n = 宝物种类数
W, n = map(int, input().split())

# 第一步：二进制拆分所有宝物
items = []  # 拆分后的 (重量, 价值)

for _ in range(n):
    v, w, m = map(int, input().split())  # 价值 v，重量 w，数量 m

    # 二进制拆分
    k = 1
    while k <= m:
        items.append((w * k, v * k))  # k 件打包
        m -= k
        k <<= 1
    if m > 0:
        items.append((w * m, v * m))

# 第二步：对拆分后的所有物品跑 0/1 背包
dp = [0] * (W + 1)

for w, v in items:
    # 倒序容量核心代码
    for j in range(W, w - 1, -1):
        dp[j] = max(dp[j], dp[j - w] + v)

print(dp[W])
```

## 四、分组背包

### 4.1 问题模型

> 有 $N$ 组物品和一个容量为 $V$ 的背包。**每组内最多选一件物品**。第 $i$ 组的第 $k$ 件物品体积 $w_{ik}$，价值 $v_{ik}$。求最大总价值。

**典型场景：** 配餐时每组只能选一种主食、搭配决策时每组策略互斥。

### 4.2 状态定义与转移

$$
dp[j] = \text{容量 j 能装的最大价值}
$$

对于每组，我们都要决定："这一组里到底选哪个？还是都不选？"

**转移思路：**

```
// 伪代码
for 每个组:
    for j from V down to 0:          // 组外倒序（0/1 背包的"每组选或不选"）
        for 组内每个物品:
            if j >= w:
                dp[j] = max(dp[j], dp[j-w] + v)
```

- **外层枚举组别**，每组做一次决策。
- **中间容量倒序**：保证每组最多选一个物品（用倒序是因为组内决策完成后，本组的信息已经是"最新"的了）。
- **内层遍历组内物品**：决定本组选哪一个最好。

### 4.3 模板代码（逐行注释）

```python
def group_knapsack(V, groups):
    """
    分组背包通用模板
    :param V: 背包容量
    :param groups: 列表的列表，每个元素是 [(w1, v1), (w2, v2), ...] 表示该组的物品
    :return: 最大总价值
    """
    dp = [0] * (V + 1)

    # 遍历每一组
    for group in groups:
        # 容量倒序，保证每组最多选一个
        for j in range(V, -1, -1):
            # 遍历组内所有物品
            for w, v in group:
                if j >= w:  # 装得下才能考虑
                    dp[j] = max(dp[j], dp[j - w] + v)

    return dp[V]
```

**注意这里的容量循环范围：** `range(V, -1, -1)`（到 0 结束），因为组内物品的 w 可能不同，最小的 w 决定了下界。

### 4.4 例题

#### P1757 通天之分组背包

> **题意：** 有 $m$ 件物品，每件有组号 $a$、重量 $b$、价值 $c$。每组最多选一件。背包容量 $n$。求最大价值。
>
> **分析：** 输入数据不保证组号连续，需要先按组号分组。然后套分组背包模板。

```python
# 读入背包容量 n 和物品总数 m
n, m = map(int, input().split())

# 第一步：按组号分组
groups = {}  # 字典: 组号 -> [(重量, 价值), ...]

for _ in range(m):
    a, b, c = map(int, input().split())  # 组号 a，重量 b，价值 c
    if a not in groups:
        groups[a] = []
    groups[a].append((b, c))

# 第二步：跑分组背包
dp = [0] * (n + 1)

# 遍历每组
for gid, items in groups.items():
    # 容量倒序，保证每组最多选一个
    for j in range(n, -1, -1):
        # 遍历本组所有物品
        for w, v in items:
            if j >= w:
                dp[j] = max(dp[j], dp[j - w] + v)

print(dp[n])
```

## 五、总结

| 背包类型 | 物品限制 | 内层循环方向 | 核心思路 | 对应例题 |
|----------|---------|-------------|---------|---------|
| **0/1 背包** | 每件最多 1 次 | 倒序 | 选或不选 | P1048 采药 / P1060 金明 |
| **完全背包** | 每种无限次 | 正序 | 允许重复 | P1616 疯狂采药 |
| **多重背包** | 每种有限 $c_i$ 件 | 拆够后倒序 | 二进制分组 → 0/1 背包 | P1776 宝物筛选 |
| **分组背包** | 每组最多选 1 件 | 组外倒序 | 每组内做 0/1 决策 | P1757 分组背包 |

### 做题框架

1. **先判断物品限制**
   - 每件只 1 次 → **0/1 背包**
   - 每种无限次 → **完全背包**
   - 每种有限次 → **多重背包**（数据小可朴素拆，数据大用二进制优化）
   - 有分组限制 → **分组背包**

2. **套对应的模板**
3. **看特殊要求**
   - 恰好装满？初始化 `dp[0]=0, 其余=-∞`
   - 方案数？把 `max` 改成 `+`，初始化 `dp[0]=1`
   - 最小值？把 `max` 改成 `min`，初始化 `dp[0]=0, 其余=+∞`
   - 二维费用？用二维数组 `dp[c1][c2]`，每层倒序

背包 DP 是动态规划入门的第一关，也是最重要的基础。把它们吃透，后面遇到区间 DP、树形 DP、状压 DP 时，你会发现很多思想是相通的。加油！💪
