## 📌 一句话理解矩阵分解

**类比理解**：想象你有10000个用户和4499部电影组成的评分表（99%都是空的）。矩阵分解就像一个"翻译官"，把这么复杂的表格翻译成：
- 每个用户的"性格特征"（比如：喜欢动作片、讨厌爱情片）
- 每部电影的"DNA"（比如：很热血、节奏快）

这样，"性格特征+DNA"就能预测用户会给电影打多少分！

---

## 📊 二、矩阵分解的核心原理

### 2.1 问题背景

用户-项目评分矩阵 R 是这样的：

| | 电影1 | 电影2 | 电影3 | ... | 电影4499 |
|---|---|---|---|---|---|
| 用户1 | 5 | ? | 3 | ... | ? |
| 用户2 | ? | 4 | ? | ... | 2 |
| 用户3 | 3 | ? | ? | ... | ? |
| ... | ... | ... | ... | ... | ... |
| 用户10000 | ? | 5 | ? | ... | 4 |

> **目标**：预测所有的"?"，把稀疏矩阵填满！

### 2.2 核心思想

**核心公式**（通俗理解）：
```
R ≈ P^T × Q
```

翻译成人话就是：**评分矩阵 ≈ 用户特征矩阵 × 电影特征矩阵**

详细展开：
```
预测评分 = Σ(用户特征 × 电影特征)
     即：r̂_ui = p_u · q_i
```

## 👤 三、潜在特征矩阵 P 和 Q 的物理意义

### 3.1 矩阵 P（用户特征矩阵）

- **大小**：m × f（m=用户数，f=隐藏特征数）
- **含义**：每个用户对 f 种"潜在偏好"的匹配程度

**举例说明**：
```
假设 f=3（3个隐藏特征：动作指数、爱情指数、烧脑指数）

用户1的P矩阵: [0.9, 0.2, 0.7]
解释：用户1喜欢动作片(0.9)，不喜欢爱情片(0.2)，喜欢烧脑片(0.7)
```

### 3.2 矩阵 Q（电影特征矩阵）

- **大小**：n × f（n=电影数，f=隐藏特征数）
- **含义**：每部电影对 f 种"属性"的具备程度

**举例说明**：
```
电影1的Q矩阵: [0.8, 0.1, 0.6]
解释：电影1是动作片(0.8)，爱情成分少(0.1)，有点烧脑(0.6)
```

### 3.3 预测评分的计算

```
用户1对电影1的预测评分 = P_用户1 · Q_电影1
                        = 0.9×0.8 + 0.2×0.1 + 0.7×0.6
                        = 0.72 + 0.02 + 0.42
                        = 1.16（如果满分5分，说明不推荐）
```

## ⚡ 四、正则化矩阵分解的目标函数

### 4.1 为什么需要正则化？

**问题**：如果只用最小二乘法优化，模型容易"死记硬背"训练数据，导致过拟合。

**解决方案**：加入正则项，防止模型参数过大。

### 4.2 正则化目标函数（核心公式）

$$\min_{P,Q} \sum_{(u,i) \in K} (r_{ui} - p_u q_i^T)^2 + \lambda(\|p_u\|^2 + \|q_i\|^2)$$

### 4.3 公式解读

| 符号 | 含义 |
|------|------|
| $r_{ui}$ | 用户u对电影i的真实评分 |
| $p_u$ | 用户u的特征向量 |
| $q_i$ | 电影i的特征向量 |
| $\lambda$ | 正则化系数（防止过拟合） |
| K | 已评分数据集合 |

### 4.4 通俗理解

```
总损失 = 预测误差平方和 + 正则惩罚项
       = Σ(真实评分 - 预测评分)² + λ×(参数平方和)
```

**正则项的作用**：让 P 和 Q 的值不要太大，否则会被"惩罚"。

## 🔄 五、梯度下降求解方法

### 5.1 算法流程（伪代码）

```python
# ===== 矩阵分解梯度下降算法 =====
输入: 评分矩阵R, 隐藏特征数f, 学习率η, 正则系数λ, 迭代次数H
输出: 用户矩阵P, 电影矩阵Q

1. 初始化:
   P = 随机小数值矩阵 (m × f)
   Q = 随机小数值矩阵 (n × f)

2. 重复 H 次迭代:
   对每个已知评分 (u, i) 执行:
   
   a) 计算预测误差:
      error = r_ui - dot(p_u, q_i)
   
   b) 更新用户特征 (沿梯度负方向):
      p_u = p_u + η × (error × q_i - λ × p_u)
   
   c) 更新电影特征:
      q_i = q_i + η × (error × p_u - λ × q_i)

3. 返回 P, Q
```

### 5.2 参数更新公式详解

```
p_u_new = p_u_old + η × (error × q_i - λ × p_u_old)
q_i_new = q_i_old + η × (error × p_u - λ × q_i_old)
```

### 5.3 参数选择建议

| 参数 | 建议范围 | 说明 |
|------|----------|------|
| f（隐藏特征数） | 10-100 | 太少欠拟合，太多过拟合 |
| η（学习率） | 0.001-0.01 | 太大震荡，太小太慢 |
| λ（正则系数） | 0.01-0.1 | 越大越保守 |
| H（迭代次数） | 50-200 | 需观察收敛 |

## 📏 六、评价指标 MAE 和 RMSE

### 6.1 MAE（平均绝对误差）

**定义**：预测评分与真实评分之差的绝对值的平均

$$\text{MAE} = \frac{\sum_{u,i} |r_{ui} - \hat{r}_{ui}|}{|M|}$$

**特点**：
- 直观：误差是多少分就是多少分
- 对异常值不敏感
- **越小越好**

**举例**：
```
预测: [4, 5, 3, 4]
真实: [5, 4, 3, 4]
MAE = (|5-4| + |4-5| + |3-3| + |4-4|) / 4 = 0.5
```

### 6.2 RMSE（均方根误差）

**定义**：预测评分与真实评分之差的平方和的均值的平方根

$$\text{RMSE} = \sqrt{\frac{\sum_{u,i} (r_{ui} - \hat{r}_{ui})^2}{|M|}}$$

**特点**：
- 对大误差惩罚更重
- 与MAE单位相同（但通常RMSE > MAE）
- **越小越好**

**举例**：
```
预测: [4, 5, 3, 4]
真实: [5, 4, 3, 4]
RMSE = sqrt((1² + 1² + 0² + 0²) / 4) = sqrt(0.5) ≈ 0.707
```

### 6.3 两个指标对比

| 指标 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| MAE | 直观、易解释 | 对大误差不敏感 | 关注平均偏差 |
| RMSE | 惩罚大误差 | 对异常值敏感 | 关注极端错误 |

## 💻 八、Python 代码示例

### 8.1 基础版矩阵分解（numpy实现）

```python
import numpy as np
from sklearn.model_selection import train_test_split

class MatrixFactorization:
    """正则化矩阵分解"""
    
    def __init__(self, n_factors=50, lr=0.005, reg=0.02, n_epochs=100):
        self.n_factors = n_factors  # 隐藏特征数
        self.lr = lr               # 学习率
        self.reg = reg             # 正则系数
        self.n_epochs = n_epochs   # 迭代次数
    
    def fit(self, R):
        """
        R: 评分矩阵 (m x n)，0表示未评分
        """
        self.m, self.n = R.shape
        
        # 初始化P和Q矩阵（随机小数值）
        np.random.seed(42)
        self.P = np.random.normal(0, 0.1, (self.m, self.n_factors))
        self.Q = np.random.normal(0, 0.1, (self.n, self.n_factors))
        
        # 获取已知评分的位置
        self.users, self.items = np.where(R > 0)
        self.ratings = R[self.users, self.items]
        
        # 梯度下降训练
        for epoch in range(self.n_epochs):
            for u, i, r in zip(self.users, self.items, self.ratings):
                # 预测误差
                error = r - np.dot(self.P[u], self.Q[i])
                
                # 更新P和Q
                P_u = self.P[u].copy()
                self.P[u] += self.lr * (error * self.Q[i] - self.reg * self.P[u])
                self.Q[i] += self.lr * (error * P_u - self.reg * self.Q[i])
            
            # 计算训练误差
            if (epoch + 1) % 20 == 0:
                pred = self.predict_all()
                train_mae = np.abs(R[R > 0] - pred[R > 0]).mean()
                print(f"Epoch {epoch+1}/{self.n_epochs}, Train MAE: {train_mae:.4f}")
    
    def predict(self, u, i):
        """预测单个评分"""
        return np.dot(self.P[u], self.Q[i])
    
    def predict_all(self):
        """预测整个矩阵"""
        return np.dot(self.P, self.Q.T)

# ===== 使用示例 =====
# 假设 R 是 10000×4499 的评分矩阵
# R[i,j] = 用户i对电影j的评分，0表示未评分

model = MatrixFactorization(n_factors=50, lr=0.005, reg=0.02, n_epochs=100)
model.fit(R)

# 预测
predicted_rating = model.predict(user_id=0, item_id=100)
print(f"预测评分: {predicted_rating:.2f}")
```

### 8.2 使用 sklearn 的 NMF（非负矩阵分解）

```python
from sklearn.decomposition import NMF
import numpy as np

class NMFRecommender:
    """基于 sklearn NMF 的推荐系统"""
    
    def __init__(self, n_components=20):
        self.n_components = n_components
        self.model = NMF(n_components=n_components, init='random', random_state=42)
    
    def fit(self, R):
        """
        R: 评分矩阵，非负
        """
        # NMF要求输入非负矩阵
        # 对于0-5的评分，需要把0替换为非负值或直接使用
        self.R = R
        self.P = self.model.fit_transform(R)  # 用户矩阵 (m x n_components)
        self.Q = self.model.components_       # 电影矩阵 (n_components x n)
    
    def predict(self, u, i):
        return np.dot(self.P[u], self.Q[:, i])
    
    def predict_all(self):
        return np.dot(self.P, self.Q)

# ===== 使用示例 =====
model = NMFRecommender(n_components=20)
model.fit(R)

# 计算填充后的完整矩阵
filled_matrix = model.predict_all()
```

### 8.3 评估指标计算

```python
def evaluate_model(model, R_test, R_full):
    """
    计算MAE和RMSE
    model: 训练好的模型
    R_test: 测试集评分矩阵（非零位置为测试数据）
    R_full: 完整的评分矩阵（包含真实评分）
    """
    # 获取测试集位置
    test_users, test_items = np.where(R_test > 0)
    test_ratings = R_full[test_users, test_items]
    
    # 预测
    pred_ratings = model.predict_all()[test_users, test_items]
    
    # 计算MAE
    mae = np.mean(np.abs(test_ratings - pred_ratings))
    
    # 计算RMSE
    rmse = np.sqrt(np.mean((test_ratings - pred_ratings) ** 2))
    
    return mae, rmse

# ===== 使用示例 =====
mae, rmse = evaluate_model(model, test_matrix, full_matrix)
print(f"MAE: {mae:.4f}")
print(f"RMSE: {rmse:.4f}")
```

## ⚠️ 九、针对 Netflix 精简版数据的特别说明

### 9.1 数据特点

| 特点 | 说明 | 对应影响 |
|------|------|----------|
| 10000用户 × 4499电影 | 规模适中 | 无需分布式计算 |
| 只有评分矩阵 | 无用户/电影属性 | 部分高级算法不可用 |
| 稀疏矩阵 | 大部分为空 | 需要特殊处理 |

### 9.2 ✅ 可用的方法

1. **基础矩阵分解 (Basic MF)**
   - ✅ 完全可用
   - 适合入门和基准对比

2. **正则化矩阵分解 (Regularized MF)**
   - ✅ 完全可用
   - 推荐使用，防止过拟合

3. **偏置矩阵分解 (Biases MF)**
   - ✅ 完全可用
   - 考虑用户/电影偏置（有人爱打高分，有人电影本身质量高）

4. **SVD / SVD++**
   - ✅ 完全可用
   - scikit-surprise 库直接支持

5. **NMF（非负矩阵分解）**
   - ✅ 完全可用
   - sklearn 直接支持

### 9.3 ⚠️ 部分可用的方法

1. **融合聚类集成的矩阵分解 (k-CE-MF)**
   - ⚠️ 需要用户/电影属性信息
   - 论文中提出但**你的数据集没有额外属性**
   - **建议**：用评分矩阵自己做聚类辅助

2. **融合近邻的矩阵分解 (USISMF)**
   - ⚠️ 需要额外信息计算相似度
   - **建议**：只用评分矩阵计算用户/电影相似度

### 9.4 ❌ 不可用的方法

| 方法 | 原因 |
|------|------|
| 融合用户属性的矩阵分解 | 没有用户属性数据 |
| 融合电影属性的矩阵分解 | 没有电影属性数据 |
| 基于内容的推荐 | 需要项目内容特征 |

### 9.5 推荐实践方案

```python
# ===== Netflix数据推荐方案 =====

# 方案1：简单基准
- 使用 sklearn 的 NMF 或 TruncatedSVD
- 参数：n_components=20~50

# 方案2：推荐方案
- 使用 scikit-surprise 库的 SVD/SVD++
- 原因：成熟、稳定、有早停功能

# 方案3：进阶方案
- 自定义正则化矩阵分解
- 加入偏置项
- 使用交叉验证调参
```
