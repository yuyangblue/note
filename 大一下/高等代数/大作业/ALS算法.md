## ALS（交替最小二乘法）详解

---

## 核心思想

矩阵分解的目标函数：

$$\min_{P,Q} \sum_{(u,i) \in \Omega} (R_{ui} - P_u Q_i^T)^2 + \lambda(\|P\|^2 + \|Q\|^2)$$

**问题**：同时优化 P 和 Q 是非凸问题，很难直接求解。

**ALS的技巧**：固定一个，优化另一个

- 固定 Q → P 的问题变成**凸优化**（有闭式解）
- 固定 P → Q 的问题变成**凸优化**（有闭式解）

交替迭代，逐步收敛。

---

## 图解直觉

```
        损失函数 L(P, Q)
        
              *
             /|\
            / | \
           /  |  \    同时优化 P 和 Q：找不到最低点
          /   |   \
         /    |    \
        /     |     \
       
       
交替优化：
        
Step 1: 固定 Q，只动 P
        P ───────────→ 沿某个方向找到最低点
        
Step 2: 固定 P，只动 Q
        Q ───────────→ 沿另一个方向找到最低点
        
Step 3: 重复...
        
最终到达山谷 ✓
```

---

## 数学推导

### 目标函数（单个用户 u）

固定 Q，只看用户 u 的损失：

$$L_u = \sum_{i \in I_u} (R_{ui} - P_u Q_i^T)^2 + \lambda \|P_u\|^2$$

其中 $I_u$ 是用户 u 评分过的物品集合。

### 求导 = 0

$$\frac{\partial L_u}{\partial P_u} = 0$$

展开：

$$-2 \sum_{i \in I_u} (R_{ui} - P_u Q_i^T) Q_i + 2\lambda P_u = 0$$

整理成矩阵形式：

$$P_u (Q_u^T Q_u + \lambda I) = R_u^T Q_u$$

**闭式解**：

$$P_u = R_u^T Q_u (Q_u^T Q_u + \lambda I)^{-1}$$

其中：
- $Q_u$：用户 u 评分过的物品对应的 Q 行
- $R_u$：用户 u 的评分向量（已知部分）

## 代码对照

```python
# 固定 Q，更新 P 的第 u 行
for u in range(m):
    mask_u = mask[u, :]           # 用户 u 评分过的物品
    Q_u = Q[mask_u, :]            # 取出对应的 Q 行
    R_u = data[u, mask_u]         # 用户 u 的已知评分
    
    # 闭式解: P_u = (Q_u^T Q_u + λI)^{-1} Q_u^T R_u
    P[u, :] = np.linalg.solve(
        Q_u.T @ Q_u + l2_lambda * np.eye(k),  # 系数矩阵
        Q_u.T @ R_u                            # 右边
    )
```

`np.linalg.solve(A, b)` 求解 $Ax = b$，等价于 $x = A^{-1}b$

## ALS vs SGD 对比

| 对比项 | SGD（梯度下降） | ALS（交替最小二乘） |
|--------|----------------|-------------------|
| 每轮更新 | 逐个评分更新 | 整行更新 |
| 收敛速度 | 慢（需要很多轮） | 快（通常20-50轮） |
| 实现难度 | 简单 | 稍复杂 |
| 并行性 | 差 | 好（不同用户可并行） |
| 适用场景 | 小数据、在线学习 | 大数据、离线批处理 |

---

## 为什么ALS更快？

**SGD**：每个评分更新一次，假设有100万评分，每轮要更新100万次

**ALS**：每个用户/物品更新一次，假设有1万用户、1万物品，每轮只需更新2万次

```
SGD:  每轮迭代次数 = 已知评分数（可能几百万）
ALS:  每轮迭代次数 = 用户数 + 物品数（通常几万）
```

---

## 一句话总结

**ALS = 交替固定 P 和 Q，每步用闭式解直接求解，比梯度下降快得多**
