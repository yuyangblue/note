## git diff查看差异

`HEAD`（当前分支的最新提交）
`HEAD^` 代表上一个提交
`HEAD~n`表示最新提交前第n次的提交

|                        命令                        |            功能            |
| :----------------------------------------------: | :----------------------: |
|                    `git diff`                    |      查看工作目录与暂存区的差异       |
|           `git diff --cached(staged)`            | 查看暂存区与上一次提交（commit）之间的差异 |
|                 `git diff HEAD`                  | 显示工作目录中所有文件相对于 HEAD 的修改  |
|       `git diff <commit-id1> <commit-id2>`       |       查看两个提交之间的差异        |
| `git diff <commit-id1> <commit-id2> <file-path>` |      两个提交之间的单个文件差异       |


## git rm删除文件

|            命令            |          说明          |
| :----------------------: | :------------------: |
| `rm file; git add file`  | 先从工作区删除文件，然后再暂存删除内容  |
|     `git rm <file>`      |   把文件从工作区和暂存区同时删除    |
| `git rm --cached <file>` | 把文件从暂存区删除，但保留在当前工作区中 |
|      `git rm -r *`       |  递归删除某个目录下的所有子目录和文件  |
|          **提示**          |      删除后不要忘记提交       |


## `.gitignore`忽略文件

### 创建`.gitignore` 文件

1. **在项目根目录创建**：在你的项目根目录下，创建一个名为`.gitignore`的文件（注意前面有点）。在 Linux 系统下，可以使用命令 `touch .gitignore` 来创建；在 Windows 系统中，可以在文件资源管理器中新建一个文本文件，然后重命名为`.gitignore` 。

### 编写`.gitignore` 规则

1. **基本语法**
    
    - **一行一条规则**：每一行代表一条忽略规则。例如，如果你想忽略所有的`.log`文件，可以在`.gitignore`中添加一行：`*.log` 。这表示所有文件名以`.log`结尾的文件都不会被 Git 跟踪。
    - **目录忽略**：如果你想忽略某个目录，直接写目录名，例如 `temp/` 。这会忽略整个`temp`目录及其所有子目录和文件。
    - **排除特定文件**：如果你已经忽略了某个目录，但又想让其中某个特定文件被纳入版本控制，可以使用 `!` 符号。例如，你忽略了`temp/`目录，但又想跟踪`temp/important.txt`，可以这样写：
    

```bash
temp/
!temp/important.txt
```

2. **通配符使用**
    
    - **`*`**：匹配零个或多个字符，但不跨目录。例如，`*.py`匹配所有`.py`文件，但`src/*.py` 只匹配`src`目录下的`.py`文件，不会匹配`src/subdir/*.py` 。
    - **`?`**：匹配任何单个字符。例如，`f?o.txt` 可以匹配`foo.txt` 、`fao.txt`等。
    - **`**`**：匹配零个或多个目录（跨目录匹配）。例如，`src/**/*.py` 可以匹配`src`目录及其所有子目录下的`.py`文件。
    

### 应用规则

1. **重新添加文件**：当你修改了`.gitignore`文件后，已经被 Git 跟踪的文件不会自动被忽略。如果你想让这些文件被忽略，需要先从暂存区移除（但不删除文件本身），可以使用命令 `git rm --cached <file or directory>` 。例如，要忽略已经跟踪的`temp/`目录，可以运行 `git rm --cached -r temp/` ，然后重新提交更改。之后新添加的符合忽略规则的文件就不会被 Git 跟踪了。

## SSH配置和克隆仓库

|操作|命令 / 说明|
|---|---|
|生成 SSH Key|`ssh-keygen -t rsa -b 4096`<br><br>生成后会得到两个文件：<br><br>- 私钥：`id_rsa`<br><br>- 公钥：`id_rsa.pub`|
|克隆仓库|`git clone repo-address`|
|推送更新内容|`git push <remote> <branch>`|
|拉取更新内容|`git pull <remote>`|

## 关联本地仓库和远程仓库


|操作类型|命令|说明|
|---|---|---|
|添加远程仓库|Step 1: `git remote add <远程仓库别名> <远程仓库地址>`|关联本地仓库与远程仓库|
||Step 2: `git push -u <远程仓库名> <分支名>`|将本地分支推送到远程仓库并建立关联|
|查看远程仓库|`git remote -v`|查看已关联的远程仓库地址|
|拉取远程仓库内容|`git pull <远程仓库名> <远程分支名>:<本地分支名>`|拉取远程分支内容到本地分支<br><br>若远程分支名与本地分支名相同，可省略冒号及后面部分|
