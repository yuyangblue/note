    📚 Day1学习卡片：HTML基础与文档结构¶

    🎯 今日主题¶

    HTML入门：理解HTML本质，掌握文档结构五个核心标签，初探Typecho主题结构

    ⏱️ 学习时间¶

    预计：阅读10分钟 + 实操15分钟 + 检验5分钟 = 总计30分钟

    📖 核心知识点（5个关键概念）¶

    1. HTML本质：网页的骨架（2分钟）¶

    • 标记语言：用尖括号标签描述内容结构，如<p>表示段落
    • 浏览器解析：将HTML代码渲染为可视化网页，忽略多余空格和换行
    • Typecho基础：你的博客所有页面都是HTML构成，学习HTML等于掌握博客"底层代码"

    2. 文档结构标签：<!DOCTYPE>的实际作用（2分钟）¶

    标签 实际作用（不仅仅是定义） 在Typecho中的表现 可以怎么修改
    <!DOCTYPE html> 渲染模式指令：告诉浏览器使用HTML5标准模式渲染，避免怪异模式兼容问题 每个.php文件开头，通常第一行 基本不需要改，保持即可

    核心理解：这不是一个HTML标签，而是一个声明。现代网页开发中，99%的情况都使用<!DOCTYPE html>。

    3. 文档结构标签：<html>、<head>、<body>的分工（3分钟）¶

    <!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <title>我的博客标题</title>
  </head>
  <body>
    <!-- 所有可见内容在这里 -->
  </body>
</html>
    标签 实际作用 在Typecho中的位置 可以怎么修改
    <html lang="zh-CN"> 根容器+语言声明：包含整个页面，lang属性帮助搜索引擎和屏幕阅读器识别语言 index.php、post.php等模板文件的根元素 修改lang="en"可声明英文页面
    <head> 元信息控制中心：存放CSS链接、JS脚本、字符编码、SEO信息等，不显示但控制页面行为 header.php或模板<head>部分 添加自定义CSS、修改字符编码
    <body> 可见内容容器：所有用户看到的文字、图片、布局都在这里 index.php、post.php的<body>部分 修改结构、添加class用于CSS控制

    4. 创建第一个HTML文件：编辑器与文件保存（2分钟）¶

    本地测试步骤： 1. 在桌面创建test.html文件 2. 用记事本/VSCode打开，输入上面代码 3. 保存后双击用浏览器打开 4. 观察：页面标题显示"我的博客标题"，页面内容为空

    Typecho关联：Typecho的每个模板文件（.php）最终都会输出类似结构的HTML。

    5. Typecho主题结构初探：index.php的基本结构（1分钟）¶

    • 位置：/usr/themes/你的主题名/index.php
    • 作用：博客首页的模板，控制首页的HTML输出
    • 观察点：在Typecho主题文件中寻找<!DOCTYPE>、<html>、<head>、<body>标签

    💡 要点总结¶

    • ✅ HTML是标记语言：用标签描述结构，浏览器渲染成网页
    • ✅ 五个核心结构标签：<!DOCTYPE>（声明）、<html>（根）、<head>（元信息）、<body>（可见内容）
    • ✅ 标签的实际作用：理解每个标签对浏览器/搜索引擎/屏幕阅读器的意义
    • ✅ Typecho主题结构：.php模板文件最终输出HTML，index.php控制首页

    Typecho主题文件路径¶

    实际作用关联¶

    • 修改<title> → 修改header.php中的<title>标签或Typecho后台设置
    • 添加CSS链接 → 在header.php的<head>内添加<link>标签
    • 调整页面结构 → 修改index.php、post.php等文件的<body>内容

    🛠️ 30分钟实操任务：定位并理解Typecho主题中的HTML文档结构¶

    任务目标¶

    登录你的VPS，找到Typecho主题目录，查看index.php和header.php文件，识别HTML结构标签，理解Typecho模板语法。

    步骤分解¶

    第一步：登录VPS并找到主题目录（5分钟）¶

    1. SSH登录你的VPS
    2. 进入Typecho安装目录（通常）： 
      1
      cd /var/www/html/typecho  # 或你的安装路径
    3. 进入主题目录： 
      1
      cd usr/themes/
    4. 列出所有主题： 
      1
      ls -la
    5. 进入你正在使用的主题目录（如default）： 
      1
      cd 你的主题名

    第二步：查看index.php的HTML结构（5分钟）¶

    1. 查看index.php内容： 
      1
      cat index.php | head -20  # 查看前20行
    2. 寻找以下标签：
    3. <!DOCTYPE html>（通常在文件开头）
    4. <html（可能有lang属性）
    5. <head>（可能在header.php中被引入）

    6. <body>（注意<body>标签的开始）

    7. 观察Typecho模板语法：

    8. <?php ... ?>：PHP代码块
    9. <?php $this->method(); ?>：Typecho对象方法调用
    10. <?php $this->options->themeUrl(); ?>：主题文件路径函数

    第三步：查看header.php理解结构（5分钟）¶

    1. 查看header.php内容： 
      1
      cat header.php
    2. 寻找以下内容：
    3. <title>标签及其内容
    4. <link rel="stylesheet"标签（引入CSS）
    5. <meta>标签（字符编码等）

    6. <?php $this->header(); ?>：Typecho头部输出函数

    7. 关键发现：

    8. Typecho主题使用PHP输出HTML，如<title><?php $this->options->title(); ?></title>
    9. CSS链接使用路径函数：<?php $this->options->themeUrl('style.css'); ?>

    验收标准¶

    • [ ] 成功登录VPS并进入主题目录
    • [ ] 在index.php中找到<!DOCTYPE html>和<html>标签
    • [ ] 在header.php中找到<head>、<title>和<link>标签
    • [ ] 理解至少一个Typecho模板语法（如<?php $this->options->title(); ?>）
    • [ ] 能够回答：你的主题首页使用HTML5标准模式渲染吗？（通过<!DOCTYPE html>确认）

    注意事项¶

    1. 文件路径可能不同：Typecho安装路径可能为/var/www/typecho、/home/www/typecho等
    2. 主题可能不同：你使用的可能是default、handsome或其他主题
    3. 如果找不到：使用find / -name "header.php" 2>/dev/null | grep themes搜索
    4. 安全操作：只查看，不修改，今日目标是理解结构
