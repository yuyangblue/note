#### 关键设计点：

1. **双向关联**：每个 Department 知道自己的 Teachers，每个 Teacher 也知道自己所在的 Departments
    
2. **指针或引用**：使用指针（或智能指针）避免对象复制问题
    
3. **至少一个 Teacher**：在 Department 的构造函数或业务逻辑中需要确保至少有一个 Teacher
    
4. **一致性维护**：当添加或移除关系时，需要同时更新两端的集合

```cpp
#include <vector>
#include <string>

class Teacher;  // 前向声明

class Department {
private:
    std::string name;
    std::string code;
    std::vector<Teacher*> teachers;  // 指向 Teacher 的指针集合
    
public:
    Department(const std::string& name, const std::string& code)
        : name(name), code(code) {}
    
    void addTeacher(Teacher* teacher) {
        teachers.push_back(teacher);
    }
    
    // ... 其他方法
};

class Teacher {
private:
    std::string name;
    std::string id;
    std::string title;
    std::vector<Department*> departments;  // 指向 Department 的指针集合
    
public:
    Teacher(const std::string& name, const std::string& id)
        : name(name), id(id) {}
    
    void addToDepartment(Department* dept) {
        departments.push_back(dept);
        dept->addTeacher(this);  // 保持双向关联的一致性
    }
    
    // ... 其他方法
};
```
