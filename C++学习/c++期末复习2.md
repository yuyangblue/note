### 0.小tips
1. 声明外部变量的时候没有定义，所以一般情况下不会分配内存
2. 列表初始化一定要类型相同int a {3.12}会报错
3. `0xfe` 对应的二进制是 `1111 1110`
4. 函数原型 `A f( const int&);`能够执行语句a.f(100);成功（因为 `100` 可以隐式转换为 `const int` 并绑定到 `const int&`）。
5. 在 C++ 中，`unsigned`修饰符只能用于**整数类型**（如`unsigned int`、`unsigned char`等），**不能用于浮点数类型（`float`、`double`）**。
6. **`private`静态数据成员**：只能在 **类内部（成员函数、友元** 访问，**类外部无法直接访问**，必须通过类的`public`成员函数间接操作。
### 1.指针运算

```cpp
char s[50]="The Object-Oriented Programming Language!";

short *p1=(short*)(s+1),*p2=(short *)(s+17);

请指出表达式p2-p1的值是
```
1. **指针运算的原理**
    
    - 在 C++ 中，当两个指针相减时，结果是它们之间相隔的元素个数，而不是字节数的差值。指针相减要求两个指针必须指向同一个数组（或类似连续存储结构）中的元素。这里虽然 `s` 是字符数组，但通过强制类型转换将其视为 `short` 类型指针进行运算。
    - 指针相减的计算公式为：`pointer2 - pointer1 = (address2 - address1) / sizeof(type)`，其中 `address2` 和 `address1` 是两个指针所指向的地址，`type` 是指针所指向的数据类型。

2. **具体分析例题**

    - 已知 `p1=(short*)(s + 1)` 和 `p2=(short *)(s + 17)`。`s` 是字符数组，`char` 类型通常占 1 个字节。
    - `s + 1` 表示从字符数组 `s` 的第二个字符开始的地址，`s + 17` 表示从字符数组 `s` 的第 18 个字符开始的地址。
    - 经过强制类型转换后，`p1` 和 `p2` 被视为 `short` 类型指针，`short` 类型通常占 2 个字节（具体字节数取决于编译器和系统，但在一般情况下是 2 字节）。
    - 首先计算地址差值：`(s + 17) - (s + 1)=16` 字节（因为每个 `char` 占 1 字节）。
    - 然后根据指针相减公式，由于指针指向 `short` 类型，`short` 占 2 字节，所以 `p2 - p1 = 16 / sizeof(short) = 16 / 2 = 8`。
    

所以表达式 `p2 - p1` 的值是 8，答案选 B。
