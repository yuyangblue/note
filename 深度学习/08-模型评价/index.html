    模型评价¶

    训练简单模型的实验分析¶

    MNIST-1D数据集¶

    属性 说明
    类别空间 \(y \in \{0,1,...,9\}\)
    数据生成 基于一维模板的随机仿射变换 + 高斯噪声
    训练集 4000个样本
    样本维度 \(x_i \in \mathbb{R}^{40}\)
    类别平衡 每类约400个样本

    网络架构¶

    • 输入层：\(D_i = 40\)
    • 隐藏层：\(2 \times 100\) ReLU
    • 输出层：\(D_o = 10\) + softmax
    • 损失函数：\(L_{CE}\)
    • 优化器：SGD (η = 0.1)
    • Batch size：100
    • 训练轮数：150 epochs

    实验观察¶

    指标 观察结果
    测试误差 最终稳定在 ~40% 水平
    测试损失 初期下降（0-1500步），后期上升（过拟合）

    测试误差的数学描述¶

    数据生成机制¶

    \[y = \mu(x) + \epsilon, \quad \epsilon \sim N(0, \sigma^2)\]
    • \(\mu(x)\)：真实函数
    • \(\epsilon\)：高斯噪声

    误差三大来源¶

    来源 定义 特点
    固有噪声 (Noise) 数据采集的随机性 任何模型都无法消除
    模型偏差 (Bias) 表达能力受限 欠拟合
    估计方差 (Variance) 样本敏感度 过拟合

    预期损失分解¶

    第一步：不可约简误差¶

    设 \(y = \mu(x) + \epsilon\)，其中 \(E[\epsilon] = 0, Var(\epsilon) = \sigma^2\)。

    \[\mathbb{E}_y[(f(x,\phi)-y)^2] = \underbrace{(f(x,\phi)-\mu(x))^2}_{\text{可约简误差}} + \underbrace{\sigma^2}_{\text{噪声（下界）}}\]

    物理意义：即使模型完美捕捉 \(\mu(x)\)，\(\sigma^2\) 仍然存在。

    第二步：偏差-方差分解¶

    记 \(f_\mu(x) = \mathbb{E}_\mathcal{D}[f(x,\phi(\mathcal{D}))]\) 为平均模型输出。

    \[\mathbb{E}_\mathcal{D}[(f(x,\phi(\mathcal{D}))-\mu(x))^2] = \underbrace{\mathbb{E}_\mathcal{D}[(f(x,\phi(\mathcal{D}))-f_\mu(x))^2]}_{\text{Variance}} + \underbrace{(f_\mu(x)-\mu(x))^2}_{\text{Bias}^2}\]

    最终结论¶

    \[\boxed{\text{Total Error} = \text{Bias}^2 + \text{Variance} + \sigma^2}\]
    组成 含义 对应问题
    Bias² 平均预测值偏离真实函数 欠拟合
    Variance 模型对数据扰动的敏感度 过拟合
    σ² 数据固有随机性 误差下界

    偏差-方差权衡¶

    数据规模的影响¶

    \[Var(\hat{f}) \propto \frac{\sigma^2}{N}\]
    数据规模 观察
    N = 6 估计严重依赖样本点
    N = 10 拟合曲线趋于平滑
    N = 100 模型几乎收敛至 \(f_\mu\)

    偏差优化¶

    通过扩展假设空间 \(\mathcal{H}\)：

    • 增加网络层数/单元数
    • 引入高阶交互项或非线性核

    最优模型选择¶

    存在唯一 \(C^*\) 最小化：

    \[\min_C \{\text{Bias}^2(C) + \text{Var}(C) + \sigma^2\}\]

    现代泛化悖论：双重下降现象¶

    经典观点 vs 现代观察¶

    观点 描述
    经典观点 模型复杂度存在"最优极值点"，过大容量导致过拟合
    现代观察 深度学习中，模型容量越过某个阈值后，测试误差再次下降

    实验设定¶

    • 模型：双隐藏层全连接网络，通过神经元数量控制参数量 \(D\)
    • 训练：N = 10,000 样本，Adam 全批量训练
    • 核心概念：插值阈值 \(D \approx N\)

    理想情形：无噪声泛化¶

    实验观察¶

    指标 现象
    训练误差 在 \(D = N\) 处达到零点，随后保持为零
    测试误差 呈单调下降趋势（而非U型上升）

    启示：在数据纯净时，过参数化不仅不会伤害泛化，反而提升拟合质量。

    真实情形：双重下降¶

    含噪数据下的现象¶

    1. 欠参数化区：经典偏差-方差均衡，出现首个局部最优
    2. 临界区域 (D ≈ N)：模型被迫用有限自由度记忆噪声，函数震荡，误差激增
    3. 过参数化区 (D > N)：误差再次下降，进入现代泛化区间

    机制分析¶

    解空间的范式转移¶

    阶段 系统特性 解的性质
    临界点 (D ≈ N) 恰好确定 唯一但极度扭曲的解（方差极大）
    过参数化 (D ≫ N) 欠定 无数个零训练误差解

    隐式正则化¶

    优化算法（如SGD/Adam）在无数零误差解中： - 不随机选择 - 倾向于选择"范数最小"或"路径最平滑"的解

    这称为隐式正则化 (Implicit Regularization)。

    为什么参数越多越平滑？¶

    • 参数量越大，冗余自由度越多
    • 隐式正则化利用这些自由度"磨平"震荡
    • 在穿过噪声点的同时，保持整体路径能量最低

    实践启示¶

    重新理解过拟合¶

    • 真正的"危险区"不在大模型
    • 而是在容量刚好覆盖噪声的临界点
    • 隐式正则化是深度学习能够泛化的底层基石

    建议¶

    建议 说明
    超越阈值 资源允许时，将模型推向过参数化区域
    慎用显式正则 过度使用L2或Dropout可能干扰双重下降
    关注数据噪声 噪声越强，临界点的"泛化峰值"越剧烈

    超参数优化¶

    为何需要HPO¶

    问题 说明
    不可观测性 真实偏差、方差在实际任务中无法直接计算
    泛化界限 理论只能定性指导，无法给出具体参数

    超参数分类¶

    类型 示例
    模型架构超参数 隐藏层深度、神经元数量
    算法偏置超参数 学习率、动量系数、权重衰减

    定义¶

    超参数 (Hyperparameter)：在算法运行前手动设定，用于控制学习过程、定义模型容量且不直接通过梯度下降更新的参数。

    评估框架：数据集划分¶

    三种数据集¶

    数据集 用途
    训练集 (Training) 拟合模型参数 \(\phi\)
    验证集 (Validation) 评估超参数组合的泛化性能
    测试集 (Testing) 评估最终交付模型的性能（仅用一次）

    严禁"偷看"(Peeking)¶

    • 测试集只能使用一次
    • 基于测试集反馈调整超参数会导致信息泄露

    跨验证策略¶

    对于小数据集，建议使用 K-折交叉验证 以降低评价指标的随机方差。

    黑盒优化的算法挑战¶

    搜索空间的复杂性¶

    特点 说明
    多维异构 包含连续、离散、类别变量
    非凸不可微 无法通过导数优化

    昂贵的评估代价¶

    \[\text{Total Cost} \approx N_{\text{trials}} \times \text{Cost}(\text{Training})\]

    主流搜索算法¶

    算法 特点
    网格搜索 穷举遍历，易陷入"维度灾难"
    随机搜索 高维空间比网格搜索更高效
    贝叶斯优化 建立代理模型，实现"智能采样"

    高效调优的最佳实践¶

    分阶段搜索 (Coarse-to-Fine)¶

    1. 先在对数尺度上大范围采样确定区间
    2. 在局部区域精细调优

    早停机制 (Early Stopping)¶

    方法 说明
    Successive Halving / Hyperband 阶段性剔除表现差的组合
    学习曲线监控 性能停滞则提前终止该轮试验

    自动化机器学习 (AutoML)¶

    • 神经架构搜索 (NAS)
    • 自动调参工具：Optuna, Ray Tune

    经验法则¶

    学习率通常是对模型性能影响最大的超参数 建议优先从学习率的量级（10⁻⁵ 到 10⁻¹）开始搜索

    核心总结¶

    主题 关键点
    误差分解 Total Error = Bias² + Variance + σ²
    双重下降 过参数化区域误差可再次下降
    隐式正则化 优化算法倾向于选择平滑解
    超参数调优 学习率最重要，建议分阶段搜索
