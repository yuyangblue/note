## 二、优化实验设计

### 2.1 优化目标

基于以上分析，我们设定了三个优化方向：

1. **提升整体准确率**：将序列准确率从81.90%提升到**90%以上**
2. **弥补短板**：重点提升位置3的准确率，缩小各位置之间的差距
3. **增强泛化能力**：通过数据增强和正则化，让模型在各种干扰下都能"稳住"

### 2.2 优化维度

在深度学习的世界里，调参就像做菜一样，调料（超参数）放得恰到好处，菜才香。我们主要从以下几个维度进行优化：

#### 2.2.1 数据增强策略

原始模型只用了最基本的预处理：Resize + ToTensor + Normalize。这就像是"清水煮白菜"，营养够了但口感单一。我们加入了更多"调料"：

- **随机仿射变换** (`RandomAffine`)：模拟验证码可能出现的轻微旋转和平移
- **高斯模糊** (`GaussianBlur`)：模拟拍照模糊或打印模糊的情况

#### 2.2.2 优化器选择

优化器就像是登山的向导，带领模型走向"损失函数的低谷"。我们对比了三种主流优化器：

| 优化器 | 特点 | 适用场景 |
|:-------|:-----|:---------|
| Adam | 自适应学习率，收敛快 | 快速原型开发 |
| AdamW | 带权重衰减的Adam，正则化效果更好 | 需要正则化的场景 |
| SGD | 传统稳定，但需要手动调学习率 | 大型数据集、fine-tuning |

AdamW的核心改进是将权重衰减从**L2正则化**改为**真正的权重衰减**：

$$
\theta_{t+1} = \theta_t - \eta \cdot \frac{1}{\sqrt{v_t} + \epsilon} \cdot (g_t + \lambda \theta_t)
$$

而不是像Adam那样把正则化加在梯度上。

#### 2.2.3 学习率调度

学习率是深度学习最重要的超参数之一。我们对比了三种调度策略：

1. **ReduceLROnPlateau**：当指标不再改善时自动降低学习率
2. **CosineAnnealing**：余弦曲线式下降，周期性地给模型"喘息"机会
3. **StepLR**：阶梯式下降，简单粗暴但有效

CosineAnnealing的公式如下：

$$
\eta_t = \eta_{\min} + \frac{1}{2}(\eta_{\max} - \eta_{\min}) \left(1 + \cos\left(\frac{t \pi}{T}\right)\right)
$$

#### 2.2.4 正则化技巧

除了Dropout（原始模型已使用，rate=0.5），我们还引入了**Label Smoothing**：

$$
L_{smooth} = -\sum_{i} y_i \cdot \log(\hat{y}_i) \cdot (1 - \alpha) + \frac{\alpha}{K} \cdot \log(\hat{y}_i)
$$

其中 $\alpha=0.1$ 是平滑系数，$K$ 是类别数。这个技巧可以防止模型对自己的预测"过于自信"。

### 2.3 实验分组

为了系统地验证各优化策略的效果，我们将实验分为三组：

| 实验 | 优化内容 | 主要变量 |
|:-----|:---------|:---------|
| **实验1** | 数据增强优化 | 保持其他配置不变，添加增强策略 |
| **实验2** | 优化器对比 | 固定数据增强，对比不同优化器 |
| **实验3** | 综合优化 | 融合所有有效策略 + 训练策略优化 |

```

    ↓
全连接层 (128 → 256)
    ↓
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│ 分类头1  │ 分类头2  │ 分类头3  │ 分类头4  │ 分类头5  │
│(256→36) │(256→36) │(256→36) │(256→36) │(256→36) │
└─────────┴─────────┴─────────┴─────────┴─────────┘
```

**模型参数量**：约 **20.3M**，其中大部分参数来自5个独立的分类头。

## 四、优化实验过程与结果

### 4.1 实验一：数据增强优化

#### 4.1.1 增强策略说明

数据增强是防止过拟合、提升泛化能力的"神器"。我们的增强策略：

```python
train_transform = transforms.Compose([
    transforms.Resize((40, 100)),                    # 统一尺寸
    transforms.RandomAffine(
        degrees=5,                                    # ±5度随机旋转
        translate=(0.1, 0.1),                         # 随机平移（宽高10%）
        scale=(0.9, 1.1)                              # 随机缩放（90%-110%）
    ),
    transforms.GaussianBlur(
        kernel_size=3,                                # 高斯模糊核大小
        sigma=(0.1, 2.0)                              # 模糊强度
    ),
    transforms.ToTensor(),
    transforms.Normalize((0.5,), (0.5,))
])
```

这些增强的"意图"是：
- **仿射变换**：验证码可能因为采集角度、屏幕畸变而产生形变
- **高斯模糊**：模拟老旧设备、低分辨率摄像头拍摄的情况

#### 4.1.2 实验结果对比

| 指标 | 基准模型 | +数据增强 | 提升 |
|:-----|:--------:|:---------:|:----:|
| 序列准确率 | 81.90% | **86.3%** | +4.4% |
| 平均字符准确率 | 95.43% | **96.1%** | +0.7% |
| 位置3准确率 | 90.48% | **92.7%** | +2.2% |
| 验证集loss | 0.58 | **0.49** | ↓15.5% |

#### 4.1.3 分析

数据增强带来了**全面提升**！尤其是：

1. **序列准确率提升明显**（+4.4%）：增强后的训练数据更多样化，模型学到了更鲁棒的特征
2. **位置3提升最大**（+2.2%）：这印证了我们的猜测——中间位置容易受到干扰，增强训练帮助模型"学会"忽略噪声
3. **验证loss下降显著**：说明模型的泛化能力确实增强了

不过，数据增强也有"代价"——训练时间增加了约30%，因为模型需要处理更复杂的输入。但这个代价完全值得！

### 4.2 实验二：优化器对比

#### 4.2.1 实验设置

我们固定数据增强策略不变，对比三种优化器配置：

**方案A：AdamW + CosineAnnealingLR**

```python
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=0.001,
    weight_decay=0.01  # 权重衰减
)
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
    optimizer,
    T_max=30,
    eta_min=1e-6
)
```

**方案B：SGD + StepLR**

```python
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=0.01,  # SGD通常需要更高的初始学习率
    momentum=0.9,
    weight_decay=0.01
)
scheduler = torch.optim.lr_scheduler.StepLR(
    optimizer,
    step_size=10,
    gamma=0.5  # 每10个epoch学习率减半
)
```

**对照组：Adam + ReduceLROnPlateau（基准）**

```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
    optimizer,
    mode='min',
    factor=0.5,
    patience=5
)
```

#### 4.2.2 学习率曲线对比

```
学习率
  ↑
1e-2 ┤                    ████
     │              █████      █████
     │         █████              ████  ← SGD (阶梯下降)
1e-3 ┤████
     │    ████
     │        ████                  ← Adam/AdamW (平滑下降)
     │              ████████
     │
1e-6 ┼────────────────────────────────→ Epoch
     0        10        20        30
```

从学习率曲线可以看出：
- **SGD + StepLR**：学习率呈阶梯状下降，在epoch 10和20处有明显"断层"
- **AdamW + CosineAnnealing**：学习率平滑下降，避免了学习率的突变
- **Adam + ReduceLROnPlateau**：自适应下降，在epoch 15后因为验证loss不再改善而降到了5e-4

#### 4.2.3 结果对比表格

| 配置 | 序列准确率 | 平均字符准确率 | 位置3准确率 | 收敛速度 |
|:-----|:----------:|:--------------:|:-----------:|:--------:|
| Adam + ReduceLROnPlateau | 86.3% | 96.1% | 92.7% | 快 |
| **AdamW + CosineAnnealing** | **88.1%** | **96.8%** | **94.2%** | 中等 |
| SGD + StepLR | 85.7% | 95.9% | 93.1% | 慢但稳定 |

#### 4.2.4 分析

**AdamW + CosineAnnealing** 取得了最佳成绩，原因分析：

1. **权重衰减更合理**：AdamW的weight decay直接作用于参数，而不是通过梯度间接影响，更符合正则化本意
2. **学习率调度更平滑**：CosineAnnealing让学习率在训练后期缓慢下降，给模型更多时间在"平坦区域"找到好解
3. **泛化能力更强**：实验发现AdamW训练的模型在测试集上的表现更稳定

SGD虽然最终准确率略低，但它的收敛过程更稳定，波动更小——这对生产环境部署是很有价值的特性。

### 4.3 实验三：综合优化

#### 4.3.1 最终方案

综合前两个实验的发现，我们制定了"终极套餐"：

```python
# === 数据增强 ===
train_transform = transforms.Compose([
    transforms.Resize((40, 100)),
    transforms.RandomAffine(degrees=5, translate=(0.1, 0.1)),
    transforms.GaussianBlur(kernel_size=3),
    transforms.ToTensor(),
    transforms.Normalize((0.5,), (0.5,))
])

# === 优化器 ===
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=0.001,
    weight_decay=0.01
)
scheduler = torch.optim.lr_scheduler.CosineAnnealingWarmRestarts(
    optimizer,
    T_0=10,          # 第一个周期
    T_mult=2,        # 后续周期翻倍
    eta_min=1e-6
)

# === 损失函数 ===
criterion = nn.CrossEntropyLoss(label_smoothing=0.1)

# === 训练策略 ===
NUM_EPOCHS = 50
early_stopping_patience = 10
```

**关键改动**：
- 学习率调度改为 `CosineAnnealingWarmRestarts`，周期性地让学习率"重启"，帮助跳出局部最优
- 增加Label Smoothing（0.1），防止过拟合
- 训练轮数扩展到50个epoch，配合早停机制（patience=10）

### A.2 Label Smoothing Loss

```python
class LabelSmoothingCrossEntropy(nn.Module):
    def __init__(self, smoothing=0.1):
        super().__init__()
        self.smoothing = smoothing
    
    def forward(self, pred, target):
        n_classes = pred.size(-1)
        # 将真实标签的置信度从1降到(1-smoothing)
        # 将平滑值均匀分配给其他类别
        log_preds = F.log_softmax(pred, dim=-1)
        loss = -log_preds.sum(dim=-1).mean()
        nll = F.nll_loss(log_preds, target)
        return self.smoothing * (nll / n_classes) + (1 - self.smoothing) * loss
```

### A.3 早停机制

```python
class EarlyStopping:
    def __init__(self, patience=10, min_delta=0.001):
        self.patience = patience
        self.min_delta = min_delta
        self.counter = 0
        self.best_loss = None
        self.early_stop = False
    
    def __call__(self, val_loss):
        if self.best_loss is None:
            self.best_loss = val_loss
        elif val_loss > self.best_loss - self.min_delta:
            self.counter += 1
            if self.counter >= self.patience:
                self.early_stop = True
        else:
            self.best_loss = val_loss
            self.counter = 0
```
