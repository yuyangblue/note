# 深度学习：正则化（Regularization）

> 张晓平 · 武汉大学数学与统计学院 · 2026年4月23日

---

## 一、引言：为什么需要正则化

### 1.1 两个核心挑战

| 挑战 | 含义 | 本质 |
|------|------|------|
| **泛化鸿沟** (Generalization Gap) | 训练集过拟合→IID测试数据上失效 | 模型太复杂，记住噪声 |
| **分布偏移** (Distribution Shift) | 训练与测试分布不一致(OOD) | 模型缺乏外推约束 |

### 1.2 正则化的定义

- **广义**：旨在减少**测试误差**而非训练误差的所有策略
- **狭义**：通过向目标函数添加额外项，显式约束参数空间的偏好

### 1.3 正则化分类体系

| 类别 | 方法 | 核心机制 |
|------|------|---------|
| **显式正则化** | L1/L2、结构稀疏性约束 | 修改损失函数，添加惩罚项 |
| **隐式正则化** | GD/SGD的优化动力学 | 优化算法本身引入的归纳偏置 |
| **启发式方法** | 早停、集成、Dropout、数据增强等 | 各类策略性/注入型/映射型/规范型手段 |

## 二、显式正则化

### 2.1 从ERM到SRM

**经验风险最小化 (ERM)**：

$$\hat{\phi}_{MLE} = \arg\min_{\phi} \mathcal{L}_{emp}(\phi), \quad \mathcal{L}_{emp}(\phi) = \sum_{i=1}^{I} \ell(f(x_i, \phi), y_i)$$

只管训练误差最小，不管泛化→容易过拟合。

**结构风险最小化 (SRM)**：

$$\hat{\phi}_{reg} = \arg\min_{\phi} \left[\mathcal{L}_{emp}(\phi) + \lambda \Omega(\phi)\right]$$

- $\Omega(\phi)$：正则化算子，度量参数的"复杂度"
- $\lambda > 0$：平衡超参数，控制**拟合精度 vs 泛化能力**的权衡

> 📌 **核心思想**：正则化不以降低训练误差为目标，而是通过引入偏置(Bias)来显著降低方差(Variance)。

### 2.2 正则化对损失曲面的"地形改造"

| 阶段 | 描述 |
|------|------|
| 原始地形 | 局部极小值众多且深邃，模型易陷入"尖锐"谷底，对扰动敏感 |
| 正则项约束 | 产生向原点收拢的吸引力，消除参数空间中极端不合理的配置 |
| 最终地形 | 曲面更平滑与凸显，极小值点向中心偏移，提升解的鲁棒性 |

> 📌 直觉：正则化就像把一个坑坑洼洼的山地"推平"，让模型不容易掉进又窄又深的坑里（过拟合），而是待在宽浅的谷底（泛化好）。

### 2.3 最大后验估计 (MAP) 解释

显式正则化本质上是对参数 $\phi$ 引入了**先验偏好**：

| 视角 | 假设 | 结果 |
|------|------|------|
| **MLE** | 参数是未知常数，完全由样本驱动 | 无正则化 |
| **MAP** | 参数有先验分布 | $\hat{\phi} = \arg\max_{\phi} \underbrace{Pr(\mathcal{Y}|\mathcal{X},\phi)}_{\text{似然}} \cdot \underbrace{Pr(\phi)}_{\text{先验}}$ |

等价关系：$\lambda\Omega(\phi) = -\log Pr(\phi) + \text{const}$

| 正则化类型 | $\Omega(\phi)$ | 等价贝叶斯先验 | 特点 |
|-----------|---------------|---------------|------|
| **L2正则化** | $\frac{1}{2}\|\phi\|_2^2$ | 高斯先验 $\mathcal{N}(0, \sigma^2)$ | 参数趋近0但不为0 |
| **L1正则化** | $\|\phi\|_1$ | 拉普拉斯先验 $\text{Lap}(0, b)$ | **稀疏性**→参数可精确为0 |

> 📌 **记忆**：L2=高斯（圆润的钟形→参数缩到0附近但不为0）；L1=拉普拉斯（尖锐的峰→把参数压到精确0）。

### 2.4 L2正则化与梯度更新

优化目标：$J(\phi) = \mathcal{L}(\phi) + \frac{\lambda}{2}\|\phi\|_2^2$

梯度更新：

$$\phi_{t+1} = \phi_t - \eta(\nabla_\phi \mathcal{L}(\phi_t) + \lambda \phi_t) = \underbrace{(1 - \eta\lambda)}_{\text{权重衰减}} \phi_t - \eta \nabla_\phi \mathcal{L}(\phi_t)$$

三项物理意义：

| 效应 | 解释 |
|------|------|
| **权重衰减** | 每轮迭代前参数自发按比例缩减，抑制"虚假"权重增长 |
| **Lipschitz约束** | 抑制权重绝对值→降低函数斜率波动→增强模型平滑度 |
| **数值稳定性** | 避免某些维度梯度过大导致的训练不稳定 |

### 2.5 岭回归 (Ridge Regression) — L2的典型应用

模型：$f(x;\omega) = \sum_{k=0}^{K} \omega_k x^k$

损失：$J(\omega) = \|Z\omega - y\|_2^2 + \lambda\|\omega\|_2^2$

解析解：$\omega^* = (Z^TZ + \lambda I)^{-1}Z^Ty$

| 优势 | 解释 |
|------|------|
| **消除奇异性** | 样本量不足或共线性时 $Z^TZ$ 不可逆，$\lambda I$ 确保矩阵正定可逆 |
| **控制范数** | 防止多项式系数过大导致的 Runge 现象（高次多项式在边界剧烈震荡） |

## 三、隐式正则化

### 3.1 核心现象

即使不显式添加 L1/L2 正则项，模型依然展现良好泛化能力。这种泛化能力源于**优化算法本身**引入的归纳偏置。

| 视角 | 理解 |
|------|------|
| 传统 | 优化算法仅用于寻找损失函数的极小值 |
| 现代 | 离散更新轨迹（非零步长α）导致模型自发避开陡峭区域，等价于叠加了正则项 |

分析工具：**反向误差分析 (Backward Error Analysis)**

### 3.2 梯度下降的隐式正则化

#### 定理

考虑离散步 $\phi_{n+1} = \phi_n - \alpha \nabla L(\phi_n)$。当步长 $\alpha$ 较小时，迭代轨迹与以下修正损失函数的连续梯度流解路径在 $O(\alpha^2)$ 阶上一致：

$$\boxed{\tilde{L}_{GD}(\phi) = L(\phi) + \frac{\alpha}{4}\|\nabla L(\phi)\|^2}$$

**物理含义**：离散梯度下降不仅最小化 $L(\phi)$，同时也在**抑制梯度的模长**。

**泛化逻辑**：惩罚梯度模长→迫使模型趋向平滑区域→因为梯度剧烈（陡峭）的区域通常对应高梯度模长。

#### 证明思路（4步）

**Step 1**：设修正连续方程 $\dot{\phi} = g(\phi) + \alpha g_1(\phi)$，其中 $g = -\nabla L$

**Step 2**：连续路径的泰勒展开

$$\phi(t+\alpha) = \phi(t) + \alpha\dot{\phi} + \frac{\alpha^2}{2}\ddot{\phi} + O(\alpha^3)$$

**Step 3**：匹配离散迭代步——为使误差降至 $O(\alpha^3)$，二阶项系数必须抵消：

$$\alpha^2 g_1 + \frac{\alpha^2}{2}(\nabla g \cdot g) = 0 \implies g_1 = -\frac{1}{2}(\nabla g \cdot g)$$

**Step 4**：将 $g = -\nabla L$ 代入并转化为势函数：

$$g_1 = -\frac{1}{2}\nabla^2 L \cdot \nabla L = -\nabla\left(\frac{1}{4}\|\nabla L\|^2\right)$$

因此：$\dot{\phi} = -\nabla L - \alpha \nabla\left(\frac{1}{4}\|\nabla L\|^2\right) = -\nabla\left(L + \frac{\alpha}{4}\|\nabla L\|^2\right)$

> 📌 **一句话**：离散GD实际在最小化 $L + \frac{\alpha}{4}\|\nabla L\|^2$，多出来的那项就是"惩罚陡峭"。

### 3.3 随机梯度下降的隐式正则化

#### 定理

SGD 更新 $\phi_{n+1} = \phi_n - \alpha \nabla L_{B_n}(\phi_n)$，其期望路径在 $O(\alpha^2)$ 阶上等价于：

$$\boxed{\tilde{L}_{SGD}(\phi) = L(\phi) + \frac{\alpha}{4}\|\nabla L(\phi)\|^2 + \frac{\alpha}{4}\text{Var}[\nabla L_B(\phi)]}$$

其中 $\text{Var}[\nabla L_B(\phi)] = \mathbb{E}_B[\|\nabla L_B(\phi) - \nabla L(\phi)\|^2]$。

#### 与GD的对比

| | 修正损失 |
|---|---------|
| **GD** | $L + \frac{\alpha}{4}\|\nabla L\|^2$ |
| **SGD** | $L + \frac{\alpha}{4}\|\nabla L\|^2 + \frac{\alpha}{4}\text{Var}[\nabla L_B]$ |

SGD 比 GD 多了一项**梯度方差项**。

#### 证明关键步骤

**Step 1**：定义梯度随机变量 $g = \nabla L$（全量），$g_B = \nabla L_B$（小批量）。满足：

$$\mathbb{E}_B[g_B] = g, \quad \mathbb{E}_B[\|g_B\|^2] = \|g\|^2 + \text{Var}(g_B)$$

**Step 2**：期望离散更新量 $\mathbb{E}_B[\Delta\phi] = -\alpha g(\phi_n)$

**Step 3**：连续解的二阶演化 $\phi(t+\alpha) - \phi(t) = \alpha\dot{\phi} + \frac{\alpha^2}{2}\ddot{\phi} + O(\alpha^3)$

**Step 4**：处理二阶项

$$\mathbb{E}_B\left[\frac{\alpha^2}{2}\frac{d}{dt}(-g_B)\right] = \frac{\alpha^2}{2}\mathbb{E}_B[\nabla(\frac{1}{2}\|g_B\|^2)] = \frac{\alpha^2}{4}\nabla(\|g\|^2 + \text{Var}(g_B))$$

**Step 5**：合并求导，解出连续流漂移项

$$f(\phi) = -g - \frac{\alpha}{4}\nabla(\|g\|^2 + \text{Var}(g_B)) = -\nabla\left(L + \frac{\alpha}{4}\|\nabla L\|^2 + \frac{\alpha}{4}\text{Var}(g_B)\right)$$

### 3.4 梯度方差项的物理含义

$$\frac{\alpha}{4}\text{Var}[\nabla L_B] = \frac{\alpha}{4}\frac{1}{|B|}\sum_{i \in B}\|\nabla\ell_i - \nabla L\|^2$$

- **惩罚不一致性**：如果不同样本提供的梯度方向冲突，该项很大
- **SGD避开冲突区域**：寻找所有样本"达成共识"的区域
- **平坦极小值**：各样本意见一致的区域通常更平滑，对噪声鲁棒

### 3.5 超参数权衡

| 超参数 | 变化 | 对隐式正则的影响 |
|--------|------|----------------|
| 学习率 $\alpha$ ↑ | 正则更强 | $\frac{\alpha}{4}$ 系数变大 |
| 批量大小 $|B|$ ↓ | 方差大→正则更强 | 小批量=更多噪声=更多正则 |

> 📌 **一句话**：大学习率 + 小批量 = 更强的隐式正则化。这也是为什么实践中小批量SGD往往泛化更好。

## 四、其他启发式方法

### 4.1 Early Stopping（提前停止）

**核心思想**：在训练完全收敛之前适时终止训练——模型已充分学习数据模式但尚未开始拟合噪声。

| 解释视角 | 机理 |
|---------|------|
| **正则化解释** | 权重初始化为小随机值，提前停止限制权重过度增长，效果类似 L2 权重衰减 |
| **模型复杂度解释** | 控制模型的有效容量，在偏差-方差权衡曲线上达到更优位置 |

**标准流程**：
1. 定期（每T次迭代）在验证集上评估
2. 保存性能最佳的模型参数副本
3. 最终选择验证集上表现最优的模型版本

> 📌 实验观察：浅层网络训练中，中间阶段(c-d)的模型泛化最优，后期(e-f)明显过拟合。**训练损失持续下降≠模型越来越好**。

### 4.2 集成方法 (Ensemble Methods)

**核心**：组合多个基模型的预测结果，有效减小泛化误差。是最可靠的性能提升技术之一。

#### 预测策略

| 任务 | 平均法 | 投票法 |
|------|--------|--------|
| 回归 | 直接平均各模型输出 | 取预测值中位数 |
| 分类 | 平均softmax前的logits | 多数表决 |

#### 训练策略

| 策略 | 方法 | 优势 |
|------|------|------|
| **随机初始化集成** | 不同初始化点探索参数空间 | 改善训练数据稀疏区域的预测 |
| **Bagging** | 有放回采样构建多个训练子集 | 增强数据扰动鲁棒性，隐式数据平滑 |

> 📌 实验结论：集成预测比单一模型更平滑、泛化更好、抗过拟合更强。代价是额外计算和存储。

### 4.3 Dropout

**机制**：训练时以固定概率（通常50%）随机将部分隐藏单元置零。

**三大关键优势**：

| 优势 | 解释 |
|------|------|
| **降低协同适应** | 防止神经元之间形成过度依赖的"小团伙" |
| **隐式L2正则化** | 等价于对权重施加L2约束 |
| **增强鲁棒性** | 每次只用子网络，强迫每个神经元独立有用 |

**动力学解释**（实验验证）：
- 全批次GD→非理想的"扭结"现象
- Dropout移除产生扭结的神经元→函数形态变平滑
- 2000次迭代后→有效消除冗余结构

> 📌 **直觉**：Dropout像是在训练一支"每个人都得能独当一面"的团队——不能指望队友随时在，所以每个人都得学会独立完成任务。

### 4.4 噪声注入

| 注入位置 | 方法 | 正则化效果 |
|---------|------|-----------|
| **输入噪声** | 对输入加随机扰动 | 等价于对输入梯度施加正则化约束；极端=对抗训练 |
| **权重噪声** | 对权重加随机扰动 | 促使收敛到**平坦极小值**，提升参数扰动鲁棒性 |
| **标签扰动** | 随机标签噪声 / 标签平滑 | 防止模型过度自信(over-confident) |

**输入噪声的实验规律**：噪声方差 $\sigma_x^2$ 递增→函数拟合更平滑→与L2正则化效果类似。

### 4.5 贝叶斯推断

#### MLE的局限性

- 仅考虑单一最优参数点估计，忽略参数不确定性
- 可能导致过度自信(overconfident)的预测结果

#### 贝叶斯学习框架

将参数 $\phi$ 视为随机变量，计算后验分布：

$$p(\phi|D) = \frac{\prod_{i=1}^{I} p(y_i|x_i,\phi)p(\phi)}{\int \prod_{i=1}^{I} p(y_i|x_i,\phi)p(\phi) d\phi}$$

| 组件 | 作用 |
|------|------|
| 先验 $p(\phi)$ | 编码领域知识 |
| 证据项（分母） | 概率归一化 |

#### 贝叶斯预测

$$p(y|x,D) = \int p(y|x,\phi)p(\phi|D)d\phi$$

- 通过参数空间积分实现**模型平均**
- 自然产生**不确定性估计**

#### 实践挑战

| 挑战 | 原因 |
|------|------|
| 计算复杂性 | 高维参数空间精确积分不可行 |
| 证据项计算 | 涉及棘手积分 |
| 近似方法 | 变分推断、MCMC采样、Laplace近似 |
| 计算代价 | 显著增加训练和推理复杂度 |

> 📌 实验观察：小方差先验→更平滑的函数；贝叶斯模型平均→预测均值（青色曲线）+ 2σ置信区间（灰色区域）。

### 4.6 迁移学习与多任务学习

#### 迁移学习

**两阶段训练**：
1. **源任务预训练**：学习通用特征表示
2. **目标任务微调**：适应特定需求

**实现方式**：特征提取器冻结 / 全网络微调

#### 多任务学习

| 优势 | 解释 |
|------|------|
| **隐式数据增强** | 共享表征学习，任务间正则化效应 |
| **计算效率** | 单一模型服务多目标，参数共享 |

> 📌 迁移学习=一个人先学通用技能再专精；多任务学习=同时学多个相关技能互相促进。

### 4.7 自监督学习

**核心**：解决标注数据稀缺——通过 pretext tasks 自动生成监督信号。

| 方法 | 核心思想 | 典型应用 |
|------|---------|---------|
| **生成式** | 通过掩码重建学习数据内在结构 | 图像修复(Inpainting)、文本填充(MLM) |
| **对比式** | 学习样本间的相似性关系 | 正样本对（语义相关）、负样本对（语义无关） |

**对比式关键组件**：
- 正样本对：语义相关的数据视图（如同一图像的不同裁剪）
- 负样本对：语义无关的随机组合

> 📌 生成式=补全缺失部分；对比式=分辨谁和谁是同一类。无需人工标注即可构建判别性特征空间。

### 4.8 数据增强

**核心原则**：保持语义不变性 + 增加分布多样性

| 数据类型 | 典型增强操作 |
|---------|------------|
| 视觉 | 几何变换、色彩调整、随机遮挡 |
| 文本 | 同义词替换、回译增强 |
| 语音 | 时频域变换、噪声注入 |

**三维数据扩展策略**：
- 跨数据集 → 迁移学习
- 跨任务 → 多任务学习
- 单样本扩展 → 数据增强

## 五、核心公式速查

| 公式 | 内容 | 一句话总结 |
|------|------|-----------|
| **SRM** | $\min_\phi [\mathcal{L}_{emp}(\phi) + \lambda\Omega(\phi)]$ | 经验损失 + 复杂度惩罚 |
| **L2正则化** | $\Omega(\phi) = \frac{1}{2}\|\phi\|_2^2$ | 等价高斯先验，权重衰减 |
| **L1正则化** | $\Omega(\phi) = \|\phi\|_1$ | 等价拉普拉斯先验，稀疏性 |
| **L2梯度更新** | $\phi_{t+1} = (1-\eta\lambda)\phi_t - \eta\nabla\mathcal{L}$ | 每步先缩再走 |
| **岭回归解** | $\omega^* = (Z^TZ+\lambda I)^{-1}Z^Ty$ | λI保证可逆 |
| **GD隐式正则** | $\tilde{L}_{GD} = L + \frac{\alpha}{4}\|\nabla L\|^2$ | 惩罚梯度模长→避陡峭 |
| **SGD隐式正则** | $\tilde{L}_{SGD} = L + \frac{\alpha}{4}\|\nabla L\|^2 + \frac{\alpha}{4}\text{Var}[\nabla L_B]$ | 比GD多惩罚梯度方差 |
| **贝叶斯后验** | $p(\phi|D) \propto p(D|\phi)p(\phi)$ | 似然×先验 |

## 六、方法对比总览

| 方法 | 类型 | 机制 | 代价 | 适用场景 |
|------|------|------|------|---------|
| L2正则化 | 显式 | 权重衰减→平滑 | 需调λ | 通用，几乎必用 |
| L1正则化 | 显式 | 稀疏性→特征选择 | 需调λ | 需要特征筛选 |
| GD隐式正则 | 隐式 | 惩罚梯度模长 | 无额外代价 | 大学习率时更强 |
| SGD隐式正则 | 隐式 | 惩罚梯度方差 | 无额外代价 | 小批量时更强 |
| Early Stopping | 启发式 | 限制权重增长 | 需验证集 | 通用 |
| 集成方法 | 启发式 | 模型平均 | N倍计算存储 | 追求极致性能 |
| Dropout | 启发式 | 破坏协同适应 | 训练慢~2倍 | 全连接层标配 |

## 七、易错点与直觉理解

### 常见错误

| 错误 | 正确理解 |
|------|---------|
| 正则化是让训练误差变小 | 正则化牺牲训练误差来降低测试误差——以Bias换Variance |
| L1和L2效果差不多 | L1产生稀疏解（特征选择），L2产生小而分散的解（权重衰减） |
| 隐式正则化不如显式正则化 | SGD的隐式正则化在实践中非常有效，甚至优于某些显式方法 |
| Dropout只在训练时用 | 推理时不做Dropout，但需要将权重乘以保留概率(或训练时做inverted dropout) |
| 集成总是更好 | 集成提升有限，且计算代价线性增长，不一定划算 |
| 大批量训练更快所以更好 | 大批量减少隐式正则→泛化可能更差，需要调整学习率等策略补偿 |

### 关键直觉

1. **正则化 = 给模型加"约束"**：就像考试限时——限制反而逼迫出更好的策略
2. **L2 = 软约束**：把参数往0拉但不到0（橡皮筋）；**L1 = 硬约束**：直接把不重要的参数压到0（剪刀）
3. **SGD隐式正则 = 民主投票**：小批量噪声让模型寻找"各样本共识"的解，而非少数样本说了算
4. **Dropout = 团队不能靠单点**：每个人必须独立能干，不能过度依赖队友
5. **Early Stopping = 见好就收**：继续练反而走火入魔
