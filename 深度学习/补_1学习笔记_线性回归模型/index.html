    线性回归模型 - 学习笔记（完整版）¶

    本文档详细讲解 01_线性回归模型.ipynb 的每个模块。

    一、概述¶

    这个 notebook 实现了一个线性基函数回归模型，可以： - 使用多种基函数（多项式、高斯、Sigmoid）拟合数据 - 通过解析解或梯度下降两种方式求解 - 支持 L2 正则化防止过拟合

    class LinearRegression:
    def __init__(self, basis_type='polynomial', M=10, l2_lambda=0, **kwargs):
        self.basis_type = basis_type  # 基函数类型
        self.M = M                     # 基函数个数
        self.l2_lambda = l2_lambda     # 正则化系数
        self.w = None                  # 模型权重（训练后才有）
        self.params = kwargs           # 基函数的额外参数

    参数详解¶

    参数 默认值 含义 怎么选
    basis_type 'polynomial' 基函数类型 看数据形状选择
    M 10 基函数个数 越多越灵活，但容易过拟合
    l2_lambda 0 正则化强度 过拟合时加大（0.01, 0.1, 1...）
    **kwargs - 额外参数 高斯传 sigma，Sigmoid 传 c

    为什么要正则化？¶

    问题：M=10 意味着有 10 个参数 \(w_0, w_1, ..., w_9\) 可以调。参数太多会怎样？

    1
2
3
4
    训练数据：5个点
模型：10个参数

结果：曲线穿过每个点，但在点之间疯狂震荡 → 死记硬背，泛化能力差

    正则化的作用：在损失函数里加一项 \(\frac{\lambda}{2} \sum w_i^2\)

    • 原损失 = 数据误差
    • 新损失 = 数据误差 + λ × (权重平方和)
    • 效果：惩罚过大的权重 → 曲线更平滑 → 泛化更好

    打个比方：正则化就像给模型戴上"紧箍咒"，让它不能太任性。

    def _transform(self, X):
    """将原始输入 X 转换至基函数空间 Z (设计矩阵)"""
    N = X.shape[0]
    X = X.flatten()
    Z = np.zeros((N, self.M))

    # 第一个基函数通常取 phi0(x) = 1 (常数项/偏置)
    Z[:, 0] = 1.0

    if self.basis_type == 'polynomial':
        for j in range(1, self.M):
            Z[:, j] = np.power(X, j)

    elif self.basis_type == 'gaussian':
        sigma = self.params.get('sigma', 0.1)
        mus = np.linspace(0, 1, self.M - 1)
        for j in range(1, self.M):
            Z[:, j] = np.exp(-0.5 * (X - mus[j-1])**2 / sigma**2)

    elif self.basis_type == 'sigmoid':
        c = self.params.get('c', 10.0)
        ms = np.linspace(0, 1, self.M - 1)
        for j in range(1, self.M):
            Z[:, j] = 1 / (1 + np.exp(-c * (X - ms[j-1])))

    return Z

    为什么第一列是 1？¶

    1
    Z[:, 0] = 1.0

    这是偏置项（bias），相当于直线方程 \(y = b + wx\) 中的 \(b\)。

    没有这一项的话，模型必须经过原点，拟合能力受限。

    三种基函数详解¶

    1. 多项式基函数¶

Z[:, 1] = x           # φ₁(x) = x
Z[:, 2] = x²          # φ₂(x) = x²
Z[:, 3] = x³          # φ₃(x) = x³

    效果： - M=3：\(y = w_0 + w_1 x + w_2 x^2\) → 抛物线 - M=5：能拟合更复杂的波浪

    什么时候用：数据有明显的多项式趋势（比如抛物线轨迹）

    注意：高次多项式容易在边界处爆炸（Runge 现象）

    4. 基函数拟合过程详解¶
    核心思想：用"积木"搭建任意曲线¶

    问题：线性模型 \(y = wx + b\) 只能拟合直线，怎么办？

    解决：把 \(x\) 变成一组基函数 \(\phi_1(x), \phi_2(x), ..., \phi_M(x)\)

    \[y = w_0 + w_1 \phi_1(x) + w_2 \phi_2(x) + ... + w_M \phi_M(x)\]

    本质：每个基函数是一块"积木"，加权求和后拼出目标曲线。

    高斯基函数拟合过程¶

    步骤1：构造设计矩阵

    假设有 \(N\) 个数据点 \((x_i, y_i)\)，构造设计矩阵 \(\Phi\)：

    \[\Phi = \begin{bmatrix} 1 & \phi_1(x_1) & \phi_2(x_1) & \cdots & \phi_M(x_1) \\ 1 & \phi_1(x_2) & \phi_2(x_2) & \cdots & \phi_M(x_2) \\ \vdots & \vdots & \vdots & \ddots & \vdots \\ 1 & \phi_1(x_N) & \phi_2(x_N) & \cdots & \phi_M(x_N) \end{bmatrix}\]

    每列是一个基函数在所有数据点上的值。

    步骤2：求解权重

    目标是最小化 \(\|y - \Phi w\|^2\)，解析解为：

    \[w = (\Phi^T \Phi + \lambda I)^{-1} \Phi^T y\]

    # 判断数据类型的方法
def analyze_data_shape(X, y):
    """分析数据形态，推荐基函数类型"""

    dy = np.diff(y)  # 计算一阶差分
    sign_changes = np.sum(np.diff(np.sign(dy)) != 0)
    jumps = np.abs(dy) > 2 * np.std(dy)

    if np.sum(jumps) > len(dy) * 0.1:
        return "建议用 Sigmoid 基函数（台阶状数据）"
    elif sign_changes > len(dy) * 0.2:
        return "建议用高斯基函数（多峰数据）"
    else:
        return "可以尝试两种基函数对比效果"

    一句话总结： - 高斯基函数像橡皮泥，可以捏出各种光滑形状 - Sigmoid 基函数像积木台阶，适合搭建有高低落差的结构

    def fit_analytical(self, X, t):
    """使用解析解求解 (正规方程)"""
    Z = self._transform(X)
    I = np.eye(self.M)
    self.w = np.linalg.solve(Z.T @ Z + self.l2_lambda * I, Z.T @ t)

    数学推导¶

    目标：找到 \(w\) 使得预测值 \(y = Zw\) 尽可能接近真实值 \(t\)

    损失函数（带正则化）： $\(L(w) = \frac{1}{2}(Zw - t)^T(Zw - t) + \frac{\lambda}{2}w^T w\)$

    对 \(w\) 求导，令导数为 0： $\(\frac{\partial L}{\partial w} = Z^T(Zw - t) + \lambda w = 0\)$

    整理得到正规方程： $\(w = (Z^T Z + \lambda I)^{-1} Z^T t\)$

    代码对应¶

    1
2
3
4
    Z.T @ Z           # ZᵀZ
Z.T @ t           # Zᵀt
self.l2_lambda * I  # λI
np.linalg.solve(A, b)  # 求解 Aw = b，比直接求逆更稳定

    为什么用 np.linalg.solve 而不是 np.linalg.inv？¶

    1
2
3
4
5
    # 方法1：直接求逆（不推荐）
w = np.linalg.inv(Z.T @ Z + self.l2_lambda * I) @ Z.T @ t

# 方法2：求解线性方程组（推荐）
w = np.linalg.solve(Z.T @ Z + self.l2_lambda * I, Z.T @ t)

    原因： - 求逆数值不稳定，容易放大误差 - solve 内部用更稳定的算法（如 LU 分解）

    def fit_gd(self, X, t, learning_rate=0.1, epochs=1000, batch_size=None):
    """使用小批量梯度下降求解"""
    Z = self._transform(X)
    N, M = Z.shape
    self.w = np.zeros(M)

    if batch_size is None:
        batch_size = N

    for epoch in range(epochs):
        # 学习率衰减
        lr = learning_rate / (1 + 0.0001 * epoch)

        # 打乱数据
        indices = np.random.permutation(N)

        for i in range(0, N, batch_size):
            idx = indices[i:i+batch_size]
            ZB, tB = Z[idx], t[idx]

            # 计算预测值和误差
            y_pred = ZB @ self.w
            error = y_pred - tB

            # 计算梯度
            gradient = (ZB.T @ error) / len(idx) + self.l2_lambda * self.w

            # 更新参数
            self.w -= lr * gradient

        # 打印损失（每 100 轮）
        if (epoch + 1) % 100 == 0 or epoch == 0:
            y_all_pred = Z @ self.w
            data_loss = np.mean((y_all_pred - t)**2) / 2.0
            reg_loss = 0.5 * self.l2_lambda * np.sum(self.w**2)
            print(f"Epoch {epoch+1}: Loss = {data_loss + reg_loss:.6f}")

    参数详解¶

    参数 默认值 含义
    learning_rate 0.1 学习率，步子大小
    epochs 1000 训练轮数
    batch_size None 每批样本数，None=全量

    三种梯度下降对比¶

    类型 batch_size 特点
    批量梯度下降 batch_size=N 每次用全部数据，稳定但慢
    随机梯度下降 batch_size=1 每次用一个样本，快但抖
    小批量梯度下降 batch_size=32~256 折中，最常用

    梯度公式推导¶

    损失函数： $\(L = \frac{1}{2N}\sum_{i=1}^{N}(y_i - t_i)^2 + \frac{\lambda}{2}\sum_{j}w_j^2\)$

    对 \(w\) 求导： $\(\frac{\partial L}{\partial w} = \frac{1}{N}Z^T(Zw - t) + \lambda w\)$

    代码中： 

    1
2
    gradient = (ZB.T @ error) / len(idx) + self.l2_lambda * self.w
#          ↑ 数据梯度               ↑ 正则化梯度

    学习率衰减¶

    1
    lr = learning_rate / (1 + 0.0001 * epoch)

    效果： - 第 1 轮：lr = 0.1 - 第 1000 轮：lr ≈ 0.1 / 1.1 ≈ 0.09 - 第 10000 轮：lr ≈ 0.05

    为什么要衰减？ - 初期：大步走，快速接近最优解 - 后期：小步走，避免在最优解附近震荡

    def predict(self, X):
    """预测输出"""
    Z = self._transform(X)
    return Z @ self.w

    数学含义¶

    \[y = Z \cdot w = \sum_{j=0}^{M-1} w_j \phi_j(x)\]

    展开： $\(y = w_0 \cdot 1 + w_1 \phi_1(x) + w_2 \phi_2(x) + ... + w_{M-1} \phi_{M-1}(x)\)$

    import numpy as np
import matplotlib.pyplot as plt

# 生成数据
np.random.seed(42)
X = np.linspace(0, 1, 50).reshape(-1, 1)
t = np.sin(2 * np.pi * X).flatten() + np.random.normal(0, 0.1, 50)

y_pred = model.predict(X_test)

# 绑图
plt.scatter(X, t, label='训练数据')
plt.plot(X_test, y_pred, 'r-', label='预测')
plt.legend()
plt.show()

    四、总结对比¶

    方法 优点 缺点 什么时候用
    解析解 一步到位，精确 数据量大时慢（\(O(M^3)\)） 样本数 < 10000
    梯度下降 可扩展到大数据 需要调参，可能陷入局部最优 样本数 > 10000

    五、基函数选择指南¶

    数据特征 推荐基函数 参数建议
    单调变化 polynomial M=3~5
    抛物线/波浪 polynomial M=3~7
    多个峰谷 gaussian M=10, sigma=0.1
    有台阶/跳跃 sigmoid M=10, c=10
    不知道形状 gaussian 先试 M=10

    六、常见问题¶

    Q1：为什么拟合效果不好？¶

    可能原因： 1. 基函数类型不对：比如数据是抛物线，却用了 Sigmoid 2. M 太小：模型复杂度不够 3. 正则化太强：l2_lambda 太大，压制了模型 4. 数据没归一化：如果 x 范围是 [0, 1000]，高斯基函数会失效

    Q2：为什么过拟合？¶

    可能原因： 1. M 太大：基函数太多 2. 数据太少：样本数 < 参数数 3. 没有正则化：l2_lambda=0

    解决方法：加大 l2_lambda（如 0.01, 0.1, 1）

    Q3：梯度下降不收敛？¶

    可能原因： 1. 学习率太大：减小 learning_rate 2. 数据没归一化：确保 x 在 [0, 1] 范围内 3. 迭代次数不够：增加 epochs

    def create_toy_data(f, sample_size, std=0.25, noise=True):
    x = np.linspace(0, 1, sample_size).reshape(-1, 1)
    if noise:
        y = f(x) + np.random.normal(scale=std, size=x.shape)
    else:
        y = f(x)            
    return x, y.flatten()

def f(x):
    return np.sin(2 * np.pi * x)

    逐行解释¶

    代码 含义
    np.linspace(0, 1, sample_size) 在 [0, 1] 区间均匀生成 sample_size 个点
    .reshape(-1, 1) 变成列向量（N行1列），符合 sklearn 风格
    f(x) 真实函数 \(\sin(2\pi x)\)
    np.random.normal(scale=std, ...) 添加高斯噪声，标准差为 std
    .flatten() 把 y 压成一维数组

    为什么要加噪声？¶

    真实数据都有噪声。如果不加噪声，模型会死记硬背，无法测试泛化能力。

    举个例子： 

    1
2
    无噪声：所有点都在完美正弦曲线上 → 模型只需插值
有噪声：点在曲线附近随机波动 → 模型要学会"忽略噪声"，找到真实趋势

    7.3 不同多项式次数对比¶

    1

    degrees = [0, 1, 6, 9]

for i, degree in enumerate(degrees):
    model = LinearRegression(basis_type='polynomial', M=degree+1, l2_lambda=0)
    model.fit_analytical(x_train, y_train)
    y_pred = model.predict(x_test)

    为什么要对比不同次数？¶

    degree (M-1) 模型 拟合能力 预期结果
    0 \(y = w_0\) 只能是一条水平线 欠拟合
    1 \(y = w_0 + w_1 x\) 直线 欠拟合
    6 \(y = w_0 + w_1 x + ... + w_6 x^6\) 可以弯曲 比较合适
    9 10个参数 非常灵活 可能过拟合（训练数据只有10个点）

    为什么是 M = degree + 1？¶

    • degree = 0 → M = 1 → 只有 \(\phi_0(x) = 1\)（常数项）
    • degree = 1 → M = 2 → \(\phi_0(x) = 1, \phi_1(x) = x\)
    • degree = 9 → M = 10 → \(\phi_0(x) = 1, \phi_1(x) = x, ..., \phi_9(x) = x^9\)

    7.4 RMSE 误差函数¶

    1
2
3
    def rmse(a, b):
    """Rooted Mean Squared Error"""
    return np.sqrt(np.mean(np.square(a - b)))

    数学公式¶

    \[\text{RMSE} = \sqrt{\frac{1}{N}\sum_{i=1}^{N}(a_i - b_i)^2}\]

    为什么要用 RMSE？¶

    指标 公式 特点
    MSE \(\frac{1}{N}\sum(y - \hat{y})^2\) 平方后单位和 y 不同
    RMSE \(\sqrt{\text{MSE}}\) 单位和 y 相同，更容易解释
    MAE $\frac{1}{N}\sum y - \hat{y}

    例子：如果 y 是房价（万元），RMSE=5 意味着"平均误差约5万元"。

    参数选择建议¶

    参数 本例设置 一般建议
    learning_rate 0.001 从 0.1 开始，不收敛就减小
    epochs 5000 看损失曲线，稳定了就停
    batch_size 5 小数据集用小 batch，大数据集用 32~256

    什么时候用梯度下降？¶

    • 数据量大（>10000样本）：解析解太慢
    • 在线学习：数据源源不断，无法一次性加载
    • 需要正则化：某些正则化无法用解析解
