## 一、概述

这个 notebook 实现了一个**线性基函数回归模型**，可以：
- 使用多种基函数（多项式、高斯、Sigmoid）拟合数据
- 通过解析解或梯度下降两种方式求解
- 支持 L2 正则化防止过拟合

### 模块2：类定义与初始化 `__init__`

```python
class LinearRegression:
    def __init__(self, basis_type='polynomial', M=10, l2_lambda=0, **kwargs):
        self.basis_type = basis_type  # 基函数类型
        self.M = M                     # 基函数个数
        self.l2_lambda = l2_lambda     # 正则化系数
        self.w = None                  # 模型权重（训练后才有）
        self.params = kwargs           # 基函数的额外参数
```

#### 参数详解

| 参数 | 默认值 | 含义 | 怎么选 |
|-----|-------|------|-------|
| `basis_type` | `'polynomial'` | 基函数类型 | 看数据形状选择 |
| `M` | `10` | 基函数个数 | 越多越灵活，但容易过拟合 |
| `l2_lambda` | `0` | 正则化强度 | 过拟合时加大（0.01, 0.1, 1...） |
| `**kwargs` | - | 额外参数 | 高斯传 `sigma`，Sigmoid 传 `c` |

#### 为什么要正则化？

**问题**：M=10 意味着有 10 个参数 $w_0, w_1, ..., w_9$ 可以调。参数太多会怎样？

```
训练数据：5个点
模型：10个参数

结果：曲线穿过每个点，但在点之间疯狂震荡 → 死记硬背，泛化能力差
```

**正则化的作用**：在损失函数里加一项 $\frac{\lambda}{2} \sum w_i^2$

- 原损失 = 数据误差
- 新损失 = 数据误差 + λ × (权重平方和)
- 效果：惩罚过大的权重 → 曲线更平滑 → 泛化更好

**打个比方**：正则化就像给模型戴上"紧箍咒"，让它不能太任性。

### 模块3：基函数变换 `_transform`

```python
def _transform(self, X):
    """将原始输入 X 转换至基函数空间 Z (设计矩阵)"""
    N = X.shape[0]
    X = X.flatten()
    Z = np.zeros((N, self.M))
    
    # 第一个基函数通常取 phi0(x) = 1 (常数项/偏置)
    Z[:, 0] = 1.0
    
    if self.basis_type == 'polynomial':
        for j in range(1, self.M):
            Z[:, j] = np.power(X, j)
            
    elif self.basis_type == 'gaussian':
        sigma = self.params.get('sigma', 0.1)
        mus = np.linspace(0, 1, self.M - 1)
        for j in range(1, self.M):
            Z[:, j] = np.exp(-0.5 * (X - mus[j-1])**2 / sigma**2)
            
    elif self.basis_type == 'sigmoid':
        c = self.params.get('c', 10.0)
        ms = np.linspace(0, 1, self.M - 1)
        for j in range(1, self.M):
            Z[:, j] = 1 / (1 + np.exp(-c * (X - ms[j-1])))
            
    return Z
```

#### 输入输出是什么？

```
输入 X: [x1, x2, x3, ..., xN]  形状 (N,) 或 (N,1)
输出 Z: 设计矩阵              形状 (N, M)

Z 的每一列是一个基函数对所有样本的输出
```

#### 为什么第一列是 1？

```python
Z[:, 0] = 1.0
```

这是**偏置项**（bias），相当于直线方程 $y = b + wx$ 中的 $b$。

没有这一项的话，模型必须经过原点，拟合能力受限。

#### 三种基函数详解

##### 1. 多项式基函数

```python
Z[:, 0] = 1           # φ₀(x) = 1      (偏置项)
Z[:, 1] = x           # φ₁(x) = x
Z[:, 2] = x²          # φ₂(x) = x²
Z[:, 3] = x³          # φ₃(x) = x³
...
```

**效果**：
- M=3：$y = w_0 + w_1 x + w_2 x^2$ → 抛物线
- M=5：能拟合更复杂的波浪

**什么时候用**：数据有明显的多项式趋势（比如抛物线轨迹）

**注意**：高次多项式容易在边界处爆炸（Runge 现象）

##### 2. 高斯基函数（RBF）

```python
mus = [0, 0.11, 0.22, 0.33, 0.44, 0.56, 0.67, 0.78, 0.89, 1.0]  # M-1个中心点
sigma = 0.1  # 控制宽度

φ_j(x) = exp(-(x - μ_j)² / (2σ²))
```

**效果**：每个基函数是一个"小山包"

```
        φ₁    φ₅    φ₉
         ∧     ∧     ∧
        / \   / \   / \
       /   \_/   \_/   \
    ──────────────────────→ x
    0    0.5    1.0
```

**参数 `sigma` 怎么选？**
- sigma 小：山包很尖，只能影响附近 → 可能欠拟合
- sigma 大：山包很宽，影响范围大 → 可能过拟合

**什么时候用**：数据有多个局部峰谷，或者分布不规则

##### 3. Sigmoid 基函数

```python
φ_j(x) = 1 / (1 + exp(-c(x - m_j)))
```

**效果**：每个基函数是一个"阶梯"

```
        φ₁    φ₅    φ₉
         |_____|_____|
         |           |
    _____|           |_____
    ──────────────────────→ x
    0    0.5    1.0
```

**参数 `c` 怎么选？**
- c 大：阶梯很陡，接近阶跃函数
- c 小：阶梯很缓，过渡平滑

**什么时候用**：数据有明显的"台阶"或"跳跃"

### 模块4：解析解求解 `fit_analytical`

```python
def fit_analytical(self, X, t):
    """使用解析解求解 (正规方程)"""
    Z = self._transform(X)
    I = np.eye(self.M)
    self.w = np.linalg.solve(Z.T @ Z + self.l2_lambda * I, Z.T @ t)
```

#### 数学推导

**目标**：找到 $w$ 使得预测值 $y = Zw$ 尽可能接近真实值 $t$

**损失函数**（带正则化）：
$$L(w) = \frac{1}{2}(Zw - t)^T(Zw - t) + \frac{\lambda}{2}w^T w$$

**对 $w$ 求导，令导数为 0**：
$$\frac{\partial L}{\partial w} = Z^T(Zw - t) + \lambda w = 0$$

**整理得到正规方程**：
$$w = (Z^T Z + \lambda I)^{-1} Z^T t$$

#### 代码对应

```python
Z.T @ Z           # ZᵀZ
Z.T @ t           # Zᵀt
self.l2_lambda * I  # λI
np.linalg.solve(A, b)  # 求解 Aw = b，比直接求逆更稳定
```

#### 为什么用 `np.linalg.solve` 而不是 `np.linalg.inv`？

```python
# 方法1：直接求逆（不推荐）
w = np.linalg.inv(Z.T @ Z + self.l2_lambda * I) @ Z.T @ t

# 方法2：求解线性方程组（推荐）
w = np.linalg.solve(Z.T @ Z + self.l2_lambda * I, Z.T @ t)
```

**原因**：
- 求逆数值不稳定，容易放大误差
- `solve` 内部用更稳定的算法（如 LU 分解）

### 模块5：梯度下降求解 `fit_gd`

```python
def fit_gd(self, X, t, learning_rate=0.1, epochs=1000, batch_size=None):
    """使用小批量梯度下降求解"""
    Z = self._transform(X)
    N, M = Z.shape
    self.w = np.zeros(M)
    
    if batch_size is None:
        batch_size = N
    
    for epoch in range(epochs):
        # 学习率衰减
        lr = learning_rate / (1 + 0.0001 * epoch)
        
        # 打乱数据
        indices = np.random.permutation(N)
        
        for i in range(0, N, batch_size):
            idx = indices[i:i+batch_size]
            ZB, tB = Z[idx], t[idx]
            
            # 计算预测值和误差
            y_pred = ZB @ self.w
            error = y_pred - tB
            
            # 计算梯度
            gradient = (ZB.T @ error) / len(idx) + self.l2_lambda * self.w
            
            # 更新参数
            self.w -= lr * gradient
        
        # 打印损失（每 100 轮）
        if (epoch + 1) % 100 == 0 or epoch == 0:
            y_all_pred = Z @ self.w
            data_loss = np.mean((y_all_pred - t)**2) / 2.0
            reg_loss = 0.5 * self.l2_lambda * np.sum(self.w**2)
            print(f"Epoch {epoch+1}: Loss = {data_loss + reg_loss:.6f}")
```

#### 参数详解

| 参数 | 默认值 | 含义 |
|-----|-------|------|
| `learning_rate` | 0.1 | 学习率，步子大小 |
| `epochs` | 1000 | 训练轮数 |
| `batch_size` | None | 每批样本数，None=全量 |

#### 三种梯度下降对比

| 类型 | batch_size | 特点 |
|-----|------------|------|
| 批量梯度下降 | batch_size=N | 每次用全部数据，稳定但慢 |
| 随机梯度下降 | batch_size=1 | 每次用一个样本，快但抖 |
| 小批量梯度下降 | batch_size=32~256 | 折中，最常用 |

#### 梯度公式推导

**损失函数**：
$$L = \frac{1}{2N}\sum_{i=1}^{N}(y_i - t_i)^2 + \frac{\lambda}{2}\sum_{j}w_j^2$$

**对 $w$ 求导**：
$$\frac{\partial L}{\partial w} = \frac{1}{N}Z^T(Zw - t) + \lambda w$$

**代码中**：
```python
gradient = (ZB.T @ error) / len(idx) + self.l2_lambda * self.w
#          ↑ 数据梯度               ↑ 正则化梯度
```

#### 学习率衰减

```python
lr = learning_rate / (1 + 0.0001 * epoch)
```

效果：
- 第 1 轮：lr = 0.1
- 第 1000 轮：lr ≈ 0.1 / 1.1 ≈ 0.09
- 第 10000 轮：lr ≈ 0.05

**为什么要衰减？**
- 初期：大步走，快速接近最优解
- 后期：小步走，避免在最优解附近震荡

### 模块6：预测 `predict`

```python
def predict(self, X):
    """预测输出"""
    Z = self._transform(X)
    return Z @ self.w
```

#### 数学含义

$$y = Z \cdot w = \sum_{j=0}^{M-1} w_j \phi_j(x)$$

**展开**：
$$y = w_0 \cdot 1 + w_1 \phi_1(x) + w_2 \phi_2(x) + ... + w_{M-1} \phi_{M-1}(x)$$

## 三、完整使用示例

```python
import numpy as np
import matplotlib.pyplot as plt

# 生成数据
np.random.seed(42)
X = np.linspace(0, 1, 50).reshape(-1, 1)
t = np.sin(2 * np.pi * X).flatten() + np.random.normal(0, 0.1, 50)

# 训练模型
model = LinearRegression(basis_type='gaussian', M=10, l2_lambda=0.01)
model.fit_analytical(X, t)  # 或 model.fit_gd(X, t)

# 预测
X_test = np.linspace(0, 1, 100).reshape(-1, 1)
y_pred = model.predict(X_test)

# 绑图
plt.scatter(X, t, label='训练数据')
plt.plot(X_test, y_pred, 'r-', label='预测')
plt.legend()
plt.show()
```

## 六、常见问题

### Q1：为什么拟合效果不好？

可能原因：
1. **基函数类型不对**：比如数据是抛物线，却用了 Sigmoid
2. **M 太小**：模型复杂度不够
3. **正则化太强**：l2_lambda 太大，压制了模型
4. **数据没归一化**：如果 x 范围是 [0, 1000]，高斯基函数会失效

### Q2：为什么过拟合？

可能原因：
1. **M 太大**：基函数太多
2. **数据太少**：样本数 < 参数数
3. **没有正则化**：l2_lambda=0

解决方法：加大 l2_lambda（如 0.01, 0.1, 1）

### Q3：梯度下降不收敛？

可能原因：
1. **学习率太大**：减小 learning_rate
2. **数据没归一化**：确保 x 在 [0, 1] 范围内
3. **迭代次数不够**：增加 epochs
