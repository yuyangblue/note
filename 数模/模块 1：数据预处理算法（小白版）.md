## 1.1 缺失值填充

### 🎬 先讲个故事

> 你拿到一张体检表，发现"体重"列有几个空位。
> 空着不填？模型会报错。瞎填？结果会偏。
>
> 该怎么填？看下面 👇

### 🧠 四个方法从简单到复杂

| 方法 | 适合情况 | 一句话 |
|------|---------|--------|
| **均值填充** 🟢 | 缺得少（<5%）、数据正常 | 拿剩下的数算个平均填进去 |
| **中位数填充** 🟢 | 缺得少但有异常值 | 排个序取中间那位，不怕极端值 |
| **KNN 填充** 🟡 | 缺 5%~20%、变量间有关系 | 找最像的几个人取均值 |
| **随机森林填充** 🔴 | 缺 > 20%、关系复杂 | 把空列当"考题"，其他列当"线索"来预测 |

### ⚠️ 小白翻车预警

- **均值填充会低估方差**！填完之后数据波动变小了，统计检验容易"不显著"
- KNN 一定要先**标准化**，不然身高cm（150~190）和收入元（3000~30000）没法一起算距离
- 随机森林填充**跑得慢**，时间不够就换 KNN

### 💻 代码速查

```python
from sklearn.impute import SimpleImputer

# 均值填充 — 1行搞定
df_filled = SimpleImputer(strategy='mean').fit_transform(df)

# 中位数填充 — 也是1行
df_filled = SimpleImputer(strategy='median').fit_transform(df)
```

```python
from sklearn.impute import KNNImputer
from sklearn.preprocessing import StandardScaler

# KNN三步走：标准化→填充→反标准化
scaler = StandardScaler()
df_scaled = scaler.fit_transform(df)
df_filled_scaled = KNNImputer(n_neighbors=5, weights='distance').fit_transform(df_scaled)
df_filled = scaler.inverse_transform(df_filled_scaled)
```

### 🏆 速查

> **缺 < 5% 且数据正常 → 均值（最快）**
> **缺 < 5% 但有异常值 → 中位数（最稳）**
> **缺 5%~20% 且有关联 → KNN（性价比高）**
> **缺 > 20% 或关系复杂 → 随机森林（最后手段）**

## 1.2 异常值识别

### 🎬 先讲个故事

> 班上同学身高都在 160~190cm，突然来了个 999cm 的"巨人"。
> 这明显是录入错误（把 199 打成了 999）。
>
> 异常值识别的任务就是：**把这种离谱的数据揪出来**。

### 🧠 四种方法

| 方法 | 原理 | 适合 |
|------|------|------|
| **IQR 箱线法** 🟢 | 超过 $Q_3+1.5\times IQR$ 或低于 $Q_1-1.5\times IQR$ 的就是异常 | 快速筛查，不需要正态分布 |
| **3-σ 准则** 🟡 | 超过均值 ± 3 个标准差的算异常 | 数据近似正态分布时用 |
| **Z-score** 🟡 | 和 3-σ 类似，但可以自定义阈值 | 标准化后的通用异常检测 |
| **DBSCAN** 🔴 | 密度聚类，落单的就是异常 | 多维度、复杂分布 |

### 📐 公式（瞄一眼就行）

**IQR 箱线法**：
- $Q_1$ = 数据排好序，第 25% 位置的值
- $Q_3$ = 第 75% 位置的值
- $IQR = Q_3 - Q_1$
- **异常值** = $x < Q_1 - 1.5\times IQR$ 或 $x > Q_3 + 1.5\times IQR$

> **人话**：正常数据集中在中间 50% 的区域，超出这个区域太多就不正常。

### 💻 代码速查

```python
# IQR法
Q1 = df['身高'].quantile(0.25)
Q3 = df['身高'].quantile(0.75)
IQR = Q3 - Q1
outliers = df[(df['身高'] < Q1 - 1.5*IQR) | (df['身高'] > Q3 + 1.5*IQR)]

# Z-score法
from scipy import stats
z_scores = np.abs(stats.zscore(df['身高']))
outliers = df[z_scores > 3]  # 超过3个标准差
```

### 🏆 选哪个？

| 条件 | 用 |
|------|----|
| 单变量、快速筛查 | IQR |
| 数据近似正态 | 3-σ / Z-score |
| 多变量、分布复杂 | DBSCAN |
| 有异常值干扰 | 修正 Z-score（用中位数替代均值） |

## 1.3 数据标准化

### 🎬 先讲个故事

> 你有两个特征：身高（150~190cm）和收入（3000~30000元）。
> 如果直接算距离，收入差距会把身高差距"淹没"——因为收入的数字大。
>
> 标准化的目的：**去掉量纲，让所有特征在同一个尺度上说话**。

### 🧠 两种主要方法

| 方法 | 效果 | 适用算法 |
|------|------|---------|
| **Min-Max 归一化** | 压缩到 [0,1] | 神经网络、图像处理 |
| **Z-score 标准化** | 变成均值为0、标准差为1 | SVM、PCA、KNN、回归 |

### 📐 公式

**Min-Max**：$\tilde{x} = \frac{x - x_{\min}}{x_{\max} - x_{\min}}$

> **人话**：看你在最大值和最小值之间的相对位置

**Z-score**：$z = \frac{x - \mu}{\sigma}$

> **人话**：看你距离均值有几个标准差

### ⚠️ 小白翻车预警

- Min-Max **怕异常值**——如果数据有个 999，其他数据全被压到 0~0.01 去了
- **不需要"数据是正态的"才能用 Z-score**，它只是一个线性变换，不改变分布形状
- **做完标准化要保存 scaler**，预测新数据时要用同一个 scaler 转换

### 💻 代码

```python
from sklearn.preprocessing import StandardScaler, MinMaxScaler

# Z-score标准化（推荐默认用这个）
scaler = StandardScaler()
df_scaled = scaler.fit_transform(df)

# Min-Max归一化
scaler = MinMaxScaler()
df_norm = scaler.fit_transform(df)
```

## 1.4 相关性分析

### 🎬 先讲个故事

> 身高越高的人，体重通常也越重——这就是"正相关"。
> 温度越高，暖气费越低——就是"负相关"。
> 鞋码和智商——这俩没关系。
>
> 相关性分析：**量化地说"这俩变量关系有多铁"**。

### 🧠 三种相关系数

| 方法 | 度量什么 | 适合场景 |
|------|---------|---------|
| **Pearson** 🟢 | **线性**关系的强弱 | 两个连续变量、大致呈直线关系 |
| **Spearman** 🟡 | **单调**关系的强弱 | 有异常值、或者关系不是直线但一直在变 |
| **灰色关联** 🔴 | 几何形状的相似度 | 数据很少（<30条）或信息不完全 |

### 🏆 速查

> **数据正常、线性关系 → Pearson**
> **有异常值或非线性关系 → Spearman**
> **样本很少、要做综合评价 → 灰色关联**

## 1.5 数据降维

### 🎬 先讲个故事

> 你有 50 个特征，但很多是重复的或者噪音。
> 想压缩到几个关键维度，又不损失太多信息。
>
> 降维就是：**用更少的特征，表达差不多的信息**。

### 🧠 三种方法

| 方法 | 类型 | 一句话 |
|------|------|--------|
| **PCA** 🟢 | 无监督 | 找方差最大的方向投影 |
| **LDA** 🟡 | 有监督（需要标签） | 让不同类分开、同类靠拢 |
| **因子分析** 🟡 | 无监督 | 找背后隐藏的"因素" |

### 📐 PCA 核心思想

1. 标准化数据
2. 找方差最大的方向 → 第一主成分（PC1）
3. 找与 PC1 正交且方差次大的方向 → PC2
4. 取前 k 个主成分，累计方差贡献 ≥ 85% 就够

### ⚠️ 小白翻车预警

- PCA **一定要先标准化**，不然量纲大的特征会主导结果
- PCA 得到的主成分**没有物理含义**（它是特征的线性组合），解释性差
- LDA 最多降到 $K-1$ 维（$K$ 是类别数）

### 🏆 选哪个？

| 场景 | 用 |
|------|----|
| 无标签、画图看数据分布 | PCA |
| 有标签、分类任务 | LDA |
| 需要解释潜在结构 | 因子分析 |

## 🏆 模块 1 总结

| 部分 | 核心方法 | 一句话 |
|------|---------|--------|
| **缺失值填充** | 均值 → 中位数 → KNN → 随机森林 | 缺得少简单填，缺得多上高级 |
| **异常值识别** | IQR → 3-σ → Z-score → DBSCAN | 单变量 IQR，多变量 DBSCAN |
| **数据标准化** | Min-Max / Z-score | 去掉量纲，一起算 |
| **相关性分析** | Pearson / Spearman / 灰色关联 | 看变量关系铁不铁 |
| **数据降维** | PCA / LDA / 因子分析 | 用更少维度表达差不多信息 |

> **预处理做得好，建模没烦恼 🍡**
