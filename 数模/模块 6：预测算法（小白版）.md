## 第 1 部分：ARIMA 模型

### 🎬 先讲个故事

> 你手头有过去 3 年每个月的奶茶销量数据。
> 老板问你：下个月备多少料？
>
> 你想了想：销量有趋势（越卖越多），有季节（夏天多冬天少），还有随机波动。
>
> ARIMA 就是干这个的——**把趋势、季节、随机波动拆开，然后算下一个月最可能是多少**。

### 🧠 大白话解释

ARIMA 是三个东西拼起来的：

| 字母 | 全称 | 大白话 |
|------|------|--------|
| **AR** | 自回归 | 明天的销量 ≈ 前几天的销量加权求和 |
| **I** | 差分 | 数据不平稳？做减法让它变平稳 |
| **MA** | 移动平均 | 明天的销量 ≈ 过去的"猜错了多少"来修正 |

记作 $\text{ARIMA}(p, d, q)$，其中 $p$ 是看过去几天，$d$ 是差几次，$q$ 是考虑几个预测误差。

### 📐 公式（瞄一眼就行）

**AR($p$)**：当前值 = 过去 $p$ 期的线性组合 + 噪声

$$X_t = c + \phi_1 X_{t-1} + \phi_2 X_{t-2} + \cdots + \phi_p X_{t-p} + \varepsilon_t$$

**MA($q$)**：当前值 = 均值 + 过去 $q$ 个预测误差的修正

$$X_t = \mu + \varepsilon_t + \theta_1 \varepsilon_{t-1} + \cdots + \theta_q \varepsilon_{t-q}$$

**差分 $\nabla^d$**：一阶差分 = 今天 - 昨天

$$\nabla X_t = X_t - X_{t-1}$$

> **人话**：AR 说"过去会重复"，MA 说"上次猜错了这次改"，差分说"数据趋势太明显，先去掉再说"。

### ⚠️ 小白翻车预警

- ARIMA **不能直接处理季节性**！有周期性的数据要用 **SARIMA**（加个 S 就行）
- **差分不要太贪心**：一般 $d=0$ 或 $d=1$ 就够了，差多了信息全丢了
- ADF 检验的 $p<0.05$ 才说明数据平稳了，否则继续差分

### 💻 代码

```python
import numpy as np
import matplotlib.pyplot as plt
from statsmodels.tsa.arima.model import ARIMA
from statsmodels.tsa.stattools import adfuller

# ============ 1. 造点数据 ============
np.random.seed(42)
n = 120
t = np.arange(n)
trend = 0.05 * t                       # 上升趋势
seasonal = 10 * np.sin(2 * np.pi * t / 12)  # 12个月周期
noise = np.random.normal(0, 3, n)      # 随机噪声
sales = 100 + trend + seasonal + noise # 合成销量

# ============ 2. 平稳性检验 ============
adf_result = adfuller(sales)
print(f"ADF检验 p-value: {adf_result[1]:.4f}")  # p<0.05才算平稳

# ============ 3. 拟合ARIMA模型 ============
model = ARIMA(sales, order=(2, 1, 2))  # p=2, d=1, q=2
model_fit = model.fit()

# ============ 4. 预测未来12期 ============
forecast_result = model_fit.get_forecast(steps=12)
forecast_mean = forecast_result.predicted_mean   # 预测值
forecast_ci = forecast_result.conf_int(alpha=0.05)  # 95%置信区间
```

### 🎯 什么时候用 ARIMA？

| 场景 | 建议 |
|------|------|
| ✅ 时序数据有明显趋势 | ARIMA 能处理 |
| ✅ 数据量 50~200 左右 | ARIMA 最擅长 |
| ❌ 数据有强季节性 | 改用 SARIMA |
| ❌ 数据量太大（>1000） | 可能更适合 LSTM |

## 第 2 部分：指数平滑 + 灰色预测 GM(1,1)

### 🎬 先讲个故事

> 指数平滑的思路很简单：**越近的数据越重要**。
> 上个月的销量比去年的销量更能反映下个月的趋势。
>
> 灰色预测更狠：你只有**4~5 个数据点**？没问题，我照样能预测。

### 🧠 大白话解释

**指数平滑**：给每个历史数据打个折，越近折扣越小，越远折扣越大。

**Holt-Winters** = 指数平滑的升级版，能同时处理趋势 + 季节性。

**GM(1,1)**：数据很少的时候也能用——先把数据累加起来让它"变光滑"，再建一个指数方程，最后还原回去。

### 📐 公式（瞄一眼就行）

**简单指数平滑**：

$$S_t = \alpha \times (\text{今天的值}) + (1-\alpha) \times (\text{昨天的预测})$$

> **人话**：新预测 = 新数据占 $\alpha$ 权重 + 旧预测占 $1-\alpha$ 权重。$\alpha$ 越大，越相信新数据。

**GM(1,1) 核心公式**：

先累加：$x^{(1)}(k) = x^{(0)}(1) + x^{(0)}(2) + \cdots + x^{(0)}(k)$

建指数模型：$\hat{x}^{(1)}(k+1) = \left(x^{(0)}(1) - \frac{b}{a}\right)e^{-ak} + \frac{b}{a}$

再还原回去得到预测值。

### ⚠️ 小白翻车预警

- **GM(1,1) 不能做季节性预测**——它只能看出总体的上升或下降趋势
- 指数平滑的 $\alpha$ 一般用 0.1~0.3，别设太大（模型会太"激动"）
- Holt-Winters 要求数据至少有 2 个完整周期

### 🎯 什么时候用什么？

| 模型 | 适用场景 |
|------|---------|
| 简单指数平滑 | 数据平稳，无明显趋势 |
| Holt-Winters | 有趋势 + 季节性（如月度销售数据） |
| GM(1,1) | 数据很少（5~10个点），只要趋势不要季节 |

## 第 3 部分：Logistic 增长 + 随机森林 / XGBoost

### 🎬 先讲个故事

> 你卖一款新产品，刚开始销量涨得飞快，但市场总有一天会饱和。
> Logistic 模型就是描述这种 **"S 型增长"** 的——先加速，再减速，最后撞到天花板。

### 🧠 大白话解释

**Logistic 增长**：假设市场有个"天花板" $K$。一开始增长快，逼近 $K$ 时增长慢下来。

**随机森林 / XGBoost**：把过去几天的数据做成"特征"，让树模型自己学规律——不需要假设是线性还是 S 型，模型自己找。

### 💻 代码片段

```python
from scipy.optimize import curve_fit

# Logistic函数：K=天花板, r=增速, P0=起点
def logistic(t, K, r, P0):
    return K / (1 + ((K - P0) / P0) * np.exp(-r * t))

# 自动拟合参数
popt, _ = curve_fit(logistic, t, P_obs, p0=[600, 0.4, 5])
K_fit, r_fit, P0_fit = popt
print(f"预估市场天花板 K = {K_fit:.1f}")
```

### 🎯 什么时候用？

| 模型 | 场景 |
|------|------|
| Logistic | 新产品/新技术扩散、市场饱和预测 |
| 随机森林 | 数据特征多、关系复杂、不求解释性 |
| XGBoost | 精度要求极高、愿意花时间调参 |

## 第 4 部分：LSTM 神经网络

### 🎬 先讲个故事

> 传统方法预测时序时，有一个致命问题：**记不住太久远的事**。
> 比如去年同期的销售旺季——ARIMA 很难记住那么远，但 LSTM 可以。

### 🧠 大白话解释

LSTM 是一种特殊的神经网络，它有个"细胞状态"——像一条高速公路，信息可以在上面稳定地跑很多步，不会丢失。

它有三个"门"来控制信息流动：
- **遗忘门**：过去哪些信息没用了？扔掉
- **输入门**：新的信息哪些值得记住？写进去
- **输出门**：现在需要输出什么？放出来

### ⚠️ 小白翻车预警

- LSTM **不是万能的** —— 数据太少（< 200 条）效果不如 ARIMA
- **一定要归一化数据**，不然训不动
- 递归预测时误差会累积，越往后越不准

### 💻 技术要点

| 要点 | 说明 |
|------|------|
| 最少数据量 | ≥ 200 条，最好 ≥ 500 |
| 回溯窗口 | 取一个完整周期（月度数据选 12） |
| 网络深度 | 2 层 LSTM 大多数情况够用 |
| 归一化 | 必须做，推荐 MinMax |
| 替代方案 | 数据少时用 ARIMA，可解释时用 Holt-Winters |

## 第 5 部分：蒙特卡洛仿真

### 🎬 先讲个故事

> 你种了 100 亩小麦，想知道明年赚不赚钱。
> 但你不知道：明年产量多少？价格多少？有没有天灾？
>
> 蒙特卡洛的思路是：**把每个不确定因素都"随机抽"一万次，看结果分布**。

### 🧠 大白话解释

三步走：
1. **定义风险因子**：产量 ~ 正态分布，价格 ~ 对数正态，天灾 ~ 随机发生
2. **随机采样**：每个因子抽 10000 个可能值
3. **统计分析**：算平均收益、最坏情况、亏损概率

### 📐 关键指标

| 指标 | 含义 | 大白话 |
|------|------|--------|
| **VaR₉₅** | 95% 置信下最低收益 | "只有 5% 的可能比这个数差" |
| **CVaR₉₅** | 尾部平均损失 | "最差那 5% 的情况，平均亏多少" |
| **亏损概率** | 亏钱的几率 | "十次里有几次是亏的" |

### 💻 代码框架

```python
import numpy as np

n_sim = 10000

# 1. 不确定因素采样
yield_sim = np.random.normal(5000, 800, n_sim)        # 产量
price_sim = np.random.lognormal(np.log(3.5), 0.15, n_sim)  # 价格
cost_sim = np.random.uniform(7000, 10000, n_sim)      # 成本
disaster = np.random.binomial(1, 0.1, n_sim)          # 灾害概率10%

# 2. 计算每次的收益
profit_sim = yield_sim * price_sim - cost_sim

# 3. 风险分析
var_95 = np.percentile(profit_sim, 5)     # VaR: 最差5%的边界
cvar_95 = np.mean(profit_sim[profit_sim <= var_95])  # 更差情况平均
prob_loss = np.mean(profit_sim < 0) * 100  # 亏损概率
```

### 🎯 什么时候用？

| 场景 | 建议 |
|------|------|
| ✅ 题目涉及**不确定性、风险、随机** | 蒙特卡洛是标准答案 |
| ✅ C 题种植/养殖/投资决策 | **2024-C 就是这个套路** |
| ✅ 需要量化"最坏情况" | VaR + CVaR 组合 |

## 🏆 模块 6 总结：预测方法速查

| 部分 | 模型 | 一句话 | 数据量要求 |
|------|------|--------|-----------|
| 1️⃣ | ARIMA | 线性时序的基础款 | 50~200 |
| 2️⃣ | Holt-Winters | 有周期时用这个 | 至少2个周期 |
| 2️⃣ | GM(1,1) | 只有5个点也能用 | ≥4 |
| 3️⃣ | Logistic | S型增长天花板 | ≥10（拟合用） |
| 3️⃣ | RF / XGBoost | 精度优先，特征多 | ≥200 |
| 4️⃣ | LSTM | 长序列大数据 | ≥500 |
| 5️⃣ | 蒙特卡洛 | 风险评估一把手 | 不用约束，纯模拟 |

> **选型口诀**：有周期→Holt-Winters / SARIMA，数据少→GM(1,1)，增长有天花板→Logistic，精度要最高→XGBoost，风险要量化→蒙特卡洛 🍡
