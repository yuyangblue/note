## 1.1 缺失值填充

### 🎬 先讲个故事

> 你拿到一张 500 人的体检表，想分析身高和体重的关系。
> 结果发现：第 3 行的"体重"是空的，第 17 行的"身高"也是空的……
> 
> 现在问题来了——**这些空位怎么办？**
>
> 这就叫"缺失值处理"，是数据预处理的第一步。

---

### 1.1.0 三个面试题级别的概念（不用背，有个印象就行）

数据缺失有三种"病因"，知道以下就够了：

| 类型 | 中文名 | 一句话解释 | 国赛怎么办 |
|------|--------|-----------|-----------|
| **MCAR** | 完全随机缺失 | 缺就缺了，和啥都没关系（比如仪器偶尔抽风） | 随便填，问题不大 |
| **MAR** | 随机缺失 | 缺不缺少和其他已有的数据有关（比如年龄大的人不爱填体重） | 用别的列预测这个空 |
| **MNAR** | 非随机缺失 | 缺不缺少和这个值本身有关（比如体重特别重的人不写体重） | 麻烦，需要专门处理 |

> 💡 **国赛实战**：你不需要纠结到底是哪种缺失——**直接看图**👇

---

### 1.1.1 先看看缺失长什么样

```python
import pandas as pd
import seaborn as sns
import matplotlib.pyplot as plt

# 画个热力图，黄色=有数据，深红=缺失
sns.heatmap(df.isnull(), cbar=True, cmap='YlOrRd', yticklabels=False)
```

> 看一眼热力图，你就知道哪列缺得多、哪列缺得少。**缺得少的（<5%）直接填均值就行，不用纠结。**

---

### 1.1.2 方法一：均值填充 🟢 最简单、最快

#### 🧠 大白话解释

如果一列数据缺了几个值，就拿**这一列剩下的数**算个平均值，把空位填上。

**举个例子：**

```
身高列：170, 165, 空, 180, 175
                 ↓
先算剩下4个人的平均：(170+165+180+175)/4 = 172.5
                 ↓
身高列：170, 165, 172.5, 180, 175  ← 空位被填上了
```

#### 📐 公式（瞄一眼就行，知道啥意思就够了）

设第 $j$ 列有 $m$ 个非空值，它们的均值是：

$$\bar{x}_j = \frac{x_1 + x_2 + \cdots + x_m}{m}$$

填充规则：空位填 $\bar{x}_j$，已经有值的保持不变。

#### ⚠️ 小白翻车预警

均值填充有个**坑**——它会压缩数据的波动范围。

打个比方：

> 你们班真实体重范围是 45kg~90kg，方差很大。
> 但如果用了均值填充，所有空位都填成了 65kg，数据波动就变小了。
> **这就叫"低估方差"**，会影响后面做显著性检验的结果。

公式长这样（不想看可以跳过）：

$$\tilde{\sigma}_j^2 = \frac{\text{非空样本数}}{\text{总样本数}} \times \sigma_j^2 < \sigma_j^2$$

**结论**：均值填充让标准差变小，统计检验更容易"不显著"。

#### 💻 代码（直接复制就能用）

```python
from sklearn.impute import SimpleImputer

# 均值填充 —— 一行搞定
imputer = SimpleImputer(strategy='mean')
df_filled = pd.DataFrame(imputer.fit_transform(df), columns=df.columns)
```

#### 🎯 什么时候用均值填充？

| 条件 | 建议 |
|------|------|
| ✅ 缺失比例 < 5% | 用均值，快且够用 |
| ✅ 数据大致对称（不是严重偏态） | 用均值合理 |
| ❌ 有极端异常值 | **别用均值**，换中位数 |
| ❌ 缺失比例 > 20% | 均值会扭曲数据，换高级方法 |

---

### 1.1.3 方法二：中位数填充 🟢 比均值更稳

#### 🧠 大白话解释

如果数据里有**异常值**（比如某人身高写了 999），均值就会被带偏。
这时候用中位数更聪明——把数据排个序，取**中间那个数**。

**举个例子：**

```
月收入：3000, 3500, 4000, 空, 100000（大佬乱入）
      
均值 = (3000+3500+4000+100000)/4 = 27625 ❌ 被大佬带飞了
中位数 = 排队→ [3000, 3500, 4000, 100000] → 中间两个平均 = 3750 ✅ 靠谱
```

#### 📐 公式

把数据从小到大排好，取中间那位：

$$\tilde{x} = \begin{cases} \text{排中间的那个数}, & \text{奇数个} \\ \text{中间两个的平均}, & \text{偶数个} \end{cases}$$

#### 💻 代码

```python
from sklearn.impute import SimpleImputer

# 中位数填充 —— 也是1行
imputer = SimpleImputer(strategy='median')
df_filled = pd.DataFrame(imputer.fit_transform(df), columns=df.columns)
```

#### 🎯 什么时候用中位数？

| 条件 | 建议 |
|------|------|
| ✅ 有异常值 | 中位数比均值稳 10 倍 |
| ✅ 数据偏态分布（比如大多数人都矮，少数人特别高） | 中位数更代表"大多数" |
| ❌ 缺失比例 > 5% | 简单的填充方法都不太行，该上高级的了 |
| ❌ 需要做严格的统计推断 | 中位数填充也会低估方差 |

---

### 🏁 均值 vs 中位数 一句话总结

> **数据干净 → 均值（快）**
> **数据有脏东西 → 中位数（稳）**
> **缺太多（>5%）→ 换下面的大招**

### 1.1.4 方法三：KNN 填充 🟡 考虑变量关系

#### 🧠 大白话解释

均值和中位数都犯了一个"错"——**它们只看这一列自己的数**，不看别的列。

但实际数据里，各种特征之间是有关系的。比如：

> 一个人**身高 185cm**，体重那列空了。
> 凭常识你大概能猜出来：这个人大概率不瘦，体重应该在 70~85kg 左右。
> 
> KNN 就是这个思路：**找身高最像的那几个人的体重，取个平均填进去。**

#### 📐 数学原理（瞄一眼）

KNN 会算"你和别人有多像"（欧氏距离）：

$$d(\mathbf{x}_i, \mathbf{x}_{i'}) = \sqrt{(x_{i1} - x_{i'1})^2 + (x_{i2} - x_{i'2})^2 + \cdots}$$

然后找最近的 $K$ 个邻居，用他们的值做个加权平均。

#### ⚠️ 小白翻车预警

KNN 有个大坑——**量纲敏感**。

意思是说：
- 身高单位是 cm（范围 150~190）
- 体重单位是 kg（范围 40~100）
- 如果身高列直接参与距离计算，身高差异会"淹没"体重差异

**解决办法**：先标准化，再做 KNN。

#### 💻 代码

```python
from sklearn.impute import KNNImputer
from sklearn.preprocessing import StandardScaler

# 第1步：标准化（去掉量纲影响）
scaler = StandardScaler()
df_scaled = scaler.fit_transform(df)

# 第2步：KNN填充（k=5，距离加权）
knn_imputer = KNNImputer(n_neighbors=5, weights='distance')
df_filled_scaled = knn_imputer.fit_transform(df_scaled)

# 第3步：反标准化（把数据变回原来的单位）
df_filled = pd.DataFrame(scaler.inverse_transform(df_filled_scaled), columns=df.columns)
```

> 💡 注意三步走：标准化 → 填充 → 反标准化，**缺一不可**，少了第一步 KNN 结果就是错的。

#### 🎯 什么时候用 KNN？

| 条件 | 建议 |
|------|------|
| ✅ 缺失 5%~20% | KNN 性价比最高 |
| ✅ 变量间有明显相关性（比如身高和体重） | KNN 能利用这个关系 |
| ❌ 缺失 > 20% | 数据缺太多了，KNN 也救不了 |
| ❌ 变量之间几乎没关系 | KNN 和均值填充没啥区别，白费功夫 |

### 1.1.5 方法四：随机森林填充 🛑 最强的，也是最后的武器

#### 🧠 大白话解释

把"有空值的那一列"当成**要预测的目标**，用其他列当特征，训练一个随机森林来预测这个空值。

而且这个方法是**迭代的**——填了一次不够，反复填好几轮，直到结果稳定。

> 你可以理解为：**找了一群专家（100棵决策树），每个人都给个预测值，最后投票决定填多少。**

#### 💻 代码

```python
from sklearn.ensemble import RandomForestRegressor

def rf_impute(df, target_col):
    """对某一列做随机森林填充"""
    df_copy = df.copy()
    missing_idx = df_copy[target_col].isnull()
    
    # 没有缺失就不用填
    if missing_idx.sum() == 0:
        return df_copy
    
    # 用其他列当特征
    feature_cols = [c for c in df_copy.columns if c != target_col]
    
    # 训练集：没缺失的那些行
    X_train = df_copy.loc[~missing_idx, feature_cols].fillna(
        df_copy.loc[~missing_idx, feature_cols].mean()
    )
    y_train = df_copy.loc[~missing_idx, target_col]
    
    # 测试集：有缺失的那些行
    X_test = df_copy.loc[missing_idx, feature_cols].fillna(
        df_copy.loc[missing_idx, feature_cols].mean()
    )
    
    # 训练并预测
    rf = RandomForestRegressor(n_estimators=100, random_state=42)
    rf.fit(X_train, y_train)
    df_copy.loc[missing_idx, target_col] = rf.predict(X_test)
    return df_copy
```

#### 🎯 什么时候用随机森林填充？

| 条件 | 建议 |
|------|------|
| ✅ 缺失 > 20% | 只能用这个级别的了 |
| ✅ 变量关系复杂、非线性 | 随机森林能捕捉到 |
| ❌ 数据量特别小（<100行） | 随机森林容易过拟合 |
| ❌ 比赛只剩 30 分钟交卷 | 跑起来太慢，KNN 更合适 |

---

### 🏆 缺失值填充：终极速查表

```
数据缺 < 5% 且 分布正常  →  均值填充  （1秒，一行代码）
数据缺 < 5% 但有异常值  →  中位数填充（1秒，一行代码）
数据缺 5%~20% 且有关联  →  KNN填充   （5秒，三步走）
数据缺 > 20% 或关系复杂  →  随机森林  （30秒，迭代慢但准）
```

### ✍️ 国赛评卷人想看到什么？

> 1️⃣ **先可视化缺失**（热力图）——让评委一眼看到你懂数据
> 2️⃣ **解释你为什么选这个方法**——不要直接甩代码，要说"因为缺失只有3%，所以用均值填充"
> 3️⃣ **承认方法的局限性**——说了"均值填充会低估方差"，比闷头填完强
> 4️⃣ **如果缺得多（>20%），不要只用一种方法**——可以用 KNN 和随机森林各填一次，对比结果说结论稳健
