# 📘 模块 4：综合评价算法（小白友好版）

> **这是干啥的？**
> 给你 N 个方案 / 城市 / 产品，每个有 M 个指标，让你排个名。
> C 题第二问高频考点，国赛评委最认 **熵权-TOPSIS**。

---

## Part 1：熵权-TOPSIS ⭐⭐⭐⭐⭐ 最推荐

### 🎬 先讲个故事

> 你要从 5 个城市里选一个建工厂，考虑：地价、劳动力、交通、环境。
> 但每个指标的重要程度不一样——怎么客观地定权重？怎么排名？
>
> **熵权法**帮你说哪个指标更重要，**TOPSIS** 帮你算哪个城市最接近"理想城市"。

### 🧠 两步走

**Step 1：熵权法 → 自动算权重**

核心思想：一个指标的数据越分散（熵越小），权重越大。

> **人话**：如果所有城市"交通"得分都差不多，那这个指标就没区分度，权重低。
> 如果"地价"差异很大，那它很重要，权重高。

**Step 2：TOPSIS → 排名**

> **人话**：先虚构一个"理想方案"（所有指标最优），再虚构一个"最差方案"（所有指标最劣）。
> 看每个方案离理想有多近、离最劣有多远——**近理想、远最劣**的就排前面。

### 📐 公式（瞄一眼）

**Step 1：指标正向化**（统一成"越大越好"）

- 成本类（越小越好）：$x' = \max - x$
- 区间类（比如 pH 6~8 最好）：用分段函数转

**Step 2：标准化**

$$z_{ij} = \frac{x_{ij}}{\sqrt{\sum x_{ij}^2}}$$

**Step 3：熵权法**

$$H_j = -\frac{1}{\ln n}\sum p_{ij}\ln(p_{ij}), \quad w_j = \frac{1-H_j}{\sum(1-H_j)}$$

> **人话**：$H_j$ 小 → 数据分散 → $w_j$ 大 → 这个指标重要

**Step 4：TOPSIS 排序**

找"理想解" $Z^+$（每列最大值）和"最劣解" $Z^-$（每列最小值），算距离：

$$D_i^+ = \sqrt{\sum w_j(z_{ij} - Z_j^+)^2}, \quad D_i^- = \sqrt{\sum w_j(z_{ij} - Z_j^-)^2}$$

最终得分：$C_i = \frac{D_i^-}{D_i^- + D_i^+}$

> **人话**：$C_i$ 越接近 1，排名越靠前。

### 💻 代码

```python
import numpy as np

def entropy_topsis(X, indicator_types=None):
    """
    X: n×m 矩阵（n个对象，m个指标）
    indicator_types: 指标类型列表，'max'=越大越好, 'min'=越小越好
    """
    n, m = X.shape
    
    # 1. 正向化（统一成越大越好）
    if indicator_types:
        for j in range(m):
            if indicator_types[j] == 'min':
                X[:, j] = X[:, j].max() - X[:, j]
    
    # 2. 标准化
    Z = X / np.sqrt((X**2).sum(axis=0))
    
    # 3. 熵权法
    P = Z / Z.sum(axis=0)  # 计算比重
    H = -np.sum(P * np.log(P + 1e-10), axis=0) / np.log(n)  # 信息熵
    W = (1 - H) / (1 - H).sum()  # 权重
    
    # 4. TOPSIS
    Z_plus = Z.max(axis=0)   # 理想解
    Z_minus = Z.min(axis=0)  # 最劣解
    D_plus = np.sqrt(((Z - Z_plus)**2 * W).sum(axis=1))
    D_minus = np.sqrt(((Z - Z_minus)**2 * W).sum(axis=1))
    scores = D_minus / (D_plus + D_minus)
    
    return scores, W
```

### 🎯 什么时候用？

| 情况 | 建议 |
|------|------|
| ✅ C 题第二问评价排序 | **首选！国赛评委最喜欢** |
| ✅ 指标 > 3 个，想客观定权重 | 熵权法自动算 |
| ✅ 需要排名+解释性 | TOPSIS 结果直观 |
| ❌ 数据量太少（< 5 个对象） | 熵权法不太准 |

## Part 2：层次分析法 AHP ⭐⭐⭐⭐

### 🎬 先讲个故事

> 你买房要考虑：价格、地段、学区、环境……
> 但这些指标没法用数据量化，只能凭"感觉"比——AHP 就是帮你把"感觉"变成数字的。

### 🧠 大白话

AHP = 人脑打分 + 数学检验

1. **建层次**：目标 → 准则 → 方案
2. **两两比较**：比如"价格 vs 地段，哪个更重要？重要多少？"（1~9 分制）
3. **计算权重**：从比较矩阵算出来
4. **一致性检验**：检查你的打分有没有自相矛盾（比如 A>B, B>C, 但 C>A）

### ⚠️ 小白翻车预警

- **一定要做一致性检验**，CR < 0.1 才算有效
- 指标太多（>7 个）时 AHP 比较矩阵会非常庞大，不推荐
- **主观性争议**——国赛评委可能会质疑你的打分依据

### 🎯 什么时候用 AHP？

| 情况 | 建议 |
|------|------|
| ✅ 定性指标多、没数据 | AHP 适合凭经验打分 |
| ✅ 需要展示指标体系层次 | AHP 的层次结构清晰 |
| ❌ 有客观数据可用 | 用熵权法更好（更客观） |
| ❌ 指标太多（>7） | 两两比较会把人逼疯 |

## Part 3：灰色关联分析 ⭐⭐⭐⭐

### 🎬 先讲个故事

> 你有 3 年的月度数据，想知道"哪个指标和最终结果长得最像"。
> 灰色关联就是**看两个序列的几何形状有多像**——形状越像，关联度越高。

### 🧠 大白话

1. 选一个"参考序列"（比如综合得分）
2. 把其他序列和它一起画图
3. 看每条线和参考线之间的"距离"——离得近就关联大

### 🎯 什么时候用？

| 情况 | 建议 |
|------|------|
| ✅ 样本很少（< 30 条） | 灰色关联小样本很好用 |
| ✅ 数据不完全、信息少 | 灰色系统的看家本领 |
| ✅ 需要定指标权重（综合评价） | 算关联度后归一化就是权重 |
| ❌ 数据量大且有客观数据 | 熵权法更优 |

## Part 4：模糊综合评价 ⭐⭐⭐

### 🎬 先讲个故事

> "这个方案好不好？"——答案不是"好"或"不好"，而是"70% 好，30% 一般"。
>
> 模糊综合评价就是处理这种**模糊概念**的。

### 🧠 大白话

1. 给每个指标分几个等级（好/中/差）
2. 用隶属度函数把实际值转成"属于各等级的程度"
3. 加权汇总，得到最终评价向量

### 🎯 什么时候用？

| 情况 | 建议 |
|------|------|
| ✅ 指标边界模糊（如"舒适度"） | 模糊评价很合适 |
| ✅ 需要输出"优秀率 / 合格率" | 结果直接给分布 |
| ❌ 有明确客观数据 | 用 TOPSIS 更简单直接 |
