**RISC-V 常规立即数只有 12 位，不够放完整 32 位常数，所以必须用 “lui + addi” 组合构造完整 32 位常数。**

---
## 1. lui 指令（Load Upper Immediate）

```
lui rd, constant
```

作用：

- 把 **20 位立即数** 放到寄存器的 **bit[31:12]**
- **bit [11:0] 清 0**
- 64 位架构自动把 **bit 31** 符号扩展到 **bit[63:32]**（寄存器是64位）

一句话：

**lui = 把常数放到 “高 20 位”**

---

## 2. 例子：lui x19, 976

976 = 0x3D0（十六进制）

所以：

```plaintext
lui x19, 976   // 0x003D0
```

寄存器 x19 变成：

```plaintext
0x00000000000003D0
```

---

## 3. 配合 addidi 构建完整 32 位常数

再做一步：

```plaintext
addi x19, x19, 128   // 0x500
```

现在 x19 = 0x003D0 + 0x500 = **0x00000000000008D0**

最终效果：

```plaintext
x19 = 0x8D0   (32 位完整常数)
```

---

## 4. 为什么必须用 lui？

因为 RISC-V 普通指令（I-type）只有 **12 位立即数**。

12 位 = ±2048

不够存任何 32 位的大常数。

所以构造 32 位常数必须：

- lui 放高位 20 位
- addi 补低位 12 位

组合得到：

**20 + 12 = 32 位**
