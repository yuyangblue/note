# 一、同步核心知识点

## 1. 共享内存与数据竞争

- **场景**：两个处理器（P1、P2）共享同一块内存区域，例如 P1 先写入数据，P2 再读取该数据。
- **风险**：如果 P1 和 P2 之间没有同步机制，就会产生**数据竞争（Data race）**。
- **后果**：程序的最终结果完全依赖于内存访问的执行顺序，导致行为不可预测。

## 2. 硬件支持的必要性

- 解决数据竞争需要**硬件层面的原子操作支持**：
    
    - **原子读写内存操作**：将 “读 - 改 - 写” 这类复合操作封装为不可分割的最小执行单元。
    - **互斥保证**：在原子操作的读与写步骤之间，不允许其他处理器访问该内存位置。
    
## 3. 原子操作的实现形式

- **单指令实现**：通过一条专用指令完成原子操作，例如：
    
    - 寄存器 ↔ 内存的原子交换（Atomic Swap）
    
- **指令对实现**：通过一对特殊指令实现，例如 “加载保留 / 条件存储”（Load-Reserve/Store-Conditional），同样能保证操作的原子性。

# 二、具体指令
## 1. 加载保留指令 `lr.d rd, (rs1)`

- **功能**：
    
    1. 从 `rs1` 寄存器指向的内存地址加载一个双字（doubleword）到目标寄存器 `rd`。
    2. 同时在硬件层面为该内存地址**设置一个保留标记（Reservation）**，监控后续对该地址的写操作。
    
- **本质**：开启一段 “乐观读取”，并标记要保护的内存位置。

## 2. 条件存储指令 `sc.d rd, (rs1), rs2`

- **功能**：
    
    1. 尝试将 `rs2` 寄存器的值存储到 `rs1` 指向的内存地址。
    2. **成功条件**：自上次 `lr.d` 执行后，该内存地址**未被其他处理器或中断修改**。
    
    - 成功时：存储完成，并在 `rd` 中写入 **0**。
    
    3. **失败条件**：该内存地址在 `lr.d` 之后被修改过。
    
    - 失败时：存储操作被**放弃**，并在 `rd` 中写入**非零值**。
    
- **本质**：只有在 “读取后未被篡改” 的前提下，写入才会生效，从而保证了 “读 - 改 - 写” 操作的原子性

#### Example 1: 原子交换（Atomic Swap）

用于测试并设置锁变量，实现原子的 “读旧值 - 写新值” 操作。

```asm
again:
    lr.d  x10, (x20)      # 从x20指向的地址加载旧值到x10，并标记保留
    sc.d  x11, (x20), x23 # 尝试将x23的新值存入x20地址；x11接收状态(0=成功)
    bne   x11, x0, again  # 若存储失败(x11≠0)，则重试
    addi  x23, x10, 0     # 将读取到的旧值(x10)保存到x23
```

- **核心逻辑**：通过循环重试，保证 “加载 - 存储” 操作的原子性。
- **结果**：`x23` 保存了交换前的旧值，`x20` 地址被更新为 `x23` 原来的新值。

---

#### Example 2: 自旋锁（Lock）与解锁（Unlock）

实现一个经典的**测试并设置（Test-and-Set）自旋锁**，`x20` 指向锁变量地址（0 = 未锁，1 = 已锁）。

##### 加锁流程：


```asm
    addi  x12, x0, 1      # 将锁定值(1)存入x12
again:
    lr.d  x10, (x20)      # 读取锁变量到x10，并标记保留
    bne   x10, x0, again  # 若锁已被占用(x10≠0)，则循环等待（自旋）
    sc.d  x11, (x20), x12 # 尝试将锁置为1；x11接收状态
    bne   x11, x0, again  # 若存储失败，重试
```

- **流程**：先检查锁是否为 0（未锁），若为 0 则尝试原子地将其置为 1；若失败则重试，直到成功获取锁。
- **特点**：在锁被占用时，CPU 会不断循环检查，属于**忙等待（Busy Waiting）**。

##### 解锁流程：


```asm
    sd    x0, 0(x20)      # 将0写入锁变量地址，释放锁
```

- **逻辑**：直接将锁变量写回 0，无需原子操作（因为只有锁持有者会执行解锁）。
