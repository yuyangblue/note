## 存储层次结构的基本概念

### 核心内容讲解

计算机存储层次结构从快速小容量到慢速大容量的金字塔式层级：

- **Processor（处理器）**：位于顶端，是数据的最终使用者
- **SRAM Memory**：对应L1/L2缓存，容量小（KB级别）但速度极快（0.5-2.5ns）
- **DRAM Memory**：对应主内存，容量中等（GB级别），速度适中（50-70ns）
- **Disk（磁盘）**：位于底端，容量最大（TB级别）但速度最慢（毫秒级）

箭头表示数据在各层之间的传递方向。

### 核心概念

1. **时间局部性（Temporal Locality）**：最近访问的项很可能再次被访问。例如循环中的指令会被反复执行。

2. **空间局部性（Spatial Locality）**：相邻地址的项很可能被一起访问。例如顺序执行的指令流、数组数据的顺序访问。

3. **存储层次结构的本质**：在速度、容量、成本三者之间做权衡，利用局部性原理让"大容量"和"高速"看似同时实现。

---

### 常见问题解答

**Q：存储层次结构层级图解中各层是什么意思？**

```
CPU（处理器）
    ↓
Cache（缓存）—— SRAM，最快最小最贵
    ↓
Memory（主存/内存）—— DRAM，中等速度和容量
    ↓
Disk（磁盘/硬盘）—— 最慢最大最便宜
```

**Memory（主存）是什么？**
- 就是平时说的"内存条"，通常几GB到几十GB
- 技术上使用DRAM（动态RAM），断电数据丢失
- 作用：给CPU一个"足够快"的数据源，避免CPU直接访问慢速磁盘

**各层关系类比**：
- Memory = 图书馆（大，但找书要时间）
- Cache = 书桌（小，但伸手就能拿到）
- CPU想要的书，优先从书桌拿，没有再去图书馆

## 利用局部性原理

### 核心内容讲解
![[Pasted image 20260509162020.png|208]]
存储层次结构通过数据复制实现"缓存"的假象：

**命中流程**：
- 数据请求首先到达最上层
- 如果数据存在（Hit命中），直接从上层返回
- 命中率高意味着大部分访问由上层服务，性能好

**未命中流程**：
- 如果数据不在上层（Miss未命中），从下一层获取整个数据块
- 获取的数据块被复制到上层供后续访问使用
- 未命中惩罚（Miss Penalty）= 从下层获取数据的时间

**块（Block/Line）** 是数据复制的基本单位，大小影响性能。

### 核心概念

1. **块（Block）**：存储层次中数据复制的基本单位，大小是关键设计参数

2. **命中率的计算**：Hit ratio = hits / accesses = 1 - Miss ratio

3. **块大小的权衡**：
   - 大块利用空间局部性好
   - 但增加未命中惩罚和可能的缓存污染

## 存储技术对比

### 核心内容讲解

四种存储技术的性能与成本对比：

| 存储类型 | 访问时间 | 成本（$/GB） |
|---------|---------|-------------|
| SRAM | 0.5ns – 2.5ns | $2000 – $5000 |
| DRAM | 50ns – 70ns | $20 – $75 |
| 磁盘 | 5ms – 20ms | $0.20 – $2 |
| **理想内存** | SRAM速度 | 磁盘成本 |

**现实**：物理上不可能同时获得SRAM的速度和磁盘的成本，因此需要存储层次结构。

### 核心概念

1. **SRAM**：静态随机存取存储器，无需刷新，但占用面积大、功耗高
   - 存储1位需要6个晶体管（6T结构）
   - 面积约为DRAM的4-6倍
   - 用触发器结构稳定存储，速度快但成本高

2. **DRAM**：动态随机存取存储器，使用电容存储，需要周期性刷新
   - 存储1位只需1个晶体管+1个电容（1T1C结构）
   - 面积小、成本低，但需要刷新

3. **存储层次结构的驱动力**：弥补各种存储技术之间的速度、容量、成本差距

## DRAM技术详解

![[第五章6.png]]

### 核心内容讲解

DRAM使用电容存储电荷来表示比特位：

**存储单元结构**：
- 每个存储单元由1个晶体管 + 1个电容组成
- 电容的电荷随时间泄漏，必须周期性刷新

**刷新机制**：
- 刷新按"行"进行
- 读出整行数据再写回
- 行缓冲区（Row Buffer）允许在同一行内连续读取多个字

**行缓冲区位置**：
- 行缓冲区位于DRAM芯片内部，是每个Bank独立的临时存储区域
- 访问流程：选中某行 → 整行数据进入行缓冲区 → 根据列地址取具体数据
- 作用：访问同一行的多个数据时无需反复激活行，提高速度

### 核心概念

1. **DRAM行缓冲区**：允许并行读取和刷新多个字，提高突发访问效率

2. **Bank结构**：多个独立的banks可同时激活不同行，实现并行访问
   - 单Bank限制：同一时间只能有一行处于激活状态
   - 多Bank优势：每个Bank有独立的行缓冲区，可同时激活不同行
   - 地址中的BankID字段用于选择访问哪个Bank

3. **刷新开销**：DRAM需要周期性刷新，这段时间无法响应访问请求
   - 刷新=充电：读出整行 → 放大信号 → 写回（重新充电）
   - 刷新期间存储阵列被占用，CPU必须等待
   - 典型DRAM每64ms刷新所有行，约5-10%时间无法访问

## DRAM高级组织

### 核心内容讲解

DRAM内部以矩形阵列组织数据位：

**DDR（双倍数据率）**：
- 在时钟的上升沿和下降沿都传输数据
- 相同频率下带宽翻倍

**QDR（四倍数据率）**：
- 分离输入输出通道
- 进一步提高带宽

**突发模式**：从一行连续读取多个字时减少延迟

### 核心概念

1. **DDR代数演进**：DDR → DDR2 → DDR3 → DDR4 → DDR5，持续提升传输速率和能效

2. **DRAM分银行（Banking）**：多个独立banks可同时激活不同行，提升整体带宽

3. **同步DRAM**：使用时钟同步，允许突发传输连续字而无需每次发送地址

## 存储带宽提升技术

![[第五章10.png]]

### 内存带宽优化：三种核心方法

在 CPU 访存过程中，缓存缺失时需要从主存取数据。为了加快这个过程，通常有三种优化策略：
---
#### a. 宽缓存组织（Wide Cache）

**思路**：把数据通路加宽，一次从主存取回多个字。

```
原本：一次拿 1 个字（4B）
宽缓存：一次拿 4 个字（16B）
```

- **优点**：减少访问次数，单次缺失处理更快
- **缺点**：连线增多，功耗增大
---
#### b. 多银行缓存（Multi-banked Cache）

**思路**：把缓存分成多个独立的"银行"，每个银行可以同时工作。

```
银行1 | 银行2 | 银行3 | 银行4
  A1  |  A2  |  A3  |  A4    ← 交叉存放
```

- **优点**：支持多个访问并行进行
- **缺点**：单次传输反而更慢（要分多次交叉传输）
---
#### 带宽对比计算

| 方案 | 缺失惩罚 | 带宽 |
|------|----------|------|
| 4-word 宽缓存 | 1 + 15 + 1 = 17 周期 | 16B ÷ 17 = **0.94 B/周期** |
| 4-bank 交叉缓存 | 1 + 15 + 4×1 = 20 周期 | 16B ÷ 20 = **0.80 B/周期** |

> **结论**：单次大块数据传输选宽缓存；多个并发访问选多银行。
---
### 一句话总结

- **宽组织** → 减少单次延迟
- **交叉组织** → 增加并发带宽

影响DRAM性能的三个关键因素：

1. **Row Buffer（行缓冲区）**：允许在同一行内连续读取多个字而无需重新激活行，提高突发模式效率

## 闪存存储

![[第五章11.png]]

### 核心内容讲解

闪存是一种非易失性半导体存储，性能介于磁盘和DRAM之间：

**优点**：
- 比磁盘快100–1000倍
- 更小、更低功耗、更耐用（无机械部件）

**缺点**：
- 成本高于磁盘（介于磁盘和DRAM之间）

### 核心概念

1. **闪存bits会随访问次数增加而磨损（通常1000次后）**

2. **磨损均衡（Wear Leveling）**：将数据映射到不同物理块以延长寿命

3. **应用场景**：SSD、移动设备存储、嵌入式系统

---

## 闪存类型

### 核心内容讲解

闪存分为两大类型：

**NOR闪存**：
- 存储单元类似NOR门
- 支持随机读写访问
- 用于嵌入式系统的指令存储

**NAND闪存**：
- 存储单元类似NAND门
- 密度更高（每单位面积更多位）
- 只能按块访问
- 用于USB驱动器、存储卡、SSD

**NOR门与NAND门**：
- NOR门 = OR门 + NOT门（或非门）：两个输入都为0时输出1
- NAND门 = AND门 + NOT门（与非门）：至少一个输入为0时输出1
- NOR闪存单元并联连接 → 支持独立随机访问
- NAND闪存单元串联连接 → 只能按块访问，但密度更高

### 核心概念

1. **NOR vs NAND**：NOR适合随机读取（代码执行），NAND适合大容量顺序访问（数据存储）

2. **闪存不能直接替代需要频繁写入的RAM或磁盘应用**

3. **磨损均衡**：操作系统或控制器负责将写操作分散到不同物理块

## 磁盘存储结构

![[第五章13.png]]

### 核心内容讲解

磁盘是非易失性、旋转式磁性存储：

**核心结构**：
- **Track（磁道）**：同心圆状的数据轨道
- **Sector（扇区）**：磁道上的数据单元（通常512字节或4KB）
- **Cylinder（柱面）**：多层盘片上相同位置的磁道集合

### 核心概念

1. **磁盘是存储层次中的最底层**：提供最大容量和最低成本，但访问时间最长（毫秒级）

2. **磁盘访问涉及机械运动**：寻道时间 + 旋转延迟 + 传输时间

3. **局部性原理在磁盘调度中的应用**：电梯算法、提前排队等优化

## 磁盘扇区与访问

![[第五章14.png]]

### 核心内容讲解

每个扇区记录的内容：
- **Sector ID**：扇区标识
- **Data（512字节）**：实际数据
- **ECC（纠错码）**：用于隐藏缺陷和记录错误
- **同步字段和间隙**：用于分隔和同步

**磁盘访问步骤**：
1. **队列延迟**：等待其他访问完成
2. **寻道时间**：移动磁头到正确磁道
3. **旋转延迟**：等待扇区旋转到磁头下方
4. **数据传输**：实际读写数据
5. **控制器开销**：控制器处理请求的时间

**寻道时间详解**：
- 磁头装在移动臂上，需要机械运动才能到达目标磁道
- 过程：启动加速 → 移动 → 减速 → 精确定位
- 磁道宽度极窄（约0.1微米），需要精准定位
- 平均寻道时间4-10ms，是磁盘访问的主要延迟来源之一

**旋转延迟详解**：
- 磁头到位后，需要等待目标扇区旋转过来
- 目标扇区位置随机，平均需等待半圈
- 计算：转一圈时间 ÷ 2（如15000转/分 → 半圈2ms）

### 核心概念

1. **ECC（Error Correcting Code）**：检测和纠正数据传输或存储中的错误，提高数据可靠性

2. **扇区是磁盘I/O的基本单位**

3. **访问时间组成**：寻道时间 + 旋转延迟 + 传输时间 + 控制器开销

## 缓存基础概念

![[第五章17.png]]

### 核心内容讲解

缓存位于CPU和主存之间，利用局部性原理加速数据访问：

**核心问题**：
- **如何知道数据是否存在？** → 通过标签（Tag）判断
- **数据应该放在哪里？** → 由映射规则决定

缓存接收一系列地址访问X₁...Xₙ₋₁, Xₙ：
- 如果数据已在缓存中（命中），直接返回
- 如果数据不在缓存中（未命中），从主存取回并放入缓存

### 核心概念

1. **缓存是存储层次中第一个处理命中/未命中概念的层级**

2. **命中时间（Hit Time）**：从缓存中获取数据所需的时间

3. **未命中惩罚（Miss Penalty）**：缓存未命中时从下一层获取数据的时间

## 直接映射缓存

![[第五章18.png]]

### 核心内容讲解

直接映射缓存中，**每个内存块只能放在唯一的位置**：

**映射规则**：
```
缓存位置 = 块地址 mod 缓存块数
```

例如：缓存有8块，地址22 mod 8 = 6 → 地址22的块只能放在第6块。

**地址结构**：
- **Index（索引）**：选择缓存中的具体行
- **Tag（标签）**：用于区分映射到同一位置的不同块
- **Offset（偏移）**：块内选择具体字节

### 核心概念

1. **直接映射的优点**：硬件简单、查找速度快

2. **直接映射的缺点**：可能产生**冲突未命中**——两个频繁访问的块如果映射到同一位置，会相互替换

3. **地址划分示例**：32位地址，1KB缓存，16字节块 → Index占6位，Offset占4位

---

### 常见问题解答

**Q：为什么需要映射？**

核心问题：Cache太小，内存太大
- 内存容量远大于Cache，不能把所有数据都放进Cache
- 如果不映射（全相联），每次要搜索所有Cache位置，硬件复杂、成本高
- **映射的作用：缩小搜索范围**——用地址计算出"数据应该在哪个Cache位置"

| 映射方式 | 需要搜索的位置 | 硬件复杂度 |
|---------|--------------|-----------|
| 直接映射 | 1个位置 | 最简单 |
| 2路组相联 | 2个位置 | 中等 |
| 全相联 | 所有位置 | 最复杂 |

**Q：映射在什么时候发生？**

映射不是"预先设置"的，而是在**每次CPU访问数据时动态发生**：

```
CPU发出请求："我要读地址 10110 的数据"
        ↓
第一步：地址分解 → Tag=10, Index=110
        ↓
第二步：映射查找 → Index=110 指向 Cache位置110  ← 映射在这里发生！
        ↓
    ┌───检查Valid和Tag───┐
    │                     │
  命中                   缺失
    ↓                     ↓
直接返回            从Memory加载到该位置
```

**Q：怎么确认Cache位置存的就是我们要的数据？**

映射只告诉我们"去哪个位置找"，但那个位置存的不一定是我们要的数据。需要三步验证：

```
Cache每个位置的结构：
┌────────┬─────────┬──────────────┐
│ Valid  │   Tag   │     Data     │
│ 有效位  │  标签   │     数据     │
└────────┴─────────┴──────────────┘
```

验证流程：
1. **Index定位**：用地址的Index部分定位Cache位置
2. **检查Valid**：Valid=1才表示位置有效
3. **比较Tag**：Tag匹配才确认是你要的数据

**类比**：映射（Index）像"座位号"告诉你去哪找，Tag像"姓名牌"确认是不是你要找的人。

## 标签与有效位

### 核心内容讲解

缓存需要额外信息来确定存储的数据是否有效：

**标签（Tag）**：
- 存储块地址的高位部分
- 用于在缓存查找时判断该行是否包含请求的地址

**有效位（Valid Bit）**：
- 1 = 数据有效
- 0 = 无有效数据（初始状态或已被清除）
- 系统启动时所有有效位为0

### 核心概念

1. **命中条件**：标签匹配 AND 有效位为1

2. **标签存储开销**：每行需要额外的Tag位

3. **有效位的作用**：区分"空"和"存有旧数据"的缓存行

### 核心内容讲解

**访问地址22（二进制10 110）**：

```
Index = 110 (6)

块地址 = 22
```
结果：Miss（未命中），加载包含地址22的块到Index=6的行。

**访问地址26（二进制11 010）**：
```
Index = 010 (2)
Tag = 11
```
结果：Miss（未命中），加载块到Index=2的行。

**再次访问**

| Word addr | Binary | Tag | Index | Hit/Miss |
| --------- | ------ | --- | ----- | -------- |
| 22        | 10110  | 10  | 110   | Hit      |
| 26        | 11010  | 11  | 010   | Hit      |

### 核心概念

1. **时间局部性体现**：刚访问过的数据再次访问会命中

2. **不同Index的访问互不影响**

3. **首次访问 → Miss，再次访问 → Hit**

## 地址细分

![[第五章26.png]]

### 核心内容讲解

以64KB缓存、64字节块为例：

**64位地址划分**：
```
| Tag (48位) | Index (10位) | Offset (6位) |
```

**32位地址划分**：
```
| Tag (16位) | Index (10位) | Offset (6位) |
```

**计算过程**：
```
块数 = 64KB ÷ 64B = 1024块
Index位数 = log₂(1024) = 10位
Offset位数 = log₂(64) = 6位（已包含块内字节选择）
Tag位数 = 地址总位数 - 10 - 6
```

**注意**：Offset已经包含了块内字节定位，不需要单独的Byte Offset！

**查找过程**：
1. 用Index从缓存中选择一行
2. 比较该行的Tag与地址的Tag
3. 检查Valid位
4. 如果都匹配，用Offset从数据块中选择字节

### 核心概念

1. **地址划分取决于缓存大小和块大小**

2. **Offset用于选择块内的具体字节**，位数由块大小决定

---

### Offset详解

**Offset由块大小决定**：
- Offset位数 = log₂(块大小)
- 块大小4字节 → Offset 2位（块内位置0-3）
- 块大小32字节 → Offset 5位（块内位置0-31）
- 块大小128字节 → Offset 7位（块内位置0-127）

**Offset的本质**：
- Offset就是地址除以块大小后的余数
- 用二进制看：就是地址的最低几位
- 作用：告诉缓存"要取块内的第几个字节"

**完整地址划分计算示例**：
```
给定：缓存总容量4096字节，128块，32位地址

1. 块大小 = 4096 ÷ 128 = 32字节
2. Offset位数 = log₂(32) = 5位
3. Index位数 = log₂(128) = 7位
4. Tag位数 = 32 - 7 - 5 = 20位

地址划分：| Tag(20位) | Index(7位) | Offset(5位) |
```

## 块大小考虑

![[第五章28.png]]

### 核心内容讲解

**增加块大小的优点**：
- 更好利用空间局部性
- 减少强制性未命中（首次访问）

**增加块大小的缺点**：

2. **现代缓存典型块大小**：32-64字节

3. **污染效应**：大块可能将其他有用的块挤出缓存

## 缓存未命中处理

![[第五章29.png]]

### 核心内容讲解

**命中时**：CPU正常执行

**未命中时**：
1. 停顿CPU流水线（Stall）
2. 从下一层取数据块
3. 存入缓存
4. 恢复CPU执行

**指令缓存未命中**：重启取指阶段

**数据缓存未命中**：依赖该数据的指令停顿，其他指令可能继续（乱序执行）

### 核心概念

1. **未命中惩罚（Miss Penalty）** 是评估缓存设计的重要指标

2. **非阻塞缓存**：允许CPU在未命中期间继续执行其他指令

3. **关键路径影响**：如果未命中发生在指令依赖链上，整体性能影响更大

---

## 写直达策略

![[第五章30.png]]

### 核心内容讲解

**写直达（Write-Through）**：
- 写命中时，同时更新缓存和主存
- 主存始终与缓存一致

**问题**：每次写都需要访问主存，性能开销大

**解决方案：写缓冲（Write Buffer）**：
- 数据先写入缓存和写缓冲
- CPU可立即继续执行
- 后台将数据写入主存

**性能计算**：
```
CPI = 1 + 0.1 × 100 = 11（10%存储指令，每次写需100周期）
```

### 核心概念

1. **写缓冲缓解了写直达的性能问题**

2. **写缓冲满时的停顿**：需要等待写缓冲清空

3. **一致性优势**：缓存和内存始终一致，便于多处理器共享

---

## 写回策略

![[第五章31.png]]

### 核心内容讲解

**写回（Write-Back）**：
- 写命中时，只更新缓存
- 主存在块被替换时才更新
- 需要**脏位（Dirty Bit）** 跟踪修改状态

**替换脏块时**：
- 先将脏块写回主存，将脏块的内容存到主存对应的位置上
- 再加载新块，将主存的数据加载到新地址的脏块
- 可用写缓冲优化

### 核心概念
1. **写回策略显著减少对主存的写操作**
2. **脏位**：标记缓存行是否被修改过， 表示这个块**被修改过**，但还没写回主存。
3. **适合写密集型工作负载**：减少总线带宽占用

---

## 写分配策略

![[第五章32.png]]

### 核心内容讲解

**写分配（Write Allocate）**：
- 写未命中时，先将块从主存加载到缓存
- 再执行写操作
- 后续对同块其他数据的访问会命中

**非写分配（No-Write Allocate / Write Around）**：
- 写未命中时，直接写主存
- 不加载块到缓存

**组合策略**：
- 写回缓存 + 写分配（常见）
- 写直达缓存 + 非写分配（常见）

### 核心概念

1. **写分配与写回配合效果最佳**：一旦块在缓存中，后续写操作都在缓存内完成

2. **非写分配适合只写一次的大块数据**

3. **组合选择影响整体性能**：需要根据工作负载特性选择

FastMATH是一款**嵌入式MIPS处理器**，采用12级流水线，每个周期都要访问指令和数据：

**缓存配置**：

## Intrinsity FastMATH缓存结构详解

![[第五章34.png]]

### 核心内容讲解

这张图展示了FastMATH缓存的**完整寻址与数据读取逻辑**，是理解cache工作原理的关键图：

**地址划分（32位地址）** ：

表格

|字段|位数|作用|
|---|---|---|
|Tag|18位|标识是哪个主存块|
|Index|8位|选中256行中的哪一行|
|Byte offset|4位|选中块内16字中的哪个字节|

> 为什么是这些位数？256块 → Index需要 log2​256=8 位；每块16字 = 64字节 → Byte offset需要 log2​64=6 位，其中低2位选字节、高4位选字；剩余 32−8−6=18 位做Tag。

**Cache每行结构**：

表格

|字段|位数|说明|
|---|---|---|
|V（有效位）|1位|标记该行是否存有有效数据|
|Tag|18位|与地址的Tag比对，确认是否命中|
|Data|512位|16字 × 32位/字 = 512位|

**命中判断流程**：

1. **Index** → 选中cache的某一行
2. 取出该行的 **Tag** 与地址的Tag → 送入**比较器（Compare）**
3. 比较结果 **AND** 有效位V → 输出 **Hit信号**
    - Tag匹配 **且** V=1 → Hit ✅
    - Tag不匹配 **或** V=0 → Miss ❌
4. **Byte offset** → 通过**多路选择器（Mux）** 从512位Data中选出32位输出

### 核心概念

1. **命中 = Tag匹配 + 有效位为1**：两个条件缺一不可，V=0说明该行还没加载过数据
    
2. **比较器是直接映射cache的核心**：只需要1个比较器（因为Index已经唯一确定了一行），硬件简单
    
3. **Mux（多路选择器）的作用**：一块数据有16个字，Byte offset告诉Mux该输出哪个字

## 主存支持缓存

![[第五章35.png]]

### 核心内容讲解

DRAM作为主存需要与缓存协同工作：

**DRAM特性**：
- 使用DRAM作为主存
- 固定宽度输出（如1字）
- 通过固定宽度的时钟总线连接
- 总线时钟通常慢于CPU时钟

**块读取时序（Example cache block read）**：
```
1 bus cycle：地址传输
15 bus cycles per DRAM access：DRAM访问延迟
1 bus cycle per data transfer：数据传输
```

**4字块、1字宽DRAM的Miss Penalty计算**：
```
Miss Penalty = 1 + 4×15 + 4×1 = 65 bus cycles
              ↑      ↑      ↑
           地址传输 访问延迟  数据传输

带宽 = 16 bytes / 65 cycles = 0.25 B/cycle
```

### 核心概念

1. **DRAM访问延迟远大于CPU周期**：这是引入缓存的主要原因

2. **块传输效率**：一次传输多个字可以提高带宽利用率

3. **Miss Penalty的组成**：地址传输 + DRAM访问 + 数据传输

## 测量缓存性能

![[第五章36.png]]

### 核心内容讲解

**CPU时间的组成**：
```
CPU Time = Program execution cycles + Memory stall cycles
         = 程序执行周期        + 存储停顿周期
```

**存储停顿周期的来源**：
- 主要来自缓存未命中
- 包括从下一层取数据的时间

**存储停顿周期计算**：
```
Memory stall cycles = Memory accesses/Program × Miss rate × Miss penalty
                    = (每程序内存访问次数) × (未命中率) × (未命中惩罚)

或者：

Memory stall cycles = Instructions/Program × Misses/Instruction × Miss penalty
                    = (每程序指令数) × (每指令未命中次数) × (未命中惩罚)
```

### 核心概念

1. **缓存性能直接影响CPU执行时间**

2. **减少未命中率或未命中惩罚都能提升性能**

3. **Miss Penalty是关键指标**：取决于主存速度和块传输效率

## Cache Performance Example（缓存性能计算）
![[第五章37.png]]
### 已知条件

表格

|参数|值|含义|
|---|---|---|
|I-cache miss rate|2%|每条指令取指时，2%的概率未命中|
|D-cache miss rate|4%|每次访问数据时，4%的概率未命中|
|Miss penalty|100 cycles|每次未命中要额外等100个周期|
|Base CPI（理想cache）|2|假设cache永远命中，平均每条指令2个周期|
|Load & Store占比|36%|所有指令中，36%需要访问内存（读/写数据）|

### 计算过程

**第一步：算I-cache带来的额外周期**

每条指令都取指，所以：

> I-cache额外周期 = 2% × 100 = **2 cycles/instruction**

**第二步：算D-cache带来的额外周期**

不是每条指令都访问数据，只有36%（Load/Store）才需要：

> D-cache额外周期 = 36% × 4% × 100 = **1.44 cycles/instruction**

⚠️ 这就是为什么要乘0.36——因为D-cache只在访问数据时才有机会miss，而64%的指令根本不碰数据。

**第三步：算实际CPI**

> Actual CPI = Base CPI + I-cache额外 + D-cache额外
> 
> = 2 + 2 + 1.44 = **5.44**

**第四步：算理想CPU快多少倍**

> 理想CPU（无miss）比实际CPU快 = 5.44 / 2 = **2.72倍**

### 一句话总结

虽然cache未命中率看着很小（2%、4%），但因为miss penalty高达100个周期，最终实际性能比理想情况慢了**将近3倍**。所以**降低miss penalty和miss rate**是cache设计的核心目标。

## 平均内存访问时间

![[第五章38.png]]

### 核心内容讲解

**AMAT（Average Memory Access Time，平均内存访问时间）**：

$$AMAT = Hit\ Time + Miss\ Rate \times Miss\ Penalty$$

**命中时间（Hit time）**：
- 命中时的访问时间
- 包括标签比较和数据选择的时间
- 对性能同样重要！

**未命中率（Miss Rate）**：
- 未命中次数 / 总访问次数
- 通常以百分比表示

**未命中惩罚（Miss Penalty）**：
- 未命中时额外花费的时间
- 从下一层获取数据所需周期

### AMAT计算示例

**给定参数**：
```
CPU时钟周期 = 1ns
命中时间 = 1 cycle
未命中惩罚 = 20 cycles
I-cache未命中率 = 5%
```

**计算过程**：
```
AMAT = Hit time + Miss rate × Miss penalty
     = 1 + 0.05 × 20
     = 1 + 1
     = 2ns
```

**等价表示**：平均每条指令需要2个周期

### 核心概念

1. **命中时间同样重要**：即使牺牲少量命中率换取更快的命中时间，也可能改善整体性能

2. **AMAT是评估存储层次性能的统一指标**

3. **优化方向**：
   - 降低未命中率（Miss Rate ↓）
   - 降低未命中惩罚（Miss Penalty ↓）
   - 降低命中时间（Hit Time ↓）

## Performance Summary（性能总结）

### 三个关键趋势

**① 当CPU性能提升时**
→ Miss penalty变得**更显著**
> CPU越快，每个周期时间越短，但内存速度没跟上，所以miss时浪费的周期占比更大了

**② 当Base CPI降低时**
→ 内存停顿占的时间比例**更大**
> 举个直观例子：
> 
> - Base CPI = 10，miss额外5个周期 → miss占比 5/15 = 33%
> - Base CPI = 1，miss额外5个周期 → miss占比 5/6 = 83%
> 
> CPU本身执行变快了，但miss浪费的时间没变，所以miss的"拖后腿"效应反而更明显了

**③ 当时钟频率提高时**
→ 内存停顿浪费的CPU周期数**更多**
> 时钟频率翻倍 = 每个周期时间减半 = 同样的内存访问时间折算成更多CPU周期

### 最终结论

> **评估系统性能时，绝对不能忽视cache行为。**
### 一句话总结

CPU发展越来越快，内存速度跟不上，cache的命中率对整体性能的影响只会越来越大。过去可以忽略的miss，现在可能让CPU慢好几倍——所以cache设计越来越重要。

相联度决定块在缓存中的可选位置数量：

## 相联性谱

![[第五章42.png]]

### 核心内容讲解

**对于8条目的缓存，不同相联度的结构对比：**

| 相联度 | 结构名称 | 可选位置数 | 需要的比较器 |
|--------|---------|-----------|------------|
| 1-way | 直接映射 | 1个 | 1个 |
| 2-way | 2路组相联 | 2个 | 2个 |
| 4-way | 4路组相联 | 4个 | 4个 |
| 8-way | 全相联 | 8个 | 8个 |

**直接映射（1-way）**：
- 8个块 = 8个独立位置
- 每块只有唯一位置
- Index决定位置，Tag验证身份

**2路组相联**：
- 4个组 × 2路 = 8块
- Index定位组，组内2选1
- 减少冲突未命中

**4路组相联**：
- 2个组 × 4路 = 8块
- Index定位组，组内4选1
- 更高的灵活性

**全相联（8-way）**：
- 整个缓存作为1个组
- 块可放在任意位置
- 需要8路并行比较

### 核心概念

1. **相联度越高，灵活性越大**：块可以放在更多位置

2. **代价是硬件复杂度**：需要更多比较器

3. **选择取决于权衡**：性能 vs 成本/延迟

## 相联性对比示例

![[第五章43.png]]

### 核心内容讲解

**访问序列**：0, 8, 0, 6, 8（5次访问）

**直接映射缓存的地址分析**：
```
块大小 = 1 word
缓存块数 = 8

块地址计算：Block = Memory address / Block size

地址0 → 块0 → Index = 0
地址8 → 块8 → Index = 8 mod 8 = 0  ← 与地址0冲突！
地址6 → 块6 → Index = 6
```

**访问过程分析**：

表格

| 访问  | 地址  | Index | Tag | 命中 / 缺失 | 说明                                 |
| --- | --- | ----- | --- | ------- | ---------------------------------- |
| 0   | 0   | 0     | 0   | Miss    | 首次访问，加载 Mem [0] 到块 0               |
| 8   | 8   | 0     | 1   | Miss    | Index=0 被 Mem [0] 占用，替换后加载 Mem [8] |
| 0   | 0   | 0     | 0   | Miss    | Index=0 被 Mem [8] 占用，再次替换          |
| 6   | 6   | 6     | 0   | Miss    | Index=6 独立位置                       |
| 8   | 8   | 0     | 1   | Miss    | Index=0 被 Mem [0] 占用，替换 Mem [0]    |

**结果**：5次访问全部Miss！

### 核心概念

1. **直接映射的冲突问题**：地址0和地址8映射到同一位置

2. **反复冲突导致性能下降**：同一个Index的位置被反复替换

3. **这就是高相联度缓存的价值**：提供更多可选位置

## 相联性示例续

![[第五章44.png]]

### 2路组相联示例

**访问序列**：0, 8, 0, 6, 8

**缓存结构**：4组 × 2路 = 8块
- Index = Block mod 4

**访问过程**：

| 访问  | Index | 组内块 0  | 组内块 1  | 命中 / 缺失          |
| --- | ----- | ------ | ------ | ---------------- |
| 0   | 0     | -      | -      | Miss（加载 Mem [0]） |
| 8   | 0     | Mem[0] | -      | Miss（加载 Mem [8]） |
| 0   | 0     | Mem[0] | Mem[8] | **Hit!**         |
| 6   | 2     | -      | -      | Miss（加载 Mem [6]） |
| 8   | 0     | Mem[8] | Mem[6] | **Hit!**         |

**结果**：2次Hit，3次Miss → 显著改善！

### 全相联示例

**访问序列**：0, 8, 0, 6, 8

**缓存结构**：1个组 × 8路

| 访问 | 缓存内容 | 命中/缺失 |
|------|---------|----------|
| 0 | [Mem[0]] | Miss |
| 8 | [Mem[0], Mem[8]] | Miss |
| 0 | [Mem[0], Mem[8]] | **Hit** |
| 6 | [Mem[0], Mem[8], Mem[6]] | Miss |
| 8 | [Mem[0], Mem[8], Mem[6]] | **Hit** |

**结果**：2次Hit，3次Miss → 与2路相同

### 核心概念

1. **2路组相联大幅减少冲突**：地址0和8在同一组但不同路
2. **全相联最灵活**：块可放在任意位置
3. **实际权衡**：全相联硬件成本高，2-4路是常见选择

## 组相联缓存硬件结构

![[第五章46.png]]

### 核心内容讲解

**地址划分（4路组相联示例）**：
```
┌────────────────────┬──────────┬──────────┐
│       Tag          │  Index   │  Offset  │
│     22位           │   8位    │   2位    │
│  (31...12)        │ (11...3) │  (1...0) │
└────────────────────┴──────────┴──────────┘
```

**硬件结构**：
```
1. Index选组：用8位Index从256个组中选择1个
        ↓
2. Tag并行比较：4路的Tag同时与地址Tag比较
        ↓
3. 命中判断：比较结果+Valid位 → 与门 → 或门 → Hit
        ↓
4. 数据选择：4选1多路选择器 → 输出数据
```

**关键组件**：

| 组件           | 作用                 |
| ------------ | ------------------ |
| Tag 比较器 (×4) | 同时比较 4 路 Tag       |
| 与门 (×4)      | Tag 匹配 AND Valid=1 |
| 或门           | 任意一路命中即为命中         |
| 4 选 1 多路选择器  | 根据命中信号选择数据         |

### 核心概念

1. **相联度越高，比较器越多**：硬件复杂度线性增长

2. **并行比较是性能关键**：所有候选块同时检查

3. **延迟随相联度增加**：更多比较器可能拉长关键路径

## 替换策略概述

![[第五章47.png]]

### 核心内容讲解

当缓存组已满且需要加载新块时，必须选择一个块替换：

**直接映射缓存**：
- 没有选择——只有唯一位置
- 必须替换该位置的原块

**组相联缓存**：
- 优先选择**无效块**（Valid=0）
- 如果没有无效块，在组内选择一个块替换

常见策略包括 **LRU**（最久未用）、**随机**、**FIFO**（先进先出）和**伪LRU**，核心权衡是命中率 vs 硬件复杂度。关于各策略的具体实现细节和对比，详见后文"缓存替换策略详解"一节。

### 核心概念

1. **替换策略直接影响缓存命中率**

2. **高相联度缓存常用伪LRU**：权衡性能和硬件成本

## 多级缓存

![[第五章48.png]]

### 核心内容讲解

现代处理器采用**多级缓存**来平衡速度和容量：

**第一级缓存（L1）**：
- 紧邻CPU
- 最小，但最快
- 通常1-2个周期访问

**第二级缓存（L2）**：
- 服务于L1的未命中
- 更大，但稍慢
- 比主存快但比L1慢

**第三级缓存（L3，可选）**：
- 高端系统才有
- 更大，进一步减少主存访问

### 性能对比

**单级L1缓存的问题**：
```
基础CPI = 1
L1未命中率 = 2%
未命中惩罚 = 100ns / 0.25ns = 400周期
有效CPI = 1 + 0.02 × 400 = 9
```

**添加L2缓存后**：
- 大幅减少对主存的访问
- 有效降低平均内存访问时间

### 核心概念

1. **多级缓存通过在速度和容量间取得更好平衡**

2. **L1命中率高但容量小，L2容量大但命中率较低**

3. **全局未命中率**：通过L1后仍miss的比例（到达主存）

## 多级缓存示例

![[第五章49.png]]![[第五章50.png]]

### 给定参数

```
CPU基础CPI = 1，时钟频率 = 4GHz
每指令L1未命中率 = 2%
主存访问时间 = 100ns
```

**周期换算**：
```
时钟周期 = 1 / 4GHz = 0.25ns
100ns = 100 / 0.25 = 400周期
```

### 仅L1缓存

```
Miss penalty = 400 cycles
有效CPI = 1 + 0.02 × 400 = 9
```

### 添加L2缓存后

**L2参数**：
```
L2访问时间 = 5ns = 20周期
L2全局未命中率 = 0.5%（到达主存的比例）
```

**Miss penalty 分析**：

| 情况               | 惩罚     |
| ---------------- | ------ |
| L1 命中            | 0 周期   |
| L1 Miss, L2 Hit  | 20 周期  |
| L1 Miss, L2 Miss | 400 周期 |

**CPI计算**：
```
CPI = 基础CPI + L1 Miss×L2惩罚 + 全局Miss×内存惩罚
    = 1 + 0.02 × 20 + 0.005 × 400
    = 1 + 0.4 + 2.0
    = 3.4
```

**性能提升**：
```
性能比 = 9 / 3.4 = 2.6倍
```

### 核心概念

1. **L2缓存带来2.6倍性能提升**：显著减少主存访问

2. **每级设计目标不同**：
   - L1：最小化命中时间（快速）
   - L2/L3：最小化未命中率（高效）

3. **包含策略（Inclusive vs Exclusive）**：L2是否包含L1的内容

## 多级缓存设计考虑

![[第五章51.png]]

### 各级缓存的设计重点

**L1缓存（第一级）**：
- 核心目标：**最小化命中时间**
- 块大小相对较小
- 通常离CPU最近，必须快

**L2/L3缓存（后续级）**：
- 核心目标：**低未命中率**，避免访问主存
- 命中时间的影响较小（因为已经在L1之后）
- 可以用更大的块和更高的相联度

### 设计权衡

| 特性 | L1 | L2/L3 |
|------|-----|-------|
| 容量 | 小（32-64KB） | 大（256KB-数MB） |
| 块大小 | 较小 | 较大 |
| 相联度 | 通常4-8路 | 可能更高 |
| 命中时间 | 关键（1-2周期） | 次要（数十周期） |
| 未命中率目标 | 尽量低 | 尽量低 |

### 典型配置

```
L1块大小 < L2块大小  （如：32B vs 64B）
L1容量 < L2容量      （如：32KB vs 256KB）
L1相联度 ≥ L2相联度  （L1需要更快的访问）
```

### 核心概念

1. **分离vs统一L2**：统一缓存灵活但可能成为瓶颈

2. **片上vs片外L2**：片上更快但占用芯片面积

3. **性能优化需要全局考虑**：不能只优化某一层

## 与高级CPU的交互

![[第五章52.png]]

### 核心内容讲解

**乱序执行CPU（Out-of-order）**可以在缓存未命中期间继续执行其他指令：

**关键机制**：
- 待存储指令留在Load/Store单元

```
传统观点：
缓存Miss → CPU停顿 → 性能下降

现代观点：
```

**坏消息**：分析变得复杂，需要系统模拟

### 核心概念

1. **乱序执行减少缓存Miss的直接影响**：CPU不必完全停顿

2. **性能分析更复杂**：取决于程序的数据依赖关系

3. **评估需要模拟**：很难用公式精确预测

## 缓存性能综合练习

![[第五章54.png]]

### 核心内容讲解

这道题综合考查缓存性能分析的三种方法，我们需要仔细分析每种情况。

**已知条件**：
```
CPU基础CPI = 2.0，时钟周期 = 0.5ns
L1 cache未命中率 = 5%
L2 cache未命中率 = 33%（相对于L1未命中）
L1 miss penalty = 20周期
L2 miss penalty = 200周期
```

### AMAT方法

**公式**：
$$AMAT = T_{L1} + MissRate_{L1} \times (T_{L2} + MissRate_{L2} \times T_{L2,miss})$$

**代入计算**：
```
AMAT = 0.5ns + 0.05 × (10ns + 0.33 × 100ns)
     = 0.5ns + 0.05 × 43ns
     = 0.5ns + 2.15ns
     = 2.65ns
```

### CPI方法

**公式**：
$$CPI = CPI_{base} + IC_{miss} \times Penalty$$

**计算过程**：
```
总CPI = 2.0 + (L1 miss贡献) + (L2 miss贡献)
      = 2.0 + 0.05 × 10ns + 0.05 × 0.33 × 100ns
      = 2.0 + 0.5ns + 1.65ns
      = 2.0 + 2.15ns（换算为周期）
      = 2.0 + 4.3 cycles
      = 6.3 cycles
```

### 局部性分析

**空间局部性视角**：
- 大块提高空间局部性，减少 compulsory miss
- 但增大块会减少块数量，增加 conflict miss

**时间局部性视角**：
- LRU替换策略依赖时间局部性
- 热点数据应尽量保留在缓存中

### 一句话总结

三种分析方法本质等价，选择取决于具体问题的表述方式。考试时注意单位换算和条件判断。

## AMAT深入分析

![[第五章55.png]]

### 核心内容讲解

AMAT（Average Memory Access Time）是评估存储层次性能的核心指标。

**AMAT = Hit Time + Miss Rate × Miss Penalty**

### 公式分解

**Hit Time（命中时间）**：
- 从CPU发出请求到收到数据的时间
- 包括地址解码、标签比较、数据选择

**Miss Rate（未命中率）**：
- 未命中次数 / 总访问次数
- 通常以百分比表示

**Miss Penalty（未命中惩罚）**：
- 未命中时从下一层获取数据的时间
- 包括传输延迟和处理开销

### 多级缓存的AMAT

对于两级存储系统：
$$AMAT = T_{L1} + MR_{L1} \times T_{L2}$$
$$= T_{L1} + MR_{L1} \times (T_{L2,hit} + MR_{L2} \times T_{mem})$$

### 性能优化方向

| 优化目标 | 方法 | 代价 |
|---------|------|------|
| 降低Hit Time | 简化设计、减少级数 | 可能增加Miss Rate |
| 降低Miss Rate | 增大缓存、提高相联度 | 增加Hit Time |
| 降低Miss Penalty | 预取、多级流 水线 | 增加复杂度 |

### 核心概念

1. **AMAT是存储系统设计的北极星指标**：所有优化最终都反映在AMAT上

2. **三个因素相互制约**：不能只优化某一个指标

3. **瓶颈分析**：找出最影响AMAT的因素优先优化

---

## 多级缓存性能深入分析

![[第五章56.png]]

### 核心内容讲解

多级缓存的全局未命中率计算是考试重点。

### 全局未命中率 vs 局部未命中率

**局部未命中率（Local Miss Rate）**：
$$MissRate_{L2,local} = Misses_{L2} / Accesses_{L2}$$

**全局未命中率（Global Miss Rate）**：
$$MissRate_{L2,global} = Misses_{L2} / Total\ Accesses$$
$$= MissRate_{L1} \times MissRate_{L2,local}$$

### 示例计算

```
L1: 1000次访问，50次miss
L2: 50次访问，10次miss

L1局部未命中率 = 50/1000 = 5%
L2局部未命中率 = 10/50 = 20%
L2全局未命中率 = 50/1000 × 10/50 = 5% × 20% = 1%
```

### CPI计算

$$CPI = CPI_{base} + MF_{L1} \times MP_{L1} + MF_{L2} \times MP_{L2} + MF_{mem} \times MP_{mem}$$

其中：
- $MF_x$ = 全局未命中率（到达该层的miss频率）
- $MP_x$ = 该层的miss penalty（该层到下一层的访问时间）

### 设计建议

1. **L1目标**：最小化命中时间（快速）
2. **L2目标**：最小化全局未命中率（高效）
3. **L3目标**：进一步减少主存访问

### 核心概念

1. **全局未命中率才是真正影响性能的因素**

2. **L1的miss会成为L2的访问**，所以L2的局部miss率 ≠ 全局miss率

3. **优化L2时关注全局miss率**：减少到达主存的miss次数

## 缓存性能优化策略

![[第五章57.png]]

### 核心内容讲解

基于三种未命中类型，我们可以制定针对性优化策略。

### 未命中类型与优化

| 未命中类型 | 原因 | 解决方案 |
|-----------|------|----------|
| 强制性(Compulsory) | 首次访问某块 | 预取、大块 |
| 容量(Capacity) | 缓存容量不足 | 增大容量 |
| 冲突(Conflict) | 相联度不足 | 提高相联度 |

### 容量未命中分析

**判断方法**：如果全相联缓存仍有未命中，则存在容量未命中

**示例**：
```
工作集大小 = 100KB
缓存大小 = 64KB
→ 必然存在容量未命中
```

### 冲突未命中分析

**判断方法**：直接映射 vs 全相联，全相联减少的未命中 = 冲突未命中

**示例**：
```
直接映射未命中率 = 10.3%
全相联未命中率 = 4.3%
→ 冲突未命中 = 6.0%
```

### 综合优化示例

**原始设计**：
```
Cache: 16KB, 1-way, 32B块
Miss Rate: 8.5%
```

**优化后的权衡**：
```
方案A: 32KB块 → 减少capacity miss，但增加miss penalty
方案B: 4-way组相联 → 减少conflict miss
方案C: 预取 → 减少compulsory miss
```

### 核心概念

1. **先分析未命中类型，再选择优化策略**

2. **不要过度优化**：有些miss是不可避免的

3. **测试验证**：理论分析与实际测量结合

## 性能分析实战

![[第五章58.png]]

### 核心内容讲解

让我们通过综合例题来巩固缓存性能分析。

### 题目

给定系统参数，计算实际CPU时间：
```
Base CPI = 1.0
Clock rate = 4GHz
I-cache miss rate = 2%
D-cache miss rate = 4%
Miss penalty = 100ns
Load/Store指令比例 = 30%
```

### 解答

**第一步：周期换算**
```
时钟周期 = 1 / 4GHz = 0.25ns
Miss penalty = 100ns / 0.25ns = 400周期
```

**第二步：计算存储停顿**
```
I-cache停顿 = 1.0 × 2% × 400 = 8 cycles/instr
D-cache停顿 = 30% × 4% × 400 = 4.8 cycles/instr
总停顿 = 12.8 cycles/instr
```

**第三步：计算实际CPI**
```
实际CPI = 1.0 + 12.8 = 13.8
```

**第四步：计算CPU时间**
```
T = CPI × IC × T_cycle
  = 13.8 × IC × 0.25ns
```

### 性能提升估算

如果改善缓存性能后，D-cache miss率降到2%：
```
新D-cache停顿 = 30% × 2% × 400 = 2.4 cycles/instr
新CPI = 1.0 + 8 + 2.4 = 11.4
性能提升 = 13.8 / 11.4 ≈ 1.21倍
```

### 核心概念

1. **单位换算是计算的第一步**：ns和cycles之间需要时钟周期转换

2. **Load/Store比例只影响D-cache**：因为只有这些指令访问数据

3. **性能瓶颈分析**：找出最大的停顿来源优先优化

## 页表项详细结构

![[第五章70.png]]

### 核心内容讲解

页表项（PTE）是页表的基本单位，存储虚拟页到物理页的映射信息。

### 典型PTE格式（32位系统）

```
┌────────────┬────────┬─────┬────┬─────┬────────┐
│ 物理页号    │  保留  | G   | A  | D   │  保护  │
│ (20位)     │        |     |    |     │ (4位)  │
└────────────┴────────┴─────┴────┴─────┴────────┘
```

### 各字段详解

| 字段 | 位数 | 说明 |
|------|------|------|
| **物理页号 (PPN)** | 20位 | 物理地址的高位部分 |
| **有效位 (V)** | 1位 | 页是否在内存中（有时隐含） |
| **保护位** | 4位 | 读/写/执行权限 |
| **脏位 (D)** | 1位 | 页是否被修改过 |
| **访问位 (A)** | 1位 | 页是否被访问过（用于替换） |
| **全局位 (G)** | 1位 | 内核页，不因进程切换清除TLB |

### PTE中的关键位

**有效位（Valid）**：
- 0 = 页不在内存中（可能触发page fault）
- 1 = 页在内存中

**脏位（Dirty Bit）**：
- 0 = 页面未被修改（干净）
- 1 = 页面已被修改（替换时需写回磁盘）

**访问位（Access/Reference Bit）**：
- 每次访问页面时由硬件设置
- OS周期性清零，用于LRU替换算法

### 核心概念

1. **页表项是虚拟内存系统的核心数据结构**：包含翻译和保护所需的所有信息

2. **PTE大小通常为4字节**：32位系统可寻址4GB虚拟空间，每进程最多1M个PTE

3. **TLB缓存的是PTE的内容**：而不是完整的物理地址

## 页表结构

![[第五章71.png]]![[第五章72.png]]

### 核心内容讲解

**页表**：存储虚拟页到物理页的映射

**页表项（PTE）内容**：
- **有效位**：页是否在内存中

## 多级页表结构

![[第五章73.png]]

### 核心内容讲解

单级页表的问题：32位系统4GB虚拟空间，每进程需要4MB的页表（4GB / 4KB × 4字节）。

### 多级页表的优势

**页目录 + 页表的两级结构**：
- 页目录指向页表
- 页表指向实际页面
- 不需要将整个4GB空间都映射到页表中

### 两级页表工作流程

```
虚拟地址结构（32位，4KB页）：
┌──────────┬──────────┬──────────┐
│ 页目录索引 │  页表索引  │  页内偏移  │
│  (10位)  │  (10位)  │  (12位)  │
└──────────┴──────────┴──────────┘

1. 用页目录索引找到页目录项（PDE）
2. 用页目录项指向的页表
3. 用页表索引找到页表项（PTE）
4. 用PTE中的物理页号 + 页内偏移 = 物理地址
```

### 页目录项格式

```
┌─────────────────┬────┬────┐
│ 页表基地址       │ A  | V  │
│ (20位)          │    │    │
└─────────────────┴────┴────┘
```

### 64位系统的多级页表

**四级页表**（x86-64）：
```
┌─────┬─────┬─────┬─────┬────────┐
│ PML4│ PDPT│  PD │  PT │ Offset │
│ 9位 │ 9位 │ 9位 │ 9位 │  12位  │
└─────┴─────┴─────┴─────┴────────┘
```

### 核心概念

1. **多级页表只保留使用的页表**：节省内存

2. **页目录常驻内存**：其他页表可以按需加载

3. **每次地址转换需要多次内存访问**：TLB的重要性凸显

- 引用位周期性清零

3. **写时复制（COW）**：延迟物理复制直到真正写入

**问题**：地址转换需要额外的内存访问

## TLB结构详解

![[第五章76.png]]

### 核心内容讲解

TLB是一种特殊的相联缓存，专门用于存储虚拟地址到物理地址的转换。

### TLB条目结构

```
┌────────┬────────────┬──────────┬─────┬────┐
│  Valid │ Virtual Page#│ Physical Page#│ D  │ G │
│  (1位) │   (VPN)    │   (PPN)    │(1位)│(1位)│
└────────┴────────────┴──────────┴─────┴────┘
```

### TLB查找过程

**并行查找**：
1. 用虚拟页号（VPN）与所有TLB条目的Tag并行比较
2. 同时检查Valid位
3. 任一匹配即为命中

**全相联 vs 组相联**：
- 小TLB：通常全相联（16-64条目）
- 大TLB：通常组相联（128-512条目）

### TLB的上下文切换

**问题**：不同进程有不同的虚拟地址空间

**解决方案**：
- ASID（Address Space ID）标记每个TLB条目属于哪个进程
- TLB查找时同时匹配VPN和ASID
- 切换进程时可能需要部分或全部刷新TLB

### TLB Miss处理流程

```
TLB Miss
    ↓
┌─────────────────────────────┐
│ 页在内存中？                 │
└─────────────────────────────┘
    ↓ 是                       ↓ 否
从页表加载PTE到TLB         触发Page Fault
    ↓                       （OS处理磁盘I/O）
重试指令                      更新页表
                              ↓
                            重试指令
```

### 核心概念

1. **TLB的全相联查找确保最快的转换速度**

2. **ASID机制允许TLB条目在上下文切换后保留**

3. **TLB miss的开销远小于page fault**：但仍需优化

## 块放置策略

![[第五章83.png]]

### 核心内容讲解

### 核心概念

1. **选择放置策略是复杂度和性能之间的权衡**

2. **缓存层级越高**：越倾向于使用较低相联度

3. **虚拟内存**：天然使用全相联（页可放在任何帧）

## 缓存替换策略详解

![[第五章85.png]]

### 核心内容讲解

当缓存组已满且需要加载新块时，必须选择一个块替换。不同的替换策略有不同的性能特点。

### 替换策略对比

| 策略 | 全称 | 原理 | 优点 | 缺点 |
|------|------|------|------|------|
| **LRU** | Least Recently Used | 替换最久未访问的块 | 性能最优 | 高相联度实现复杂 |
| **随机** | Random | 随机选择 | 硬件简单 | 性能不稳定 |
| **FIFO** | First In First Out | 替换最早进入的块 | 实现简单 | 不考虑使用频率 |
| **PLRU** | Pseudo LRU | 近似LRU | 性能接近LRU，硬件简单 | 实际效果略差 |

### LRU实现详解

**LRU的核心思想**：保留最近使用的数据，替换最久未使用的

**2路组相联LRU实现**：
- 只需1个bit记录哪个路最近被使用
- 命中时更新bit
- 替换时选择bit=0的路

**4路组相联LRU实现**：
- 需要3个bit维护使用顺序
- 实现复杂度增加
- 通常使用树形LRU

### 伪LRU（PLRU）

**树形伪LRU**：
```
      root (1 bit)
     /        \
   L1a        L1b
  (1 bit)    (1 bit)
 /    \      /    \
0     1     0     1
→0   →1    →2    →3  (选择bit=0的子树)
```

**特点**：
- 每次访问更新O(log n)个bit
- 替换选择O(log n)个bit
- 硬件简单，性能接近真LRU

### 核心概念

1. **替换策略直接影响缓存命中率**

2. **2-4路用LRU，8路及以上用PLRU或随机**

3. **实测差异不大**：在现代工作负载下，各种策略的性能差距通常在1-2%

---

## 替换策略的选择

![[第五章86.png]]

### 核心内容讲解

替换策略的选择需要考虑硬件成本和性能收益。

### 不同相联度的策略选择

| 相联度 | 推荐策略 | 原因 |
|--------|---------|------|
| 1-way | 无需选择（唯一位置） | - |
| 2-way | LRU | 只需1 bit，简单高效 |
| 4-way | LRU或PLRU | 树形LRU 3 bit，可接受 |
| 8-way | PLRU | 7 bit树形实现 |
| 16-way+ | 随机或PLRU | 真LRU硬件太复杂 |

### 性能对比数据

**SPEC2000测试结果**（典型值）：
```
直接映射：Miss Rate基准
2路LRU：减少约10-15%
4路LRU：减少约15-20%
8路PLRU：减少约18-22%
全相联：最优但不可行
```

### 实际考虑因素

**缓存污染（Cache Pollution）**：
- 大块可能将热点数据替换出去
- LRU有助于保留热点数据

**公平性**：
- 某些访问模式可能导致某一路总被替换
- 随机策略可以避免这种不公平

### 核心概念

1. **大多数现代处理器使用PLRU或随机策略**：在复杂度和性能间取得平衡

2. **相联度越高，LRU的优势越不明显**：因为候选块更多

3. **实测比理论分析更重要**：不同工作负载差异很大

## 未命中类型

![[第五章87.png]]

### 核心内容讲解

**三类未命中**：

**强制性未命中（Compulsory/Cold Start）**：
- 首次访问某个块
- 无法避免

**容量未命中（Capacity）**：
- 缓存容量有限
- 被替换的块稍后又被访问

**冲突未命中（Conflict）**：
- 只存在于非全相联缓存
- 多个块竞争同一组的位置

### 核心概念

1. **区分未命中类型有助于针对性优化**

2. **冷启动未命中**：可通过预取减少
3. **容量未命中**：可通过增加缓存大小解决
4. **冲突未命中**：可通过增加相联度解决

## 缓存设计权衡

![[第五章88.png]]

### 核心内容讲解

| 设计改变 | 对未命中率的影响 | 负面性能影响 |
|----------|------------------|--------------|
| 增加缓存大小 | 减少容量未命中 | 可能增加访问时间 |
| 增加相联度 | 减少冲突未命中 | 可能增加访问时间 |
| 增加块大小 | 减少强制性未命中 | 增加未命中惩罚；过大可能增加污染 |

### 核心概念

1. **缓存设计是权衡的艺术**：没有最优解，只有适合特定场景的方案

2. **所有优化都有代价**：需要根据工作负载特性选择

3. **仿真和测量是关键**：理论分析需要实验验证

## 虚拟索引与物理标签

![[第五章89.png]]

### 核心内容讲解

缓存可以使用虚拟地址的某些部分作为索引，这涉及到缓存与虚拟内存的交互问题。

### 地址类型

**虚拟地址（VA）**：程序使用的地址
**物理地址（PA）**：实际内存地址

### 缓存索引方式

**PIPT（Physical Index, Physical Tag）**：
- 用物理地址的Index选择缓存行
- 用物理地址的Tag进行比较
- 最简单，但无法与TLB查找并行

**VIVT（Virtual Index, Virtual Tag）**：
- 用虚拟地址的Index和Tag
- 可以与TLB查找并行
- 别名问题严重：不同虚拟地址映射同一物理地址

**VIPT（Virtual Index, Physical Tag）**：
- 用虚拟地址的Index（与TLB查找并行）
- 用物理地址的Tag比较（正确性保证）
- 常见的折中方案

### VIPT的条件

**约束**：Index位不能超过页内偏移的位数

```
如果Offset = 6位（64B块），则Index最多6位
否则Index会跨越页边界，导致别名问题
```

### 核心概念

1. **VIPT允许并行TLB查找和缓存索引**

2. **Index位受页大小限制**：避免别名问题

3. **大缓存需要特殊技术**：如页着色（page coloring）

---

## 别名问题与解决

![[第五章90.png]]

### 核心内容讲解

别名（Alias）问题发生在多个虚拟地址映射到同一物理地址时。

### 别名问题示例

```
进程A: VA1 = 0x1000 → PA = 0x5000（假设）
进程B: VA2 = 0x2000 → PA = 0x5000（同一物理页）

如果缓存使用VIVT：
- VA1的数据可能放在Cache位置X
- VA2的数据可能放在Cache位置Y
- 修改一处不会更新另一处 → 数据不一致！
```

### 别名的影响

**数据一致性问题**：
- 共享内存中的数据可能被不同缓存行缓存
- 修改后其他副本不会更新

**缓存污染**：
- 同一物理数据占用多个缓存行
- 降低有效缓存容量

### 解决方案

**方案1：VIPT + 操作系统保证**
- 操作系统确保别名映射到同一缓存位置
- 实现：保证不同VA的Index部分相同

**方案2：物理标签**
- Tag使用物理地址
- 保证同一物理地址只有一个缓存副本

**方案3：别名消除（Anti-aliasing）**
- 硬件检测别名
- 将别名虚拟地址映射到同一缓存行

### 核心概念

1. **别名问题是VIVT缓存的主要挑战**

2. **VIPT通过限制Index位避免大多数别名问题**

3. **操作系统配合是解决别名问题的关键**

## 页着色技术

![[第五章91.png]]

### 核心内容讲解

页着色（Page Coloring）是解决大缓存Index位数不足的技术。

### 问题背景

**大缓存的Index需求**：
```
64KB缓存，64B块，4路组相联：
- 块数 = 64KB / 64B = 1024块
- 组数 = 1024 / 4 = 256组
- 需要8位Index
```

**页内偏移限制**：
```
4KB页 → Offset = 12位
只能使用12位中的低几位作为Index
```

### 页着色原理

**概念**：利用物理页号的高位来扩展Index

```
物理页号（PPN）结构：
┌──────────┬──────────┬────────┐
│ 颜色位   │ 实际高位  │ 颜色内偏移 │
│ (k位)   │          │         │
└──────────┴──────────┴────────┘

颜色 = PPN的高k位
选择缓存组时：Index ⊕ Color
```

### 页着色的效果

**无页着色**：
- 同一颜色的页只能使用部分缓存组
- 可能导致更多冲突

**有页着色**：
- 不同颜色的页可以使用不同缓存组
- 更均匀地利用缓存容量

### 核心概念

1. **页着色扩展了VIPT缓存的有效Index位数**

2. **需要操作系统和硬件协同实现**

3. **颜色数量 = 缓存组数 / 组内块数**

## Cache与虚拟内存的交互

![[第五章92.png]]

### 核心内容讲解

Cache和虚拟内存在现代处理器中密切交互，设计时需要考虑多方面因素。

### 交互流程

```
CPU产生虚拟地址(VA)
    ↓
┌─────────┐
│ TLB查找 │ ← 并行进行
└─────────┘
    ↓ (TLB命中后)
获得物理地址(PA)
    ↓
┌─────────┐
│ Cache   │ ← 用PA的Index和Tag查找
│ 查找    │
└─────────┘
    ↓
命中/未命中
```

### 关键设计决策

**1. 统一缓存 vs 分离缓存**

| 类型 | 优点 | 缺点 |
|------|------|------|
| 统一缓存 | 灵活利用容量 | 竞争可能导致miss |
| 分离缓存 | 并行访问I/D | 容量分配不灵活 |

**2. 包含缓存 vs 排除缓存**

| 类型 | 特性 | 适用场景 |
|------|------|---------|
| Inclusive | L2包含L1内容 | 简化一致性 |
| Exclusive | L2不含L1内容 | 最大化容量 |

**3. 写策略**

- 虚拟内存使用写回（延迟写磁盘）
- Cache可以使用写回或写直达

### 性能考虑

**TLB miss的影响**：
- TLB miss → 10-100周期
- Cache miss → 100-1000周期
- Page fault → 百万周期

**TLB miss和Cache miss的叠加**：
- 如果先查TLB再查Cache，可能串行延迟
- VIPT允许并行查找，但Tag比较需要PA

### 核心概念

1. **虚拟索引物理标签（VIPT）是常见的折中**

2. **TLB和Cache的设计相互影响**

3. **现代处理器使用多级TLB减少miss惩罚**

## 虚拟内存Cache设计

![[第五章93.png]]

### 核心内容讲解

虚拟内存系统的Cache设计需要解决几个独特问题。

### 问题1：上下文切换

**TLB刷新开销**：
- 每次上下文切换需要刷新TLB
- 刷新太频繁 → TLB miss率上升
- 刷新太慢 → 进程间保护减弱

**解决方案**：
- 使用ASID（地址空间ID）标记TLB条目
- 允许TLB条目在上下文切换后保留

### 问题2：共享内存

**共享映射的一致性**：
- 多个进程共享物理页面
- 每个进程的虚拟地址可能不同
- 需要确保缓存一致性

**操作系统职责**：
- 管理共享页面的映射
- 处理页面换出时的脏位检查

### 问题3：大页（Huge Pages）

**大页的优势**：
- 减少TLB miss率
- 减少页表大小
- 适合大内存应用

**大页的挑战**：
- 内存碎片化
- 页表管理复杂
- 分配不灵活

### 核心概念

1. **ASID机制允许TLB在上下文切换时保留有效条目**

2. **共享内存需要操作系统精心管理**

3. **大页是减少TLB miss的有效方法**

## 可靠性度量

- 减少MTTR：改进诊断和修复工具

1. **可靠性设计是多层次的**：ECC纠错、冗余备份等

3. **高可用系统需要同时优化MTTF和MTTR**

## 汉明纠错码

2. **SEC（Single Error Correction）**：纠正单比特错误

3. **校验位数计算**：满足 2^p ≥ d + p + 1

## 汉明码编码

![[第五章61.png]]

2. **校验位位置**：2的幂次方（1, 2, 4, 8, 16...）

3. **覆盖规则**：位号等于校验位号之和

## 存储系统可靠性设计

![[第五章66.png]]

### 可靠性指标详解

**FIT (Failures In Time)**：
- 衡量设备故障频率的单位
- 1 FIT = 每10亿小时发生1次故障
- 现代磁盘典型值：0.6-1.5 FIT

- 年度故障率

- 宇宙射线、α粒子等导致比特翻转

**纠错码的作用**：
- 检测错误
- 纠正单比特错误（SEC）
- 检测双比特错误（SEC-DED）

### 系统级可靠性设计

**RAID（Redundant Array of Independent Disks）**：
| 级别 | 冗余方式 | 容忍故障 | 应用场景 |
|------|---------|---------|---------|
| RAID 0 | 无 | 0 | 性能优先 |
| RAID 1 | 镜像 | 1 | 高可靠性 |
| RAID 5 | 奇偶校验 | 1 | 平衡方案 |
| RAID 6 | 双奇偶 | 2 | 云计算 |

**多副本策略**：
- 跨地域复制
- 通常3副本保证高可用

### 核心概念

1. **可靠性设计需要从硬件到软件的多层考虑**

2. **MTTF/MTBF是评估存储设备的关键指标**

3. **企业级存储系统采用多层次的冗余设计**

## 内存一致性

![[第五章97.png]]

### 核心内容讲解

**内存一致性**：定义多处理器系统中内存操作的可见顺序

**假设**：
- 写操作完成后，只有所有处理器都看到时才完成
- 处理器不重排写操作与其他访问

2. **宽松一致性模型**：允许更多优化，但增加编程复杂度

## DRAM刷新机制

### 核心内容讲解

DRAM需要周期性刷新来维持数据，这是DRAM与SRAM的关键区别。

### DRAM存储单元的工作原理

**DRAM Cell（DRAM单元）**：
```
┌────────┐
│电容     │ ← 存储电荷：充电=1，放电=0
└────┬───┘
     │
   ┌─┴─┐
   │ Tr │ ← 晶体管：控制访问
   └─┬─┘
     │
─────电容───┘
```

**问题**：电容会随时间放电（泄漏），需要周期性刷新

### 刷新机制详解

**刷新过程**：
1. 激活（Activate）某一行
2. 读取整行数据（电荷转化为电压）
3. 放大信号
4. 写回（Recharge）整行

**刷新频率**：
```
典型刷新周期：64ms
行数：8192行
刷新间隔：64ms / 8192 ≈ 7.8μs
```

**刷新开销**：
```
刷新一行时间 ≈ 100ns
刷新8192行总时间 ≈ 0.8ms
开销比例 ≈ 0.8ms / 64ms ≈ 1.25%
```

### 自刷新（Self-Refresh）

**概念**：DRAM进入低功耗状态，自己执行刷新

**应用场景**：
- 系统休眠
- 移动设备的低功耗模式
- DRAM保留但CPU停止

**优势**：
- 刷新完全由DRAM内部完成
- 无需内存控制器干预
- 可优化刷新功耗

### 刷新与性能

**刷新对性能的影响**：
```
刷新期间：DRAM无法响应访问请求
CPU等待：刷新操作阻塞内存访问

优化方向：
1. 分散刷新：定期刷新而非集中刷新
2. 局部刷新：只刷新需要刷新的行
3. 刷新跳过：已访问的行在短时间内无需刷新
```

### 核心概念

1. **DRAM刷新是保持DRAM数据完整性的必要操作**

2. **刷新开销约1-2%**：可接受的功耗代价

3. **自刷新支持低功耗模式**：移动设备的重要特性

## 存储层次综合练习

![[第五章105.png]]

### 核心内容讲解

让我们通过综合例题来巩固本章的核心概念。

### 综合题目

**系统配置**：
```
L1 Cache: 32KB, 4路组相联, 64B块, 命中时间2周期
L2 Cache: 256KB, 8路组相联, 64B块, 命中时间10周期
主存: 访问时间100ns
CPU: 4GHz, Base CPI = 1.0
Load/Store指令比例: 30%
L1 Miss Rate = 5%, L2 Miss Rate = 20%
```

### 求解

**1. 计算AMAT**：
```
AMAT = T_L1 + MR_L1 × T_L2
     = 2 + 0.05 × (10 + 0.20 × 100ns)

时钟周期 = 0.25ns
T_L2 = 10周期 = 2.5ns
T_mem = 100ns/0.25ns = 400周期 = 100ns

AMAT = 2 + 0.05 × (2.5 + 0.20 × 100)
     = 2 + 0.05 × 22.5
     = 2 + 1.125
     = 3.125周期
```

**2. 计算实际CPI**：
```
Miss Penalty转换：
L2 miss → 主存 = 100ns = 400周期

CPI = Base + I_cache_miss + D_cache_miss
     = 1.0 + (1.0 × 5% × 10) + (30% × 5% × 10) + (30% × 5% × 20% × 400)
     = 1.0 + 0.5 + 0.15 + 1.2
     = 2.85
```

**3. 判断优化方案**

哪个优化更有效？

| 优化方案 | 预期效果 | 计算验证 |
|---------|---------|---------|
| A: L1 Miss率→3% | 减少miss | 2.85→2.4 |
| B: L2 Miss率→10% | 减少L2→Mem访问 | 2.85→1.95 |

### 常见考点总结

1. **地址计算**：
   - 块地址 = floor(地址 / 块大小)
   - Index = (块地址) mod (组数)
   - Tag = 块地址 / 组数

2. **性能指标**：
   - AMAT = Hit Time + Miss Rate × Miss Penalty
   - CPI = Base CPI + Memory Stall Cycles

3. **未命中类型**：
   - 强制性：首次访问
   - 容量：工作集 > 缓存大小
   - 冲突：直接映射 vs 全相联

### 核心概念

1. **综合题需要熟练掌握各公式和单位换算**

2. **先理解问题，再选择合适的方法求解**

3. **优化选择要基于计算结果判断**

## 总结

![[第五章106.png]]

### 核心要点

1. **局部性原理**是存储层次结构的基础

2. **缓存**通过保持最近使用的数据来减少访问延迟

3. **直接映射、组相联、全相联**是三种基本的放置策略

4. **写直达和写回**是两种主要的写策略

5. **多级缓存**在容量和速度间取得更好平衡

6. **虚拟内存**使用磁盘作为主存的扩展

7. **TLB**加速虚拟地址到物理地址的转换

8. **缓存一致性**是多处理器系统的关键问题

### 设计权衡

- 更高的相联度 → 更低未命中率，但更复杂
- 更大的缓存 → 更低容量未命中，但可能更慢
- 更大的块 → 更好空间局部性，但增加未命中惩罚

理解这些权衡对于设计高效的计算机系统至关重要。
