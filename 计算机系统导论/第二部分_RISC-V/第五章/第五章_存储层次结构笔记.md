## 存储层次结构的基本概念

![[第五章2.png]]

### 核心内容讲解

这张图展示了计算机存储层次结构的核心原理，呈现从快速小容量到慢速大容量的金字塔式层级：

- **Processor（处理器）**：位于顶端，是数据的最终使用者
- **SRAM Memory**：对应L1/L2缓存，容量小（KB级别）但速度极快（0.5-2.5ns）
- **DRAM Memory**：对应主内存，容量中等（GB级别），速度适中（50-70ns）
- **Disk（磁盘）**：位于底端，容量最大（TB级别）但速度最慢（毫秒级）

箭头表示数据在各层之间的传递方向。

### 核心概念

1. **时间局部性（Temporal Locality）**：最近访问的项很可能再次被访问。例如循环中的指令会被反复执行。

2. **空间局部性（Spatial Locality）**：相邻地址的项很可能被一起访问。例如顺序执行的指令流、数组数据的顺序访问。

3. **存储层次结构的本质**：在速度、容量、成本三者之间做权衡，利用局部性原理让"大容量"和"高速"看似同时实现。

## 利用局部性原理

![[第五章3.png]]

### 核心内容讲解

存储层次结构通过数据复制实现"缓存"的假象：

**命中流程**：
- 数据请求首先到达最上层
- 如果数据存在（Hit命中），直接从上层返回
- 命中率高意味着大部分访问由上层服务，性能好

**未命中流程**：
- 如果数据不在上层（Miss未命中），从下一层获取整个数据块
- 获取的数据块被复制到上层供后续访问使用
- 未命中惩罚（Miss Penalty）= 从下层获取数据的时间

**块（Block/Line）**是数据复制的基本单位，大小影响性能。

### 核心概念

1. **块（Block）**：存储层次中数据复制的基本单位，大小是关键设计参数

2. **命中率的计算**：Hit ratio = hits / accesses = 1 - Miss ratio

3. **块大小的权衡**：
   - 大块利用空间局部性好
   - 但增加未命中惩罚和可能的缓存污染

## 存储技术对比

![[第五章5.png]]

### 核心内容讲解

四种存储技术的性能与成本对比：

| 存储类型 | 访问时间 | 成本（$/GB） |
|---------|---------|-------------|
| SRAM | 0.5ns – 2.5ns | $2000 – $5000 |
| DRAM | 50ns – 70ns | $20 – $75 |
| 磁盘 | 5ms – 20ms | $0.20 – $2 |
| **理想内存** | SRAM速度 | 磁盘成本 |

**现实**：物理上不可能同时获得SRAM的速度和磁盘的成本，因此需要存储层次结构。

### 核心概念

1. **SRAM**：静态随机存取存储器，无需刷新，但占用面积大、功耗高

2. **DRAM**：动态随机存取存储器，使用电容存储，需要周期性刷新

3. **存储层次结构的驱动力**：弥补各种存储技术之间的速度、容量、成本差距

## DRAM技术详解

![[第五章6.png]]

### 核心内容讲解

DRAM使用电容存储电荷来表示比特位：

**存储单元结构**：
- 每个存储单元由1个晶体管 + 1个电容组成
- 电容的电荷随时间泄漏，必须周期性刷新

**刷新机制**：
- 刷新按"行"进行
- 读出整行数据再写回
- 行缓冲区（Row Buffer）允许在同一行内连续读取多个字

### 核心概念

1. **DRAM行缓冲区**：允许并行读取和刷新多个字，提高突发访问效率

2. **Bank结构**：多个独立的banks可同时激活不同行，实现并行访问

3. **刷新开销**：DRAM需要周期性刷新，这段时间无法响应访问请求

---

## DRAM高级组织

![[第五章7.png]]

### 核心内容讲解

DRAM内部以矩形阵列组织数据位：

**DDR（双倍数据率）**：
- 在时钟的上升沿和下降沿都传输数据
- 相同频率下带宽翻倍

**QDR（四倍数据率）**：
- 分离输入输出通道
- 进一步提高带宽

**突发模式**：从一行连续读取多个字时减少延迟

### 核心概念

1. **DDR代数演进**：DDR → DDR2 → DDR3 → DDR4 → DDR5，持续提升传输速率和能效

2. **DRAM分银行（Banking）**：多个独立banks可同时激活不同行，提升整体带宽

3. **同步DRAM**：使用时钟同步，允许突发传输连续字而无需每次发送地址

---

## 存储带宽提升技术

![[第五章10.png]]

### 核心内容讲解

三种增加内存带宽的方法：

**a. 宽缓存组织（Wide cache）**：
- 数据通路加宽，一次传输多个字
- 缺点：增加连线和功耗

**b. 多银行缓存（Multi-banked cache）**：
- 将缓存分为多个独立银行
- 允许同时进行多个访问
- 4银行交叉存储可每个周期提供4字节

**带宽计算示例**：
```
4-word宽缓存：Miss penalty = 1 + 15 + 1 = 17周期，带宽 = 16B/17周期 = 0.94 B/周期
4-bank交叉缓存：Miss penalty = 1 + 15 + 4×1 = 20周期，带宽 = 16B/20周期 = 0.8 B/周期
```

### 核心概念

1. **带宽优化的核心思想**：通过并行化增加单位时间内的数据传输量

2. **组织方式的选择**：宽组织适合减少延迟，交叉组织适合增加带宽

## DRAM性能因素

![[第五章9.png]]

### 核心内容讲解

影响DRAM性能的三个关键因素：

1. **Row Buffer（行缓冲区）**：允许在同一行内连续读取多个字而无需重新激活行，提高突发模式效率

2. **Synchronous DRAM（同步DRAM）**：使用时钟同步，允许突发传输连续字而无需每次发送地址

3. **DRAM Banking（分银行）**：同时访问多个DRAM，提高带宽

### 核心概念

1. **这些优化都是为了弥补DRAM与SRAM之间约100倍的访问速度差距**

2. **DDR技术**：在时钟上升沿和下降沿都传输数据，相同频率下带宽翻倍

3. **现代DRAM**：通常包含多个banks，支持并行访问

## 闪存存储

![[第五章11.png]]

### 核心内容讲解

闪存是一种非易失性半导体存储，性能介于磁盘和DRAM之间：

**优点**：
- 比磁盘快100–1000倍
- 更小、更低功耗、更耐用（无机械部件）

**缺点**：
- 成本高于磁盘（介于磁盘和DRAM之间）

### 核心概念

1. **闪存bits会随访问次数增加而磨损（通常1000次后）**

2. **磨损均衡（Wear Leveling）**：将数据映射到不同物理块以延长寿命

3. **应用场景**：SSD、移动设备存储、嵌入式系统

---

## 闪存类型

![[第五章12.png]]

### 核心内容讲解

闪存分为两大类型：

**NOR闪存**：
- 存储单元类似NOR门
- 支持随机读写访问
- 用于嵌入式系统的指令存储

**NAND闪存**：
- 存储单元类似NAND门
- 密度更高（每单位面积更多位）
- 只能按块访问
- 用于USB驱动器、存储卡、SSD

### 核心概念

1. **NOR vs NAND**：NOR适合随机读取（代码执行），NAND适合大容量顺序访问（数据存储）

2. **闪存不能直接替代需要频繁写入的RAM或磁盘应用**

3. **磨损均衡**：操作系统或控制器负责将写操作分散到不同物理块

---

## 磁盘存储结构

![[第五章13.png]]

### 核心内容讲解

磁盘是非易失性、旋转式磁性存储：

**核心结构**：
- **Track（磁道）**：同心圆状的数据轨道
- **Sector（扇区）**：磁道上的数据单元（通常512字节或4KB）
- **Cylinder（柱面）**：多层盘片上相同位置的磁道集合

### 核心概念

1. **磁盘是存储层次中的最底层**：提供最大容量和最低成本，但访问时间最长（毫秒级）

2. **磁盘访问涉及机械运动**：寻道时间 + 旋转延迟 + 传输时间

3. **局部性原理在磁盘调度中的应用**：电梯算法、提前排队等优化

---

## 磁盘扇区与访问

![[第五章14.png]]

### 核心内容讲解

每个扇区记录的内容：
- **Sector ID**：扇区标识
- **Data（512字节）**：实际数据
- **ECC（纠错码）**：用于隐藏缺陷和记录错误
- **同步字段和间隙**：用于分隔和同步

**磁盘访问步骤**：
1. **队列延迟**：等待其他访问完成
2. **寻道时间**：移动磁头到正确磁道
3. **旋转延迟**：等待扇区旋转到磁头下方
4. **数据传输**：实际读写数据
5. **控制器开销**：控制器处理请求的时间

### 核心概念

1. **ECC（Error Correcting Code）**：检测和纠正数据传输或存储中的错误，提高数据可靠性

2. **扇区是磁盘I/O的基本单位**

3. **访问时间组成**：寻道时间 + 旋转延迟 + 传输时间 + 控制器开销

## 磁盘访问实例

![[第五章15]]

### 核心内容讲解

给定参数计算平均读取时间：
- 512B扇区，15000转/分
- 4ms平均寻道时间
- 100MB/s传输速率
- 0.2ms控制器开销

**计算过程**：
```
4ms（寻道）+ 2ms（半圈旋转延迟）+ 0.005ms（传输）+ 0.2ms（控制器）
= 6.2ms
```

如果实际平均寻道时间为1ms：
```
1ms + 2ms + 0.005ms + 0.2ms = 3.2ms
```

### 核心概念

1. **实际平均寻道时间通常小于标称值**：局部性和OS调度可以减少磁头移动

2. **智能控制器**：将逻辑相邻扇区放在物理相邻位置

3. **磁盘缓存**：预取扇区以避免寻道和旋转延迟

## 缓存基础概念

![[第五章17]]

### 核心内容讲解

缓存位于CPU和主存之间，利用局部性原理加速数据访问：

**核心问题**：
- **如何知道数据是否存在？** → 通过标签（Tag）判断
- **数据应该放在哪里？** → 由映射规则决定

缓存接收一系列地址访问X₁...Xₙ₋₁, Xₙ：
- 如果数据已在缓存中（命中），直接返回
- 如果数据不在缓存中（未命中），从主存取回并放入缓存

### 核心概念

1. **缓存是存储层次中第一个处理命中/未命中概念的层级**

2. **命中时间（Hit Time）**：从缓存中获取数据所需的时间

3. **未命中惩罚（Miss Penalty）**：缓存未命中时从下一层获取数据的时间

---

## 直接映射缓存

![[第五章18]]

### 核心内容讲解

直接映射缓存中，**每个内存块只能放在唯一的位置**：

**映射规则**：
```
缓存位置 = 块地址 mod 缓存块数
```

例如：缓存有8块，地址22 mod 8 = 6 → 地址22的块只能放在第6块。

**地址结构**：
- **Index（索引）**：选择缓存中的具体行
- **Tag（标签）**：用于区分映射到同一位置的不同块
- **Offset（偏移）**：块内选择具体字节

### 核心概念

1. **直接映射的优点**：硬件简单、查找速度快

2. **直接映射的缺点**：可能产生**冲突未命中**——两个频繁访问的块如果映射到同一位置，会相互替换

3. **地址划分示例**：32位地址，1KB缓存，16字节块 → Index占6位，Offset占4位

---

## 标签与有效位

![[第五章19]]

### 核心内容讲解

缓存需要额外信息来确定存储的数据是否有效：

**标签（Tag）**：
- 存储块地址的高位部分
- 用于在缓存查找时判断该行是否包含请求的地址

**有效位（Valid Bit）**：
- 1 = 数据有效
- 0 = 无有效数据（初始状态或已被清除）
- 系统启动时所有有效位为0

### 核心概念

1. **命中条件**：标签匹配 AND 有效位为1

2. **标签存储开销**：每行需要额外的Tag位

3. **有效位的作用**：区分"空"和"存有旧数据"的缓存行

## 缓存示例：初始状态

![[第五章20]]

### 核心内容讲解

缓存配置：8块、每块1字、直接映射

**初始状态表**：
| Index | V | Tag | Data |
|-------|---|-----|------|
| 000-111 | N | - | - |

所有缓存行都无效（V=N）。

### 核心概念

1. **初始状态是理解缓存行为的基础**——首次访问任何块都会产生未命中（冷启动未命中）

2. **有效位为N表示该行未加载任何有效数据**

3. **冷启动未命中（强制性未命中）**：无法避免，只能通过预取优化

## 缓存示例：访问序列

![[第五章21]]![[第五章22]]![[第五章23]]

### 核心内容讲解

**访问地址22（二进制10 110）**：
```
Index = 110 (6)
Tag = 10
块地址 = 22
```
结果：Miss（未命中），加载包含地址22的块到Index=6的行。

**访问地址26（二进制11 010）**：
```
Index = 010 (2)
Tag = 11
```
结果：Miss（未命中），加载块到Index=2的行。

**再次访问**：
| Word addr | Binary | Hit/Miss | Block |
|-----------|--------|----------|-------|
| 22 | 10 110 | **Hit** | 110 |
| 26 | 11 010 | **Hit** | 010 |

### 核心概念

1. **时间局部性体现**：刚访问过的数据再次访问会命中

2. **不同Index的访问互不影响**

3. **首次访问 → Miss，再次访问 → Hit**

## 缓存冲突示例

![[第五章24]]![[第五章25]]

### 核心内容讲解

继续访问序列：
| Word addr | Binary | Hit/Miss |
|-----------|--------|----------|
| 16 | 10 000 | Miss |
| 3 | 00 011 | Miss |
| 16 | 10 000 | **Hit** |

**冲突情况**：
- 地址18（10 010）映射到Index=2
- 但Index=2已被地址26占用（Tag=11）
- 访问18会**替换**Index=2的块
- 导致后续访问26时变为Miss

### 核心概念

1. **冲突未命中**：多个块竞争同一缓存位置

2. **直接映射的局限性**：两个经常访问的块如果映射到同一位置，会严重影响命中率

3. **解决方案**：使用组相联或全相联缓存

## 地址细分

![[第五章26]]

### 核心内容讲解

以64KB缓存、32位地址、64字节块为例：

```
| Tag (52位) | Index (10位) | Block Offset (6位) | Byte Offset (2位) |
```

**查找过程**：
1. 用Index从缓存中选择一行
2. 比较该行的Tag与地址的Tag
3. 检查Valid位
4. 如果都匹配，用Offset从数据块中选择字节

### 核心概念

1. **地址划分取决于缓存大小和块大小**

2. **块内偏移（Block Offset）**用于选择块内的具体字节或字

3. **Byte Offset**用于选择字节（对于4字节字，Byte Offset是2位）

## 块大小考虑

![[第五章28]]

### 核心内容讲解

**增加块大小的优点**：
- 更好利用空间局部性
- 减少强制性未命中（首次访问）

**增加块大小的缺点**：
- 固定容量下块数减少 → 更多竞争 → 增加未命中
- 增加未命中惩罚（传输更大块）
- 可能导致缓存污染

### 核心概念

1. **块大小的选择是权衡**：过大可能导致性能反而下降

2. **现代缓存典型块大小**：32-64字节

3. **污染效应**：大块可能将其他有用的块挤出缓存

---

## 缓存未命中处理

![[第五章29]]

### 核心内容讲解

**命中时**：CPU正常执行

**未命中时**：
1. 停顿CPU流水线（Stall）
2. 从下一层取数据块
3. 存入缓存
4. 恢复CPU执行

**指令缓存未命中**：重启取指阶段

**数据缓存未命中**：依赖该数据的指令停顿，其他指令可能继续（乱序执行）

### 核心概念

1. **未命中惩罚（Miss Penalty）**是评估缓存设计的重要指标

2. **非阻塞缓存**：允许CPU在未命中期间继续执行其他指令

3. **关键路径影响**：如果未命中发生在指令依赖链上，整体性能影响更大

---

## 写直达策略

![[第五章30]]

### 核心内容讲解

**写直达（Write-Through）**：
- 写命中时，同时更新缓存和主存
- 主存始终与缓存一致

**问题**：每次写都需要访问主存，性能开销大

**解决方案：写缓冲（Write Buffer）**：
- 数据先写入缓存和写缓冲
- CPU可立即继续执行
- 后台将数据写入主存

**性能计算**：
```
CPI = 1 + 0.1 × 100 = 11（10%存储指令，每次写需100周期）
```

### 核心概念

1. **写缓冲缓解了写直达的性能问题**

2. **写缓冲满时的停顿**：需要等待写缓冲清空

3. **一致性优势**：缓存和内存始终一致，便于多处理器共享

---

## 写回策略

![[第五章31]]

### 核心内容讲解

**写回（Write-Back）**：
- 写命中时，只更新缓存
- 主存在块被替换时才更新
- 需要**脏位（Dirty Bit）**跟踪修改状态

**替换脏块时**：
- 先将脏块写回主存
- 再加载新块
- 可用写缓冲优化

### 核心概念

1. **写回策略显著减少对主存的写操作**

2. **脏位**：标记缓存行是否被修改过

3. **适合写密集型工作负载**：减少总线带宽占用

---

## 写分配策略

![[第五章32]]

### 核心内容讲解

**写分配（Write Allocate）**：
- 写未命中时，先将块从主存加载到缓存
- 再执行写操作
- 后续对同块其他数据的访问会命中

**非写分配（No-Write Allocate / Write Around）**：
- 写未命中时，直接写主存
- 不加载块到缓存

**组合策略**：
- 写回缓存 + 写分配（常见）
- 写直达缓存 + 非写分配（常见）

### 核心概念

1. **写分配与写回配合效果最佳**：一旦块在缓存中，后续写操作都在缓存内完成

2. **非写分配适合只写一次的大块数据**

3. **组合选择影响整体性能**：需要根据工作负载特性选择

---

## 平均内存访问时间

![[第五章38]]

### 核心内容讲解

**AMAT = Hit Time + Miss Rate × Miss Penalty**

**命中时间（Hit Time）**：命中时的访问时间

**未命中率（Miss Rate）**：未命中次数/总访问次数

**未命中惩罚（Miss Penalty）**：未命中时额外花费的时间

**示例**：
```
CPU: 1ns时钟周期
命中时间 = 1周期
未命中惩罚 = 20周期
未命中率 = 5%
AMAT = 1 + 0.05 × 20 = 2ns
```

### 核心概念

1. **命中时间同样重要**：即使牺牲少量命中率换取更快的命中时间，也可能改善整体性能

2. **AMAT是评估存储层次性能的统一指标**

3. **优化方向**：降低命中率、降低未命中惩罚、降低命中时间

---

## 缓存性能示例

![[第五章37]]

### 核心内容讲解

**给定参数**：
- I-cache未命中率 = 2%
- D-cache未命中率 = 4%
- 未命中惩罚 = 100周期
- 基础CPI = 2
- 加载/存储占指令比例 = 36%

**计算**：
```
每指令I-cache未命中周期 = 0.02 × 100 = 2
每指令D-cache未命中周期 = 0.36 × 0.04 × 100 = 1.44
实际CPI = 2 + 2 + 1.44 = 5.44
```

**性能差距**：理想CPU比实际快 5.44/2 = 2.72倍

### 核心概念

1. **缓存未命中显著影响CPI**：特别是在高时钟频率处理器中

2. **数据缓存的影响**：由于数据访问比例高，D-cache未命中影响可能更大

3. **性能分析的重要性**：理解瓶颈才能有效优化

### 核心内容讲解

FastMATH是一款嵌入式MIPS处理器：

**缓存配置**：
- 分离I-cache和D-cache（Split Cache）
- 每缓存16KB = 256块 × 16字/块

**未命中率（SPEC2000）**：

- 加权平均: 3.2%

### 核心概念

1. **指令访问局部性通常优于数据访问**：程序在循环中反复执行指令

2. **分离缓存优势**：I-cache和D-cache可同时访问

3. **SPEC基准测试**：用于评估处理器的标准工作负载

## 相联缓存概念

![[第五章40]]

### 核心内容讲解

相联度决定块在缓存中的可选位置数量：

**直接映射（Direct Mapped）**：
- 每块只有1个可选位置
- 硬件简单，速度快
- 可能产生冲突未命中

**n路组相联（n-way Set Associative）**：
- 每块有n个可选位置（在同一组内）
- 组号 = 块地址 mod 组数

**全相联（Fully Associative）**：
- 块可放在任意位置
- 冲突未命中最少
- 需要搜索所有条目，只适合小缓存（如TLB）

### 核心概念

1. **更高的相联度减少冲突未命中**

2. **但增加复杂度和潜在延迟**

3. **2路和4路组相联是常见的平衡选择**

---

## 相联缓存示例

![[第五章41]]

### 核心内容讲解

对比4块缓存的三种组织方式：

**直接映射**：
- 只能放在唯一位置
- 搜索1个标签

**2路组相联**：
- 可放在组内2个位置之一
- 搜索2个标签

**全相联**：
- 可放在任意位置
- 搜索所有4个标签

### 核心概念

1. **相联度越高，需要的比较器越多**

2. **但能减少冲突未命中**

3. **全相联缓存适合做TLB**：容量小但需要快速定位

---

## 相联度权衡

![[第五章45]]

### 核心内容讲解

**增加相联度的效果**：
- 减少冲突未命中
- 但可能增加访问时间

**实验数据（64KB D-cache，16-word blocks，SPEC2000）**：
| 相联度 | 未命中率 |
|--------|----------|
| 1-way | 10.3% |
| 2-way | 8.6% |
| 4-way | 8.3% |
| 8-way | 8.1% |

### 核心概念

1. **增加相联度收益递减**：8-way相比4-way改善很小

2. **成本增加**：需要更多比较器，增加延迟和功耗

3. **L1缓存常用4-8路**：延迟敏感，需要平衡

## 替换策略

![[第五章47]]

### 核心内容讲解

当缓存组已满时，必须选择一个块替换：

**最近最少使用（LRU）**：
- 替换最长时间未访问的块
- 性能最好，但高相联度下实现复杂

**随机（Random）**：
- 随机选择替换目标
- 实现简单，硬件开销低
- 实践中性能与LRU接近

**LRU近似**：
- 用简化硬件实现接近LRU的效果
- 现代处理器常用

### 核心概念

1. **替换策略影响缓存命中率**

2. **选择取决于硬件复杂度和性能需求的权衡**

3. **伪LRU**：在大相联度缓存中常用的简化实现

---

## 多级缓存

![[第五章48]]

### 核心内容讲解

现代处理器采用多级缓存：

**单级L1缓存问题**：
```
基础CPI = 1
未命中率 = 2%
未命中惩罚 = 100ns / 0.25ns = 400周期
有效CPI = 1 + 0.02 × 400 = 9
```

**多级缓存设计**：
- **L1缓存**：小而快（32-64KB，1-2周期）
- **L2缓存**：大而慢（256KB-数MB，数十周期）
- **L3缓存**：可选，更大更慢

### 核心概念

1. **多级缓存通过在速度和容量间取得更好平衡**

2. **L1命中率高但容量小，L2容量大但命中率较低**

3. **全局未命中率**：通过L1后仍miss的比例

---

## 多级缓存示例

![[第五章49]]![[第五章50]]

### 核心内容讲解

**添加L2缓存后**：
- L2访问时间 = 5ns = 20周期
- L2全局未命中率 = 0.5%
- 内存惩罚 = 500周期

**计算**：
```
CPI = 1 + 0.02×20 + 0.005×500 = 3.4
性能提升 = 9 / 3.4 = 2.6倍
```

### 核心概念

1. **多级缓存显著改善整体性能**

2. **每级的设计目标不同**：
   - L1：最小化命中时间
   - L2/L3：最小化未命中率

3. **包含策略（Inclusive vs Exclusive）**：L2是否包含L1的内容

---

## 多级缓存考虑

![[第五章51]]

### 核心内容讲解

**L1缓存设计重点**：
- 最小化命中时间
- 块大小相对较小

**L2/L3缓存设计重点**：
- 低未命中率
- 命中时间影响较小

**典型配置**：
- L1块大小 < L2块大小
- L1容量 < L2容量
- L1相联度可能更高

### 核心概念

1. **分离vs统一L2**：统一缓存灵活但可能成为瓶颈

2. **片上vs片外L2**：片上更快但占用芯片面积

3. **性能优化需要全局考虑**：不能只优化某一层

## 虚拟内存概念

![[第五章68]]

### 核心内容讲解

**虚拟内存**：将主存作为磁盘的缓存

**核心特性**：
- 由CPU硬件和操作系统共同管理
- 程序使用独立的虚拟地址空间
- 私有代码和数据相互保护
- CPU和OS将虚拟地址转换为物理地址

### 核心概念

1. **页（Page）**：虚拟内存的块单位

2. **页错误（Page Fault）**：虚拟内存的未命中

3. **虚拟内存提供内存保护和地址空间扩展**

---

## 地址转换

![[第五章69]]

### 核心内容讲解

**虚拟地址结构**：
```
| 虚拟页号 (VPN) | 页内偏移 |
```

**物理地址结构**：
```
| 物理页号 (PPN) | 页内偏移 |
```

**转换过程**：
- 用VPN索引页表
- 获取物理页号
- 与页内偏移组合形成物理地址

### 核心概念

1. **页内偏移直接保留**：只有页号需要转换

2. **页表项（PTE）**：存储虚拟页到物理页的映射

3. **页大小**：通常4KB，也可支持大页（Huge Pages）

---

## 页表结构

![[第五章71]]![[第五章72]]

### 核心内容讲解

**页表**：存储虚拟页到物理页的映射

**页表项（PTE）内容**：
- **有效位**：页是否在内存中
- **物理页号**：如果页在内存中
- **其他状态位**：引用位、脏位等

**地址转换流程**：
1. 从页表基址寄存器获取页表位置
2. 用VPN索引页表
3. 检查有效位
4. 获取物理页号
5. 与页内偏移组合

### 核心概念

1. **每个进程有自己的页表**：由页表基址寄存器指向

2. **页表是虚拟内存系统的核心数据结构**

3. **大地址空间需要多级页表**：避免页表占用过多内存

## 页替换与写操作

![[第五章74]]

### 核心内容讲解

**LRU替换**：
- 使用引用位（Reference Bit）跟踪页访问
- 引用位周期性清零
- 引用位=0的页最近未被使用

**写操作**：
- 磁盘写入需要数百万周期
- 写直达不实际
- 使用**写回策略**
- PTE中的**脏位（Dirty Bit）**标记已修改的页

### 核心概念

1. **虚拟内存必须使用写回策略**

2. **脏位开销**：写入时需要设置，增加延迟

3. **写时复制（COW）**：延迟物理复制直到真正写入

## 页错误处理

![[第五章79]]

### 核心内容讲解

**页错误处理步骤**：
1. 用产生错误的虚拟地址找到PTE
2. 在磁盘上定位页
3. 选择要替换的页（如果是脏页，先写回磁盘）
4. 将页读入内存并更新页表
5. 使进程再次可运行
6. 重启产生错误的指令

### 核心概念

1. **页错误处理由OS执行**：需要数百万周期

2. **页错误是虚拟内存的主要开销**：需要尽量减少

3. **透明页错误**：操作系统可自动处理，不影响程序

## TLB概述

![[第五章75]]

### 核心内容讲解

**问题**：地址转换需要额外的内存访问

**解决方案**：TLB（Translation Lookaside Buffer）
- 页表的硬件缓存
- 典型参数：
  - 16-512个条目
  - 命中时间：0.5-1周期
  - 未命中惩罚：10-100周期
  - 未命中率：0.01%-1%

### 核心概念

1. **TLB缓存最近使用的虚拟地址到物理地址的转换结果**

2. **TLB是相联缓存**：需要搜索所有条目

3. **软件vs硬件处理TLB未命中**：RISC-V采用软件处理

## TLB与缓存交互

![[第五章80]]

### 核心内容讲解

**虚拟索引缓存**：
- 缓存使用虚拟地址的索引部分
- 与TLB查找并行进行
- 减少延迟

**问题：别名（Aliasing）**：
- 不同虚拟地址映射到同一物理地址
- 但可能在缓存不同位置

**解决方案**：
- 操作系统确保别名虚拟地址映射到相同缓存位置
- 或使用物理地址作为缓存标签

### 核心概念

1. **TLB和缓存的交互设计影响处理器性能**

2. **虚拟索引物理标签（VIPT）**：常见的折中方案

3. **别名问题**：共享内存时需要特别处理

### 核心内容讲解

**TLB未命中情况**：

**页在内存中**：

**页不在内存中（页错误）**：

## 存储层次统一框架

存储层次各层级遵循相同原则：

## 块放置策略

![[第五章83]]

### 核心内容讲解

**放置选项**：
- **直接映射**：每个块只有1个位置选择
- **n路组相联**：每个块有n个位置选择（在同一组内）
- **全相联**：每个块可放在任意位置

**权衡**：
- 更高相联度 → 更低未命中率
- 更高相联度 → 增加复杂度和成本

### 核心概念

1. **选择放置策略是复杂度和性能之间的权衡**

2. **缓存层级越高**：越倾向于使用较低相联度

3. **虚拟内存**：天然使用全相联（页可放在任何帧）

## 块查找策略

![[第五章84]]

### 核心内容讲解

**查找方法对比**：

| 相联度 | 定位方法 | 标签比较次数 |
|--------|---------|-------------|
| 直接映射 | 索引到1个位置 | 1 |
| n路组相联 | 索引到组 + 搜索组内n个标签 | n |
| 全相联 | 搜索所有标签 | 所有条目 |

### 核心概念

1. **查找策略影响硬件复杂度和访问延迟**

2. **硬件缓存**：减少比较次数以降低成本

3. **虚拟内存**：全表查找使全相联可行

## 未命中类型

![[第五章87]]

### 核心内容讲解

**三类未命中**：

**强制性未命中（Compulsory/Cold Start）**：
- 首次访问某个块
- 无法避免

**容量未命中（Capacity）**：
- 缓存容量有限
- 被替换的块稍后又被访问

**冲突未命中（Conflict）**：
- 只存在于非全相联缓存
- 多个块竞争同一组的位置

### 核心概念

1. **区分未命中类型有助于针对性优化**

2. **冷启动未命中**：可通过预取减少
3. **容量未命中**：可通过增加缓存大小解决
4. **冲突未命中**：可通过增加相联度解决

## 缓存设计权衡

![[第五章88]]

### 核心内容讲解

| 设计改变 | 对未命中率的影响 | 负面性能影响 |
|----------|------------------|--------------|
| 增加缓存大小 | 减少容量未命中 | 可能增加访问时间 |
| 增加相联度 | 减少冲突未命中 | 可能增加访问时间 |
| 增加块大小 | 减少强制性未命中 | 增加未命中惩罚；过大可能增加污染 |

### 核心概念

1. **缓存设计是权衡的艺术**：没有最优解，只有适合特定场景的方案

2. **所有优化都有代价**：需要根据工作负载特性选择

3. **仿真和测量是关键**：理论分析需要实验验证

## 可靠性度量

![[第五章59]]

### 核心内容讲解

**可靠性指标**：
- **MTTF（Mean Time To Failure）**：平均故障前时间
- **MTTR（Mean Time To Repair）**：平均修复时间
- **MTBF = MTTF + MTTR**：平均故障间隔时间
- **可用性 = MTTF / (MTTF + MTTR)**

**提升可用性**：
- 增加MTTF：故障避免、容错、预测
- 减少MTTR：改进诊断和修复工具

### 核心概念

1. **可靠性设计是多层次的**：ECC纠错、冗余备份等

2. **MTTF是衡量组件可靠性的关键指标**

3. **高可用系统需要同时优化MTTF和MTTR**

## 汉明纠错码

![[第五章60]]

### 核心内容讲解

**汉明距离**：两个比特模式之间不同位的数量

**纠错能力**：
- 最小距离 = 2：检测单比特错误
- 最小距离 = 3：纠正单比特错误，检测双比特错误

### 核心概念

1. **汉明码通过在数据位之间插入校验位**：实现错误检测和纠正

2. **SEC（Single Error Correction）**：纠正单比特错误

3. **校验位数计算**：满足 2^p ≥ d + p + 1

## 汉明码编码

![[第五章61]]

### 核心内容讲解

**SEC（Single Error Correction）编码规则**：
1. 从左到右给所有位编号
2. 所有2的幂次位置是校验位
3. 每个校验位覆盖一组数据位

**校验位覆盖**：
- p1: 覆盖位1,3,5,7,9,11...
- p2: 覆盖位2,3,6,7,10,11...
- p4: 覆盖位4,5,6,7,12,13...

### 核心概念

1. **通过校验位覆盖关系的异或运算确定错误位置**

2. **校验位位置**：2的幂次方（1, 2, 4, 8, 16...）

3. **覆盖规则**：位号等于校验位号之和

## 汉明码解码

![[第五章62]]

### 核心内容讲解

**解码过程**：
- 计算校验位的值
- 校验位值表示错误位的位置
- 校验位=0000：无错误
- 校验位=1010：第10位翻转

**示例**：
- 校验位=0000 → 无错误
- 校验位=1010 → 第10位需要翻转纠正

### 核心概念

1. **纠错码在存储系统中保护数据完整性**

2. **高可靠性应用**：如服务器、存储系统

3. **延迟和面积开销**：需要权衡

## SEC/DED码

![[第五章63]]

### 核心内容讲解

**SEC/DED（Single Error Correction, Double Error Detection）**：
- 添加整个字的奇偶校验位pn
- 使汉明距离 = 4

**解码规则**：
| H（校验位） | pn | 结果 |
|------------|-----|------|
| 偶数 | 偶数 | 无错误 |
| 奇数 | 奇数 | 可纠正的单比特错误 |
| 偶数 | 奇数 | pn位错误 |
| 奇数 | 偶数 | 发生了双比特错误 |

### 核心概念

1. **ECC DRAM使用SEC/DED**：用8位保护每64位数据

2. **能检测双比特错误**：提高可靠性

3. **成本增加**：需要额外的校验位

## 缓存一致性定义

![[第五章94]]

### 核心内容讲解

**形式化一致性定义**：
1. **P写X，P读X（无其他写干预）** → 读返回写的值
2. **P₁写X，P₂读X（足够晚）** → P₂读返回P₁写的值
3. **P₁写X，P₂写X** → 所有处理器以相同顺序看到写操作

### 核心概念

1. **一致性确保多处理器系统中的内存操作行为可预测**

2. **程序员的可见顺序**：可能与执行顺序不同

3. **同步原语**：确保正确的内存顺序

## 缓存一致性协议

![[第五章95]]

### 核心内容讲解

**缓存一致性协议类型**：

**监听协议（Snooping）**：
- 每个缓存监视总线事务
- 缓存和内存记录块的共享状态

**目录协议（Directory-based）**：
- 使用目录记录每个内存块的共享状态
- 减少共享内存的带宽需求

### 核心概念

1. **协议选择影响系统复杂度和可扩展性**

2. **监听协议**：适合小规模系统，开销与核心数成正比

3. **目录协议**：适合大规模系统，需要额外存储

## 失效协议

![[第五章96]]

### 核心内容讲解

**写失效协议**：
1. 缓存获得块的独占访问权（要写入时）
2. 在总线上广播失效消息
3. 其他缓存中的副本被标记为无效
4. 后续读取在其他缓存中未命中
5. 拥有缓存提供更新后的值

### 核心概念

1. **失效协议是维护多处理器缓存一致性的常用方法**

2. **写直达 vs 写回缓存**：一致性实现不同

3. **总线流量**：大量无效化消息可能成为瓶颈

## 内存一致性

![[第五章97]]

### 核心内容讲解

**内存一致性**：定义多处理器系统中内存操作的可见顺序

**假设**：
- 写操作完成后，只有所有处理器都看到时才完成
- 处理器不重排写操作与其他访问

**结论**：
- P写X然后写Y → 所有看到新Y的处理器也看到新X
- 处理器可以重排读操作，但不能重排写操作

### 核心概念

1. **程序员应使用同步原语（锁、屏障）确保正确的内存顺序**

2. **宽松一致性模型**：允许更多优化，但增加编程复杂度

3. **内存屏障指令**：显式控制内存顺序

## TLB组织对比

![[第五章99]]

### 核心内容讲解

**ARM Cortex-A53 TLB**：
- 虚拟地址：48位
- 物理地址：40位
- 微TLB：全相联，10条目，循环替换
- 主TLB：64条目，4路组相联
- 页面大小：可变（4KB-1GB）

**Intel Core i7 TLB**：
- 虚拟地址：48位
- 物理地址：44位
- L1 I-TLB：128条目（小页），7条目/线程（大页）
- L1 D-TLB：64条目（小页），32条目（大页）
- L2 TLB：512条目，4路组相联

### 核心概念

1. **两者都采用两级TLB结构**：微TLB小而快，主TLB更大

2. **大页支持**：减少TLBmiss率，但增加内存开销

3. **TLBmiss处理**：现代处理器通常由硬件处理

## 常见陷阱

![[第五章103]]![[第五章104]]

### 核心内容讲解

**陷阱1：字节寻址vs字寻址**
- 示例：32字节块、4字节块、字节36映射到块1
- 字36映射到块4（如果一个字=4字节）

**陷阱2：忽略存储系统效果**
- 示例：按行vs按列遍历数组
- 大步长导致空间局部性差

**陷阱3：共享L2/L3缓存的多处理器**
- 更少相联度 → 更多冲突未命中
- 更多核心 → 需要增加相联度

**陷阱4：用AMAT评估乱序处理器性能**
- 忽略非阻塞访问的影响
- 应通过仿真评估性能

### 核心概念

1. **理解存储层次对写出高效代码至关重要**

2. **性能评估需要考虑具体处理器架构**

3. **测试很重要**：不能仅靠理论分析

## 总结

![[第五章106]]

### 核心要点

1. **局部性原理**是存储层次结构的基础

2. **缓存**通过保持最近使用的数据来减少访问延迟

3. **直接映射、组相联、全相联**是三种基本的放置策略

4. **写直达和写回**是两种主要的写策略

5. **多级缓存**在容量和速度间取得更好平衡

6. **虚拟内存**使用磁盘作为主存的扩展

7. **TLB**加速虚拟地址到物理地址的转换

8. **缓存一致性**是多处理器系统的关键问题

### 设计权衡

- 更高的相联度 → 更低未命中率，但更复杂
- 更大的缓存 → 更低容量未命中，但可能更慢
- 更大的块 → 更好空间局部性，但增加未命中惩罚

理解这些权衡对于设计高效的计算机系统至关重要。
