## 存储层次结构的基本概念

### 核心内容讲解

计算机存储层次结构从快速小容量到慢速大容量的金字塔式层级：

- **Processor（处理器）**：位于顶端，是数据的最终使用者
- **SRAM Memory**：对应L1/L2缓存，容量小（KB级别）但速度极快（0.5-2.5ns）
- **DRAM Memory**：对应主内存，容量中等（GB级别），速度适中（50-70ns）
- **Disk（磁盘）**：位于底端，容量最大（TB级别）但速度最慢（毫秒级）

箭头表示数据在各层之间的传递方向。

### 核心概念

1. **时间局部性（Temporal Locality）**：最近访问的项很可能再次被访问。例如循环中的指令会被反复执行。

2. **空间局部性（Spatial Locality）**：相邻地址的项很可能被一起访问。例如顺序执行的指令流、数组数据的顺序访问。

3. **存储层次结构的本质**：在速度、容量、成本三者之间做权衡，利用局部性原理让"大容量"和"高速"看似同时实现。

---

### 常见问题解答

**Q：存储层次结构层级图解中各层是什么意思？**

```
CPU（处理器）
    ↓
Cache（缓存）—— SRAM，最快最小最贵
    ↓
Memory（主存/内存）—— DRAM，中等速度和容量
    ↓
Disk（磁盘/硬盘）—— 最慢最大最便宜
```

**Memory（主存）是什么？**
- 就是平时说的"内存条"，通常几GB到几十GB
- 技术上使用DRAM（动态RAM），断电数据丢失
- 作用：给CPU一个"足够快"的数据源，避免CPU直接访问慢速磁盘

**各层关系类比**：
- Memory = 图书馆（大，但找书要时间）
- Cache = 书桌（小，但伸手就能拿到）
- CPU想要的书，优先从书桌拿，没有再去图书馆

## 利用局部性原理

### 核心内容讲解
![[Pasted image 20260509162020.png|208]]
存储层次结构通过数据复制实现"缓存"的假象：

**命中流程**：
- 数据请求首先到达最上层
- 如果数据存在（Hit命中），直接从上层返回
- 命中率高意味着大部分访问由上层服务，性能好

**未命中流程**：
- 如果数据不在上层（Miss未命中），从下一层获取整个数据块
- 获取的数据块被复制到上层供后续访问使用
- 未命中惩罚（Miss Penalty）= 从下层获取数据的时间

**块（Block/Line）** 是数据复制的基本单位，大小影响性能。

### 核心概念

1. **块（Block）**：存储层次中数据复制的基本单位，大小是关键设计参数

2. **命中率的计算**：Hit ratio = hits / accesses = 1 - Miss ratio

3. **块大小的权衡**：
   - 大块利用空间局部性好
   - 但增加未命中惩罚和可能的缓存污染

## 存储技术对比

### 核心内容讲解

四种存储技术的性能与成本对比：

| 存储类型 | 访问时间 | 成本（$/GB） |
|---------|---------|-------------|
| SRAM | 0.5ns – 2.5ns | $2000 – $5000 |
| DRAM | 50ns – 70ns | $20 – $75 |
| 磁盘 | 5ms – 20ms | $0.20 – $2 |
| **理想内存** | SRAM速度 | 磁盘成本 |

**现实**：物理上不可能同时获得SRAM的速度和磁盘的成本，因此需要存储层次结构。

### 核心概念

1. **SRAM**：静态随机存取存储器，无需刷新，但占用面积大、功耗高
   - 存储1位需要6个晶体管（6T结构）
   - 面积约为DRAM的4-6倍
   - 用触发器结构稳定存储，速度快但成本高

2. **DRAM**：动态随机存取存储器，使用电容存储，需要周期性刷新
   - 存储1位只需1个晶体管+1个电容（1T1C结构）
   - 面积小、成本低，但需要刷新

3. **存储层次结构的驱动力**：弥补各种存储技术之间的速度、容量、成本差距

## DRAM技术详解

![[第五章6.png]]

### 核心内容讲解

DRAM使用电容存储电荷来表示比特位：

**存储单元结构**：
- 每个存储单元由1个晶体管 + 1个电容组成
- 电容的电荷随时间泄漏，必须周期性刷新

**刷新机制**：
- 刷新按"行"进行
- 读出整行数据再写回
- 行缓冲区（Row Buffer）允许在同一行内连续读取多个字

**行缓冲区位置**：
- 行缓冲区位于DRAM芯片内部，是每个Bank独立的临时存储区域
- 访问流程：选中某行 → 整行数据进入行缓冲区 → 根据列地址取具体数据
- 作用：访问同一行的多个数据时无需反复激活行，提高速度

### 核心概念

1. **DRAM行缓冲区**：允许并行读取和刷新多个字，提高突发访问效率

2. **Bank结构**：多个独立的banks可同时激活不同行，实现并行访问
   - 单Bank限制：同一时间只能有一行处于激活状态
   - 多Bank优势：每个Bank有独立的行缓冲区，可同时激活不同行
   - 地址中的BankID字段用于选择访问哪个Bank

3. **刷新开销**：DRAM需要周期性刷新，这段时间无法响应访问请求
   - 刷新=充电：读出整行 → 放大信号 → 写回（重新充电）
   - 刷新期间存储阵列被占用，CPU必须等待
   - 典型DRAM每64ms刷新所有行，约5-10%时间无法访问

## DRAM高级组织

### 核心内容讲解

DRAM内部以矩形阵列组织数据位：

**DDR（双倍数据率）**：
- 在时钟的上升沿和下降沿都传输数据
- 相同频率下带宽翻倍

**QDR（四倍数据率）**：
- 分离输入输出通道
- 进一步提高带宽

**突发模式**：从一行连续读取多个字时减少延迟

### 核心概念

1. **DDR代数演进**：DDR → DDR2 → DDR3 → DDR4 → DDR5，持续提升传输速率和能效

2. **DRAM分银行（Banking）**：多个独立banks可同时激活不同行，提升整体带宽

3. **同步DRAM**：使用时钟同步，允许突发传输连续字而无需每次发送地址

## 存储带宽提升技术

![[第五章10.png]]

### 内存带宽优化：三种核心方法

在 CPU 访存过程中，缓存缺失时需要从主存取数据。为了加快这个过程，通常有三种优化策略：

---
#### a. 宽缓存组织（Wide Cache）

**思路**：把数据通路加宽，一次从主存取回多个字。

```
原本：一次拿 1 个字（4B）
宽缓存：一次拿 4 个字（16B）
```

- **优点**：减少访问次数，单次缺失处理更快
- **缺点**：连线增多，功耗增大
---
#### b. 多银行缓存（Multi-banked Cache）

**思路**：把缓存分成多个独立的"银行"，每个银行可以同时工作。

```
银行1 | 银行2 | 银行3 | 银行4
  A1  |  A2  |  A3  |  A4    ← 交叉存放
```

- **优点**：支持多个访问并行进行
- **缺点**：单次传输反而更慢（要分多次交叉传输）
---
#### 带宽对比计算

| 方案 | 缺失惩罚 | 带宽 |
|------|----------|------|
| 4-word 宽缓存 | 1 + 15 + 1 = 17 周期 | 16B ÷ 17 = **0.94 B/周期** |
| 4-bank 交叉缓存 | 1 + 15 + 4×1 = 20 周期 | 16B ÷ 20 = **0.80 B/周期** |

> **结论**：单次大块数据传输选宽缓存；多个并发访问选多银行。
---
### 一句话总结

- **宽组织** → 减少单次延迟
- **交叉组织** → 增加并发带宽

影响DRAM性能的三个关键因素：

1. **Row Buffer（行缓冲区）**：允许在同一行内连续读取多个字而无需重新激活行，提高突发模式效率

## 闪存存储

![[第五章11.png]]

### 核心内容讲解

闪存是一种非易失性半导体存储，性能介于磁盘和DRAM之间：

**优点**：
- 比磁盘快100–1000倍
- 更小、更低功耗、更耐用（无机械部件）

**缺点**：
- 成本高于磁盘（介于磁盘和DRAM之间）

### 核心概念

1. **闪存bits会随访问次数增加而磨损（通常1000次后）**

2. **磨损均衡（Wear Leveling）**：将数据映射到不同物理块以延长寿命

3. **应用场景**：SSD、移动设备存储、嵌入式系统

---

## 闪存类型

### 核心内容讲解

闪存分为两大类型：

**NOR闪存**：
- 存储单元类似NOR门
- 支持随机读写访问
- 用于嵌入式系统的指令存储

**NAND闪存**：
- 存储单元类似NAND门
- 密度更高（每单位面积更多位）
- 只能按块访问
- 用于USB驱动器、存储卡、SSD

**NOR门与NAND门**：
- NOR门 = OR门 + NOT门（或非门）：两个输入都为0时输出1
- NAND门 = AND门 + NOT门（与非门）：至少一个输入为0时输出1
- NOR闪存单元并联连接 → 支持独立随机访问
- NAND闪存单元串联连接 → 只能按块访问，但密度更高

### 核心概念

1. **NOR vs NAND**：NOR适合随机读取（代码执行），NAND适合大容量顺序访问（数据存储）

2. **闪存不能直接替代需要频繁写入的RAM或磁盘应用**

3. **磨损均衡**：操作系统或控制器负责将写操作分散到不同物理块

## 磁盘存储结构

![[第五章13.png]]

### 核心内容讲解

磁盘是非易失性、旋转式磁性存储：

**核心结构**：
- **Track（磁道）**：同心圆状的数据轨道
- **Sector（扇区）**：磁道上的数据单元（通常512字节或4KB）
- **Cylinder（柱面）**：多层盘片上相同位置的磁道集合

### 核心概念

1. **磁盘是存储层次中的最底层**：提供最大容量和最低成本，但访问时间最长（毫秒级）

2. **磁盘访问涉及机械运动**：寻道时间 + 旋转延迟 + 传输时间

3. **局部性原理在磁盘调度中的应用**：电梯算法、提前排队等优化

## 磁盘扇区与访问

![[第五章14.png]]

### 核心内容讲解

每个扇区记录的内容：
- **Sector ID**：扇区标识
- **Data（512字节）**：实际数据
- **ECC（纠错码）**：用于隐藏缺陷和记录错误
- **同步字段和间隙**：用于分隔和同步

**磁盘访问步骤**：
1. **队列延迟**：等待其他访问完成
2. **寻道时间**：移动磁头到正确磁道
3. **旋转延迟**：等待扇区旋转到磁头下方
4. **数据传输**：实际读写数据
5. **控制器开销**：控制器处理请求的时间

**寻道时间详解**：
- 磁头装在移动臂上，需要机械运动才能到达目标磁道
- 过程：启动加速 → 移动 → 减速 → 精确定位
- 磁道宽度极窄（约0.1微米），需要精准定位
- 平均寻道时间4-10ms，是磁盘访问的主要延迟来源之一

**旋转延迟详解**：
- 磁头到位后，需要等待目标扇区旋转过来
- 目标扇区位置随机，平均需等待半圈
- 计算：转一圈时间 ÷ 2（如15000转/分 → 半圈2ms）

### 核心概念

1. **ECC（Error Correcting Code）**：检测和纠正数据传输或存储中的错误，提高数据可靠性

2. **扇区是磁盘I/O的基本单位**

3. **访问时间组成**：寻道时间 + 旋转延迟 + 传输时间 + 控制器开销‼️‼️‼️

3. **磁盘缓存**：预取扇区以避免寻道和旋转延迟

## 缓存基础概念

![[第五章17.png]]

### 核心内容讲解

缓存位于CPU和主存之间，利用局部性原理加速数据访问：

**核心问题**：
- **如何知道数据是否存在？** → 通过标签（Tag）判断
- **数据应该放在哪里？** → 由映射规则决定

缓存接收一系列地址访问X₁...Xₙ₋₁, Xₙ：
- 如果数据已在缓存中（命中），直接返回
- 如果数据不在缓存中（未命中），从主存取回并放入缓存

### 核心概念

1. **缓存是存储层次中第一个处理命中/未命中概念的层级**

2. **命中时间（Hit Time）**：从缓存中获取数据所需的时间

3. **未命中惩罚（Miss Penalty）**：缓存未命中时从下一层获取数据的时间

## 直接映射缓存

![[第五章18.png]]

### 核心内容讲解

直接映射缓存中，**每个内存块只能放在唯一的位置**：

**映射规则**：
```
缓存位置 = 块地址 mod 缓存块数
```

例如：缓存有8块，地址22 mod 8 = 6 → 地址22的块只能放在第6块。

**地址结构**：
- **Index（索引）**：选择缓存中的具体行
- **Tag（标签）**：用于区分映射到同一位置的不同块
- **Offset（偏移）**：块内选择具体字节

### 核心概念

1. **直接映射的优点**：硬件简单、查找速度快

2. **直接映射的缺点**：可能产生**冲突未命中**——两个频繁访问的块如果映射到同一位置，会相互替换

3. **地址划分示例**：32位地址，1KB缓存，16字节块 → Index占6位，Offset占4位

---

### 常见问题解答

**Q：为什么需要映射？**

核心问题：Cache太小，内存太大
- 内存容量远大于Cache，不能把所有数据都放进Cache
- 如果不映射（全相联），每次要搜索所有Cache位置，硬件复杂、成本高
- **映射的作用：缩小搜索范围**——用地址计算出"数据应该在哪个Cache位置"

| 映射方式 | 需要搜索的位置 | 硬件复杂度 |
|---------|--------------|-----------|
| 直接映射 | 1个位置 | 最简单 |
| 2路组相联 | 2个位置 | 中等 |
| 全相联 | 所有位置 | 最复杂 |

**Q：映射在什么时候发生？**

映射不是"预先设置"的，而是在**每次CPU访问数据时动态发生**：

```
CPU发出请求："我要读地址 10110 的数据"
        ↓
第一步：地址分解 → Tag=10, Index=110
        ↓
第二步：映射查找 → Index=110 指向 Cache位置110  ← 映射在这里发生！
        ↓
    ┌───检查Valid和Tag───┐
    │                     │
  命中                   缺失
    ↓                     ↓
直接返回            从Memory加载到该位置
```

**Q：怎么确认Cache位置存的就是我们要的数据？**

映射只告诉我们"去哪个位置找"，但那个位置存的不一定是我们要的数据。需要三步验证：

```
Cache每个位置的结构：
┌────────┬─────────┬──────────────┐
│ Valid  │   Tag   │     Data     │
│ 有效位  │  标签   │     数据     │
└────────┴─────────┴──────────────┘
```

验证流程：
1. **Index定位**：用地址的Index部分定位Cache位置
2. **检查Valid**：Valid=1才表示位置有效
3. **比较Tag**：Tag匹配才确认是你要的数据

**类比**：映射（Index）像"座位号"告诉你去哪找，Tag像"姓名牌"确认是不是你要找的人。

## 标签与有效位

### 核心内容讲解

缓存需要额外信息来确定存储的数据是否有效：

**标签（Tag）**：
- 存储块地址的高位部分
- 用于在缓存查找时判断该行是否包含请求的地址

**有效位（Valid Bit）**：
- 1 = 数据有效
- 0 = 无有效数据（初始状态或已被清除）
- 系统启动时所有有效位为0

### 核心概念

1. **命中条件**：标签匹配 AND 有效位为1

2. **标签存储开销**：每行需要额外的Tag位

3. **有效位的作用**：区分"空"和"存有旧数据"的缓存行

### 核心内容讲解

**访问地址22（二进制10 110）**：
```
Index = 110 (6)
Tag = 10
块地址 = 22
```
结果：Miss（未命中），加载包含地址22的块到Index=6的行。

**访问地址26（二进制11 010）**：
```
Index = 010 (2)
Tag = 11
```
结果：Miss（未命中），加载块到Index=2的行。

**再次访问**

| Word addr | Binary | Tag | Index | Hit/Miss |
| --------- | ------ | --- | ----- | -------- |
| 22        | 10110  | 10  | 110   | Hit      |
| 26        | 11010  | 11  | 010   | Hit      |

### 核心概念

1. **时间局部性体现**：刚访问过的数据再次访问会命中

2. **不同Index的访问互不影响**

3. **首次访问 → Miss，再次访问 → Hit**

## 地址细分

![[第五章26.png]]

### 核心内容讲解

以64KB缓存、64字节块为例：

**64位地址划分**：
```
| Tag (48位) | Index (10位) | Offset (6位) |
```

**32位地址划分**：
```
| Tag (16位) | Index (10位) | Offset (6位) |
```

**计算过程**：
```
块数 = 64KB ÷ 64B = 1024块
Index位数 = log₂(1024) = 10位
Offset位数 = log₂(64) = 6位（已包含块内字节选择）
Tag位数 = 地址总位数 - 10 - 6
```

**注意**：Offset已经包含了块内字节定位，不需要单独的Byte Offset！

**查找过程**：
1. 用Index从缓存中选择一行
2. 比较该行的Tag与地址的Tag
3. 检查Valid位
4. 如果都匹配，用Offset从数据块中选择字节

### 核心概念

1. **地址划分取决于缓存大小和块大小**

2. **Offset用于选择块内的具体字节**，位数由块大小决定

---

### Offset详解

**Offset由块大小决定**：
- Offset位数 = log₂(块大小)
- 块大小4字节 → Offset 2位（块内位置0-3）
- 块大小32字节 → Offset 5位（块内位置0-31）
- 块大小128字节 → Offset 7位（块内位置0-127）

**Offset的本质**：
- Offset就是地址除以块大小后的余数
- 用二进制看：就是地址的最低几位
- 作用：告诉缓存"要取块内的第几个字节"

**⚠️ 字地址 vs 字节地址**：
- **字地址**：地址编号以"字"为单位，1个地址=1个字（如64位整数=8字节），offset只需计算块内能放几个字
  - 例：块大小1字 → offset = log₂(1) = 0位（块内就1个字，不需要偏移）
  - 例：块大小2字 → offset = log₂(2) = 1位
- **字节地址**：地址编号以"字节"为单位，1个地址=1字节，offset要定位到块内的具体字节
  - 例：块大小64字节 → offset = log₂(64) = 6位（其中低2位选字节，高4位选字）
- **判断方法**：题目说"字地址"→ offset位数 = log₂(块大小/字大小)；题目说"字节地址"→ offset位数 = log₂(块大小字节数)
- **易错点**：字地址下offset通常更短（因为单位更大），不要把字地址当成字节地址来算！

**完整地址划分计算示例**：
```
给定：缓存总容量4096字节，128块，32位地址

1. 块大小 = 4096 ÷ 128 = 32字节
2. Offset位数 = log₂(32) = 5位
3. Index位数 = log₂(128) = 7位
4. Tag位数 = 32 - 7 - 5 = 20位

地址划分：| Tag(20位) | Index(7位) | Offset(5位) |
```

## 缓存未命中处理

![[第五章29.png]]

### 核心内容讲解

**命中时**：CPU正常执行

**未命中时**：
1. 停顿CPU流水线（Stall）
2. 从下一层取数据块
3. 存入缓存
4. 恢复CPU执行

**指令缓存未命中**：重启取指阶段

**数据缓存未命中**：依赖该数据的指令停顿，其他指令可能继续（乱序执行）

### 核心概念

1. **未命中惩罚（Miss Penalty）** 是评估缓存设计的重要指标

2. **非阻塞缓存**：允许CPU在未命中期间继续执行其他指令

3. **关键路径影响**：如果未命中发生在指令依赖链上，整体性能影响更大

## 写直达策略

![[第五章30.png]]

### 核心内容讲解

**写直达（Write-Through）**：
- 写命中时，同时更新缓存和主存
- 主存始终与缓存一致

**问题**：每次写都需要访问主存，性能开销大

**解决方案：写缓冲（Write Buffer）**：
- 数据先写入缓存和写缓冲
- CPU可立即继续执行
- 后台将数据写入主存

**性能计算**：
```
CPI = 1 + 0.1 × 100 = 11（10%存储指令，每次写需100周期）
```

### 核心概念

1. **写缓冲缓解了写直达的性能问题**

2. **写缓冲满时的停顿**：需要等待写缓冲清空

3. **一致性优势**：缓存和内存始终一致，便于多处理器共享

---

## 写回策略

![[第五章31.png]]

### 核心内容讲解

**写回（Write-Back）**：
- 写命中时，只更新缓存
- 主存在块被替换时才更新
- 需要**脏位（Dirty Bit）** 跟踪修改状态

**替换脏块时**：
- 先将脏块写回主存，将脏块的内容存到主存对应的位置上
- 再加载新块，将主存的数据加载到新地址的脏块
- 可用写缓冲优化

### 核心概念
1. **写回策略显著减少对主存的写操作**
2. **脏位**：标记缓存行是否被修改过， 表示这个块**被修改过**，但还没写回主存。
3. **适合写密集型工作负载**：减少总线带宽占用

---

## 写分配策略

![[第五章32.png]]

### 核心内容讲解

**写分配（Write Allocate）**：
- 写未命中时，先将块从主存加载到缓存
- 再执行写操作
- 后续对同块其他数据的访问会命中

**非写分配（No-Write Allocate / Write Around）**：
- 写未命中时，直接写主存
- 不加载块到缓存

**组合策略**：
- 写回缓存 + 写分配（常见）
- 写直达缓存 + 非写分配（常见）

### 写分配的带宽开销计算

写分配意味着**写 miss 时要额外读一整块**，这会产生读内存流量，容易忽略：

```
假设：1000条指令中有100次写，D-cache miss rate = 2%，块大小 = 64B

写miss次数 = 100 × 2% = 2次
写miss带来的读内存流量 = 2 × 64B = 128B  ← 这部分容易漏算！
```

**各类内存流量汇总模板**：

| 流量来源 | 计算方式 | 适用策略 |
|---------|---------|---------|
| I-cache miss 取块 | 指令数 × I-miss rate × 块大小 | 所有 |
| D-cache 读 miss 取块 | 读次数 × D-miss rate × 块大小 | 所有 |
| D-cache 写 miss 取块（写分配） | 写次数 × D-miss rate × 块大小 | 写分配 |
| 写直达写内存 | 写次数 × 每次写字节数 | 写直达 |
| 脏块写回 | 替换次数 × 脏块比例 × 块大小 | 写回 |

> ⚠️ **易错点**：写分配下，写 miss 先取块（读内存）再写 cache；写直达下，每次写还要额外直写内存。两种流量别混。

### 核心概念

1. **写分配与写回配合效果最佳**：一旦块在缓存中，后续写操作都在缓存内完成

2. **非写分配适合只写一次的大块数据**

3. **组合选择影响整体性能**：需要根据工作负载特性选择

### 核心内容讲解

FastMATH是一款**嵌入式MIPS处理器**，采用12级流水线，每个周期都要访问指令和数据：

**缓存配置**：

- **分离缓存（Split Cache）** ：I-cache和D-cache各自独立，可以同时访问

## Intrinsity FastMATH缓存结构详解

![[第五章34.png]]

### 核心内容讲解

这张图展示了FastMATH缓存的**完整寻址与数据读取逻辑**，是理解cache工作原理的关键图：

**地址划分（32位地址）** ：

表格

|字段|位数|作用|
|---|---|---|
|Tag|18位|标识是哪个主存块|
|Index|8位|选中256行中的哪一行|
|Byte offset|4位|选中块内16字中的哪个字节|

> 为什么是这些位数？256块 → Index需要 log2​256=8 位；每块16字 = 64字节 → Byte offset需要 log2​64=6 位，其中低2位选字节、高4位选字；剩余 32−8−6=18 位做Tag。

**Cache每行结构**：

表格

|字段|位数|说明|
|---|---|---|
|V（有效位）|1位|标记该行是否存有有效数据|
|Tag|18位|与地址的Tag比对，确认是否命中|
|Data|512位|16字 × 32位/字 = 512位|

**命中判断流程**：

1. **Index** → 选中cache的某一行
2. 取出该行的 **Tag** 与地址的Tag → 送入**比较器（Compare）**
3. 比较结果 **AND** 有效位V → 输出 **Hit信号**
    - Tag匹配 **且** V=1 → Hit ✅
    - Tag不匹配 **或** V=0 → Miss ❌
4. **Byte offset** → 通过**多路选择器（Mux）** 从512位Data中选出32位输出

### 核心概念

1. **命中 = Tag匹配 + 有效位为1**：两个条件缺一不可，V=0说明该行还没加载过数据
    
2. **比较器是直接映射cache的核心**：只需要1个比较器（因为Index已经唯一确定了一行），硬件简单
    
3. **Mux（多路选择器）的作用**：一块数据有16个字，Byte offset告诉Mux该输出哪个字

## 主存支持缓存

![[第五章35.png]]

### 核心内容讲解

DRAM作为主存需要与缓存协同工作：

**DRAM特性**：
- 使用DRAM作为主存
- 固定宽度输出（如1字）
- 通过固定宽度的时钟总线连接
- 总线时钟通常慢于CPU时钟

**块读取时序（Example cache block read）**：
```
1 bus cycle：地址传输
15 bus cycles per DRAM access：DRAM访问延迟
1 bus cycle per data transfer：数据传输
```

**4字块、1字宽DRAM的Miss Penalty计算**：
```
Miss Penalty = 1 + 4×15 + 4×1 = 65 bus cycles
              ↑      ↑      ↑
           地址传输 访问延迟  数据传输

带宽 = 16 bytes / 65 cycles = 0.25 B/cycle
```

### 核心概念

1. **DRAM访问延迟远大于CPU周期**：这是引入缓存的主要原因

2. **块传输效率**：一次传输多个字可以提高带宽利用率

3. **Miss Penalty的组成**：地址传输 + DRAM访问 + 数据传输

## 测量缓存性能

![[第五章36.png]]

### 核心内容讲解

**CPU时间的组成**：
```
CPU Time = Program execution cycles + Memory stall cycles
         = 程序执行周期        + 存储停顿周期
```

**存储停顿周期的来源**：
- 主要来自缓存未命中
- 包括从下一层取数据的时间

**存储停顿周期计算**：
```
Memory stall cycles = Memory accesses/Program × Miss rate × Miss penalty
                    = (每程序内存访问次数) × (未命中率) × (未命中惩罚)

或者：

Memory stall cycles = Instructions/Program × Misses/Instruction × Miss penalty
                    = (每程序指令数) × (每指令未命中次数) × (未命中惩罚)
```

### 核心概念

1. **缓存性能直接影响CPU执行时间**

2. **减少未命中率或未命中惩罚都能提升性能**

3. **Miss Penalty是关键指标**：取决于主存速度和块传输效率

## Cache Performance Example（缓存性能计算）

### 已知条件

表格

|参数|值|含义|
|---|---|---|
|I-cache miss rate|2%|每条指令取指时，2%的概率未命中|
|D-cache miss rate|4%|每次访问数据时，4%的概率未命中|
|Miss penalty|100 cycles|每次未命中要额外等100个周期|
|Base CPI（理想cache）|2|假设cache永远命中，平均每条指令2个周期|
|Load & Store占比|36%|所有指令中，36%需要访问内存（读/写数据）|

### 计算过程

**第一步：算I-cache带来的额外周期**

每条指令都取指，所以：

> I-cache额外周期 = 2% × 100 = **2 cycles/instruction**

**第二步：算D-cache带来的额外周期**

不是每条指令都访问数据，只有36%（Load/Store）才需要：

> D-cache额外周期 = 36% × 4% × 100 = **1.44 cycles/instruction**

⚠️ 这就是为什么要乘0.36——因为D-cache只在访问数据时才有机会miss，而64%的指令根本不碰数据。

**第三步：算实际CPI**

> Actual CPI = Base CPI + I-cache额外 + D-cache额外
> 
> = 2 + 2 + 1.44 = **5.44**

**第四步：算理想CPU快多少倍**

> 理想CPU（无miss）比实际CPU快 = 5.44 / 2 = **2.72倍**

### 一句话总结

虽然cache未命中率看着很小（2%、4%），但因为miss penalty高达100个周期，最终实际性能比理想情况慢了**将近3倍**。所以**降低miss penalty和miss rate**是cache设计的核心目标。

## 平均内存访问时间

![[第五章38.png]]

### 核心内容讲解

**AMAT（Average Memory Access Time，平均内存访问时间）**：

$$AMAT = Hit\ Time + Miss\ Rate \times Miss\ Penalty$$

**命中时间（Hit time）**：
- 命中时的访问时间
- 包括标签比较和数据选择的时间
- 对性能同样重要！

**未命中率（Miss Rate）**：
- 未命中次数 / 总访问次数
- 通常以百分比表示

**未命中惩罚（Miss Penalty）**：
- 未命中时额外花费的时间
- 从下一层获取数据所需周期

### AMAT计算示例

**给定参数**：
```
CPU时钟周期 = 1ns
命中时间 = 1 cycle
未命中惩罚 = 20 cycles
I-cache未命中率 = 5%
```

**计算过程**：
```
AMAT = Hit time + Miss rate × Miss penalty
     = 1 + 0.05 × 20
     = 1 + 1
     = 2ns
```

**等价表示**：平均每条指令需要2个周期

### 核心概念

1. **命中时间同样重要**：即使牺牲少量命中率换取更快的命中时间，也可能改善整体性能

2. **AMAT是评估存储层次性能的统一指标**

3. **优化方向**：
   - 降低未命中率（Miss Rate ↓）
   - 降低未命中惩罚（Miss Penalty ↓）
   - 降低命中时间（Hit Time ↓）

## Performance Summary（性能总结）

### 三个关键趋势

**① 当CPU性能提升时**
→ Miss penalty变得**更显著**
> CPU越快，每个周期时间越短，但内存速度没跟上，所以miss时浪费的周期占比更大了

**② 当Base CPI降低时**
→ 内存停顿占的时间比例**更大**
> 举个直观例子：
> 
> - Base CPI = 10，miss额外5个周期 → miss占比 5/15 = 33%
> - Base CPI = 1，miss额外5个周期 → miss占比 5/6 = 83%
> 
> CPU本身执行变快了，但miss浪费的时间没变，所以miss的"拖后腿"效应反而更明显了

**③ 当时钟频率提高时**
→ 内存停顿浪费的CPU周期数**更多**
> 时钟频率翻倍 = 每个周期时间减半 = 同样的内存访问时间折算成更多CPU周期

### 最终结论

> **评估系统性能时，绝对不能忽视cache行为。**
### 一句话总结

CPU发展越来越快，内存速度跟不上，cache的命中率对整体性能的影响只会越来越大。过去可以忽略的miss，现在可能让CPU慢好几倍——所以cache设计越来越重要。

### 核心内容讲解

相联度决定块在缓存中的可选位置数量：

**直接映射（Direct Mapped）**：
- 每块只有1个可选位置
- 硬件简单，速度快
- 可能产生冲突未命中

**n路组相联（n-way Set Associative）**：
- 每块有n个可选位置（在同一组内）
- 组号 = 块地址 mod 组数

**全相联（Fully Associative）**：
- 块可放在任意位置
- 冲突未命中最少
- 需要搜索所有条目，只适合小缓存（如TLB）

### 核心概念

1. **更高的相联度减少冲突未命中**

2. **但增加复杂度和潜在延迟**

3. **2路和4路组相联是常见的平衡选择**

## 相联缓存示例

![[第五章41.png]]

### 核心内容讲解

对比4块缓存的三种组织方式：

**直接映射**：
- 只能放在唯一位置
- 搜索1个标签

**2路组相联**：
- 可放在组内2个位置之一
- 搜索2个标签

**全相联**：
- 可放在任意位置
- 搜索所有4个标签

### 核心概念

1. **相联度越高，需要的比较器越多**

2. **但能减少冲突未命中**

3. **全相联缓存适合做TLB**：容量小但需要快速定位

## 相联性谱

![[第五章42.png]]

### 核心内容讲解

**对于8条目的缓存，不同相联度的结构对比：**

| 相联度 | 结构名称 | 可选位置数 | 需要的比较器 |
|--------|---------|-----------|------------|
| 1-way | 直接映射 | 1个 | 1个 |
| 2-way | 2路组相联 | 2个 | 2个 |
| 4-way | 4路组相联 | 4个 | 4个 |
| 8-way | 全相联 | 8个 | 8个 |

**直接映射（1-way）**：
- 8个块 = 8个独立位置
- 每块只有唯一位置
- Index决定位置，Tag验证身份

**2路组相联**：
- 4个组 × 2路 = 8块
- Index定位组，组内2选1
- 减少冲突未命中

**4路组相联**：
- 2个组 × 4路 = 8块
- Index定位组，组内4选1
- 更高的灵活性

**全相联（8-way）**：
- 整个缓存作为1个组
- 块可放在任意位置
- 需要8路并行比较

### 核心概念

1. **相联度越高，灵活性越大**：块可以放在更多位置

2. **代价是硬件复杂度**：需要更多比较器

3. **选择取决于权衡**：性能 vs 成本/延迟

## 相联性对比示例

![[第五章43.png]]

### 核心内容讲解

**访问序列**：0, 8, 0, 6, 8（5次访问）

**直接映射缓存的地址分析**：
```
块大小 = 1 word
缓存块数 = 8

块地址计算：Block = Memory address / Block size

地址0 → 块0 → Index = 0
地址8 → 块8 → Index = 8 mod 8 = 0  ← 与地址0冲突！
地址6 → 块6 → Index = 6
```

**访问过程分析**：

表格

| 访问  | 地址  | Index | Tag | 命中 / 缺失 | 说明                                 |
| --- | --- | ----- | --- | ------- | ---------------------------------- |
| 0   | 0   | 0     | 0   | Miss    | 首次访问，加载 Mem [0] 到块 0               |
| 8   | 8   | 0     | 1   | Miss    | Index=0 被 Mem [0] 占用，替换后加载 Mem [8] |
| 0   | 0   | 0     | 0   | Miss    | Index=0 被 Mem [8] 占用，再次替换          |
| 6   | 6   | 6     | 0   | Miss    | Index=6 独立位置                       |
| 8   | 8   | 0     | 1   | Miss    | Index=0 被 Mem [0] 占用，替换 Mem [0]    |

**结果**：5次访问全部Miss！

### 核心概念

1. **直接映射的冲突问题**：地址0和地址8映射到同一位置

2. **反复冲突导致性能下降**：同一个Index的位置被反复替换

3. **这就是高相联度缓存的价值**：提供更多可选位置

## 相联性示例续

![[第五章44.png]]

### 2路组相联示例

**访问序列**：0, 8, 0, 6, 8

**缓存结构**：4组 × 2路 = 8块
- Index = Block mod 4

**访问过程**：

| 访问  | Index | 组内块 0  | 组内块 1  | 命中 / 缺失          |
| --- | ----- | ------ | ------ | ---------------- |
| 0   | 0     | -      | -      | Miss（加载 Mem [0]） |
| 8   | 0     | Mem[0] | -      | Miss（加载 Mem [8]） |
| 0   | 0     | Mem[0] | Mem[8] | **Hit!**         |
| 6   | 2     | -      | -      | Miss（加载 Mem [6]） |
| 8   | 0     | Mem[8] | Mem[6] | **Hit!**         |

**结果**：2次Hit，3次Miss → 显著改善！

### 全相联示例

**访问序列**：0, 8, 0, 6, 8

**缓存结构**：1个组 × 8路

| 访问 | 缓存内容 | 命中/缺失 |
|------|---------|----------|
| 0 | [Mem[0]] | Miss |
| 8 | [Mem[0], Mem[8]] | Miss |
| 0 | [Mem[0], Mem[8]] | **Hit** |
| 6 | [Mem[0], Mem[8], Mem[6]] | Miss |
| 8 | [Mem[0], Mem[8], Mem[6]] | **Hit** |

**结果**：2次Hit，3次Miss → 与2路相同

### 核心概念

1. **2路组相联大幅减少冲突**：地址0和8在同一组但不同路
2. **全相联最灵活**：块可放在任意位置
3. **实际权衡**：全相联硬件成本高，2-4路是常见选择

## 组相联缓存硬件结构

![[第五章46.png]]

### 核心内容讲解

**地址划分（4路组相联示例）**：
```
┌────────────────────┬──────────┬──────────┐
│       Tag          │  Index   │  Offset  │
│     22位           │   8位    │   2位    │
│  (31...12)        │ (11...3) │  (1...0) │
└────────────────────┴──────────┴──────────┘
```

**硬件结构**：
```
1. Index选组：用8位Index从256个组中选择1个
        ↓
2. Tag并行比较：4路的Tag同时与地址Tag比较
        ↓
3. 命中判断：比较结果+Valid位 → 与门 → 或门 → Hit
        ↓
4. 数据选择：4选1多路选择器 → 输出数据
```

**关键组件**：

| 组件           | 作用                 |
| ------------ | ------------------ |
| Tag 比较器 (×4) | 同时比较 4 路 Tag       |
| 与门 (×4)      | Tag 匹配 AND Valid=1 |
| 或门           | 任意一路命中即为命中         |
| 4 选 1 多路选择器  | 根据命中信号选择数据         |

### 核心概念

1. **相联度越高，比较器越多**：硬件复杂度线性增长

2. **并行比较是性能关键**：所有候选块同时检查

3. **延迟随相联度增加**：更多比较器可能拉长关键路径

## 替换策略概述

![[第五章47.png]]

### 核心内容讲解

当缓存组已满且需要加载新块时，必须选择一个块替换：

**直接映射缓存**：
- 没有选择——只有唯一位置
- 必须替换该位置的原块

**组相联缓存**：
- 优先选择**无效块**（Valid=0）
- 如果没有无效块，在组内选择一个块替换

常见策略包括 **LRU**（最久未用）、**随机**、**FIFO**（先进先出）和**伪LRU**，核心权衡是命中率 vs 硬件复杂度。关于各策略的具体实现细节和对比，详见后文"缓存替换策略详解"一节。

### 核心概念

1. **替换策略直接影响缓存命中率**

2. **高相联度缓存常用伪LRU**：权衡性能和硬件成本

## 多级缓存

![[第五章48.png]]

### 核心内容讲解

现代处理器采用**多级缓存**来平衡速度和容量：

**第一级缓存（L1）**：
- 紧邻CPU
- 最小，但最快
- 通常1-2个周期访问

**第二级缓存（L2）**：
- 服务于L1的未命中
- 更大，但稍慢
- 比主存快但比L1慢

**第三级缓存（L3，可选）**：
- 高端系统才有
- 更大，进一步减少主存访问

### 性能对比

**单级L1缓存的问题**：
```
基础CPI = 1
L1未命中率 = 2%
未命中惩罚 = 100ns / 0.25ns = 400周期
有效CPI = 1 + 0.02 × 400 = 9
```

**添加L2缓存后**：
- 大幅减少对主存的访问
- 有效降低平均内存访问时间

### 核心概念

1. **多级缓存通过在速度和容量间取得更好平衡**

2. **L1命中率高但容量小，L2容量大但命中率较低**

3. **全局未命中率**：通过L1后仍miss的比例（到达主存）

---

## 多级缓存示例

![[第五章49.png]]![[第五章50.png]]

### 给定参数

```
CPU基础CPI = 1，时钟频率 = 4GHz
每指令L1未命中率 = 2%
主存访问时间 = 100ns
```

**周期换算**：
```
时钟周期 = 1 / 4GHz = 0.25ns
100ns = 100 / 0.25 = 400周期
```

### 仅L1缓存

```
Miss penalty = 400 cycles
有效CPI = 1 + 0.02 × 400 = 9
```

### 添加L2缓存后

**L2参数**：
```
L2访问时间 = 5ns = 20周期
L2全局未命中率 = 0.5%（到达主存的比例）
```

**Miss penalty 分析**：

| 情况               | 惩罚     |
| ---------------- | ------ |
| L1 命中            | 0 周期   |
| L1 Miss, L2 Hit  | 20 周期  |
| L1 Miss, L2 Miss | 400 周期 |

**CPI计算**：
```
CPI = 基础CPI + L1 Miss×L2惩罚 + 全局Miss×内存惩罚
    = 1 + 0.02 × 20 + 0.005 × 400
    = 1 + 0.4 + 2.0
    = 3.4
```

**性能提升**：
```
性能比 = 9 / 3.4 = 2.6倍
```

### 核心概念

1. **L2缓存带来2.6倍性能提升**：显著减少主存访问

2. **每级设计目标不同**：
   - L1：最小化命中时间（快速）
   - L2/L3：最小化未命中率（高效）

3. **包含策略（Inclusive vs Exclusive）**：L2是否包含L1的内容

## 多级缓存设计考虑

![[第五章51.png]]

### 各级缓存的设计重点

**L1缓存（第一级）**：
- 核心目标：**最小化命中时间**
- 块大小相对较小
- 通常离CPU最近，必须快

**L2/L3缓存（后续级）**：
- 核心目标：**低未命中率**，避免访问主存
- 命中时间的影响较小（因为已经在L1之后）
- 可以用更大的块和更高的相联度

### 设计权衡

| 特性 | L1 | L2/L3 |
|------|-----|-------|
| 容量 | 小（32-64KB） | 大（256KB-数MB） |
| 块大小 | 较小 | 较大 |
| 相联度 | 通常4-8路 | 可能更高 |
| 命中时间 | 关键（1-2周期） | 次要（数十周期） |
| 未命中率目标 | 尽量低 | 尽量低 |

### 典型配置

```
L1块大小 < L2块大小  （如：32B vs 64B）
L1容量 < L2容量      （如：32KB vs 256KB）
L1相联度 ≥ L2相联度  （L1需要更快的访问）
```

### 核心概念

1. **分离vs统一L2**：统一缓存灵活但可能成为瓶颈

2. **片上vs片外L2**：片上更快但占用芯片面积

3. **性能优化需要全局考虑**：不能只优化某一层

## 与高级CPU的交互

![[第五章52.png]]

### 核心内容讲解

**乱序执行CPU（Out-of-order）** 可以在缓存未命中期间继续执行其他指令：

**关键机制**：
- 待存储指令留在Load/Store单元
- 依赖指令在保留站中等待
- 独立指令继续执行

### 缓存未命中的影响

```
传统观点：
缓存Miss → CPU停顿 → 性能下降

现代观点：
缓存Miss → 其他独立指令继续执行 → 性能影响取决于程序数据流
```

**好消息**：许多指令可以在Miss期间完成

**坏消息**：分析变得复杂，需要系统模拟

### 核心概念

1. **乱序执行减少缓存Miss的直接影响**：CPU不必完全停顿

2. **性能分析更复杂**：取决于程序的数据依赖关系

3. **评估需要模拟**：很难用公式精确预测

### 例题1：矩阵局部性（类似5.1）

以下C代码中，同一行元素连续存储，每个字为64位整数：

```c
for (I=0; I<16; I++)
  for (J=0; J<4000; J++)
    A[I][J] = B[J][I] + A[I][J];
```

**问题：**
1. 在一个32字节的Cache块中可以存放多少个64位整数？
2. 对哪个变量的访问显示了时间局部性？对哪个显示了空间局部性？
3. 如果改用Matlab（列优先存储），对哪个变量的访问显示了空间局部性？
4. 假设Cache块大小为16字节，存储所有被访问的矩阵元素需要多少个Cache块？（C存储 vs Matlab存储）

---

**参考答案：**

1. 32B ÷ 8B = **4个**
2. **空间局部性**：`A[I][J]`——内层循环J递增，C行优先，A[I][J]和A[I][J+1]在同一行连续存储，同一Cache块
   **时间局部性**：不明显（此题没有反复访问同一元素的变量；如果外层循环有复用才算）
3. Matlab列优先：`B[J][I]`显示空间局部性——J递增时B[J][I]和B[J+1][I]在同一列，列优先下连续
4. 被访问的元素：
   - A: 16×4000 = 64000个元素（整个矩阵都被访问）
   - B: 4000×16 = 64000个元素（整个矩阵都被访问）
   - 总元素 = 128000个，每块放2个（16B÷8B=2）
   - 需要的块数 = 128000÷2 = **64000块**（C存储和Matlab存储访问的元素总数相同，只是布局不同）
   - ⚠️ 如果题目问的是"覆盖所有被访问地址所需的Cache块数"而非"同时存储"，则C行优先和Matlab列优先结果一样，因为访问的总元素集合相同

### 例题3：Cache总位数计算（类似5.3）

直接映射cache，字节寻址，64位地址和字。

**5.3-A**：块大小4个字的16KiB cache

```
块大小 = 4字 × 8B = 32B
块数 = 16KiB ÷ 32B = 16384 ÷ 32 = 512块

地址划分：
Offset = log₂(32) = 5位
Index = log₂(512) = 9位
Tag = 64 - 9 - 5 = 50位

每行 = Tag(50) + Valid(1) + 数据(32×8=256) = 307位
总位数 = 307 × 512 = 157184位
```

**5.3-B**：块大小8个字的32KiB cache

```
块大小 = 8字 × 8B = 64B
块数 = 32KiB ÷ 64B = 32768 ÷ 64 = 512块

地址划分：
Offset = log₂(64) = 6位
Index = log₂(512) = 9位
Tag = 64 - 9 - 6 = 49位

每行 = Tag(49) + Valid(1) + 数据(64×8=512) = 562位
总位数 = 562 × 512 = 287744位
```

> 💡 对比5.3-A和5.3-B：数据量翻倍（16→32KiB），但总位数从157184→287744，只增加了83%而非100%，因为**Tag+Valid的元数据开销被更大的数据块分摊了**（每行数据从256位→512位，但元数据从51位→50位，几乎不变）。

### 例题4：写策略带宽计算（类似5.7）

| 参数 | 值 |
|------|-----|
| 每1000条指令数据读次数 | 300 |
| 每1000条指令数据写次数 | 150 |
| 指令Cache失效率 | 0.5% |
| 数据Cache失效率 | 3% |
| 块大小 | 32字节 |
| 读/写操作宽度 | 32位 |
| 基础CPI | 2 |

**5.7-A**：写直达 + 写分配

```
读内存（取块）：
  I-cache miss: 1000 × 0.5% × 32B = 160B
  D-cache 读miss: 300 × 3% × 32B = 288B
  D-cache 写miss（写分配取块）: 150 × 3% × 32B = 144B
  读内存小计 = 592B

写内存（写直达）：
  每次写直写: 150 × 4B = 600B

总带宽 = 592 + 600 = 1192B
BPC = 1192 ÷ (1000 × 2) = 0.596 BPC
```

**5.7-B**：写回 + 写分配，脏块比例25%

```
读内存（取块）= 592B（同上）

写内存（脏块写回）：
  D-cache 总miss = 300×3% + 150×3% = 9 + 4.5 = 13.5次
  脏块写回 = 13.5 × 25% × 32B = 108B

总带宽 = 592 + 108 = 700B
BPC = 700 ÷ 2000 = 0.35 BPC
```

> 💡 对比：写回比写直达节省约41%带宽（0.596→0.35），因为写直达每次写都打内存（150次×4B=600B），而写回只在驱逐脏块时才写回（108B），差距主要来自这部分。
