## 第一题：二进制原码减法计算

**题目**：计算(1011 1001 - 0111 1010)的二进制结果

##### 1. 原码规则回顾

8位原码的最高位是符号位：

- 0代表正数，数值位为真实值
- 1代表负数，数值位为绝对值的二进制

##### 2. 错误答案分析

你填写的`00111111`是直接将两个二进制数按无符号数相减的结果：

```plaintext
  1011 1001
- 0111 1010
= 0011 1111
```

这种计算方式忽略了原码的符号位规则，是典型的概念混淆。

##### 3. 正确计算步骤

```python
# 将原码转换为十进制
num1 = - (0b0111001)  # 10111001的数值位取反加1得到-57
num2 = 0b01111010     # 01111010是正数，值为122

# 十进制计算
result_dec = num1 - num2  # -57 - 122 = -179

# 转换为8位原码（注意：这里产生了下溢）
result_bin = bin(result_dec & 0xFF)[2:].zfill(8)  # 得到10110011
```

最终正确结果：`1011 0011`

## 第二题：溢出判断

**题目**：判断计算结果是否有上溢或下溢

##### 1. 8位原码的取值范围

- 最大值：`0111 1111`（+127）
- 最小值：`1111 1111`（-127）

##### 2. 错误原因分析

你判断为"无溢出"，但实际计算结果`-179`已经小于8位原码能表示的最小值`-127`，属于**下溢**（Underflow）。

##### 3. 溢出判断逻辑

```python
def check_overflow(result, bits=8):
    min_val = - (2 ** (bits-1) - 1)
    max_val = 2 ** (bits-1) - 1
    if result < min_val:
        return "下溢"
    elif result > max_val:
        return "上溢"
    else:
        return "无溢出"

# 测试
print(check_overflow(-179))  # 输出：下溢
```

### 1. 类型转换规则回顾

当不同类型的操作数进行比较时，C语言会遵循"通常算术转换"规则：

- 无符号类型 > 有符号类型
- 长类型 > 短类型
```
最高优先级 👑 
├─ 长双精度 (long double) 
├─ 双精度 (double) 
├─ 单精度 (float) 
└─ 整数类型转换层级 
	├─ 无符号长整型 (unsigned long) 
	├─ 长整型 (long) 
	├─ 无符号整型 (unsigned int) 
	└─ 整型 (int) ← 最低优先级
```


这个时候由于无符号**转化成有符号**的优先级更高，因此还是比较的有符号
