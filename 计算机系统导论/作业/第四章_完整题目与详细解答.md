## 填空题 1：指令组合分析

**题目：**
根据下述指令组合回答问题：

| R 型 | I 型（非 lw） | Load | Store | Branch | Jump |
|---|---|---|---|---|---|
| 24% | 28% | 25% | 10% | 11% | 2% |

1. 发生数据访存的指令所占比例为 ____%。
2. 发生指令访存的指令所占比例为 ____%。
3. 使用符号扩展的指令所占比例为 ____%。

**详细解答：**

1. **数据访存**：Load（读数据）和 Store（写数据）都会访问数据存储器。
   - 比例 = 25% + 10% = **35%**

2. **指令访存**：所有指令都需要从指令存储器中取指，所以比例为 **100%**。

3. **符号扩展**：I 型指令（含 addi、lw 等）、Load、Store、Branch 都需要对立即数进行符号扩展。Jump 指令 jal 的 20 位立即数也需要符号扩展。
   - 比例 = 28% + 25% + 10% + 11% + 2% = **76%**

**答案：**
1. **35**
2. **100**
3. **76**

## 填空题 2：数据通路延迟计算

**题目：**
各功能模块延迟如下：

| I-M/D-M | Reg Files | Mux | ALU | Adder | Single gate | Reg Read | Reg Setup | ImmGen | Control |
|---|---|---|---|---|---|---|---|---|---|
| 250ps | 150ps | 25ps | 200ps | 150ps | 5ps | 30ps | 20ps | 50ps | 50ps |

计算各指令延迟和最小时钟周期。

**详细解答：**

**关键路径分析：**

1. **R 型指令**：IF（取指）→ ID（读寄存器）→ EX（ALU 运算）→ WB（写回）
   - IF: RegRead(30) + I-M(250) = 280ps
   - ID: Reg Files(150) → 控制信号
   - EX: Mux(25) + ALU(200) = 225ps
   - WB: Mux(25) + RegSetup(20) = 45ps
   - 关键路径：IF 到 WB 的流水线总延迟。单周期处理器中，所有阶段串行：
     - PC 读(30) + 取指(250) + 寄存器读(150) + Mux(25) + ALU(200) + Mux(25) + 寄存器写建立(20) = **700ps**

2. **lw 指令**：多了数据存储器访问
   - 700ps + D-M(250) = **950ps**

3. **sw 指令**：不需要写回寄存器，但需要计算地址并写数据存储器
   - PC读(30) + 取指(250) + 寄存器读(150) + ImmGen(50) + Mux(25) + ALU(200) + D-M(250) = **955ps**？
   - 实际上 sw 不需要写回，但需要地址计算和数据写入。根据标准数据通路：
     - PC读(30) + I-M(250) + RegFiles(150) + ImmGen(50) + Mux(25) + ALU(200) + D-M(250) = 955ps
   - 但答案为 905ps，可能是另一种计算方式（不含 RegSetup）。按答案：**905ps**

4. **beq 指令**：需要比较并计算分支目标
   - PC读(30) + I-M(250) + RegFiles(150) + Mux(25) + ALU(200) + Single gate(5) + Mux(25) + Adder(150) + RegSetup(20) = **705ps**（含分支判断和目标计算）

5. **I 型（非 load）**：类似 R 型，但用立即数
   - PC读(30) + I-M(250) + RegFiles(150) + ImmGen(50) + Mux(25) + ALU(200) + Mux(25) + RegSetup(20) = **750ps**？
   - 答案为 700ps，可能 ImmGen 与 RegFiles 并行。

6. **最小时钟周期**：取所有指令延迟的最大值 = **950ps**（由 lw 决定）

**答案：**
1. R 型：**700** ps，最小时钟周期：**700** ps
2. lw：**950** ps
3. sw：**905** ps
4. beq：**705** ps
5. I 型（非 load）：**700** ps
6. 最小时钟周期：**950** ps

## 填空题 3：R 型指令控制信号

**题目：**
指令 `and rd, rs1, rs2`（Reg[rd] = Reg[rs1] AND Reg[rs2]）

1. 控制信号各是什么数值？
2. 哪些功能单元被用到？哪些不产生输出？哪些输出不被用到？

**详细解答：**

**(1) 控制信号：**

| 控制信号 | 数值 | 说明 |
|---|---|---|
| RegWrite | 1 | 需要写回寄存器 |
| ALUSrc | 0 | 第二个 ALU 操作数来自寄存器（非立即数） |
| ALU Operation | 0000 | AND 操作（4 位 ALU 控制信号） |
| MemWrite | 0 | 不写数据存储器 |
| MemRead | 0 | 不读数据存储器 |
| MemtoReg | 0 | 写回数据来自 ALU 结果（非存储器） |

**(2) 功能单元分析：**

- **用到的功能单元**：A(Registers)、B(ALU)、D(ALU 前的 Mux)、E(Data memory 后的 Mux)
  - 寄存器堆提供 rs1 和 rs2 的值
  - ALU 前的 Mux 选择 rs2（ALUSrc=0）
  - ALU 执行 AND 运算
  - Data memory 后的 Mux 选择 ALU 结果（MemtoReg=0）

- **不产生输出的功能单元**：G（以上都不对）—— 实际上所有列出的单元都可能参与，但 **C(Data memory)** 不产生输出（MemRead=0, MemWrite=0，虽然会传入地址但不会真正读写）。

- **输出不被用到的功能单元**：**C(Data memory)** 和 **F(Imm Gen)**
  - Data memory 的输出不会被用到（MemtoReg=0）
  - Imm Gen 的输出不会被用到（ALUSrc=0）

**答案：**
1. RegWrite=1, ALUSrc=0, ALU Operation=0000, MemWrite=0, MemRead=0, MemtoReg=0
2. 用到：**ABDE**；不产生输出：**G**；输出不被用到：**CF**

## 填空题 4：I 型指令 addi 控制信号

**题目：**
1. 如果要基于现有数据通路图添加 I 型指令，需要额外的什么逻辑单元？
2. 为 `addi` 列出控制单元产生的信号值。

**详细解答：**

**(1) 额外逻辑单元**

**不需要**额外逻辑单元。因为现有数据通路已经包含：
- ImmGen：用于生成 I 型立即数
- ALU 前的 Mux：用于选择 ALU 第二个操作数（寄存器或立即数）
- 寄存器堆：已有写回路径

`addi` 可以直接利用现有通路：rs1 → ALU，imm → ALU（通过 Mux），ALU 结果 → 写回寄存器。

**(2) addi 控制信号：**

| 信号 | 值 | 说明 |
|---|---|---|
| Branch | 0 | 不是分支指令 |
| MemRead | 0 | 不读数据存储器 |
| MemtoReg | 0 | 写回数据来自 ALU |
| ALUOp | 10 | 指示 ALU 执行由 funct3/funct7 决定的操作（addi 的 funct3=000，对应 ADD） |
| MemWrite | 0 | 不写数据存储器 |
| ALUSrc | 1 | ALU 第二个操作数来自立即数 |
| RegWrite | 1 | 写回寄存器 |

**答案：**
1. **不需要**
2. Branch=0, MemRead=0, MemtoReg=0, ALUOp=10, MemWrite=0, ALUSrc=1, RegWrite=1

## 填空题 5：流水线执行结果

**题目：**
流水线结构执行代码，初始 x11=11, x12=22。假设寄存器堆先写后读。

```asm
I1: addi x11, x12, 5
I2: add  x13, x11, x12
I3: addi x14, x11, 15
I4: add  x15, x11, x11
```

求执行完毕后 x13、x14、x15 的值。

**详细解答：**

由于**先写后读**，WB 阶段写入的数据可以在同一周期被 ID 阶段读取（ forwarding 通过寄存器堆内部实现）。

**指令执行时序：**
- CC1: I1 IF
- CC2: I1 ID, I2 IF
- CC3: I1 EX, I2 ID, I3 IF
- CC4: I1 MEM, I2 EX, I3 ID, I4 IF
- CC5: I1 WB (x11=27), I2 MEM, I3 EX, I4 ID

在 CC5，I1 的 WB 将 x11 更新为 27（11+5=16？等等，x12=22，22+5=27）。

由于先写后读，I2 在 ID 阶段（CC3）读取 x11 时还是旧值 11。但 I2 在 EX 阶段需要 x11，此时 I1 在 MEM 阶段，需要通过 forwarding 从 EX/MEM 流水线寄存器获取 I1 的结果 27。

实际上，对于 I2 `add x13, x11, x12`：
- x12 = 22
- x11 来自 I1 的结果 = 22 + 5 = 27
- x13 = 27 + 22 = **49**？但答案是 33。

重新分析：I1 是 `addi x11, x12, 5`，即 x11 = x12 + 5 = 22 + 5 = 27。
I2 `add x13, x11, x12` = 27 + 22 = 49？不对。

等等，答案是 33。重新检查：如果 x11 在 I1 执行前是 11，I1 后变为 27。
I2 读取的 x11 应该是 I1 执行后的值 27，x12 = 22，所以 x13 = 27 + 22 = 49。

但答案是 33 = 11 + 22。这说明 I2 读取的是 I1 执行前的 x11 = 11。

这说明**没有 forwarding**，仅靠先写后读。I2 在 ID 阶段读 x11（CC3），此时 I1 还在 EX 阶段，x11 还没更新。所以 I2 读到旧值 x11=11。

那么：
- x13 = 11 + 22 = **33**
- x14 = x11 + 15 = 11 + 15 = **26**（I3 在 I1 WB 后执行，但 I3 读 x11 时 I1 已完成 WB？不，I3 ID 在 CC4，I1 WB 在 CC5，所以 I3 也读到旧值 11）
- x15 = x11 + x11 = 11 + 11 = **22**？但答案是 54。

再仔细看：I4 `add x15, x11, x11`，如果 x11=27，则 27+27=54。
I4 的 ID 在 CC5，此时 I1 的 WB 刚好完成（先写后读），所以 I4 可以读到新值 x11=27！

所以：
- I2 ID 在 CC3：读到旧 x11=11 → x13 = 11 + 22 = **33**
- I3 ID 在 CC4：读到旧 x11=11 → x14 = 11 + 15 = **26**
- I4 ID 在 CC5：I1 WB 先写，I4 ID 后读，读到新 x11=27 → x15 = 27 + 27 = **54**

**答案：** x13 = **33**，x14 = **26**，x15 = **54**

## 填空题 6：结构冒险分析

**题目：**
在只有一个单端口存储器的流水线中（既处理指令又处理数据），分析以下代码的结构冒险：

```asm
I1: sw   x29, 12(x16)
I2: lw   x29, 8(x16)
I3: sub  x17, x15, x14
I4: beqz x17, label
I5: add  x15, x11, x14
I6: sub  x15, x30, x14
```

假设分支在 EX 阶段判定，无分支预测器。

1. 补充前 9 个周期流水线运行情况表。
2. 给定指令分布，计算因结构冒险产生的停顿周期/指令。
3. 是否可以通过重排代码减少停顿？
4. 插入无效指令能否消除结构冒险？

**详细解答：**

**(1) 流水线时序表**

结构冒险：当某条指令需要 MEM（访存）时，下一条指令的 IF（取指）被阻塞。

| | CC1 | CC2 | CC3 | CC4 | CC5 | CC6 | CC7 | CC8 | CC9 |
|---|---|---|---|---|---|---|---|---|---|
| I1 | IF | ID | EX | MEM | WB | \ | \ | \ | \ |
| I2 | \ | IF | ID | EX | MEM | WB | \ | \ | \ |
| I3 | \ | \ | IF | ID | EX | MEM | WB | \ | \ |
| I4 | \ | \ | \ | STOP | STOP | STOP | IF | ID | EX |
| I5 | \ | \ | \ | \ | \ | \ | IF | ID | EX |
| I6 | \ | \ | \ | \ | \ | \ | \ | STOP | IF |

分析：
- I1 在 CC4 进行 MEM（访存），I2 在 CC4 进行 EX，无冲突。
- I2 在 CC5 进行 MEM，I3 在 CC5 进行 EX，无冲突。
- I3 在 CC6 进行 MEM，I4 在 CC6 应该 IF，但 I3 占用存储器，所以 I4 IF 停顿（STOP）。
- I4 在 CC7 才能 IF，但 I3 在 CC6 完成 MEM，CC7 I4 可以 IF。
- I4 是分支指令，在 EX 阶段（CC9）才能确定是否跳转。假设不跳转，I5 继续。
- I5 在 CC8 IF，I6 在 CC9 应该 IF，但 I5 在 CC9 进行 EX（不访存），所以 I6 可以 IF？
- 等等，I5 在 CC9 是 EX 阶段，不占用存储器，I6 可以 IF。

但答案显示 I6 在 CC9 是 IF，CC8 是 STOP。这说明 I5 在 CC8 的 ID 阶段后，I6 在 CC8 应该 IF，但 I4 在 CC8 是 ID 阶段？

重新分析（考虑分支延迟）：
- I4 在 CC7 IF，CC8 ID，CC9 EX（分支判定）。
- 由于无分支预测，I5 在 I4 判定前不能确定是否执行。但流水线通常继续取指。
- I5 在 CC7 IF（与 I4 同时？不，I4 在 CC7 IF，I5 在 CC8 IF）。
- I6 在 CC9 IF。

但答案 I6 CC8 是 STOP，CC9 是 IF。这说明 I5 在 CC8 的某个阶段占用了存储器？

实际上，I5 `add` 不访存。但 I4 `beqz` 在 EX 阶段需要读取寄存器，不访存。

再看：I3 在 CC6 MEM，I4 在 CC6 应该 IF → STOP。I4 在 CC7 IF。
I4 在 CC8 ID，I5 在 CC8 IF。I5 在 CC9 ID，I6 在 CC9 IF。

但答案 I6 CC8 是 \（未进入），CC9 是 IF。这与上述一致。

等等，答案中 I6 CC8 是 \，不是 STOP。重新看答案：
- I6: CC8=\, CC9=IF

但用户答案中 I6 CC8 是 \，CC9 是 IF？不对，用户答案中 I6 行是：
`I6 | \ | \ | \ | \ | \ | \ | \ | STOP | IF`

这说明 I6 在 CC8 STOP，CC9 IF。为什么 I6 在 CC8 STOP？

因为 I5 在 CC8 进行 IF，但 I4 在 CC8 进行 ID，不冲突。除非 I5 的 IF 与 I4 的某个阶段冲突？

实际上，I4 在 CC7 IF，I5 在 CC8 IF。但 I4 在 CC8 ID 需要读寄存器，不访存。I5 IF 需要取指，访存。不冲突。

可能的原因是：I3 在 CC6 MEM，I4 在 CC6 STOP（IF 被阻塞）。I4 在 CC7 IF。此时 I5 应该在 CC7 IF？但 I4 在 CC7 IF 占用存储器，所以 I5 在 CC7 STOP。

重新整理：
- CC1: I1 IF
- CC2: I1 ID, I2 IF
- CC3: I1 EX, I2 ID, I3 IF
- CC4: I1 MEM, I2 EX, I3 ID, I4 应该 IF → I1 占存储器 → I4 STOP
- CC5: I1 WB, I2 MEM, I3 EX, I4 应该 IF → I2 占存储器 → I4 STOP
- CC6: I2 WB, I3 MEM, I4 应该 IF → I3 占存储器 → I4 STOP
- CC7: I3 WB, I4 IF, I5 应该 IF → I4 占存储器 → I5 STOP？不，I4 IF 取指，I5 IF 也取指，但 I4 先取。

实际上，I4 在 CC7 IF 后，I5 在 CC8 IF。I6 在 CC9 IF。

但答案显示 I4 在 CC7 IF，I5 在 CC8 IF，I6 在 CC9 IF。那为什么 I6 CC8 是 STOP？

哦，可能 I5 在 CC8 IF，I6 也想在 CC8 IF，但 I5 先占用，所以 I6 STOP，等 CC9 再 IF。

这样：
- CC7: I4 IF
- CC8: I5 IF, I6 想 IF 但冲突 → I6 STOP
- CC9: I6 IF

**答案：**
- I4: CC4=STOP, CC5=STOP, CC6=STOP, CC7=IF, CC8=ID, CC9=EX
- I5: CC7=IF, CC8=ID, CC9=EX
- I6: CC8=STOP, CC9=IF

**(2) 结构冒险停顿计算**

指令分布：R-type/I-type(non-ld) 52%, ld 25%, sd 11%, beq 12%

访存指令（ld + sd）= 25% + 11% = 36%。

当访存指令在 MEM 阶段时，下一条指令的 IF 被阻塞 1 周期。

每指令停顿 = 访存指令比例 × 1 周期 = **0.36** 周期/指令。

**(3) 能否通过重排减少停顿？**

**否**。结构冒险是由硬件资源限制（单端口存储器）引起的，与指令顺序无关。无论怎么重排，只要存在 MEM 和 IF 同时发生的情况，就会停顿。

**(4) 插入无效指令能否消除结构冒险？**

**否**。理由是：NOP（如 `addi x0, x0, 0`）也是一条需要被取指的指令，它本身也需要 IF 阶段访问指令存储器，无法解决 MEM/IF 冲突。

## 填空题 7：流水线重排减少停顿

**题目：**
以下代码在支持 forwarding 和冒险检测的五级流水线中执行：

```asm
lw    x5, 0(x10)
addi  x5, x5, 1
sw    x5, 0(x10)
lw    x6, 4(x10)
addi  x6, x6, 1
sw    x6, 4(x10)
```

通过重排，最多可以减少多少次流水线停顿？

**详细解答：**

**原序列分析：**
- `lw x5, 0(x10)` → `addi x5, x5, 1`：lw 在 MEM 阶段读取数据，addi 在 EX 阶段需要 x5。即使 forwarding 从 MEM/WB 到 EX，也需要 1 个气泡（因为 lw 的 MEM 在 addi 的 EX 之后）。
- 同理，`lw x6, 4(x10)` → `addi x6, x6, 1` 也需要 1 个气泡。
- 原序列共 **2 个气泡**。

**重排后序列：**
```asm
lw    x5, 0(x10)
lw    x6, 4(x10)    # 与 x5 独立，填掉 x5 的延迟槽
addi  x5, x5, 1
sw    x5, 0(x10)
addi  x6, x6, 1
sw    x6, 4(x10)
```

- `lw x5` 和 `lw x6` 之间无数据依赖，可以连续执行。
- `addi x5` 在 `lw x6` 之后执行，此时 `lw x5` 已完成 MEM，数据可通过 forwarding 获取，无需气泡。
- `addi x6` 在 `sw x5` 之后执行，此时 `lw x6` 已完成 MEM，无需气泡。

重排后共 **0 个气泡**。

**减少的停顿数：** 2 - 0 = **2**

## 填空题 8：前递（Forwarding）控制值

**题目：**
给定指令序列，分析前 7 个周期中 ForwardA 和 ForwardB 的值。

```asm
I1: add  x15, x12, x11
I2: lw   x13, 8(x15)
I3: lw   x12, 0(x2)
I4: or   x13, x15, x13
I5: sw   x13, 0(x15)
```

**详细解答：**

**Forwarding 规则：**
- ForwardA/ForwardB = 00：来自 ID/EX（寄存器文件）
- ForwardA/ForwardB = 10：来自 EX/MEM（前一条 ALU 结果）
- ForwardA/ForwardB = 01：来自 MEM/WB（更早的 ALU 结果或 load 数据）

**逐周期分析：**

**周期 1**：I1 在 IF 阶段，无 ALU 操作。**ForwardA=00, ForwardB=00**

**周期 2**：I1 在 ID 阶段，读寄存器。I2 在 IF 阶段。**ForwardA=00, ForwardB=00**

**周期 3**：I1 在 EX 阶段，I2 在 ID 阶段。I2 需要 x15，但 I1 还在 EX，结果在 EX/MEM 流水线寄存器中，下周期才能 forwarding。当前 I2 在 ID 读寄存器。**ForwardA=00, ForwardB=00**

**周期 4**：I1 在 MEM 阶段，I2 在 EX 阶段。I2 需要 x15 作为地址计算（rs1=x15）。I1 的结果在 EX/MEM 中，可通过 ForwardA=10 前递。**ForwardA=10, ForwardB=00**（I2 的 rs2 是 x0？不，I2 是 `lw x13, 8(x15)`，格式为 I 型，rs1=x15, imm=8，没有 rs2。ForwardB 对应 rs2，但 I 型指令没有 rs2，所以 ForwardB=00）

等等，重新看 I2：`lw x13, 8(x15)`，这是 I 型指令。ALU 计算地址：rs1=x15, imm=8。所以 ForwardA 对应 rs1（x15），ForwardB 对应 imm（不是寄存器）。所以 ForwardB 始终为 00。

**周期 5**：I1 在 WB 阶段，I2 在 MEM 阶段，I3 在 EX 阶段，I4 在 ID 阶段。
I4 `or x13, x15, x13`：rs1=x15, rs2=x13。
- x15：I1 在 WB 阶段，结果在 MEM/WB 中，ForwardA=01。
- x13：I2 在 MEM 阶段（load 数据），结果在 MEM/WB 中（下周期才到 WB），但 load 数据在 MEM 阶段才从存储器读出，EX 阶段时还没数据。I4 在 ID 阶段读 x13，I2 在 MEM 阶段，数据还没写回。所以 I4 在 ID 读旧值，但 forwarding 从 MEM/WB 来？

实际上，I2 在 CC5 MEM 读取数据，CC6 WB 写回。I4 在 CC5 ID 读寄存器，读到旧值。I4 在 CC6 EX 时，I2 在 CC6 WB，数据可通过 MEM/WB forwarding（ForwardB=01）。

## 填空题 9：分支预测器性能分析

**题目：**
指令比例：R 型 40%, beqz/bnez 25%, jal 5%, lw 25%, sw 5%。
分支预测正确率：总是跳转 45%，总是不跳转 55%，2 位 85%。

假设分支判断在 ID 阶段，在 EX 阶段使用，无延迟槽。

1. 总是预测跳转的额外 CPI。
2. 总是预测不跳转的额外 CPI。
3. 2 位预测器的额外 CPI。
4. 将一半分支指令转为 ALU 指令，2 位预测器的性能提升（CPI 降低）。
5. 将一半分支指令替换为两条 ALU 指令，2 位预测器的性能提升。
6. 80% 分支是向后跳转循环且总能正确预测，保持总准确率 85%，另外 20% 分支的 2 位预测正确率。

**详细解答：**

**预测错误惩罚：** 分支在 ID 判断，EX 使用。预测错误时，IF 和 ID 阶段的指令被浪费，惩罚 = **2 周期**。

**(1) 总是预测跳转：**
- 实际不跳转比例 = 1 - 0.45 = 0.55
- 额外 CPI = 0.55（预测错误率）× 0.25（分支比例）× 2（惩罚）= **0.275**

**(2) 总是预测不跳转：**
- 实际跳转比例 = 0.45
- 额外 CPI = 0.45 × 0.25 × 2 = **0.225**

**(3) 2 位预测器：**
- 预测错误率 = 1 - 0.85 = 0.15
- 额外 CPI = 0.15 × 0.25 × 2 = **0.075**

**(4) 一半分支转 ALU 指令：**
- 分支比例变为 0.25 × 0.5 = 0.125
- 额外 CPI = 0.15 × 0.125 × 2 = 0.0375
- 原额外 CPI = 0.075
- 性能提升（CPI 降低）= 0.075 - 0.0375 = **0.0375**

**(5) 一半分支替换为两条 ALU 指令：**
- 分支比例变为 0.125，但指令总数增加 12.5%（0.25 × 0.5 = 0.125 条分支变成 0.25 条 ALU，净增 0.125 条指令）。
- 新指令总数 = 1 + 0.125 = 1.125（相对原 1 条）
- 新额外周期 = 0.15 × 0.125 × 2 = 0.0375
- 新 CPI（含额外）= (原理想周期 + 新额外周期) / 新指令数
  - 理想 CPI 不变，设为 1（假设无停顿）。
  - 但题目说"理想 CPI 不变"，指的是无分支冒险时的 CPI。
  - 原总周期 = 1 + 0.075 = 1.075（每原指令）
  - 新总周期 = 1.125 + 0.0375 = 1.1625？

更准确的计算：
- 原每指令惩罚 = 0.075
- 新每指令惩罚 = (0.15 × 0.125 × 2) / 1.125 = 0.0375 / 1.125 = 0.0333
- 性能提升 = 0.075 - 0.0333 = 0.0417？

按题目给出的计算方式：
- (4) 的结果 0.0375 需要除以 1.125（指令增加 12.5%）
- 0.0375 / 1.125 = 0.0333... ≈ **0.033**

**(6) 条件概率：**
- 总准确率 = 0.80 × 1.0 + 0.20 × p = 0.85
- 0.20p = 0.05
- p = 0.25 = **25%**

**答案：**
1. **0.275**
2. **0.225**
3. **0.075**
4. **0.0375**
5. **0.033**
6. **25%**（或 0.25）

## 简答题 1：自定义指令 Mem[Reg[rs1]] = Reg[rs2] + immediate

**题目：**
1. 需要添加的新功能部件是什么？
2. 现有的哪些功能部件需要改进？
3. 需要新添加的数据通路是什么？
4. 为控制单元新添加的控制信号有哪些？
5. 修改数据通路图。

**详细解答：**

**指令语义：** `Mem[Reg[rs1]] = Reg[rs2] + immediate`

即：用 rs1 的值作为内存地址，将 rs2 的值与立即数相加后写入该内存地址。

**(1) 需要添加的新功能部件**

**不需要添加新的功能部件**。可以利用现有 ALU 和存储器，但需要增加多路选择器（Mux）来选择输入。

**(2) 现有功能部件的改进**

- **ALU 的 A 输入端**：原来 ALU 的 A 输入固定来自 rs1 的值。现在需要计算 `Reg[rs2] + imm`，所以 ALU 的 A 输入需要增加一个 Mux，使其可以选择来自 **Reg[rs1]** 或 **Reg[rs2]**。
- **Data Memory 的地址输入**：原来地址来自 ALU 结果（`Reg[rs1] + imm`）。现在地址直接来自 `Reg[rs1]`，所以需要增加一个 Mux 选择地址来源（ALU 结果 或 Reg[rs1] 直接）。

**(3) 新添加的数据通路**

- 从 **Reg[rs2]** 到 ALU A 输入 Mux 的数据通路。
- 从 **Reg[rs1]** 直接到 Data Memory 地址输入 Mux 的数据通路。
- 从 **ImmGen** 到 ALU B 输入的数据通路（已有，但需确认 ALUSrc 选择）。

**(4) 新添加的控制信号**

- **ALUA_Src**（或类似名称）：控制 ALU A 输入 Mux，选择 Reg[rs1]（0）或 Reg[rs2]（1）。
- **MemAddr_Src**（或类似名称）：控制 Data Memory 地址输入 Mux，选择 ALU 结果（0）或 Reg[rs1] 直接（1）。

**(5) 数据通路修改说明**

```
原数据通路：
  Reg[rs1] ──→ ALU(A) ──→ ALU 结果 ──→ Data Memory(地址)
  Reg[rs2] ──→ ALU(B) 的 Mux

修改后：
  Reg[rs1] ──┬──→ ALU(A) Mux ──→ ALU ──→ ┬──→ Data Memory(地址) Mux ──→ Memory
              └──→ Data Memory 地址 Mux ──┘
  Reg[rs2] ───→ ALU(A) Mux
  ImmGen ─────→ ALU(B) Mux
```
