
## 填空题

### 第1题
**题目：** 假定1011 1001和0111 1010是以原码形式存放的有符号8位二进制整数。

#### 详细计算过程：

**第一步：理解原码含义**
```
1011 1001(原码) → 符号位=1（负数），数值部分=011 1001 = 57 → 真值 = -57
0111 1010(原码) → 符号位=0（正数），数值部分=111 1010 = 122 → 真值 = +122
```

**第二步：转换为补码进行运算**
```
A = 1011 1001(原) → 补码转换：
  符号位保持不变：1
  数值位取反：011 1001 → 100 0110
  末位加1：100 0110 + 1 = 100 0111
  ∴ A(补) = 1100 0111

B = 0111 1010(原) → 正数补码=原码
  ∴ B(补) = 0111 1010
```

**第三步：执行补码减法 A - B = A + (-B)**
```
先求 -B 的补码（对B的补码求补）：
  B(补) = 0111 1010
  各位取反：1000 0101
  末位加1：1000 0101 + 1 = 1000 0110
  ∴ (-B)(补) = 1000 0110

现在做加法：
    1100 0111  (A的补码，即-57)
  + 1000 0110  ((-B)的补码，即-122)
  -----------
  1 0100 1101  （最高位1是进位，8位运算中丢弃）
  
  结果 = 0100 1101 = +77
```

**第四步：溢出判断**
```
实际应该等于：-57 - 122 = -179
但计算结果是：+77

8位补码可表示范围：[-128, +127]
-179 < -128 → 超出下限 → 下溢 ✅
```

1. 计算（1011 1001-0111 1010）的结果为 \_\_\_\_\_ （以二进制形式给出）
   - **你的答案：** 00111111
   - **正确答案：** 1011 0011（注：此答案存在争议，详见下方分析）
   - ❌ **错因分析：** 
     - 原码不能直接做减法，必须转换为补码运算
     - -57 - 122 = -179，超出8位有符号整数表示范围[-128, +127]
     - 严格来说，溢出时结果无意义，应标注"下溢"

2. 是否有上溢或下溢？ \_\_\_\_\_ （上溢/下溢/无溢出，三选一填入空格）
   - **你的答案：** 无溢出
   - **正确答案：** 下溢
   - ✅ **修正后答案：** 下溢
   - **详细说明：** 8位原码可表示范围为[-127, +127]，8位补码为[-128, +127]。-179 < -128，无论采用哪种编码均发生下溢。

> **关于参考答案 `1011 0011` 的分析：**
> - 如果当作补码：1011 0011 = -77
> - 如果当作原码：1011 0011 = -51
> - 都不等于 -179，因此该答案本身存在争议
> - 最严谨的回答：标注"下溢"，结果无效

```

按IEEE 754单精度格式分解：
┌─────────────┬──────────────┬─────────────────────────┐
│ 0           │ 00011000     │ 00000000000000000000000 │
```

## 简答题

### 第1题：DEC PDP-8浮点格式
**题目：** 假定采用一种类似DEC PDP-8的格式（最左边12位是指数，以二进制补码形式存储，最右边24位是小数，同样以二进制补码形式存储），写出-1.5625x10^-1的二进制位模式。没有隐含1。与IEEE 754标准的单精度和双精度进行比较，评估这个36位模式的范围和精确度。

#### 完整推导：
```
−1.5625×10^(-1) = −0.15625

转换为二进制：
-0.15625 = -0.00101₂ = -0.101×2^(-2)

12位指数（补码）: −2 → 111111111110
24位小数（补码）: −0.101 → 
  原码: 1.101000000000000000000000
  反码: 1.010111111111111111111111
  补码: 1.011000000000000000000000

完整36位: 111111111110 101100000000000000000000
```

**你的解答：**
```
111111111101110000000000000000000000

范围：36 位自定义（12 位补码指数，范围 -2047~+2047）远大于单精度（-126~+127），略大于双精度（-1022~+1023），可表示更大/更小数值；

精确度：36 位自定义（24 位无隐含 1 尾数，~7-8 位十进制有效数字）略优于单精度（~6-7 位），但远低于双精度（~15-17 位）。
```

**✅ 修正后答案：**
```
111111111110 101100000000000000000000

范围：~10^(-623) ~ 10^(616)，比单精度和双精度都大
精度：约 7 位十进制有效数字，与单精度基本相当，远低于双精度
```

**错因分析：**
- 指数部分写成了 `111111111101`（对应-3），应为 `111111111110`（对应-2）
- 小数部分的补码计算有误

**参考答案：**
- −1.5625×10^-1 = −0.15625，二进制为-0.00101 = -0.101×2^-2
- 12位指数补码为：111111111110
- 24位定点小数补码为：1011 0000 0000 0000 0000 0000
- 范围 ~ 10^−623∼10^616，比单双精度都大
- 精度 ~ 约 7 位十进制有效数字，比单精度略低，比双精度低很多

**得分：14.00/20.00**

## 总结

### 总得分：63.00/100.00

### 主要错因及改进建议：

| 错因类型 | 涉及题目 | 核心问题 | 改进建议 |
|---------|---------|---------|---------|
| 原码/补码运算混淆 | 第1题 | 未将原码转为补码进行加减运算 | 牢记：计算机中所有加减运算都必须用补码进行 |
| IEEE 754指数偏移计算错误 | 第3题、第5题 | 偏移量（bias）取值或公式应用有误 | 记住公式：bias = 2^(k-1) - 1，其中k为指数位数 |
| 浮点格式位数理解偏差 | 第4题、第5题 | 不同格式的k（指数位）和n（尾数位）混淆 | 做题前先明确各格式的位数分配 |
| 定点小数补码表示 | 简答题 | 小数部分的补码求法不熟练 | 练习：原码→反码→补码的转换步骤 |

### 重点复习建议：
1. **原码与补码的转换**：负数原码转补码 = 符号位不变 + 数值位取反 + 末位加1
2. **IEEE 754标准**：掌握单精度（1+8+23）、半精度（1+5+10）的格式和bias计算
3. **溢出判断**：结果超出表示范围时，应标注"上溢"或"下溢"，而非强行给出数值
4. **浮点格式转换**：先还原真实值，再重新编码到目标格式
