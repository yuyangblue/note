## 填空题 1：RISC-V 指令字段分析

**题目：**
对于下列每条 RISC-V 指令分析后完成填空。所有的值均用二进制表示；字段名称从 opcode、funct3、funct7、rs1、rs2、rd 和 imm 中选择填入。

**(1) `or x5, x6, x7`**

**详细解答：**
`or` 是 R 型指令（opcode = `0110011`，funct3 = `110`，funct7 = `0000000`）。
- rd = x5 = `00101`
- rs1 = x6 = `00110`
- rs2 = x7 = `00111`

**答案：**
- 指令位 [11-7] 是 **rd** 字段，值为 `00101`
- 指令位 [19-15] 是 **rs1** 字段，值为 `00110`
- 指令位 [24-20] 是 **rs2** 字段，值为 `00111`

**(2) `addi x30, x10, -8`**

**详细解答：**
`addi` 是 I 型指令（opcode = `0010011`，funct3 = `000`）。
- rd = x30 = `11110`
- rs1 = x10 = `01010`
- imm = -8（12 位有符号立即数）= `111111111000`

**答案：**
- 指令位 [11-7] 是 **rd** 字段，值为 `11110`
- 指令位 [19-15] 是 **rs1** 字段，值为 `01010`
- 指令位 [31-20] 是 **imm** 字段，值为 `111111111000`

**(3) `sw x31, 4(x30)`**

**详细解答：**
`sw` 是 S 型指令（opcode = `0100011`，funct3 = `010`）。
- rs1 = x30 = `11110`
- rs2 = x31 = `11111`
- imm = 4（12 位立即数，拆分为 imm[11:5] 和 imm[4:0]）
  - imm[11:5] = `0000000`
  - imm[4:0] = `00100`
  - 合并表示：`000000000100`

**答案：**
- 指令位 [31-25:11-7] 是 **imm** 字段，值为 `000000000100`
- 指令位 [19-15] 是 **rs1** 字段，值为 `11110`
- 指令位 [24-20] 是 **rs2** 字段，值为 `11111`

## 填空题 2：加减法溢出判断

**题目：**
假设寄存器 x5 和 x6 分别保存值 `0x80000000` 和 `0xD0000000`。

1. 执行 `add x30, x5, x6` 后，x30 的值是多少？是否发生溢出？
2. 执行 `sub x30, x5, x6` 后，x30 的值是多少？是否发生溢出？

**详细解答：**

**(1) `add x30, x5, x6`**
- x5 = `0x80000000`（最高位为 1，负数）
- x6 = `0xD0000000`（最高位为 1，负数）
- 相加：`0x80000000 + 0xD0000000 = 0x150000000`
- 截断 32 位：`0x50000000`（最高位为 0，正数）
- **溢出判断**：两个负数相加，结果为正数，符号位发生变化，**发生溢出**。答案：是。

**(2) `sub x30, x5, x6`**
- `sub` 等价于 `x5 + (-x6)`
- `-x6`：对 `0xD0000000` 取反加 1 = `0x30000000`
- `0x80000000 + 0x30000000 = 0xB0000000`
- **溢出判断**：被减数 x5 为负，减数 x6 为负，结果 `0xB0000000` 仍为负。
  两个同号数相减（或异号数相加）不会发生溢出。**未发生溢出**。答案：否。

**答案：**
1. `0x50000000`，**是**
2. `0xB0000000`，**否**

## 填空题 3：递归函数逆向工程

**题目：**
某程序定义函数 `long func(long n)`，编译后生成如下 RV32 汇编代码：

```asm
func:
    addi sp,sp,-16
    sw   ra,12(sp)
    sw   s0,8(sp)
    addi s0,a0,0
    li   a5,1
    ble  a0,a5,.L1
    sw   s1,4(sp)
    addi a0,a0,-1
    call func
    addi s1,a0,0
    addi a0,s0,-2
    call func
    add  a0,s1,a0
    lw   s1,4(sp)
.L1:
    lw   ra,12(sp)
    lw   s0,8(sp)
    addi sp,sp,16
    jr   ra
```

主函数输出 `result: 8`，推导 N 的值。

**详细解答：**

将汇编翻译为 C 伪代码：
```c
long func(long n) {
    if (n <= 1) return n;   // ble a0,a5,.L1; a5=1
    long t1 = func(n-1);
    long t2 = func(n-2);
    return t1 + t2;
}
```

这是**斐波那契数列**（Fibonacci）！
- `func(0) = 0`
- `func(1) = 1`
- `func(2) = func(1) + func(0) = 1`
- `func(3) = 2`
- `func(4) = 3`
- `func(5) = 5`
- `func(6) = 8`

程序输出 `result: 8`，所以 `func(N) = 8`，即 **N = 6**。

## 填空题 4：C 代码转汇编填空

**题目：**
将 C 代码片段补充对应的汇编代码空白处。

```c
// A 和 B 分别为整型数组
if (A[i] >= B[j])
    i++;
else
    j++;
```

```asm
# A 和 B 的起始地址放在 x10 和 x11 中，i 和 j 放在 x5 和 x28
.L5:
    slli    x6,x5,____        # i * 4
    add     x6,____,x6        # &A[i]
    lw      x7,0(x6)          # A[i]
    slli    x29,x28,____      # j * 4
    add     x29,____,x29      # &B[j]
    lw      x30,0(x29)        # B[j]
    ____    x7,x30,.L3        # if A[i] < B[j] goto .L3
    addi    x5,x5,1           # i++
    beq     ____,____,.L2     # goto .L2
.L3:
    addi    x28,x28,1         # j++
.L2:
```

**详细解答：**

1. `slli x6, x5, 2`：整型数组元素大小为 4 字节，左移 2 位即乘 4。
2. `add x6, x10, x6`：基址 x10 + 偏移量。
3. `slli x29, x28, 2`：同理，j 乘 4。
4. `add x29, x11, x29`：基址 x11 + 偏移量。
5. C 条件是 `A[i] >= B[j]`，汇编分支到 `.L3`（else 分支，即 `j++`）的条件是 `A[i] < B[j]`，所以用 **blt**（Branch Less Than）。
6. `i++` 后需要跳过 `j++`，用 `beq x0, x0, .L2`（无条件跳转，伪指令）。

**答案：**
- `2`
- `x10`
- `2`
- `x11`
- `blt`
- `x0, x0, .L2`

## 填空题 5：跳转指令地址范围

**题目：**
假设 PC = `0x20000000`。

1. 使用 `jal` 指令可以到达的最小和最大目标地址是多少？
2. 使用 `beq` 指令可以到达的最小和最大目标地址是多少？

**详细解答：**

**`jal` 指令：**
- 20 位有符号立即数，左移 1 位（半字对齐），跳转范围 $\pm 2^{20}$ 字 = $\pm 1$ MB。
- 偏移范围：$-0x100000 \sim +0xFFFFE$（字节偏移）。
- 最小目标地址：`0x20000000 - 0x100000 = 0x1FF00000`
- 最大目标地址：`0x20000000 + 0xFFFFE = 0x200FFFFE`

**`beq` 指令：**
- 12 位有符号立即数，左移 1 位，跳转范围 $\pm 2^{12}$ 字 = $\pm 4$ KB。
- 偏移范围：$-0x1000 \sim +0xFFE$（字节偏移）。
- 最小目标地址：`0x20000000 - 0x1000 = 0x1FFFF000`
- 最大目标地址：`0x20000000 + 0xFFE = 0x20000FFE`

**答案：**
1. 最小：`0x1FF00000`，最大：`0x200FFFFE`
2. 最小：`0x1FFFF000`，最大：`0x20000FFE`

## 填空题 6：while 循环汇编填空

**题目：**
补全 C 代码及其对应的汇编代码。

```c
int i = 0, j = 0;
while(________) {
    if(A[i] >= B[j])
        i++;
    else
        j++;
}
```

```asm
# A 和 B 的起始地址放在 x10 和 x11 中，N 放在 x12 中，i 和 j 放在 x5 和 x28
        li      x5,0
        li      x28,0
.L4:
        bge     x5,x12,.L1
        bge     x28,x12,.L1
.L5:
        slli    x6,x5,2
        add     x6,x10,x6
        lw      x7,0(x6)
        slli    x29,x28,2
        add     x29,x11,x29
        lw      x30,0(x29)
        blt     x7,x30,.L3
        addi    x5,x5,1
        beq     x0,x0,.L2
.L3:
        addi    x28,x28,1
.L2:
        ________________
.L1:
```

**详细解答：**

C 代码中两个 `bge` 分别判断 `i >= N` 和 `j >= N`，只要有一个满足就退出循环。所以循环继续条件是 **`i < N && j < N`**（或 `(i < N) && (j < N)` 等等价形式）。

`.L2` 标签后需要跳回循环开头 `.L4`，使用无条件跳转：`beq x0, x0, .L4`（或 `jal x0, .L4`）。

**答案：**
- while 条件：`(i < N) && (j < N)`（或 `i < N && j < N` 等）
- 空白处：`beq x0, x0, .L4`（或 `jal x0, .L4`）

## 填空题 7：二维数组逆向工程

**题目：**
已知 `long P[M][N]; long Q[N][M];`，数组在内存中连续存放。`sizeof(long)=4`。根据汇编代码确定 M 和 N。

```asm
sum_element:
    add  a4,gp,-604      # gp-604 是数组 P 的起始地址
    slli a5,a0,4         # i * 16
    add  a5,a5,a1        # i*16 + j
    slli a5,a5,2         # (i*16 + j) * 4 = i*64 + j*4
    add  a5,a4,a5
    lw   a3,0(a5)        # P[i][j]
    slli a1,a1,3         # j * 8
    add  a1,a1,a0        # j*8 + i
    slli a1,a1,2         # (j*8 + i) * 4 = j*32 + i*4
    add  a4,a4,a1
    lw   a0,512(a4)      # Q[j][i]
    add  a0,a3,a0
    ret
```

**详细解答：**

**分析 P[i][j] 的偏移：**
- `slli a5, a0, 4` → $i \times 16$
- `add a5, a5, a1` → $i \times 16 + j$
- `slli a5, a5, 2` → $(i \times 16 + j) \times 4 = i \times 64 + j \times 4$
- 二维数组行优先偏移公式：$i \times (N \times 4) + j \times 4$
- 对比得：$N \times 4 = 64$ → **N = 16**

**分析 Q[j][i] 的偏移：**
- `slli a1, a1, 3` → $j \times 8$
- `add a1, a1, a0` → $j \times 8 + i$
- `slli a1, a1, 2` → $(j \times 8 + i) \times 4 = j \times 32 + i \times 4$
- Q 的行优先偏移：$j \times (M \times 4) + i \times 4$
- 对比得：$M \times 4 = 32$ → **M = 8**

**验证数组连续性：**
- P 的大小 = $M \times N \times 4 = 8 \times 16 \times 4 = 512$ 字节。
- Q 的起始地址 = P 起始地址 + 512 = `gp-604+512 = gp-92`。
- 汇编中 `lw a0, 512(a4)`，而 `a4` 初始为 `gp-604`，加 512 后正好访问 Q 数组。验证无误。

**答案：** M = **8**，N = **16**

## 填空题 8：斐波那契递归与指令编码

**题目：**
根据 `foo()` 的 RISC-V 汇编代码，补全 C 代码，并判断编码 `0x0440006f` 对应哪一行汇编指令。

```c
int foo(int n) {
    if (n == 0)
        return ____;
    else if (n == ____)
        return ____;
    else
        return ________________;
}
```

**详细解答：**

**补全 C 代码：**
分析汇编分支逻辑：
- `bne a5, zero, .L2`：如果 `n != 0` 跳到 `.L2`。
- `li a5, 0` → `return 0`。所以 `n == 0` 时返回 **0**。
- `.L2` 中：`bne a4, a5, .L4`（a5 已加载 1）。如果 `n != 1` 跳到 `.L4`。
- `li a5, 1` → `return 1`。所以 `n == 1` 时返回 **1**。
- `.L4` 中：递归调用 `foo(n-1)` 和 `foo(n-2)`，相加返回。

**指令编码分析：**
`0x0440006f` 是 `jal` 指令（opcode `1101111`）。
- 解码得 rd = x0，立即数偏移为 32 字节。
- 这是一个无条件跳转（`j label` 伪指令，实际编码为 `jal x0, offset`）。
- 在汇编中，第 11 行 `j .L3` 和第 17 行 `j .L3` 都可能是目标。
- 通过相对地址计算：该偏移对应从当前位置跳转到 `.L3`（第 30 行标签），经计算恰好匹配第 **11** 行的 `j .L3`。

**答案：**
- C 代码空缺：`0`、`1`、`1`、`foo(n-1) + foo(n-2)`（或 `foo(n-2) + foo(n-1)`）
- 指令编码对应行号：**11**

## 简答题 1：数组操作 RISC-V 汇编

**题目：**
将 `B[8] = A[i] + A[j];` 转换为 RISC-V32 指令。假设 `i` 在 x28，`j` 在 x29，数组 A 基址在 x10，B 基址在 x11，元素大小 4 字节。

**详细解答：**

```asm
    slli x28, x28, 2       # i * 4
    add  x30, x10, x28     # &A[i]
    lw   x28, 0(x30)       # x28 = A[i]
    slli x29, x29, 2       # j * 4
    add  x31, x10, x29     # &A[j]
    lw   x29, 0(x31)       # x29 = A[j]
    add  x30, x28, x29     # x30 = A[i] + A[j]
    sw   x30, 32(x11)      # B[8] = x30 (8*4=32)
```

**注意：** 虽然可以使用 `lw x28, x28(x10)` 这类偏移寻址，但 RISC-V 标准汇编中偏移必须是立即数，不支持寄存器偏移，因此需要先 `add` 计算地址。

## 简答题 2：原子加法汇编实现

**题目：**
编写 RV32 汇编实现多线程安全的原子加法函数 `void atomic_add(int *addr, int val)`。`addr` 在 x10，`val` 在 x11。

**详细解答：**

使用 Load-Reserved / Store-Conditional（LR/SC）指令对实现原子操作：

```asm
atomic_add:
retry:
    lr.w   x5, (x10)       # 加载并保留地址处的值到 x5
    add    x6, x5, x11     # x6 = x5 + val
    sc.w   x7, x6, (x10)   # 条件存储，成功 x7=0，失败 x7≠0
    bnez   x7, retry       # 如果失败，重试
    ret
```

**原理：**
- `lr.w` 读取内存值并标记保留集（reservation set）。
- `sc.w` 仅在保留未被其他处理器打断时成功写入，并返回 0。
- `bnez` 检测失败时循环重试，保证原子性。

## 简答题 3：新指令 rpt 设计

**题目：**
考虑新指令 `rpt x29, loop`，功能：
```c
if (x29 > 0) {
    x29 = x29 - 1;
    goto loop;
}
```

1. 最适合的指令格式是什么？（给出理由）
2. 写出完成相同功能的最短 RISC-V 指令序列。

**详细解答：**

**(1) 指令格式选择**

最适合 **SB-type（分支指令格式）**。
- 理由：该指令需要判断条件（`x29 > 0`）并执行跳转（`goto loop`），同时还需要指定寄存器操作数（`x29`）。SB-type 提供了两个寄存器比较和一个 12 位有符号立即数偏移（用于跳转目标），正好匹配 "条件判断 + 相对跳转" 的需求。
- 也可以考虑 I-type（用于立即数操作），但 I-type 不支持条件跳转，因此 SB-type 更合适。

**(2) 最短指令序列**

```asm
loop:
    bge  x0, x29, exit    # if (0 >= x29) i.e., x29 <= 0, exit
    addi x29, x29, -1     # x29 = x29 - 1
    j    loop             # goto loop
exit:
```

**说明：**
- `bge x0, x29, exit` 等价于 `if (x29 <= 0) goto exit`，即 `x29 > 0` 的反向判断。
- 共 3 条指令，是最短实现。
