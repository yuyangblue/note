## 填空题 1：原码减法与溢出判断

**题目：**
假定 `1011 1001` 和 `0111 1010` 是以原码形式存放的有符号 8 位二进制整数。

1. 计算 `（1011 1001 - 0111 1010）` 的结果（以二进制形式给出）。
2. 是否有上溢或下溢？（上溢/下溢/无溢出，三选一）

**详细解答：**

**步骤 1：解析原码**
- `1011 1001`：符号位为 1（负数），数值位 `011 1001` = 32+16+8+1 = **57**。所以该数为 **-57**。
- `0111 1010`：符号位为 0（正数），数值位 `111 1010` = 64+32+16+8+2 = **122**。所以该数为 **+122**。

**步骤 2：原码减法运算**
原码减法规则：$A - B = A + (-B)$。先求 $-B$ 的原码：
- $-B = -122$，原码为 `1111 1010`。

现在进行两个负数相加（被减数 `-57` 的原码 `1011 1001` 与 `-122` 的原码 `1111 1010`）：
- 符号位相同（均为 1），结果符号位为 1。
- 数值位相加：`011 1001` + `111 1010` = `1 011 0011`（产生进位）。

**步骤 3：判断溢出**
8 位原码的数值位只有 7 位。上述相加结果数值位产生了 8 位（`1011 0011`），超出 7 位数值位表示范围，发生**下溢**（Underflow）。

**步骤 4：截取结果**
保留 8 位：`1011 0011`（符号位 1，数值位 0110011 = 51），表示 **-51**。但这是溢出后的截断结果。

**答案：**
1. `1011 0011`
2. **下溢**

## 填空题 2：关系表达式求值

**题目：**
请写出以下关系表达式的值（用 1 表示 True，0 表示 False）。

| 关系表达式 | 结果类型（无符号 / 带符号） | 结果（1/0） |
|---|---|---|
| `0 == 0U` | 无符号 | 1 |
| `-1 < 0U` | 无符号 | 0 |
| `2147483647U > -2147483647 - 1` | 无符号 | 0 |
| `2147483647 > (int)2147483647U` | 带符号 | 0 |
| `(unsigned)-1 > -2` | 无符号 | 1 |

**详细解答：**

C 语言中，当无符号数与有符号数比较时，有符号数会被**隐式转换**为无符号数。

1. **`0 == 0U`**：两边都是 0，相等。类型为无符号（unsigned int）。**结果：1**。

2. **`-1 < 0U`**：`-1` 被转为无符号数 `0xFFFFFFFF`（即 4294967295），远大于 0。所以为 False。**结果类型：无符号，结果：0**。

3. **`2147483647U > -2147483647 - 1`**：右边 `-2147483647 - 1` = `INT_MIN` = `-2147483648`。与无符号数比较时，转换为无符号 `0x80000000` = `2147483648`。左边 `2147483647U` 不大于 `2147483648`。False。**结果类型：无符号，结果：0**。

4. **`2147483647 > (int)2147483647U`**：右边强转为 `int` 后为 `2147483647`。左边等于右边，`>` 不成立。False。**结果类型：带符号，结果：0**。

5. **`(unsigned)-1 > -2`**：`(unsigned)-1` = `0xFFFFFFFF` = 4294967295。`-2` 转无符号为 `0xFFFFFFFE` = 4294967294。4294967295 > 4294967294。True。**结果类型：无符号，结果：1**。

## 填空题 3：IEEE-754 单精度浮点数解析

**题目：**
如果是 IEEE-754 单精度浮点数，则根据位模式 `0x0C00 0000`，填写：
- 符号位：____
- 指数字段的二进制位序：____
- 尾数字段的二进制位序：____
- 其代表的十进制数（用 $10^x$ 形式）：____

**详细解答：**

将 `0x0C00 0000` 展开为二进制：
```
0x0C00 0000 = 0000 1100 0000 0000 0000 0000 0000 0000
```

IEEE-754 单精度格式：1 位符号 | 8 位指数 | 23 位尾数。

- **符号位**（bit 31）：`0`（正数）
- **指数域**（bits 30-23）：`0001 1000` = 24（十进制）
- **尾数域**（bits 22-0）：`000 0000 0000 0000 0000 0000`（全 0）

**计算实际值：**
- 实际指数 $E = 24 - 127 = -103$
- 尾数 $M = 1.0$（隐含 1，尾数全 0）
- 值 $= 1.0 \times 2^{-103} \approx 9.86 \times 10^{-32}$

**答案：**
- 符号位：`0`
- 指数：`0001 1000`
- 尾数：`000 0000 0000 0000 0000 0000`
- 十进制数：$9.86 \times 10^{-32}$（或 $1 \times 10^{-31}$ 等近似形式）

## 填空题 4：9 位浮点格式转换

**题目：**
考虑两种基于 IEEE-754 浮点格式的 9 位浮点表示：
- **格式 A**：1 符号位，$k=5$ 阶码位，$n=3$ 小数位
- **格式 B**：1 符号位，$k=4$ 阶码位，$n=4$ 小数位

将格式 A 位模式转换成最接近的格式 B 表示（向 $+\infty$ 舍入）。

| 格式 A 位模式 | 格式 A 值 | 格式 B 位模式 | 格式 B 值 |
|---|---|---|---|
| `101110001` | $-(9/16)$ | `101100010` | $-(9/16)$ |
| `010110101` | 208 | `011101010` | 208 |
| `100111110` | $-(7/1024)$ | `100000111` | $-(7/1024)$ |
| `000000101` | $5/2^{17}$ | `000000001` | $1/2^{10}$ |
| `111011000` | $-4096$ | `111101111` | $-248$ |
| `011000100` | 768 | `011110000` | $+INF$ |

**详细解答：**

**格式参数推导：**
- 格式 A：5 位阶码，偏移量 $\text{Bias}_A = 2^{5-1}-1 = 15$。阶码 `00000` 表示非规格化数（指数 $1-15=-14$，尾数不隐含 1）或特殊值。
- 格式 B：4 位阶码，偏移量 $\text{Bias}_B = 2^{4-1}-1 = 7$。阶码 `0000` 表示非规格化数（指数 $1-7=-6$）。

**逐行推导：**

1. **`101110001`**：符号 1，阶码 `01110`=14，小数 `001`。
   - $E_A = 14-15 = -1$，值 $= -(1.001)_2 \times 2^{-1} = -9/16$。
   - 转格式 B：$-9/16 = -(1.0010)_2 \times 2^{-1}$。$E_B = -1+7 = 6 = `0110`$。位模式 `1 0110 0010` = `101100010`。

2. **`010110101`**：符号 0，阶码 `10110`=22，小数 `101`。
   - $E_A = 22-15 = 7$，值 $= (1.101)_2 \times 2^7 = 1.625 \times 128 = 208$。
   - 转格式 B：$208 = (1.1010)_2 \times 2^7$。$E_B = 7+7 = 14 = `1110`$。位模式 `0 1110 1010` = `011101010`。

3. **`100111110`**：符号 1，阶码 `00111`=7，小数 `110`。
   - $E_A = 7-15 = -8$，值 $= -(1.110)_2 \times 2^{-8} = -7/1024$。
   - 转格式 B：$-7/1024 = -(1.1100)_2 \times 2^{-8}$。但格式 B 最小非规格化指数为 $-6$，无法直接表示 $2^{-8}$。需用非规格化：$-(0.0000111)_2 \times 2^{-6} = -(7/2^{13})$？不对。
   - 实际上，利用格式 B 非规格化：阶码 `0000`（指数 $-6$），尾数不隐含 1。$-7/1024 = -(0.111)_2 \times 2^{-9} = -(0.000000111)_2 \times 2^{-6}$？经精确换算，格式 B 用 `100000111` 恰好表示 $-(0.111)_2 \times 2^{-9}$ 的等效非规格化形式，最终值为 $-7/1024$。

4. **`000000101`**：符号 0，阶码 `00000`，小数 `101`。
   - 格式 A 非规格化：指数 $-14$，值 $= (0.101)_2 \times 2^{-14} = 5 \times 2^{-17}$。
   - 转格式 B：$5 \times 2^{-17}$ 是正数，远小于格式 B 最小正非规格化数 $2^{-10}$（下溢）。向 $+\infty$ 舍入，取最小正非规格化数 `000000001` = $2^{-10}$。

5. **`111011000`**：符号 1，阶码 `11011`=27，小数 `000`。
   - $E_A = 27-15 = 12$，值 $= -(1.000)_2 \times 2^{12} = -4096$。
   - 格式 B 最大可表示值：阶码 `1110`=14（保留 `1111` 给 INF），$E_B=14-7=7$，最大 $=(1.1111)_2 \times 2^7 = 31 \times 8 = 248$。
   - $-4096$ 超出格式 B 范围，负数向 $+\infty$ 舍入（即向 0 靠近），取最大可表示负数 `111101111` = $-248$。

6. **`011000100`**：符号 0，阶码 `11000`=24，小数 `100`。
   - $E_A = 24-15 = 9$，值 $= (1.100)_2 \times 2^9 = 1.5 \times 512 = 768$。
   - 格式 B 最大正值约 248，768 上溢。向 $+\infty$ 舍入得正无穷 `011110000`（阶码 `1111` 表示 INF）。

## 填空题 5：半精度浮点数编码

**题目：**
IEEE 754-2008 半精度格式：16 位宽，符号位 1 位，指数 5 位（偏移 15），尾数 10 位（隐含 1）。写出 $-1.5625 \times 10^{-1}$ 的二进制位模式。

**详细解答：**

1. **转二进制**：$-1.5625 \times 10^{-1} = -0.15625$。
   - $0.15625_{10} = 0.00101_2$（因为 $1/8 + 1/32 = 0.125 + 0.03125 = 0.15625$）。
   - 规格化：$0.00101_2 = 1.01_2 \times 2^{-3}$。

2. **各字段：**
   - **符号位**：1（负数）
   - **指数**：$E = -3 + 15 = 12 = `01100`$
   - **尾数**：取规格化后小数部分 `01`，后面补 8 个 0 → `01 0000 0000`

**答案：**
- 符号：`1`
- 指数：`01100`
- 尾数：`01 0000 0000`

## 填空题 6：单精度浮点数十六进制表示

**题目：**
假定采用 IEEE 754 单精度格式，写出十进制数 $63.25$ 的十六进制表示。

**详细解答：**

1. **转二进制**：$63.25 = 111111.01_2$。
2. **规格化**：$111111.01_2 = 1.1111101_2 \times 2^5$。
3. **字段：**
   - 符号：0
   - 指数：$5 + 127 = 132 = `10000100`$
   - 尾数：`1111101` + 16 个 0 → `11111010000000000000000`
4. **拼接**：`0 10000100 11111010000000000000000`
   - 分组：`0100 0010 0111 1101 0000 0000 0000 0000`
   - 十六进制：`0x427D0000`

**答案：** `0x427D0000`

## 填空题 7：IBM 单精度浮点数格式

**题目：**
IBM 单精度浮点数：4 字节，符号位 1 位，指数 7 位（偏移 64，移码），尾数 24 位（16 进制小数，以 16 为底，不隐含前导位）。写出 $63.25$ 的二进制表示。

**详细解答：**

IBM 浮点数的尾数是以 16 为基的，格式为 $0.\text{hex} \times 16^{E}$。

1. **转十六进制**：$63.25 = 3F.4_{16}$。
2. **规格化**：$3F.4_{16} = 0.3F4_{16} \times 16^2$。
3. **字段：**
   - 符号：0（正数）
   - 指数：$2 + 64 = 66 = `1000010`$
   - 尾数：24 位表示 `0.3F4`。即 `0011 1111 0100` + 12 个 0 → `001111110100000000000000`
4. **拼接**：`0 1000010 001111110100000000000000`
   - 即 `01000010 00111111 01000000 00000000`
   - 十六进制：`0x423F4000`

**答案：** `01000010 00111111 01000000 00000000` 或 `0x423F4000`

## 简答题 1：DEC PDP-8 浮点格式分析

**题目：**
假定采用一种类似 DEC PDP-8 的格式（最左边 12 位是指数，以二进制补码形式存储，最右边 24 位是小数，同样以二进制补码形式存储），写出 $-1.5625 \times 10^{-1}$ 的二进制位模式。没有隐含 1。与 IEEE 754 标准的单精度和双精度进行比较，评估这个 36 位模式的范围和精确度。

**详细解答：**

### 1. 位模式计算

$-1.5625 \times 10^{-1} = -0.15625 = -0.00101_2$。

规格化表示（为便于存储）：$-0.00101_2 = -0.101_2 \times 2^{-2}$。

这里尾数部分存的是定点小数 $-0.101_2$（即 $-5/8$），指数部分存 $-2$。

- **指数（12 位补码）**：$-2$ 的补码为 `111111111110`。
- **小数（24 位补码）**：$-0.101_2$ 的补码表示。
  - 正数 $0.101_2$ 的 24 位原码：`0101 0000 0000 0000 0000 0000`（最高位为符号位 0，其余为小数）。
  - 负数补码：取反加 1，得 `1011 0000 0000 0000 0000 0000`。

**完整 36 位模式：**
```
111111111110 101100000000000000000000
```

### 2. 与 IEEE 754 比较

**范围：**
- 12 位指数补码范围：$-2048 \sim +2047$（即 $-2^{11} \sim 2^{11}-1$）。
- 尾数 24 位定点小数范围：$-1 \sim +(1-2^{-23})$。
- 综合数值范围约为 $2^{-2048} \sim 2^{+2047}$，即约 $10^{-616} \sim 10^{+616}$。
- **单精度**范围：约 $10^{-38} \sim 10^{+38}$。
- **双精度**范围：约 $10^{-308} \sim 10^{+308}$。
- **结论**：该 36 位自定义格式的**范围远大于单精度和双精度**。

**精确度：**
- 24 位定点小数（无隐含 1），有效二进制位数约 24 位，相当于约 **7 位十进制有效数字**。
- **单精度**精确度：约 6~7 位十进制有效数字。
- **双精度**精确度：约 15~17 位十进制有效数字。
- **结论**：该格式**精确度与单精度相当（略低），远低于双精度**。
