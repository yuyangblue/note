## 2024-2025 学年第一学期

### 一、单项选择题

**1. 不改变处理器型号，只增加处理器数量，则有（A）**
- **答案：A. 提高吞吐率**
- **解析：** 增加处理器数量属于多处理器/多核架构，可在单位时间内完成更多任务，从而提高系统吞吐率（Throughput）。但单个任务的响应时间（Latency）并不会因此减少，甚至可能因通信开销略有增加。

**2. （D）能被 GPU 和所有线程块共享**
- **答案：D. GPU 存储器（全局存储器）**
- **解析：** GPU 的存储器层次中，寄存器、私有存储器、局部存储器都是线程私有的；只有 GPU 全局存储器（Global Memory）可被所有线程块及所有线程共享。

**3. 有利于实现指令流水线的是（D）**
- **答案：D. I、II、III**
- **解析：**
  - I. 指令格式规整且长度一致：便于取指和译码阶段统一处理，有利于流水线。
  - II. 算术逻辑指令的数据来自于寄存器：避免访存操作，使执行阶段规整。
  - III. 只有 Load/Store 指令才能对操作数进行存储访问：将访存操作限制在特定指令中，简化流水线的访存阶段控制。

**4. RISC-V 转移指令目标地址（C）**
- **答案：C. 0x2010**
- **解析：** RISC-V 的 J-type（jal）和 B-type 指令中，立即数字段表示的是**半字偏移**，硬件会自动将其左移 1 位（乘以 2）得到字节偏移。题目中“相对位移量字段的内容为 0x08”，实际字节偏移为 0x08 × 2 = 0x10。当前 PC = 0x2000，目标地址 = 0x2000 + 0x10 = **0x2010**。

**5. 8 位补码运算（题目不完整，补充典型分析）**
- r1 = 0xF9 = -7, r2 = 0xFB = -5, r3 = 0x91 = -111, r4 = 0xEE = -18
- 若问“r1 × r2”：(-7) × (-5) = 35，8 位补码范围 -128 ~ 127，35 可表示，不溢出。
- 若问“r3 + r4”：(-111) + (-18) = -129，超出 -128，**溢出**。

### 二、性能计算

**1. Cache 缺失对 CPI 的影响（8 分）**

已知：
- 全命中 CPI = 1
- Load/Store 占比 50%（只有它们可能访存）
- Cache 缺失率 = 2%
- 缺失代价 = 200 时钟周期

只有 Load/Store 可能触发 Cache 缺失。每条指令因 Cache 缺失带来的额外时钟周期为：

$$
\text{额外 CPI} = 0.50 \times 0.02 \times 200 = 2
$$

实际平均 CPI：

$$
\text{CPI}_{\text{实际}} = 1 + 2 = 3
$$

与全命中相比，性能下降为原来的 **1/3**（即性能是全命中时的 33.3%）。

**2. Amdahl 定律加速比（7 分）**

| 步骤 | 占比 | 可优化 | 加速比 |
|------|------|--------|--------|
| S1   | 11%  | 否     | —      |
| S2   | 18%  | 是     | 5      |
| S3   | 23%  | 是     | 20     |
| S4   | 48%  | 是     | 1.6    |

**(1) 总体加速比**

优化后执行时间比例：

$$
T_{\text{new}} = 0.11 + \frac{0.18}{5} + \frac{0.23}{20} + \frac{0.48}{1.6}
$$

$$
= 0.11 + 0.036 + 0.0115 + 0.30 = 0.4575
$$

总体加速比：

$$
S = \frac{1}{0.4575} \approx \boxed{2.19}
$$

**(2) 原因说明**
- S4 占比最大（48%），但加速比仅 1.6 倍，对总体性能提升有限；
- S3 虽可加速 20 倍，但占比仅 23%，且优化后只占 1.15%，贡献相对较小；
- S1 不可优化（11%），成为瓶颈之一；
- **加速比受限于不可优化部分和占比大但加速有限的步骤**，符合 Amdahl 定律：总体加速比受串行部分约束。

### 三、指令系统

**1. 操作码最小长度（2 分）**

列出的指令共 9 条：add, and, lw, sw, addi, beq, lui, jal, stw。
取消 funct3/funct7，仅使用操作码区分，需要能编码至少 9 种状态。

$$
\lceil \log_2 9 \rceil = 4
$$

**答案：4 位**（2^4 = 16 >= 9）。

**2. 最多寄存器数量（2 分）**

指令长度 64 位，opcode 7 位，imm1 和 imm2 各 11 位。

剩余位数：64 - 7 - 11 - 11 = **35 位**。

X 型指令需支持最多 5 个寄存器字段（如 add 需 rd1/rs1/rs2，stw 需 rs1/rs3，统一格式下可设计为 5 个寄存器槽位）：

$$
5 \times k = 35 \Rightarrow k = 7
$$

寄存器数量 = **2^7 = 128 个**。

> 注：若采用 4 个寄存器字段，则 4k = 35 无整数解；5 个 7 位字段恰好填满，且能覆盖所有指令的寄存器需求。

**3. stw 机器码（4 分）**

指令格式（7+7+7+7+7+11+11 = 64 位）：
`opcode | rd1 | rs1 | rs2 | rs3 | imm1 | imm2`

操作码顺序编码（从 0 开始）：
- 0: add, 1: and, 2: lw, 3: sw, 4: addi, 5: beq, 6: lui, 7: jal, **8: stw**

stw 操作码：8 = `0001000`（7 位）

stw x8, 0, 4(x5)：
- rs1 = x5 = `0000101`
- rs3 = x8 = `0001000`
- rd1, rs2 未使用，填 `xxxxxxx`
- imm1 = 0 = `00000000000`
- imm2 = 4 = `00000000100`

机器码：

```
0001000 xxxxxxx 0000101 xxxxxxx 0001000 00000000000 00000000100
```

### 四、数据表示与运算

**1. 死循环分析（2 分）**

```c
int f1(unsigned n) {
    int sum = 1, power = 1;
    for (unsigned i = 0; i <= n - 1; i++) { ... }
}
```

- **当 n = 0 时**：`i <= n - 1` 即 `i <= 0xFFFFFFFF`（unsigned 下溢为最大值）。条件恒成立，**会死循环**。
- **若 i 和 n 都改为 int**：`i <= n - 1` 即 `i <= -1`。i 初始为 0，0 <= -1 为假，循环不执行，返回 sum = 1。**不会死循环**。

**2. f1(24) 与 f2(24) 是否相等（4 分）**

- **f1(24)**：int 精确计算
  $$
  f(24) = 2^{25} - 1 = 33,554,431
  $$
  32 位 int 可精确表示（最大 2^31-1）。

- **f2(24)**：float（IEEE 754 单精度，23 位尾数 + 1 位隐含位 = 24 位有效位）
  $$
  2^{25} - 1 = \underbrace{111\dots1}_{25\text{个}1}
  $$
  需要 25 位有效位，超出 float 的 24 位精度，会被舍入到最接近的可表示数 **2^25 = 33,554,432**。

**结论：不相等**。f1(24) = 33,554,431，f2(24) = 33,554,432.0。

**3. 0x7F800000 与 f2 最大 n（4 分）**

- **0x7F800000**：
  - 符号位 0，指数 11111111（255），尾数全 0
  - **表示 +∞（正无穷）**

- **f2(n) 不溢出的最大 n**：
  float 最大正规数约为 (2 - 2^-23) * 2^127 = 2^128 - 2^104。
  f2(n) = 2^(n+1) - 1 <= 2^128 - 2^104 时安全。
  当 n + 1 = 128（即 n = 127）时，2^128 - 1 > 最大正规数，溢出为 +∞。
  **最大 n = 126**。

### 五、单周期 CPU

```asm
addi x6, x6, 4      // 0x00000030
addi x19, x19, 1    // 0x00000034
jal  x0, FOR        // 0x00000038
```

**1. 能否正确执行？（2 分）**

**不能。** 原因：`jal` 指令需将返回地址（PC+4）写入目标寄存器 rd。此处 rd = x0（零寄存器），若数据通路未对 x0 做特殊处理（硬连线为 0），PC+4 会被写入 x0，破坏零寄存器恒为 0 的约定，导致后续依赖 x0 的指令出错。

**2. 最简单的修改方案（4 分）**

**方案：将寄存器 x0 的输出端恒置 0（或增加写使能屏蔽逻辑，当写寄存器号为 0 时不写入）。**

原因：RISC-V 架构约定 x0 寄存器值恒为 0，且不可写入。通过在寄存器堆输出端加硬连线 0 的逻辑，可确保任何写入 x0 的操作都被丢弃，从而保证 `jal x0, FOR`（即伪指令 `j FOR`）能正确跳转而不破坏寄存器状态。

**3. 重构 SB 型指令扩大分支范围（4 分）**

原 SB 型指令：imm[12:1] 左移 1 位，范围 +-4 KB。
要扩大到 32 字节寻址（即字节偏移范围更大），需要增加立即数字段位数。

- **修改/增加的功能部件**：
  1. **立即数扩展单元**：支持新的、更宽的立即数扩展（如从 13 位有效位增加到更多位）。
  2. **PC 更新逻辑**：支持更宽偏移量的加法。
  3. **控制信号**：增加新的分支类型控制信号，区分普通分支和扩大范围分支。

- **新 SB 指令数据通路图要点**：
  - 在 ID 阶段，立即数扩展单元输出更宽的分支偏移量。
  - PC 加法器计算 `PC + 扩展后的偏移量`。
  - 比较器（Branch 判断）结果与新的控制信号共同决定是否用新 PC 值更新 PC。

```
        +-------------+
   PC --+   加法器    +---> 新 PC 候选
        |  (PC+4/偏移) |
        +-------------+
              ^
        +-------------+
  Inst --+ 立即数扩展  +---> 扩展后的分支偏移
        |  (新 SB 型) |
        +-------------+
              ^
        +-------------+
  Reg --+   比较器    +---> Branch 控制信号
        |  (rs1,rs2)  |
        +-------------+
```

### 六、流水线 CPU

代码：
```asm
sw   x0, 0(x2)       // I1
add  x10, x11, x12   // I2
addi x10, x13, 10    // I3
sub  x14, x15, x10   // I4
```

**1. 多周期流水线状态示意图（5 分）**

| 周期 | IF  | ID  | EX  | MEM | WB  |
|------|-----|-----|-----|-----|-----|
| C1   | I1  |     |     |     |     |
| C2   | I2  | I1  |     |     |     |
| C3   | I3  | I2  | I1  |     |     |
| C4   | I4  | I3  | I2  | I1  |     |
| C5   |     | I4  | I3  | I2  | I1  |
| C6   |     |     | I4  | I3  | I2  |
| C7   |     |     |     | I4  | I3  |
| C8   |     |     |     |     | I4  |

**2. 冒险检测单元相关内容（5 分）**

检测到的数据冒险：
- **I3 (addi) -> I4 (sub)**：I3 写 x10，I4 读 x10（rs2）。
  - I3 在 EX 阶段时，I4 在 ID 阶段。
  - 检测逻辑：`EX/MEM.RegWrite == 1` 且 `EX/MEM.RegisterRd == ID/EX.RegisterRs2`（即 x10）。

- **I2 (add) -> I3 (addi)**：I2 写 x10，但 I3 的源寄存器是 x13，不读 x10，**无冒险**。

**3. 前递单元输入输出（5 分）**

- **ForwardA / ForwardB 输入**：
  - `EX/MEM.RegisterRd`, `EX/MEM.RegWrite`
  - `MEM/WB.RegisterRd`, `MEM/WB.RegWrite`
  - `ID/EX.RegisterRs1`, `ID/EX.RegisterRs2`

- **输出控制**：
  - 对 I4 的 rs2（x10）：I3 在 EX/MEM 阶段，结果可前递。
  - `ForwardB = 01`（从 EX/MEM 前递到 ALU 输入 B）。

### 七、Cache

参数：2 路组相联，16 KiB，块大小 32 B，LRU。

**1. 求和阶段命中率（STRIDE=1）（4 分）**

- Cache 组数 = 16 KiB / (2 * 32 B) = 256 组
- 块大小 32 B，int 为 4 B，每块容纳 **8 个 int**
- 顺序访问：`array[0], array[1], array[2]...`

访问模式：每 8 个元素为一组，首次访问块内第 1 个元素时 **Cold Miss**，随后 7 次 **Hit**。

$$
\text{命中率} = \frac{7}{8} = \boxed{87.5\%}
$$

> 注：数组 50000 个 int（200 KB）远大于 Cache（16 KiB），但**求和阶段是顺序单次遍历**，每个块仅被访问一次，因此替换策略不影响，仍为 7/8 命中。

**2. 平均访存时间（3 分）**

$$
\text{AMAT} = t_c + (1 - H) \times t_m = 4\text{ns} + 0.125 \times 40\text{ns} = 4 + 5 = \boxed{9\text{ ns}}
$$

或按定义：
$$
0.875 \times 4 + 0.125 \times (4 + 40) = 3.5 + 5.5 = 9\text{ ns}
$$

**3. STRIDE=100 的情况（3 分）**

- STRIDE = 100 个 int = 400 字节。
- 400 mod 32 = 16 != 0，每次访问都落在**不同的块**中。
- 每块仅被访问一次，且块内无连续访问（下次访问已跳到 100 个元素之后，远超块容量 8 个元素）。

**结果：**
- **命中率约 0%**（几乎全部 Cold Miss）
- **平均访存时间约 44 ns**（或 40 ns，取决于是否计入 Cache 命中时间，但几乎每次都要访问主存）

**原因：** 空间局部性完全丧失。步长过大导致每次访问都跨越块边界，Cache 预取的块无法被后续访问重用。

### 八、虚拟存储器

参数：虚拟地址 256 TiB = 2^48，页面 64 KiB = 2^16，物理内存 64 GiB = 2^36，TLB 全相联 128 项。

| 字段 | 计算 | 位数 |

| 页内偏移 (offset) | log2(64 KiB) | **16 位** |

**2. TLB 能容纳的最大物理内存页数（3 分）**

TLB 有 128 个表项，每项映射一个虚拟页到一个物理页。

**最大物理内存页数 = 128 页**

对应内存容量：128 * 64 KiB = **8 MiB**。

2. **第 2 次访存**：用得到的物理地址访问实际数据。

**共需 2 次内存访问**。

### 九、输入输出系统（6 分）

**向量中断处理基本过程：**

1. **中断请求**：外设通过 IR0~IR7 向中断控制器发出请求。
2. **中断屏蔽检查**：若总开关 INTE = 0，所有请求被屏蔽；若 INTE = 1，进入下一步。
3. **优先级判优**：中断控制器对多个同时到达的请求进行优先级排队（如固定优先级或轮转）。
4. **向 CPU 发请求**：控制器将最高优先级的中断请求发送给 CPU。
5. **CPU 响应**：CPU 完成当前指令后，发出中断响应信号（INTA）。
6. **向量寻址**：中断控制器将对应中断源的**向量地址**（或中断类型号）送上数据总线。
7. **保存现场**：CPU 自动保存 PC、PSW 等关键寄存器到堆栈。
8. **转入服务程序**：CPU 根据向量地址查中断向量表，获取中断服务程序入口地址，跳转执行。
9. **中断处理**：执行中断服务程序（ISR），处理外设请求。
10. **恢复现场与返回**：ISR 结束执行，恢复之前保存的 PC 和寄存器，执行 `IRET` 返回主程序。
11. **不允许嵌套**：由于系统禁止中断嵌套，在处理过程中新的中断请求被屏蔽，直到当前中断处理完毕。

### 二、性能计算

**1. 平均 CPI（4 分）**

$$
\text{MIPS} = \frac{f}{\text{CPI} \times 10^6} \Rightarrow 1500 = \frac{2 \times 10^9}{\text{CPI} \times 10^6}
$$

$$
\text{CPI} = \frac{2000}{1500} = \boxed{\frac{4}{3} \approx 1.33}
$$

**2. 新平均 CPI（6 分）**

存储子系统速率不变，原访存延迟 = 1 个时钟周期（2 GHz 时）= 0.5 ns。
时钟频率提高到 4 GHz，新时钟周期 = 0.25 ns。
访存延迟变为 0.5 / 0.25 = **2 个时钟周期**。

访存指令额外增加 1 个周期：
- 30% 指令各 1 次访存：增加 0.30 * 1 = 0.30
- 5% 指令各 2 次访存：增加 0.05 * 2 = 0.10

$$
\text{新 CPI} = \frac{4}{3} + 0.30 + 0.10 = \frac{4}{3} + 0.4 = \frac{26}{15} \approx \boxed{1.73}
$$

### 六、流水线 CPU

**1. 转发控制单元输入信号（4 分）**

除 `Control.branch` 外，还需：
- `ID/EX.RegisterRs1`、`ID/EX.RegisterRs2`（当前指令源寄存器号）
- `EX/MEM.RegisterRd`、`EX/MEM.RegWrite`（前条指令写回信息）
- `MEM/WB.RegisterRd`、`MEM/WB.RegWrite`（前两条指令写回信息）
- 可选：`EX/MEM.MemRead`（用于检测 load-use 冒险）

**2. ForwardX 工作逻辑（4 分）**

以 ForwardA（ALU 输入 A）为例：
1. **EX 冒险**：若 `EX/MEM.RegWrite == 1` 且 `EX/MEM.RegisterRd != 0` 且 `EX/MEM.RegisterRd == ID/EX.RegisterRs1`，则 `ForwardA = 01`（从 EX/MEM 前递）。
2. **MEM 冒险**：若 `MEM/WB.RegWrite == 1` 且 `MEM/WB.RegisterRd != 0` 且 `MEM/WB.RegisterRd == ID/EX.RegisterRs1` 且非 EX 冒险条件，则 `ForwardA = 10`（从 MEM/WB 前递）。
3. **无冒险**：`ForwardA = 00`（从寄存器堆读取）。

**3. 数据冒险（3 分）**

代码：
```asm
add  x12, x1, x2
ld   x10, 0(x5)
sub  x10, x3, x2
beq  x10, x11, target
```

- **ld -> beq**：`ld` 写 x10，`beq` 读 x10。Load-Use 冒险。
- **sub -> beq**：`sub` 写 x10，`beq` 读 x10。RAW 冒险。

> 注：若分支比较提前到 ID 阶段，上述冒险需停顿解决，因为结果尚未产生。

**4. 第 5 周期转发控制信号（4 分）**

| 信号 | 值 | 说明 |
|------|-----|------|
| ForwardA | 00 或 10 | 取决于 sub 是否需要前递 |
| ForwardB | 01 | beq 的 rs2（x11）无冒险，但若 x10 来自 sub，需前递 |

> 具体值需结合完整调度表确定。核心逻辑：检查 EX/MEM 和 MEM/WB 阶段的目的寄存器是否与 ID/EX 的源寄存器匹配。

### 七、Cache / 虚拟存储综合

**1. 虚拟地址 0x00350 分析（7 分）**

- 虚拟地址 24 位，页大小 256 B（2^8），偏移 8 位。
- 0x00350 = `0000 0000 0011 0101 0000`
- **VPN** = 高 16 位 = `0x003`
- **页内偏移** = `0x50`

> 注：后续 PTE 地址、物理地址、TLB 状态需依赖页表基址和页表内容，原题未提供完整页表映射，请补充后计算。

**2. 第 0 个 PTE 的 PPN 与用户页表起始地址相同（2 分）**

解释：操作系统通常将**页表自身**也视为一个普通页进行管理。第 0 个 PTE 指向的物理页就是页表所在的物理页，因此其 PPN 等于页表起始物理地址的物理页号。这是自映射（Self-mapping）或页表自引用机制，便于操作系统通过虚拟地址访问页表本身。

**3. UPTB+4*(MA>>8) 含义（2 分）**

- `MA>>8`：将物理地址右移 8 位，即除以 256，得到**物理页号**（页大小 256 B）。
- `4*(MA>>8)`：每个页表项 4 字节，乘以页号得到页表内偏移。
- `UPTB + ...`：用户页表基址 + 偏移 = **某物理页对应的 PTE 地址**。

**4. 增加虚拟内存后的新 CPI（2 分）**

- 原 CPI = 2
- 每 1000 条指令：10 次指令 TLB 缺失 + 20 次数据 TLB 缺失 = 30 次 TLB 缺失
- 每次 TLB 缺失惩罚 = 20 周期

$$
\text{新 CPI} = 2 + \frac{30}{1000} \times 20 = 2 + 0.6 = \boxed{2.6}
$$

**5. 虚拟内存对性能的影响及原因（3 分）**

- **性能变化**：CPI 从 2 增加到 2.6，**性能下降**。
- **原因**：TLB 缺失引入了额外的 20 周期处理开销。
- **为什么要用虚拟内存**：
  1. **隔离性**：每个进程拥有独立地址空间，防止越界访问。
  2. **扩展性**：程序可使用超过物理内存容量的地址空间。
  3. **灵活性**：支持内存共享、按需调页（Demand Paging）、简化链接与加载。
  4. **安全性**：操作系统通过页表控制内存访问权限。

### 八、虚拟存储器

**1. 地址位数与页号（2 分）**

- 虚拟地址 4 GB = 2^32，共 **32 位**。低 12 位为页内偏移，**高 20 位为虚拟页号**。
- 物理地址 1 MB = 2^20，共 **20 位**。低 12 位为页内偏移，**高 8 位为物理页号**。

**2. Cache 地址划分（4 分）**

**直接映射，16 行，块大小 32 B：**
- 块内偏移：5 位（log2(32)）
- 行索引：4 位（log2(16)）
- 标记：20 - 4 - 5 = **11 位**
- 位置：高 11 位 | 中 4 位 | 低 5 位

**8 路组相联，16 行，块大小 64 B：**
- 组数 = 16 / 8 = 2 组
- 块内偏移：6 位（log2(64)）
- 组索引：1 位（log2(2)）
- 标记：20 - 1 - 6 = **13 位**
- 位置：高 13 位 | 中 1 位 | 低 6 位

**3. 虚拟地址 00C60H 分析（3 分）**

00C60H = `0000 0000 1100 0110 0000`
- 虚拟页号：高 20 位 = `0000 0000 1100 = 0x00C`
- 页内偏移：`0110 0000 = 0x60`

> 注：该页是否在主存、物理地址及 Cache 命中情况需依赖页表和 Cache 内容，原题未提供完整页表/Cache 映射，请补充后判断。
