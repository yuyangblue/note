    一、实验介绍¶

    1. 实验内容¶

    本实验将使用毕昇LLM应用开发平台，通过知识库构建基于医学知识内容的医学大模型，并与其进行交互。学生可以向大模型提问，对话，回答问答题目等方式学习医学知识内容，同时可以上传其他知识文件进行学习

    2. 实验要点¶

    • 掌握毕昇LLM应用开发平台的简单使用

    • 了解知识库大模型应用的处理方式

    3. 实验环境¶

    • 希冀平台

    • 毕昇LLM应用开发平台实验环境

    二、实验步骤¶

    2.1 实验环境介绍¶

    BISHENG毕昇是一款面向企业级场景的开源 LLM 应用开发平台，目前已获得大量行业头部组织及世界 500 强企业的深度使用。命名致敬活字印刷术发明人毕昇，寓意让智能应用像当年印刷术一样低成本、大规模地“活”在千行百业。平台核心亮点是“专家级品味的通用 Agent 灵思”：通过自研 AGL（Agent Guidance Language）框架，把领域专家的偏好、经验与业务逻辑直接编译成 Agent 的“思维模板”，使 AI 在复杂任务中具备人类专家级的理解深度与决策质量。

    BISHENG workflow 采用独立、完备的应用编排内核，一套画布即可同时支撑对话、流程、批处理等多类型任务，无需像同类产品那样拆分成 chatflow 与 workflow 两套体系；Human-in-the-loop 机制允许用户在流程任意节点实时干预、多轮反馈，真正实现“人”与“智能”的螺旋式共创。画圈即成环、并列即并行、多选即批量的直觉化连线方式，让业务人员无需代码即可把成环、并行、判断、跑批等逻辑自由组合，快速落地文档审核、固定版式报告生成、多智能体协作、制度差异比对、工单问答、客服辅助、会议纪要、简历筛选、通话记录分析、非结构化数据治理、知识挖掘、数据分析等数百种高复杂度企业场景。

    下面介绍BISHENG界面的使用方法，工作台窗口允许用户使用日常模式和灵思Linsight模型。日常模式可以基于知识库和联网工具进行搜索和对话，基于Qwen2大模型生成回复。灵思Agent通过 AGL（Agent Guidance Language）框架，将领域专家的偏好、经验与业务逻辑融入 AI 之中，让 Agent 在处理任务时能具备 「专家级理解」。

    构建页面允许用户新建新的应用和修改已有的应用，应用支持工作流、助手、技能等模块，并提供了大量的demo模板。创建好的应用上线后，可以在会话中使用，通过新建对应应用的会话在相关界面使用。

    BISHENG平台同时支持文档支持库和QA知识库的创建，用户可以选择已有的embedding模型构建需要的知识库，并在工作台、LLM应用中使用想要的知识库。

    在模型页面可以看到平台支持的模型供应商和模型列表，包括模型名称，类型，状态等内容，详细信息由管理员账户操作。

    更多使用功能可以参考BISHENG平台的官方手册说明，里面还有丰富的案例和详细的平台说明报告

    2.2 医学大模型知识库构建¶

    首先，我们为大模型提供构建好的医学知识库，选择知识->文档知识库->创建，构建自定义知识库内容

    填写知识库名称和知识库描述内容，embedding模型试验Qwen3-Embedding-8B模型向量化文本内容。

    上传文件到知识库，这里使用附件的sft-20k.txt文件，下载到本地并进行上传。该文件包含200条医学知识问答对。

    上传完毕点击下一步，对文档进行切分

    • 切分通用设定

      • 预期切分长度：表示切分字符串的字符数

      • 重叠区长度：表示切分窗口与上一个切分窗口的重叠区字符长度

    • 切分规则

      • 使用单换行和双换行进行切分，也可以根据文档的结构自定义切分规则

    • PDF文档解析策略

      • 强制开启OCR

      • 开启公式识别

    由于txt文件的内容为问答对，这里我们以1个字符为切分长度，0字符为重叠长度进行切分，使用正则表达式进行切分，正则表达式为\s*},\s*\{\s*，选择后方切分，这样可以将json格式的每一项切分出来，更合适问答对的组成方式。

    点击左下角预览分段结果进行切分预览。

    可以看到分段结果都是完整的问答对格式，分段结果符合要求则可以进行下一步，不符合要求则重新制定切分规则和字符设置。

    点击下一步通过embedding模型对其进行向量化，存入向量数据库。解析中表示正在请求大模型处理。

    状态为已完成则表示处理完成，可以在应用中使用。

    点击查看进入分段管理该知识库，可以对分段编辑和删除，编辑支持自定义分段。

    2.3 医学大模型应用设计¶

    回到bisheng主界面，选择构建->新建应用->助手->自定义助手

    设置助手名称，在下方以推荐格式完成助手描述，该描述有助于后续生成该应用的开场白内容和引导问题内容的生产。

    • 名称：医学大模型应用

    • 描述：你是一名医学专家，非常擅长解决用户的相关问题，请根据用户要求结合相关知识回答问题。

    平台会根据描述内容，生产助手画像，为助手生成开场白、引导问题、工具和技能。可以选择是否使用生成内容，或对生成内容进行修改。

    可以选择需要使用的内容，然后关闭，这里我们仅使用开场白。关闭该界面后，即可进入助手配置和调试界面。该界面分为三个部分，从左到右依次为助手画像描述、助手基础配置、助手调试界面。

    进入基础配置，我们选择AI模型配置，使用ProdSvc/qwen2.5:32b模型。然后修改开场引导中的引导问题，这里我们给出三个实例（两个常见药品使用问题，和就诊问题）

    • 养胃舒胶囊的注意事项

    • 克林霉素磷酸酯外用溶液的有效期是多久

    • 怎么治疗干性坏疽

    知识库选择创建的医学知识库。工具和技能无需添加，以下是完整的配置信息

    完成配置后，可以在右侧进行测试，比如我们点击引导问题进行提问

    我们也可以换一种描述方式，让大模型结合知识库进行回答，例如：“我得了干性坏疽，怎么治疗”

    完成测试后，点击保存并上线即可在会话->应用名称 进入该应用进行使用

    三、实验任务¶

    1. 参考实验步骤，尝试扩充医学知识库，并让助手回答更复杂的问题。

    2. 撰写实验报告，需要在实验报告中完成以下任务

      1. 要求完成实验任务1实验过程全部记录下来，提供截图和文字描述

      2. 思考题1：在 bisheng 平台中，我们对于问答对使用正则表达式的方式进行切分，请问有没有更好的切分方式或者额外使用什么样的预处理方法处理数据文件

      3. 思考题2：知识库未收录的内容不宜向用户生成错误或者不全面正确的内容，如何避免这种情况的发生，请说出你的想法。
